# <a name="data-migration"></a>การโยกย้ายข้อมูล

FastTrack สามารถช่วยคุณโยกย้ายจดหมายและไฟล์ข้อมูลในสภาพแวดล้อมต้นทางของคุณไปยัง Office 365 (Exchange Online, SharePoint Online และ OneDrive for Business)

ชนิดของความช่วยเหลือที่เรามีจะขึ้นอยู่กับจํานวนสิทธิ์การใช้งาน Office 365 ของคุณ:

  - **สําหรับผู้เช่า Office 365 ที่มีสิทธิ์การใช้งาน 150-499** รายการ : FastTrack จะให้แนวทางการโยกย้ายเท่านั้น คุณมีหน้าที่รับผิดชอบในการโยกย้ายข้อมูล เราแนะแนวคุณผ่านเอกสารประกอบที่ช่วยให้คุณวางแผนและใช้เครื่องมือฟรีเพื่อโยกย้ายด้วยตนเอง
  - **สําหรับผู้เช่า Office 365 ที่มีสิทธิ์การใช้งาน 500 สิทธิ์หรือมากกว่า**: FastTrack มอบแนวทางการโยกย้ายและบริการการโยกย้ายข้อมูล เรามีแนวทางเพื่อช่วยให้คุณวางแผนการโยกย้ายของคุณ กําหนดค่าสภาพแวดล้อมต้นทางและผู้เช่า Office 365 ของคุณ และใช้ประโยชน์จากบริการการโยกย้ายข้อมูลของเราเพื่อโยกย้ายข้อมูลของคุณ คุณสร้างและจัดตารางเวลาเหตุการณ์การโยกย้ายของคุณ เราจะเปิดใช้เหตุการณ์การโยกย้ายตามกําหนดการของคุณ ตรวจสอบความคืบหน้าของเหตุการณ์และรายงานสถานะ

> [!NOTE]
> ถ้าคุณซื้อหรือต่ออายุแผนเชิงพาณิชย์ก่อนวันที่ 1/9/2017 คุณต้องมีสิทธิ์การใช้งานเพียง 150 สิทธิ์จึงจะเข้าเกณฑ์ในบริการการโยกย้ายข้อมูลได้ For education plans, only your paid faculty and staff licenses are eligible for data migration services.

### <a name="considerations"></a>ข้อพิจารณา

  - สภาพแวดล้อมต้นทางของคุณต้องตรงตามความต้องการเฉพาะเพื่อโยกย้ายข้อมูลไปยัง Office 365 ดู [ผลิตภัณฑ์และความสามารถ](products-and-capabilities.md) เพื่อดูข้อมูลเพิ่มเติมเกี่ยวกับความคาดหวังของสภาพแวดล้อมต้นทางใน Exchange, SharePoint และ OneDrive for Business
  - เราต้องการการเข้าถึงและสิทธิ์ที่เหมาะสมต่อสภาพแวดล้อมต้นทางของคุณและผู้เช่า Office 365 เพื่อให้บริการการโยกย้ายข้อมูล
  - บริการการโยกย้ายข้อมูลของเราไม่ได้ออกแบบมาหรือมีไว้เพื่อข้อมูลภายใต้ข้อกฎหมายหรือข้อบังคับพิเศษ เมื่อเราโยกย้ายข้อมูลของคุณ ข้อมูลนั้นอาจถูกถ่ายโอน จัดเก็บ และประมวลผลได้จากทุกที่ที่เราดูแลอยู่ (ยกเว้นในกรณีที่เป็นอย่างอื่นยกเว้นโครงการการโยกย้าย FastTrack ของคุณ)
  - เราไม่สามารถรับประกันความเร็วของการโยกย้ายจดหมายหรือไฟล์
  - ปัญหาที่ไม่ได้คาดไว้ (เช่น ไม่สามารถอ่านได้ หรือรายการเสียหายในสภาพแวดล้อมต้นทาง) อาจป้องกันไม่ให้เราความสามารถในการโยกย้ายรายการข้อมูลของคุณบางส่วนได้
  - ปัจจัยภายนอกที่อยู่นอกเหนือการควบคุมของเรา (เช่น การเปลี่ยนแปลงส่วนติดต่อการเขียนโปรแกรมแอปพลิเคชัน (API)) ของบริษัทอื่นอาจส่งผลให้เกิดการเปลี่ยนแปลง ความล่าช้า หรือการระงับบริการการโยกย้ายข้อมูลของเรา

### <a name="migration-service-availability"></a>ความพร้อมใช้งานของบริการการโยกย้าย

  - **ลูกค้าเชิงพาณิชย์และลูกค้ารัฐบาลสหราชอาณาจักร:** เราให้บริการการโยกย้ายข้อมูลตลอด 24 ชั่วโมงทุกวัน (7) วันต่อสัปดาห์ (24 ชั่วโมง 7 วัน)
  - **For US Government/DOD customers:** เราให้บริการการโยกย้ายข้อมูลตลอด 24 ชั่วโมงต่อวัน 5 (5) วันธุรกิจต่อสัปดาห์ (24x5)

## <a name="migration-to-exchange-online"></a>การโยกย้ายไปยัง Exchange Online

เมื่อคุณเลือกที่จะใช้ FastTrack เพื่อโยกย้ายอีเมลของคุณไปยัง Exchange Online เรามีแนวทางการโยกย้ายและบริการการโยกย้ายข้อมูล เรามีแนวทางเพื่อช่วยให้คุณวางแผนการโยกย้ายกําหนดค่าสภาพแวดล้อมต้นทางและ Exchange Online ของคุณ และใช้ประโยชน์จากบริการการโยกย้ายข้อมูลของเราเพื่อโยกย้ายกล่องจดหมายของคุณ คุณสร้างและจัดตารางเวลาเหตุการณ์การโยกย้ายของคุณ เราจะเปิดใช้เหตุการณ์การโยกย้ายตามกําหนดการของคุณ ตรวจสอบความคืบหน้าของเหตุการณ์และรายงานสถานะ เมื่อเหตุการณ์การโยกย้ายของคุณเสร็จสมบูรณ์ คุณสามารถคาดหวังให้จดหมายจากกล่องจดหมายต้นทางที่เหมาะสมและกล่องจดหมายต้นทางที่มีสิทธิ์ของสภาพแวดล้อมต้นทางของคุณถูกโยกย้ายไปยัง Exchange Online

### <a name="considerations"></a>ข้อพิจารณา

  - ก่อนการโยกย้าย คุณต้องออนบอร์ดหลัก FastTrack ให้เสร็จสมบูรณ์เพื่อ Exchange Online
      - ถ้าคุณเริ่มการออนบอร์ดด้วยตนเอง คุณต้องผ่านการตรวจสอบที่ต้องมีและโปรแกรมเบื้องต้น ดูรายละเอียด [ได้จากผลิตภัณฑ์](products-and-capabilities.md) และความสามารถ
  - FastTrack จะโยกย้ายไปยังกล่องจดหมาย Office 365 ที่ใช้งานอยู่เท่านั้น
  - คุณต้องตรงตามความต้องการเฉพาะถ้าคุณต้องการโยกย้ายจากสภาพแวดล้อม Exchange ภายในองค์กร ดูรายละเอียด [ในเบื้องต้นเกี่ยวกับ](https://go.microsoft.com/fwlink/?LinkId=787528) การปรับใช้แบบไฮบริด
  - สภาพแวดล้อมต้นทางแต่ละสภาพแวดล้อมต้องอยู่บน Service Pack (SP) ล่าสุด และระดับการอัปเดตสะสม (RU)/CUMULATIVE Update (CU) สําหรับผลิตภัณฑ์ที่เกี่ยวข้องในสภาพแวดล้อมต้นทาง
  - รายชื่อการแจกจ่าย (*วัตถุ MailEnabledGroup)* และที่ติดต่อภายนอก (วัตถุ *MailEnabledContact)* ที่มีอยู่ใน Active Directory ภายในองค์กรของคุณไม่ได้เป็นส่วนหนึ่งของการโยกย้ายข้อมูลกล่องจดหมาย อย่างไรก็ตาม คุณสามารถซิงโครไนซ์ได้โดยใช้ Azure Active Directory (Azure AD) Connect

## <a name="fasttrack-responsibilities"></a>ความรับผิดชอบ FastTrack

ผู้เชี่ยวชาญด้าน FastTrack ของเราจะจัดกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด

ผู้เชี่ยวชาญด้าน FastTrack ของเรายังปฏิบัติกิจกรรมต่อไปนี้เฉพาะการโยกย้าย Exchange โดยเฉพาะ:

  -  ให้แนวทางเพื่อช่วยให้คุณเปิดใช้งานการต่อสายจดหมาย SMTP ร่วมกันระหว่างสภาพแวดล้อมต้นทางของคุณและ Exchange Online ถ้ามี

## <a name="your-responsibilities"></a>ความรับผิดชอบของคุณ

คุณปฏิบัติกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด

คุณยังสามารถปฏิบัติกิจกรรมต่อไปนี้โดยเฉพาะการโยกย้าย Exchange:

  - การออนบอร์ดหลัก FastTrack ให้เสร็จสมบูรณ์ใน Exchange Online ถ้าคุณเริ่มการออนบอร์ดด้วยตนเอง คุณต้องผ่านการตรวจสอบที่ต้องมีและโปรแกรมเบื้องต้น ดูรายละเอียด [ได้จากผลิตภัณฑ์](products-and-capabilities.md) และความสามารถ
  -  ติดตั้งซอฟต์แวร์ไคลเอ็นต์ระดับที่เหมาะสมตามแนวทางของ Office 365 ดูรายละเอียด [ได้จากสถานที่ทํางาน](https://transform.microsoft.com/download?assetname=assets%2FMicrosoft%20365%20%20Security%20Group%20Marketing%20Field%20Advisory%20%20Renaming%20Office%20365%20SMB%20Products%20and%20Office%20365%20ProPlus.msg) ยุคใหม่
  -  ตรงตามความต้องการเฉพาะถ้าคุณต้องการโยกย้ายจากสภาพแวดล้อม Exchange ในองค์กร ดูรายละเอียด [ในเบื้องต้นเกี่ยวกับ](https://go.microsoft.com/fwlink/?LinkId=787528) การปรับใช้แบบไฮบริด
  -  ตรวจสอบให้แน่ใจว่าสภาพแวดล้อมต้นทางแต่ละสภาพแวดล้อมอยู่บน Service Pack (SP) ล่าสุด และระดับ Rollup (RU)/cumulative update (CU) ถ้ามี
  -  กําหนดค่าและตรวจสอบความถูกต้องของการกําหนดเส้นทางจดหมาย SMTP ร่วมกันระหว่างสภาพแวดล้อมต้นทางของคุณและ Exchange Online ถ้ามี
  -  ตรวจสอบให้แน่ใจว่าขนาดกล่องจดหมายต้นทางของคุณไม่เกินโควตากล่องจดหมายเป้าหมาย ขึ้นอยู่กับแพลตฟอร์มต้นทาง คุณอาจต้องจํากัดข้อมูลต้นฉบับของคุณไปที่ 85 เปอร์เซ็นต์ของโควตากล่องจดหมายเป้าหมาย
  -  โยกย้ายข้อมูลฝั่งไคลเอ็นต์ได้ถ้าต้องการ ซึ่งรวมถึงแต่ไม่จํากัดเฉพาะสมุดรายชื่อข้อมูลในไฟล์ PST ภายในเครื่อง กฎ Outlook และการตั้งค่า Outlook ภายในเครื่อง
  -  ช่วยผู้ใช้ของคุณเกี่ยวกับการแก้ไขปัญหาการโยกย้ายด้านไคลเอ็นต์

## <a name="migration-to-sharepoint-online"></a>การโยกย้ายไปยัง SharePoint Online

เมื่อคุณเลือกที่จะใช้ FastTrack เพื่อโยกย้ายไฟล์ของคุณไปยัง SharePoint Online เรามีแนวทางการโยกย้ายและบริการการโยกย้ายข้อมูล เรามีแนวทางเพื่อช่วยให้คุณวางแผนการโยกย้ายกําหนดค่าสภาพแวดล้อมต้นทางและ SharePoint Online ของคุณ และใช้ประโยชน์จากบริการการโยกย้ายข้อมูลของเราเพื่อโยกย้ายไฟล์ของคุณ คุณสร้างและจัดตารางเวลาเหตุการณ์การโยกย้ายของคุณ เราจะเปิดใช้เหตุการณ์การโยกย้ายตามกําหนดการของคุณ ตรวจสอบความคืบหน้าของเหตุการณ์และรายงานสถานะ เมื่อเหตุการณ์การโยกย้ายของคุณเสร็จสมบูรณ์ คุณสามารถคาดหวังให้ไฟล์จากแหล่งข้อมูลที่จัดเวลาไว้อย่างเหมาะสมและที่มีสิทธิ์ของสภาพแวดล้อมต้นทางของคุณถูกโยกย้ายไปยัง SharePoint Online

## <a name="considerations"></a>ข้อพิจารณา

 - การโยกย้ายทั้งหมดขึ้นอยู่กับโควตาของ SharePoint Online อ้างอิงไปยัง <a href="https://go.microsoft.com/fwlink/?LinkId=698855">ขีดจํากัดของ SharePoint</a> เพื่อดูรายละเอียด 
  - เราขอแนะนนะให้คุณจํากัดจํานวนรวมของการโยกย้ายเป็น 75 เปอร์เซ็นต์ของโควตาที่เก็บข้อมูล SharePoint Online โดยรวมที่คุณมีสิทธิ์ (รวมถึงที่เก็บข้อมูลเพิ่มเติมที่คุณอาจซื้อแยกต่างหาก)

## <a name="fasttrack-responsibilities"></a>ความรับผิดชอบ FastTrack

ผู้เชี่ยวชาญด้าน FastTrack ของเราจะจัดกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูลในกระบวนการ [และความคาดหวัง](process-and-expectations.md) ของรายละเอียด

## <a name="your-responsibilities"></a>ความรับผิดชอบของคุณ

คุณปฏิบัติกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูลในกระบวนการ [และความคาดหวัง](process-and-expectations.md) ของรายละเอียด

คุณยังจะปฏิบัติกิจกรรมต่อไปนี้โดยเฉพาะการโยกย้าย SharePoint Online:

  - เตรียมใช้งานไซต์ทีม SharePoint ทั้งหมดเพื่อตั้งเป้าหมายโดยเหตุการณ์การโยกย้ายของคุณ

## <a name="migration-to-onedrive-for-business"></a>การโยกย้ายไปยัง OneDrive for Business

เมื่อคุณเลือกที่จะใช้ FastTrack เพื่อโยกย้ายไฟล์ของคุณไปยัง OneDrive for Business เรามีแนวทางการโยกย้ายและบริการการโยกย้ายข้อมูล เรามีแนวทางเพื่อช่วยให้คุณวางแผนการโยกย้ายกําหนดค่าสภาพแวดล้อมต้นทางและ OneDrive for Business ของคุณ และใช้ประโยชน์จากบริการการโยกย้ายข้อมูลของเราเพื่อโยกย้ายไฟล์ของคุณ คุณสร้างและจัดตารางเวลาเหตุการณ์การโยกย้ายของคุณ เราจะเปิดใช้เหตุการณ์การโยกย้ายตามกําหนดการของคุณ ตรวจสอบความคืบหน้าของเหตุการณ์และรายงานสถานะ เมื่อเหตุการณ์การโยกย้ายของคุณเสร็จสมบูรณ์ คุณสามารถคาดหวังให้ไฟล์จากแหล่งข้อมูลที่จัดเวลาไว้อย่างเหมาะสมและแหล่งข้อมูลที่มีสิทธิ์ของสภาพแวดล้อมต้นทางของคุณถูกโยกย้ายไปยัง OneDrive for Business

## <a name="considerations"></a>ข้อพิจารณา

  - การโยกย้ายทั้งหมดขึ้นอยู่กับโควตาของ SharePoint Online อ้างอิงไปยัง <a href="https://go.microsoft.com/fwlink/?LinkId=698855"> ขีดจํากัดของ SharePoint</a> เพื่อดูรายละเอียด 
  - เราขอแนะนนะให้คุณจํากัดปริมาณข้อมูลโดยรวมที่คุณโยกย้ายเป็น 75 เปอร์เซ็นต์ของโควตาที่เก็บข้อมูล SharePoint Online โดยรวมที่คุณมีสิทธิ์ (รวมถึงที่เก็บข้อมูลเพิ่มเติมที่คุณอาจซื้อแยกต่างหาก)
  - FastTrack จะโยกย้ายไปยังไดรฟ์ OneDrive for Business ที่ใช้งานอยู่เท่านั้น

## <a name="fasttrack-responsibilities"></a>ความรับผิดชอบ FastTrack

ผู้เชี่ยวชาญด้าน FastTrack ของเราจะจัดกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด

## <a name="your-responsibilities"></a>ความรับผิดชอบของคุณ

คุณปฏิบัติกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด

นอกจากนี้ คุณยังสามารถกิจกรรมต่อไปนี้เพื่อโยกย้าย OneDrive for Business โดยเฉพาะ:

  - เตรียมใช้งานไซต์ OneDrive for Business ทั้งหมดที่จะตั้งเป้าหมายโดยเหตุการณ์การโยกย้ายของคุณ

## <a name="migration-to-microsoft-teams-and-microsoft-365-groups"></a>การโยกย้ายไปยัง Microsoft Teams และกลุ่ม Microsoft 365

เมื่อคุณเลือกที่จะใช้ FastTrack เพื่อโยกย้ายไฟล์ของคุณไปยัง Microsoft Teams และ Microsoft 365 Groups เราจะให้แนวทางการโยกย้ายและบริการการโยกย้ายข้อมูล เรามีแนวทางเพื่อช่วยให้คุณวางแผนการโยกย้าย กําหนดค่าสภาพแวดล้อมต้นทางและ Teams และกลุ่ม Microsoft 365 ของคุณ และใช้ประโยชน์จากบริการการโยกย้ายข้อมูลของเราเพื่อโยกย้ายไฟล์ของคุณ คุณสร้างและจัดตารางเวลาเหตุการณ์การโยกย้ายของคุณ เราจะเปิดใช้เหตุการณ์การโยกย้ายตามกําหนดการของคุณ ตรวจสอบความคืบหน้าของเหตุการณ์และรายงานสถานะ เมื่อกิจกรรมการโยกย้ายของคุณเสร็จสมบูรณ์ คุณสามารถคาดหวังให้ไฟล์จากแหล่งข้อมูลที่จัดเวลาไว้อย่างเหมาะสมและทรัพยากรที่มีสิทธิ์ของสภาพแวดล้อมต้นทางของคุณถูกโยกย้ายไปยัง Teams และ Microsoft 365 Groups ทีมแชนเนลและกลุ่ม Microsoft 365 ต้องได้รับการจัดเตรียมไว้ล่วงหน้าโดยลูกค้าก่อนที่จะสามารถโยกย้ายข้อมูลไปยังปลายทางประเภทนี้ได้ Teams และ Microsoft 365 Groups ส่งผลกระทบต่อสิทธิ์ของคุณบนปลายทางของไฟล์ Teams และ Microsoft 365 Groups ถูกสร้างขึ้นเพื่อให้การร่วมมือกัน แชนเนล Teams หรือกลุ่ม Microsoft 365 จะระบุว่าใครมีสิทธิ์เข้าถึงไฟล์เหล่านั้นเมื่อโยกย้ายไปยังปลายทางเหล่านั้น FastTrack จะไม่เพิ่มผู้ใช้หรือกลุ่มลงในแชนเนล Teams ใดๆ หรือสิทธิ์กลุ่ม Microsoft 365 ระหว่างการโยกย้าย

## <a name="considerations"></a>ข้อพิจารณา

- การโยกย้ายทั้งหมดขึ้นอยู่กับโควตาของ SharePoint Online อ้างอิงไปยัง <a href="https://go.microsoft.com/fwlink/?LinkId=698855"> ขีดจํากัดของ SharePoint</a> เพื่อดูรายละเอียด 
- เราขอแนะนนะให้คุณจํากัดจํานวนรวมของการโยกย้ายเป็น 75 เปอร์เซ็นต์ของโควตาที่เก็บข้อมูล SharePoint Online โดยรวมที่คุณมีสิทธิ์ (รวมถึงที่เก็บข้อมูลเพิ่มเติมที่คุณอาจซื้อแยกต่างหาก)

## <a name="fasttrack-responsibilities"></a>ความรับผิดชอบ FastTrack

ผู้เชี่ยวชาญด้าน FastTrack ของเราจะจัดกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด

## <a name="your-responsibilities"></a>ความรับผิดชอบของคุณ 

คุณปฏิบัติกิจกรรมมาตรฐานระหว่างโครงการการโยกย้าย อ้างอิงข้อมูลความรับผิดชอบในการโยกย้ายข้อมูล [ในกระบวนการและความคาดหวัง](process-and-expectations.md) ของรายละเอียด
นอกจากนี้ คุณยังจะปฏิบัติกิจกรรมต่อไปนี้โดยเฉพาะในการโยกย้าย Microsoft Teams และ Microsoft 365 Groups: 

- เตรียมใช้งานแชนเนล Microsoft Teams และกลุ่ม Microsoft 365 ทั้งหมดตามเป้าหมายของเหตุการณ์การโยกย้ายของคุณ

> [!NOTE]
>FastTrack ไม่ได้เตรียมใช้งาน Microsoft Teams Channels หรือ Microsoft 365 Groups ไว้ล่วงหน้า FastTrack จะไม่เพิ่มผู้ใช้หรือกลุ่มลงในแชนเนล Microsoft Teams หรือกลุ่ม Microsoft 365 คุณต้องเพิ่มผู้ใช้หรือกลุ่มของคุณลงในแชนเนล Microsoft Teams และกลุ่ม Microsoft 365 ทั้งหมดก่อนที่คุณจะโยกย้ายข้อมูลไปยังปลายทางเหล่านั้น เพื่อให้ผู้ใช้เหล่านั้นสามารถเข้าถึงเอกสารที่เพิ่งโยกย้ายได้
