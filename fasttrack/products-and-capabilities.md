# <a name="products-and-capabilities"></a>ผลิตภัณฑ์และความสามารถ

## <a name="services-and-scenarios-supported-by-fasttrack"></a>บริการและสถานการณ์ที่ได้รับการสนับสนุนโดย FastTrack

หัวข้อนี้มีรายละเอียดเกี่ยวกับสถานการณ์ปริมาณงานที่ได้รับการสนับสนุนโดย FastTrack และความคาดหวังของสภาพแวดล้อมแหล่งข้อมูลที่จำเป็นก่อนที่เราจะเริ่มต้น จากการตั้งค่าปัจจุบันของคุณเราจะทำงานร่วมกับคุณเพื่อสร้างแผนการทำงานที่จะนำสภาพแวดล้อมต้นฉบับของคุณไปใช้กับความต้องการขั้นต่ำสำหรับปฐมนิเทศที่ประสบความสำเร็จ

FastTrack ให้คำแนะนำที่จะช่วยให้คุณเป็นอันดับแรกที่มีความสามารถหลัก (ทั่วไปสำหรับบริการ Microsoft Online ทั้งหมด) และจากนั้นด้วยปฐมนิเทศบริการที่มีสิทธิ์แต่ละรายการให้ทำดังนี้

  - [ทั่ว](#general)
  - [Office 365](#office-365)
  - [การเคลื่อนไหวขององค์กร + ความปลอดภัย](#enterprise-mobility--security)
  - [Windows 10](#windows-10)
  - [เดสก์ท็อปเสมือนของ Windows](#windows-virtual-desktop)
  - [App Assure](#app-assure)
  - [Microsoft Edge ใหม่](#the-new-microsoft-edge)

> [!NOTE]
> สำหรับข้อมูลเกี่ยวกับความคาดหวังของสภาพแวดล้อมแหล่งข้อมูลสำหรับ Office ๓๖๕รัฐบาลสหรัฐอเมริกาให้ดูที่ความ[คาดหวังของสภาพแวดล้อมแหล่งที่มาสำหรับ office ๓๖๕รัฐบาลสหรัฐอเมริกา](https://docs.microsoft.com/fasttrack/us-gov-appendix-source-environment-expectations)

<strong> การรวมข้อมูลเฉพาะตัว </strong>

เรามีคำแนะนำจากระยะไกลสำหรับ:
<ul>
<li>การจัดเตรียมข้อมูลเฉพาะตัวของ active Directory ในองค์กรสำหรับการซิงโครไนซ์กับ Azure Active directory (Azure AD) รวมถึงการติดตั้งและการกำหนดค่า Azure AD Connect (แบบเดี่ยวหรือหลายฟอเรสต์) และสิทธิ์การใช้งาน (รวมถึงการอนุญาตให้ใช้งานเป็นกลุ่ม)</li>
<li>การสร้างข้อมูลประจำตัวบนระบบคลาวด์รวมถึงการนำเข้าและสิทธิ์การใช้งานจำนวนมากรวมถึงการใช้สิทธิ์การใช้งานแบบกลุ่ม</li>
<li>การเลือกและการเปิดใช้งานวิธีการรับรองความถูกต้องที่ถูกต้องสำหรับการเดินทางระบบคลาวด์ของคุณการซิงค์แฮชของรหัสผ่านการรับรองความถูกต้องแบบพาส-ทรูหรือบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS)</li>
<li>การเปิดใช้งาน AD FS สำหรับลูกค้าที่มีฟอเรสต์ของไดเรกทอรีที่ใช้งานอยู่เดียวและข้อมูลเฉพาะตัวที่ซิงโครไนซ์กับเครื่องมือการเชื่อมต่อ AD Azure สิ่งนี้จำเป็นต้องใช้ Windows Server ๒๐๑๒ R2 บริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่๒.๐หรือสูงกว่า</li>
<li>การโยกย้ายการรับรองความถูกต้องจาก AD FS ไปยัง Azure AD โดยใช้การซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องแบบพาส-ทรู</li>
<li>การโยกย้ายแอปที่รวมไว้ล่วงหน้า (เช่นแอป Azure AD gallery เวอร์ชัน-บริการ (SaaS)) จาก FS โฆษณาไปยัง Azure AD สำหรับการลงชื่อเข้าใช้ครั้งเดียว (SSO)</li>
<li>การเปิดใช้งานการรวมแอป SaaS กับ SSO จากแกลเลอรี AD Azure</li>
<li>การเปิดใช้งานการเตรียมใช้งานของผู้ใช้โดยอัตโนมัติสำหรับแอป SaaS ที่รวมไว้ล่วงหน้าตามที่แสดงอยู่ใน <a href="https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list">รายการบทช่วยสอนการรวมแอป</a> (จำกัดแอป Azure AD gallery และการจัดเตรียมขาออกเท่านั้น)  </li>
</td>

<td>  <strong>ท่าเครือข่าย </strong>  
  <br>ในฐานะที่เป็นส่วนหนึ่งของสวัสดิการ FastTrack เราขอแนะนำให้คุณปฏิบัติตามหลักปฏิบัติที่ดีที่สุดสำหรับการเชื่อมต่อกับบริการระบบคลาวด์เพื่อให้แน่ใจว่าระดับประสิทธิภาพสูงสุดของ Microsoft ๓๖๕  
  
<strong>ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่</strong> สิ่งเหล่านี้มีการตั้งค่าระดับฟอเรสต์ที่ใช้งานเป็น Windows Server ๒๐๐๓เป็นต้นไปด้วยการกำหนดค่าฟอเรสต์ต่อไปนี้:
<ul>
<li>  ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เดียว  </li>
<li>  โทโพโลยีบัญชีผู้ใช้ Active Directory และฟอเรสต์ทรัพยากร (Exchange และ/หรือ Lync ๒๐๑๐, Lync ๒๐๑๓หรือ Skype for Business)  </li>
<li>  หลายบัญชีผู้ใช้ Active Directory forests และฟอเรสต์ทรัพยากร (Exchange และ/หรือ Lync ๒๐๑๐, Lync ๒๐๑๓หรือ Skype for Business) โท  </li>
<li>  บัญชีผู้ใช้ Active Directory หลายบัญชีที่มีหนึ่งในฟอเรสต์เป็นฟอเรสต์บัญชีผู้ใช้ Active Directory แบบรวมศูนย์ที่รวม Exchange และ/หรือ Lync ๒๐๑๐, Lync ๒๐๑๓หรือ Skype for Business  </li>
<li>  หลายบัญชีผู้ใช้ Active Directory forests ที่มีองค์กร Exchange ของตนเอง  </li>
<li>  งานที่จำเป็นสำหรับการกำหนดค่าผู้เช่าและการรวมเข้ากับ Azure Active Directory ถ้าจำเป็น   </li>
</ul>
  <strong>สำคัญ</strong>  <ul>

<li>  เมื่อใช้งาน Active Directory forests หลายองค์กรที่มีหลายองค์กร Exchange ในการกำหนดค่าแบบไฮบริดของ Exchange แบบไฮบริดของผู้ใช้ที่แชร์ (UPN) ไม่ได้รับการสนับสนุน namespaces ระหว่างฟอเรสต์ต้นฉบับ คุณควรแยก SMTP namespaces หลักระหว่างองค์กรของ Exchange สำหรับข้อมูลเพิ่มเติมให้ดูที่การปรับใช้<a href="https://go.microsoft.com/fwlink/?linkid=845444">แบบไฮบริดที่มีหลายฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่</a>  </li>
<li>  สำหรับการกำหนดค่าทั้งหมดของฟอเรสต์การปรับใช้บริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) อยู่นอกขอบเขต ติดต่อ <a href="https://go.microsoft.com/fwlink/?linkid=2080150">คู่ค้าของ Microsoft</a> เพื่อขอความช่วยเหลือ  </li>
</ul></td>

<li>  การชี้ระเบียนการแลกเปลี่ยนจดหมาย (MX) ของคุณไปยัง Office ๓๖๕  </li>
<li>  การตั้งค่าฟีเจอร์ของ Office ๓๖๕ ATP ถ้าเป็นส่วนหนึ่งของบริการการสมัครใช้งานของคุณ สำหรับข้อมูลเพิ่มเติมให้ดูที่ส่วนการ <strong>ป้องกันการคุกคามขั้นสูงของ Office ๓๖๕</strong> ในตารางนี้  </li>
<li>  การตั้งค่าฟีเจอร์การป้องกันการสูญหายของข้อมูล (DLP) สำหรับโดเมนที่เปิดใช้งานจดหมายทั้งหมดที่ผ่านการตรวจสอบใน Office ๓๖๕เป็นส่วนหนึ่งของบริการการสมัครใช้งานของคุณ การดำเนินการนี้จะเสร็จสิ้นเมื่อระเบียน MX ของคุณชี้ไปยัง Office ๓๖๕  </li>
<li>  การตั้งค่าการเข้ารหัสลับข้อความของ Office ๓๖๕ (OME) สำหรับโดเมนที่เปิดใช้งานจดหมายทั้งหมดที่ผ่านการตรวจสอบใน Office ๓๖๕เป็นส่วนหนึ่งของบริการการสมัครใช้งานของคุณ การดำเนินการนี้จะเสร็จสิ้นเมื่อระเบียน MX ของคุณชี้ไปยัง Office ๓๖๕  </li>
</ul>
  <strong>หมายเหตุ:</strong> บริการการจำลองแบบกล่องจดหมาย (MRS) พยายามโยกย้ายอีเมลที่มีการจัดการสิทธิ์ในข้อมูล (IRM) จากกล่องจดหมายภายในองค์กรของคุณไปยังกล่องจดหมาย Exchange Online ที่สอดคล้องกัน ความสามารถในการอ่านเนื้อหาที่ได้รับการป้องกันหลังการโยกย้ายจะขึ้นอยู่กับการแมปลูกค้าและการคัดลอกเทมเพลตที่มีการจัดการสิทธิ์ของไดเรกทอรีที่ใช้งานอยู่ (AD RMS) ไปยังบริการการจัดการสิทธิ์ของ Azure (Azure RMS)  
<ul>
