# <a name="products-and-capabilities"></a>ผลิตภัณฑ์และความสามารถ

## <a name="services-and-scenarios-supported-by-fasttrack"></a>บริการและสถานการณ์สมมติที่ได้รับการสนับสนุนโดย FastTrack 

หัวข้อนี้มีรายละเอียดเกี่ยวกับสถานการณ์ปริมาณงานที่สนับสนุนโดย FastTrack และความคาดหวังของสภาพแวดล้อมต้นทางที่จําเป็นก่อนที่เราจะสามารถเริ่มต้นได้ ตามการตั้งค่าปัจจุบันของคุณ เราจะร่วมงานกับคุณเพื่อสร้างแผนการแก้ไขที่ปรับสภาพแวดล้อมแหล่งข้อมูลของคุณขึ้นมาเป็นความต้องการขั้นต่สุดเพื่อออนบอร์ดที่ประสบความสาเร็จ

FastTrack มอบแนวทางเพื่อช่วยให้คุณมีความสามารถหลัก (ทั่วไปสําหรับบริการ Microsoft Online ทั้งหมด) ก่อน จากนั้นด้วยการออนบอร์ดแต่ละบริการที่มีสิทธิ์:

  - [ทั่วไป](#general)
  - [ความปลอดภัยและความสอดคล้อง](#security-and-compliance)
  - [Office 365](#office-365)
  - [Enterprise Mobility + Security](#enterprise-mobility--security)
  - [Windows 10](#windows-10)
  - [เดสก์ท็อปเสมือนของ Windows](#windows-virtual-desktop)
  - [App Assure](#app-assure)
  - [Microsoft Edge](#microsoft-edge)

> [!NOTE]
> For information on source environment expectations for Office 365 US Government, see [Source Environment Expectations for Office 365 US Government](https://docs.microsoft.com/fasttrack/us-gov-appendix-source-environment-expectations).

</li>

เรามีแนวทางระยะไกลสําหรับ:
<ul>
<li>การเตรียมข้อมูลเฉพาะตัว Active Directory ภายในองค์กรเพื่อซิงโครไนซ์กับ Azure Active Directory (Azure AD) รวมถึงการติดตั้งและการกําหนดค่า Azure AD Connect (ฟอเรสต์เดียวหรือหลายฟอเรสต์) และสิทธิ์การใช้งาน (รวมถึงสิทธิ์การใช้งานแบบกลุ่ม)</li>
<li>การสร้างข้อมูลเฉพาะตัวบนระบบคลาวด์ รวมถึงการนําเข้าและการให้สิทธิ์การใช้งานเป็นกลุ่ม รวมถึงการใช้สิทธิ์การใช้งานแบบกลุ่ม</li>
<li>การเลือกและเปิดใช้งานวิธีการรับรองความถูกต้องที่ถูกต้องการเดินทางของคุณบนระบบคลาวด์ การซิงค์แฮชของรหัสผ่านการรับรองความถูกต้องแบบพาส-ผ่าน หรือบริการการติดต่อกับภายนอกของ Active Directory (AD FS)</li>
<li> การเลือกและเปิดใช้งานประสบการณ์การรับรองความถูกต้องที่สะดวกมากขึ้นให้กับผู้ใช้ของคุณด้วยการรับรองความถูกต้องที่ไม่มีรหัสผ่าน (Fast Identity Online (FIDO)2 หรือแอป Microsoft Authenticator)</li>
<li>การเปิดใช้งาน AD FS ของลูกค้าที่มีฟอเรสต์ Active Directory เดี่ยวและข้อมูลเฉพาะตัวที่ซิงโครไนซ์กับเครื่องมือ Azure AD Connect ซึ่งต้องใช้Windows Server 2012 R2 Active Directory Federation Services 2.0 หรือใหม่กว่า</li>
<li>การโยกย้ายการรับรองความถูกต้องจาก AD FS ไปยัง Azure AD โดยใช้การซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องแบบพาส-ผ่าน</li>
<li>การโยกย้ายแอปที่รวมไว้ล่วงหน้า (เช่น แอปแกลเลอรี Azure AD แบบบริการเดียว (SaaS) จาก AD FS ไปยัง Azure AD เพื่อลงชื่อเข้าใช้ครั้งเดียว (SSO)</li>
<li>การเปิดใช้งานการรวมแอป SaaS กับ SSO จากแกลเลอรี Azure AD</li>
<li>การเปิดใช้งานการเตรียมใช้งานผู้ใช้โดยอัตโนมัติกับแอป SaaS ที่รวมไว้ล่วงหน้าตามรายการบทช่วยสอนการรวมแอป <a href="https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list">(จํากัด</a> เฉพาะแอป SaaS แกลเลอรี Azure AD และการเตรียมใช้งานขาออกเท่านั้น)  </li>
</td>

<td>  <strong>การเปิดใช้งานเครือข่าย </strong>  
  <br>ในฐานะที่เป็นส่วนหนึ่งของสิทธิประโยชน์ของ FastTrack เราขอแนะนวลแนวทางปฏิบัติที่ดีที่สุดเกี่ยวกับการเชื่อมต่อกับบริการระบบคลาวด์เพื่อให้แน่ใจว่า Microsoft 365 จะมีประสิทธิภาพสูงสุด  
  
<strong>ฟอเรสต์ Active Directory</strong> ระดับฟอเรสต์ฟังก์ชันที่ตั้งค่าเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:
<ul>
<li>  ฟอเรสต์ Active Directory เดี่ยว  </li>
<li>  บัญชีผู้ใช้ Active Directory ฟอเรสต์เดียวและฟอเรสต์ทรัพยากร (การpologies ของบัญชีผู้ใช้ Active Directory และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business)  </li>
<li>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์และฟอเรสต์ทรัพยากร (การปลอมแปลง Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business)  </li>
<li>  หลายฟอเรสต์บัญชีผู้ใช้ Active Directory กับหนึ่งในฟอเรสต์เป็นฟอเรสต์บัญชีผู้ใช้ Active Directory ส่วนกลางที่มี Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business  </li>
<li>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์ แต่ละฟอเรสต์มี Exchange องค์กรของตนเอง  </li>
<li>  งานที่ต้องใช้ในการกําหนดค่าและการรวมกับ Azure Active Directory ถ้าเป็นที่ต้องการ   </li>
</ul>
  <strong>สําคัญ</strong>  <ul>
<li>  For multi-forest Active Directory scenarios, if Lync 2010, Lync 2013, or Skype for Business is deployed, it must be deployed in the same Active Directory forest as Exchange.  </li>
<li>  เมื่อปรับใช้ฟอเรสต์ Active Directory หลายฟอเรสต์กับองค์กร Exchange หลายองค์กรในการกําหนดค่าไฮบริดหลายแบบ Exchange Name หลักของผู้ใช้ที่แชร์ (UPN) ระหว่างฟอเรสต์ต้นทางไม่ได้รับการสนับสนุน Namespace SMTP หลักระหว่างองค์กร Exchange ควรแยกออกจากกันด้วย For more information, see <a href="https://go.microsoft.com/fwlink/?linkid=845444">Hybrid deployments with multiple Active Directory forests</a>.  </li>
<li>  การกําหนดค่าฟอเรสต์หลายรายการ การปรับใช้ Active Directory Federation Services (AD FS) อยู่นอกขอบเขต ติดต่อ <a href="https://go.microsoft.com/fwlink/?linkid=2080150">คู่ค้า Microsoft</a> เพื่อขอรับความช่วยเหลือเกี่ยวกับเรื่องนี้  </li>
</ul></td>

<td><strong>Azure Active Directory (Azure AD) และ Azure AD Premium</strong></td>
<td>  เรามีแนวทางระยะไกลสําหรับการรักษาความปลอดภัยข้อมูลเฉพาะตัวบนระบบคลาวด์ของคุณสําหรับสถานการณ์ต่อไปนี้  

 <br/>

<strong>โครงสร้างพื้นฐาน Secure Foundation</strong>  </ul>
<ul>
<li>  การกําหนดค่าและการเปิดใช้งานการรับรองความถูกต้องที่คาดเดาได้ยากต่อข้อมูลเฉพาะตัวของคุณ รวมถึงการป้องกันด้วยการรับรองความถูกต้องโดยใช้หลายปัจจัยของ Azure (MFA) (เฉพาะระบบคลาวด์เท่านั้น) แอป Microsoft Authenticator และการลงทะเบียนรวมของ Azure MFA และการรีเซ็ตรหัสผ่านแบบบริการตนเอง (SSPR)  </li>
<li> การปรับใช้แอป FIDO2 หรือ Microsoft Authenticator </li>
<li>  สําหรับลูกค้าที่ไม่ใช่ Azure AD Premium จะมีแนวทางในการรักษาความปลอดภัยข้อมูลเฉพาะตัวของคุณโดยใช้ค่าเริ่มต้นด้านความปลอดภัย  </li>
<li>  สําหรับลูกค้าระดับพรีเมียมของ Azure AD จะมีแนวทางในการรักษาความปลอดภัยข้อมูลเฉพาะตัวของคุณด้วยการเข้าถึงตามเงื่อนไข  </li>
