# <a name="appendix-a---migration-from-ibm-domino-to-exchange-online"></a>ภาคผนวก A - การโยกย้ายจาก IBM Domino ไปยัง Exchange Online

การย้ายข้อมูลจาก IBM Domino เพื่อแลกเปลี่ยนแบบออนไลน์รวมถึงหลายประเด็นที่สําคัญ รวมถึงสิ่งที่เกิดขึ้นระหว่างขั้นตอนต่อไปนี้: 
- [เริ่มต้นขั้นตอน](#initiate-phase)   
- [ประเมินระยะ](#assess-phase)
- [ขั้นตอนการแก้ไข](#remediate-phase)  
- [เปิดใช้งานเฟส](#enable-phase)  
- [โยกย้ายเฟส](#migrate-phase)
    
## <a name="identities"></a>Identities

คุณมีหน้าที่รับผิดชอบในการสร้างและการจัดการข้อมูลเฉพาะตัว (เมฆเท่านั้น คุณต้องทําการแมปของข้อมูลประจําตัว (ถ้าไม่มีอยู่) ระหว่าง Domino และไดเรกทอรีที่ใช้งานอยู่ในสถานที่หรือไดเรกทอรีที่ใช้งานอยู่ของ Azure ในช่วงระยะแรกของการออนบอร์ด
  
## <a name="coexistence"></a>การอยู่ร่วมกัน

ประโยชน์จากศูนย์ FastTrack สําหรับ Office 365 จัดเตรียมลําดับจดหมายแบบสองทิศทางระหว่างสภาพแวดล้อม Domino ในสถานที่และ Exchange Online ให้กับลูกค้าทั้งหมด
  
## <a name="migration"></a>โยกย้าย

กระบวนการมาตรฐานของศูนย์ FastTrack สําหรับโยกย้ายจาก Domino การแลกเปลี่ยนแบบออนไลน์เกี่ยวข้องกับการจัดเตรียมข้อมูล Domino ก่อนไปยัง Azure ก่อนที่จะโยกย้ายไปยังกล่องจดหมาย Exchange แบบออนไลน์ การย้ายข้อมูล FastTrack จําเป็นต้องมีกิจกรรมต่างๆ ที่จะดําเนินการในขั้นตอนต่างๆ ของการดูแลโดยบุคลากรของ FastTrack Center และคุณ เราครอบคลุมกิจกรรมเหล่านี้ในส่วนต่อไปนี้
  
> [!NOTE]
> ข้อมูลที่โยกย้ายผ่านบริการ FastTrack อาจถูกถ่ายโอนไปยัง จัดเก็บ และประมวลผลในสหรัฐอเมริกา หรือที่ใดก็ตามที่ Microsoft เก็บรักษาสิ่งอํานวยความสะดวกไว้ บริการ FastTrack ไม่ได้ออกแบบหรือจัดทําขึ้นเพื่อให้สอดคล้องกับข้อกําหนดทางกฎหมายหรือข้อบังคับพิเศษ 
  
## <a name="initiate-phase"></a>เริ่มต้นขั้นตอน

 **การดําเนินการที่สําคัญ**
  
- ระบุ Domino เป็นแพลตฟอร์มจดหมายต้นทาง   
- ตรวจสอบว่า ศูนย์ FastTrack ดําเนินการโยกย้าย
    
 **ความรับผิดชอบของลูกค้า**
  
- ให้ข้อมูลพื้นฐานเกี่ยวกับแพลตฟอร์มการส่งข้อความต้นทาง และยืนยันเจตนาการย้ายข้อมูลกับศูนย์ FastTrack 
- เข้าร่วมในการฝึกปฏิบัติของกระบวนการสิทธิประโยชน์ของศูนย์ FastTrack  
- ลงนามในข้อตกลงบริการ FastTrack
    
## <a name="assess-phase"></a>ประเมินระยะ

 **การดําเนินการที่สําคัญ**
  
- ศูนย์ FastTrack ดําเนินการฝึกอบรมเชิงปฏิบัติการการย้ายกับลูกค้า 
- คุณกรอกข้อมูลข้อกําหนดเบื้องต้นของการโยกย้าย เช่น แบบสอบถามการย้ายและการเตรียมใช้งานเวิร์กสเตชันของผู้ดูแลระบบ    
- การประเมินการโยกย้ายสําหรับ Domino จะดําเนินการในสภาพแวดล้อมในสถานที่ของคุณ
    
 **ความรับผิดชอบของลูกค้า**
  
- เตรียมใช้งานเวิร์กสเตชันผู้ดูแลระบบที่ FastTrack Center ใช้ดูแลงานการเริ่มทํางานและการโยกย้าย เช่น การประเมิน การสร้างแบบจําลอง การตรวจสอบ การตั้งค่าการส่งต่อระหว่างการย้าย เป็นต้น
    > [!NOTE]
    > การประเมินเป็นสิ่งสําคัญสําหรับการวางแผนที่ประสบความสําเร็จและการดําเนินการอพยพความเร็ว โดยสถาปนิกการย้ายถิ่นที่ต้องการการเข้าถึงสภาพแวดล้อมของ Domino โดยเฉพาะ คอมโพเนนต์ของเวิร์กสเตชันของผู้ดูแลระบบที่จําเป็นต้องมีไคลเอ็นต์ของ Notes ที่กําหนดค่าให้เข้าถึงเซิร์ฟเวอร์จดหมาย Domino ต้นทางทั้งหมดและเซิร์ฟเวอร์การจัดเตรียมแบบจําลอง Azure Domino 
- ให้ทีมโยกย้ายเข้าถึงเวิร์กสเตชันของผู้ดูแลระบบ บัญชี และสิทธิ์สําหรับการดําเนินการประเมินและการย้ายข้อมูล ซึ่งรวมถึงการเตรียมใช้งานหลายบัญชีในสถาน และ ใน Exchange แบบออนไลน์ที่มีสิทธิ์ระดับผู้ดูแลที่จําเป็นสําหรับการโยกย้าย    
- เปิดพอร์ตไฟร์วอลล์ พอร์ตขาออกต้องถูกเปิดระหว่างเซิร์ฟเวอร์จดหมาย Domino ต้นทางและเซิร์ฟเวอร์การจัดเตรียม Azure พอร์ตอื่น ๆ สําหรับการสื่อสาร (เช่น เวิร์กสเตชันของผู้ดูแลระบบ เซิร์ฟเวอร์ Domino ต้นทาง และเซิร์ฟเวอร์ Exchange ในสถานที่ (ถ้ามี)) ต้องเปิดด้วย 
- เปิดใช้งานการรับรองข้ามระหว่างสภาพแวดล้อม Domino ต้นทางและเซิร์ฟเวอร์จัดเตรียม Azure Domino เพื่ออํานวยความสะดวกในการจําลองแบบ งานการรับรองข้ามจะประสานงานระหว่างผู้ดูแลระบบ Domino ของลูกค้าและศูนย์ FastTrack  
- กรอกข้อมูลแบบสอบถามการย้ายซึ่งรวบรวมข้อมูลที่จําเป็นเพื่อกําหนดค่าสภาพแวดล้อมการโยกย้ายใน Azure (เช่น เครื่องมือ สคริปต์ และเซิร์ฟเวอร์การย้าย)   
- ตรวจสอบให้แน่ใจว่ากล่องจดหมายเป้าหมายใน Office 365 มีการเปิดใช้งานโพรโทคอลอินเทอร์เฟซโปรแกรมโปรแกรมประยุกต์ (MAPI) การส่งข้อความแอพลิเคชัน (MAPI)  
> [!NOTE]
> แผน Office 365 บางแผนไม่สนับสนุนโพรโทคอล MAPI กล่องจดหมายจากแผนเหล่านี้จําเป็นต้องแปลงเป็นแผนที่สนับสนุน MAPI ล่วงหน้าของเหตุการณ์การโยกย้าย แผนการเหล่านี้สามารถเปลี่ยนแปลงได้ 
  
## <a name="remediate-phase"></a>ขั้นตอนการแก้ไข

 **การดําเนินการที่สําคัญ**
  
- ศูนย์ FastTrack จะตรวจสอบรายงานการประเมินการโยกย้าย และทดสอบรายละเอียดที่คุณส่งโดยใช้แบบสอบถาม   
- รายการแก้ไขที่แนะนําโดยศูนย์ FastTrack จะต้องดําเนินการโดยคุณ
    
 **ความรับผิดชอบของลูกค้า**
  
- แก้ไขสภาพแวดล้อม Domino ตามแนวทางที่ศูนย์ FastTrack จัดเตรียม (ตัวอย่างเช่น การตั้งค่าสิทธิ์ที่จําเป็นใดๆ ที่ระบุว่าไม่มีในไฟล์เมล)  
- ตรวจสอบให้แน่ใจว่ากล่องจดหมาย Domino มีขนาดต่ํากว่าขนาดสูงสุดที่อนุญาตผ่านการย้าย
    > [!NOTE]
    >  แม้ว่า FastTrack จะโยกย้ายกล่องจดหมายได้ถึง 85% ของขนาดเป้าหมายทั้งหมดที่อนุญาต แต่การพยายามโยกย้ายกล่องจดหมายที่มีขนาดใหญ่กว่า 2 GB มีความเสี่ยงเพิ่มเติมเช่น    <br/> ระยะเวลาที่ยาวขึ้นของการย้ายข้อมูล    <br/> การใช้ทรัพยากรอื่นที่ใช้สําหรับการโยกย้ายกล่องจดหมายอื่น    <br/> อัตราความผิดพลาดที่เพิ่มขึ้นอย่างมาก 
- จัดเตรียมฐานข้อมูลเมลเข้าและรายการควบคุมการเข้าถึง (ACL) สําหรับการโยกย้าย คุณต้องทําตามขั้นตอนบางอย่างในการแก้ไขเพื่อโยกย้ายฐานข้อมูลจดหมายในและสิทธิ์ของตนเองไปยังกล่องจดหมายที่ใช้ร่วมกันในการแลกเปลี่ยนแบบออนไลน์เสร็จเรียบร้อยแล้ว ขั้นตอนเหล่านี้มีดังต่อไปนี้: 
  - ลบรายการฐานข้อมูลที่มีอยู่ในไดเรกทอรี Domino และสร้างเรกคอร์ดบุคคลใหม่
  - สร้างกลุ่มความปลอดภัยสากลเปิดใช้งานจดหมายในไดเรกทอรีที่ใช้งานอยู่ในสถานที่ที่มีการซิงโครไนส์กับ Office 365 Azure ไดเรกทอรีที่ใช้งานอยู่ และใช้ในการกําหนดค่าสิทธิ์บนกล่องจดหมายที่ใช้ร่วมกันในการแลกเปลี่ยนแบบออนไลน์ นี้ถ่ายโอนสิทธิ์ที่ตั้งค่าบนฐานข้อมูลจดหมายในไปยังกล่องจดหมายที่ใช้ร่วมกันในการแลกเปลี่ยนแบบออนไลน์
    
> [!NOTE]
> ความพร้อมและการฝึกอบรมของผู้ใช้สําหรับระบบการส่งข้อความใหม่และไคลเอนต์สามารถเริ่มต้นได้ในขณะนี้ 
  
## <a name="enable-phase"></a>เปิดใช้งานเฟส

 **การดําเนินการที่สําคัญ**
  
- ศูนย์ FastTrack: 
    - ตั้งค่าสภาพแวดล้อมการโยกย้ายใน Azure  
    - กําหนดค่าเครื่องมือการโยกย้ายบนเวิร์กสเตชันของผู้ดูแลระบบในสถานที่ 
    - กําหนดค่าและสาธิตวิธีการใช้เครื่องมือ Auto-Import  
    - ดําเนินการตรวจสอบคอมโพเนนต์การย้ายทั้งหมดและดําเนินการโยกย้ายทดสอบ
    
 **ความรับผิดชอบของลูกค้า**
  
- บุคลากรของคุณรับผิดชอบการจัดกําหนดการการย้ายกล่องจดหมายต้องเข้าใจวิธีการใช้เครื่องมือ Auto-Import คุณสามารถใช้เครื่องมือนี้เพื่อนําเข้ากําหนดการโยกย้ายลงในฐานข้อมูลการจัดกําหนดการที่ศูนย์ FastTrack ใช้เพื่อทํากิจกรรมก่อนการโยกย้าย 
- ดําเนินการกิจกรรมก่อนการย้ายข้อมูล เช่น การนําเข้ากําหนดการของผู้ใช้ วิเคราะห์รายงานการตรวจสอบ
    
กิจกรรมก่อนการโอนย้ายจะประสานงานระหว่างคุณกับศูนย์ FastTrack เริ่มต้นการจําลองแบบ Azure หลังจากที่มีการนําเข้ากําหนดการการย้ายผู้ใช้ 
    
> [!NOTE]
> เวลาที่ใช้ในการทําซ้ําขึ้นอยู่กับจํานวนข้อมูล จากนั้น FastTrack Center จะทําการตรวจสอบเพื่อตรวจสอบความพร้อมในการย้าย ผลการตรวจสอบมีให้คุณด้วยความเข้าใจว่าการแก้ไขในภายหลังเป็นสิ่งจําเป็นตามปกติ ขั้นตอนเหล่านี้ทั้งหมดเรียกว่ากิจกรรม "T ลบ" เนื่องจากต้องเริ่มต้นการโยกย้ายตามกําหนดการของผู้ใช้ 
  
## <a name="migrate-phase"></a>โยกย้ายเฟส

 **การดําเนินการที่สําคัญ**
  
- ศูนย์ FastTrack:
    - ดําเนินการย้ายถิ่นฐานและความเร็ว  
    - ดําเนินการเหตุการณ์การย้ายและกิจกรรม T-ลบ
    - ให้ความช่วยเหลือหลังการย้าย
    
 **ความรับผิดชอบของลูกค้า**
  
- ระบุกําหนดการในวิธีการที่กําหนดและรายการข้อมูลผู้ใช้เฉพาะเพื่อโยกย้ายสําหรับแต่ละเหตุการณ์การโยกย้าย
    > [!NOTE]
    > งานนี้มีความสําคัญเนื่องจากกิจกรรมก่อนโยกย้ายเกี่ยวข้องกับการแก้ไขและลองใหม่ที่เป็นไปได้ของการสร้างแบบจําลองในขั้นตอนที่แตกต่างกันก่อนวันจริงโยกย้าย (T-0) ในขณะที่กล่องจดหมายบางอย่างกําลังโยกย้าย กิจกรรม T-ลบกําลังดําเนินการกับผู้อื่น นี้ทําให้การวางแผนที่เหมาะสมและการประสานงานต้อง. 
- แก้ไขปัญหาที่ระบุในระหว่างกิจกรรม T ลบ
- แก้ไขปัญหาเซิร์ฟเวอร์ Domino ที่ส่งผลกระทบต่อกิจกรรมการย้าย 
- ดําเนินการสื่อสารของผู้ใช้ปลายทางเกี่ยวกับวันที่การย้ายข้อมูลที่กําลังจะมาถึง
- ดําเนินการกิจกรรมและการฝึกอบรมความพร้อมของผู้ใช้สําหรับระบบการส่งข้อความและลูกค้าใหม่   
- ระบุและรายงานปัญหาหลังการโยกย้าย ศูนย์ FastTrack จะให้ความช่วยเหลือหลังการย้ายข้อมูลจนกว่าจะ T+5 วันหลังจากการย้ายข้อมูล ซึ่งจะกลายเป็นความรับผิดชอบของคุณ คุณสามารถบันทึกตั๋วหลังการโยกย้ายสําหรับปัญหาต่างๆ เช่น อีเมล รายการปฏิทิน และที่ติดต่อที่ขาดหายไป หรือรายการที่ซ้ํากันในกล่องจดหมาย
    
ศูนย์ FastTrack ไม่ครอบคลุมการปรับใช้ ค่าธรรมเนียมสิทธิ์การใช้งาน หรือความช่วยเหลือที่เกี่ยวข้องกับการเตรียมไดเรกทอรี (รวมถึงการซิงโครไนส์ไดเรกทอรี Domino-to-Active Directory) Add-ons ซอฟต์แวร์การมีอยู่ร่วมกันสําหรับการทํางานร่วมกันของแอปพลิเคชัน Notes การโยกย้ายแบบบริการตนเอง หรือการเก็บถาวร
