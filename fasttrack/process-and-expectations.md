# <a name="process-and-expectations"></a>กระบวนการและความคาดหวัง

ในหัวข้อนี้เราจะครอบคลุมวิธีการมีส่วนร่วมในการให้ความช่วยเหลือในการให้ความช่วยเหลือ FastTrack การให้คำแนะนำเคล็ดลับสำหรับประสบการณ์การใช้งานที่ประสบความสำเร็จและทำความเข้าใจเกี่ยวกับความรับผิดชอบของหลักของทั้ง FastTrack และบริษัทของคุณ

## <a name="engaging-fasttrack"></a>การ FastTrack ที่น่าสนใจ

การรับความช่วยเหลือจากผู้เชี่ยวชาญ FastTrack  
  
มีสองสถานที่ที่คุณสามารถร้องขอความช่วยเหลือจากผู้เชี่ยวชาญ FastTrack:

  - ผ่านทางศูนย์การจัดการ Microsoft –ผู้ดูแลระบบผู้เช่าของคุณจะลงชื่อเข้าใช้ในศูนย์การจัดการ Microsoft แล้วคลิกที่ **ต้องการความช่วยเหลือ** วิดเจ็ตจากนั้นจึงเดินผ่านขั้นตอนการร้องขอ
  - ผ่านไซต์ FastTrack –คุณหรือคู่ค้าของคุณร้องขอความช่วยเหลือสำหรับบริษัทของคุณผ่านทางไซต์ FastTrack

เมื่อต้องการร้องขอความช่วยเหลือโดยตรงผ่านทางไซต์ FastTrack:

1.  ลงชื่อเข้าใช้[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2.  เลือก **ร้องขอความช่วยเหลือเกี่ยวกับ Microsoft ๓๖๕** จากการ **ดำเนินการด่วน** ที่ด้านบนของหน้า landing page ของคุณ
3.  ทำการ **ร้องขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วย Microsoft ๓๖๕** ฟอร์ม

เมื่อต้องการให้คู่ค้าของคุณร้องขอความช่วยเหลือในนามของคุณผ่านทางไซต์ FastTrack ให้คู่ค้าของคุณ:

1.  ลงชื่อเข้าใช้[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2.  เลือก **ร้องขอความช่วยเหลือเกี่ยวกับ Microsoft ๓๖๕** จากการ **ดำเนินการด่วน** ที่ด้านบนของหน้า landing page ของคุณ
3.  ค้นหาลูกค้าของคุณโดยการใส่ชื่อลูกค้าโดเมนหรือ TPID
4.  เลือกลูกค้าของคุณจากผลลัพธ์การค้นหา
5.  ทำการ **ร้องขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วย Microsoft ๓๖๕** ฟอร์ม

เมื่อต้องการร้องขอความช่วยเหลือสำหรับแอปให้ทำตาม [คำขอบริ](https://go.microsoft.com/fwlink/?linkid=2022721)การให้เสร็จสมบูรณ์

## <a name="fasttrack-expectations"></a>ความคาดหวัง FastTrack

FastTrack จะให้คำแนะนำแบบเป็นจุดสิ้นสุดเกี่ยวกับการย้ายไปยังและใช้ Microsoft ๓๖๕ คุณจะได้รับคำแนะนำจากระยะไกลและหลักปฏิบัติที่ดีที่สุดในการวางแผนไวร์ที่ประสบความสำเร็จ

### <a name="meeting-rhythms"></a>จังหวะการประชุม

คุณทำงานจากระยะไกลกับผู้เชี่ยวชาญ FastTrack ผ่านทางการประชุมทางเทคนิคการประชุมเชิงปฏิบัติการและจุดตรวจสอบรวมถึง:

  - การ **บรรยายทางเทคนิคและการประชุมเชิงปฏิบัติการ:** สิ่งเหล่านี้จะให้ชื่อเรื่องดำลึกและแนวทางปฏิบัติที่ดีที่สุดในการเร่งการปรับใช้ Microsoft ๓๖๕ services ของคุณ การประชุมเหล่านี้มักจะให้ภาพรวมของเอกสารประกอบเครื่องมือซอฟต์แวร์และคอนโซลการดูแลระบบที่มีให้ใช้งาน ระยะเวลามาตรฐาน:๖๐นาที
  - การ **ประชุมจุดตรวจสอบ:** รายสัปดาห์หรืออาทิตย์สัมผัสเพื่อให้แน่ใจว่าการปรับใช้ของคุณกำลังคืบหน้า ระยะเวลามาตรฐาน:30 นาที

### <a name="expected-outcomes-and-key-milestones"></a>ผลลัพธ์ที่คาดไว้และเหตุการณ์สำคัญ 

FastTrack มีแนวทางปฏิบัติที่ดีที่สุดในการปรับใช้และแนวทางปฏิบัติที่ดีที่สุดเพื่อให้การใช้งาน Microsoft ๓๖๕ขององค์กรของคุณเป็นไปตามลำดับ

 **สิ่งที่คุณสามารถคาดหวังจาก FastTrack:**

  - เราทำงานร่วมกับคุณเพื่อสร้างแผนเชิงปฏิบัติที่จะนำสภาพแวดล้อมต้นฉบับของคุณไปใช้กับความต้องการขั้นต่ำสำหรับการปฐมนิเทศและการโยกย้ายที่ประสบความสำเร็จ
  - เรามีคำแนะนำในการสร้างรากฐานที่มั่นคงสำหรับการปฐมนิเทศหลักการรวมข้อมูลประจำตัวและกิจกรรมท่าบริการ

**การมีส่วนร่วมในการดำเนินการทั่วไปได้แก่:**

  - การประชุมคิก–ทำความเข้าใจเกี่ยวกับกระบวนการกำหนดการเครื่องมือและกิจกรรมของทีม
  - การประเมิน–การพัฒนาการตรวจสอบด้าน
  - การอภิปรายเกี่ยวกับความปลอดภัย–วางแผนกลยุทธ์ด้านความปลอดภัยตามหลักปฏิบัติที่ดีที่สุดของ Microsoft security ที่แนะนำ
  - การรีวิวเกี่ยวกับความคิดเห็น–ตรวจทานรายการใดๆ
  - การประชุมเชิงปฏิบัติการความสำเร็จ–เครื่องมือแม่แบบและเส้นบอกแนวสำหรับการสร้างแผนการดำเนินการ
  - การประชุมจุดตรวจสอบ–นาทีการประชุมและจัดกำหนดการ
  - เปิดใช้งานกิจกรรม–ท่าบริการที่เสร็จสมบูรณ์
  - การโยกย้ายการประชุมเชิงปฏิบัติการความเร็ว (ไม่บังคับ-Microsoft ๓๖๕เท่านั้น) –ความพร้อมในการโยกย้าย

สำหรับข้อมูลเพิ่มเติมให้ดูที่[เรื่องราวความสำเร็จของลูกค้าของ Microsoft ๓๖๕](https://www.microsoft.com/microsoft-365/customer-stories?rtc=1)

### <a name="customer-success-tips"></a>เคล็ดลับความสำเร็จของลูกค้า

การเตรียมตัวของคุณสำหรับความช่วยเหลือของ FastTrack มีความสำคัญต่อกระบวนการที่ราบรื่นและเรียบร้อยแล้ว ก่อนที่จะมีส่วนร่วมกับ FastTrack ให้ตรวจสอบให้แน่ใจว่าได้สร้าง:

  - วันที่ที่ต้องการ (เมื่อคุณวางแผนที่จะปรับใช้บริการ)
  - จุดประสงค์ (บริการที่คุณวางแผนที่จะเปิดใช้)
  - เหตุผลทางธุรกิจ (เหตุผลที่คุณวางแผนที่จะเปิดบริการให้กับผู้ใช้)
  - เจ้าของความสำเร็จ (ผู้ที่รับผิดชอบความสำเร็จของโครงการ)

เรายังขอแนะนำให้คุณพิจารณาและระบุสิ่งต่อไปนี้:

  - ผู้มีส่วนได้เสีย-การให้การสนับสนุนและการสนับสนุนของผู้สนับสนุนระดับผู้บริหารและแชมเปี้ยน
  - สถานการณ์สมมติ–การกำหนดลำดับความสำคัญและข้อกำหนดของเกณฑ์ความสำเร็จ
  - การรับรู้–การดำเนินการของแคมเปญการสื่อสารและการวางแผนเหตุการณ์การเปิดตัว
  - การฝึกอบรม–การให้ความรู้แก่ผู้ใช้และทรัพยากร helpdesk readying

## <a name="fasttrack-and-customer-responsibilities"></a>FastTrack และความรับผิดชอบของลูกค้า 

ความรับผิดชอบที่ชัดเจนสำหรับทุกฝ่ายเป็นสิ่งจำเป็นสำหรับการปฐมนิเทศที่ราบรื่นไปยัง Microsoft ๓๖๕ ส่วนนี้จะสรุปความรับผิดชอบเหล่านั้นโดยยึดตามขั้นตอนต่างๆของการมีส่วนร่วมของ FastTrack FastTrack เป็นผู้รับผิดชอบในการให้สิทธิ์เข้าถึงผู้เชี่ยวชาญ FastTrack ตลอดระยะเวลาการสมัครใช้งานของคุณเพื่อให้มีสิทธิ์ในการใช้งาน Microsoft 365 ความคล่องตัวขององค์กร & ความปลอดภัย (EMS) และ Windows 10 service ผู้เชี่ยวชาญเหล่านี้ให้การสนับสนุนระยะไกลในช่วงเวลาทำการปกติที่ศูนย์ FastTrack อยู่และดำเนินการการโยกย้ายอีเมลและไฟล์ที่ได้รับการสนับสนุน (๕๐๐ที่นั่งหรืออื่นๆ) การสนับสนุนที่แนะนำนี้มีหลักปฏิบัติที่ดีที่สุดในการเข้าถึงแหล่งข้อมูลแบบออนไลน์และเครื่องมือที่ให้บริการด้วยตนเอง ลูกค้าหรือคู่ค้าที่ทำงานในนามของพวกเขามีหน้าที่รับผิดชอบในการจัดการโครงการการดำเนินการและการกำหนดค่าแบบมือถือการดำเนินการการจัดการการเปลี่ยนแปลงและการเพิ่มประสิทธิภาพของกระบวนการความพร้อมในไซต์และการฝึกอบรม ตารางต่อไปนี้มีรายการที่ครอบคลุมของความรับผิดชอบเหล่านี้
