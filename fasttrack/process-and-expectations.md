# <a name="process-and-expectations"></a>กระบวนการและความคาดหวัง

ในหัวข้อนี้ เราจะครอบคลุมวิธีการมีส่วนร่วมกับ FastTrack เพื่อขอความช่วยเหลือ วิธีที่ความช่วยเหลือนั้นจัดหา ให้แนะนะเคล็ดลับต่างๆ เพื่อประสบการณ์การใช้งานที่ประสบความสาเร็จ และเข้าใจความรับผิดชอบที่หลักๆ ของทั้ง FastTrack และบริษัทของคุณ

## <a name="engaging-fasttrack"></a>การมีส่วนร่วมใน FastTrack

รับความช่วยเหลือจากผู้เชี่ยวชาญด้าน FastTrack  
  
มีสองที่ที่คุณสามารถขอความช่วยเหลือจากผู้เชี่ยวชาญด้าน FastTrack:

  - ผ่านทางศูนย์การจัดการ Microsoft – ผู้ดูแลระบบผู้เช่าของคุณลงชื่อเข้าใช้ในศูนย์การจัดการ Microsoft แล้วคลิกวิดเจ็ต **ต้องการ** ความช่วยเหลือหรือไม่ จากนั้นจะผ่านขั้นตอนการร้องขอ
  - ผ่านทางไซต์ FastTrack - คุณหรือคู่ค้าของคุณขอความช่วยเหลือจากบริษัทของคุณผ่านไซต์ FastTrack

เมื่อต้องการขอความช่วยเหลือโดยตรงผ่านทางไซต์ FastTrack:

1.  ลงชื่อเข้าใช้ไซต์[FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2.  เลือก **ขอความช่วยเหลือเกี่ยวกับ Microsoft 365** จากการ **แอคชัน** ด่วนที่ด้านบนของเพจเริ่มต้นของคุณ
3.  กรอกฟอร์ม **ขอความช่วยเหลือเกี่ยวกับ Microsoft 365** ให้สมบูรณ์

เมื่อต้องการให้คู่ค้าของคุณขอความช่วยเหลือในนามของคุณผ่านไซต์ FastTrack ให้ให้คู่ค้าของคุณ:

1.  ลงชื่อเข้าใช้ไซต์[FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2.  เลือก **ขอความช่วยเหลือเกี่ยวกับ Microsoft 365** จากการ **แอคชัน** ด่วนที่ด้านบนของเพจเริ่มต้นของคุณ
3.  ค้นหาลูกค้าของคุณโดยการใส่ชื่อลูกค้า โดเมน หรือ TPID
4.  เลือกลูกค้าของคุณจากผลลัพธ์การค้นหา
5.  กรอกฟอร์ม **ขอความช่วยเหลือเกี่ยวกับ Microsoft 365** ให้สมบูรณ์

เมื่อต้องการขอความช่วยเหลือเกี่ยวกับ App Assure ให้กรอกการร้องขอ [บริการ App Assure](https://go.microsoft.com/fwlink/?linkid=2022721)ให้เสร็จสิ้น

## <a name="fasttrack-expectations"></a>ความคาดหวังของ FastTrack

FastTrack จะให้แนวทางแบบสิ้นสุดเกี่ยวกับการย้ายไปยังและใช้ Microsoft 365 คุณได้รับแนวทางระยะไกลและหลักปฏิบัติที่ดีที่สุดเกี่ยวกับการวางแผนการเริ่มการใช้งานที่ประสบความสเร็จ

### <a name="meeting-rhythms"></a>จังหวะการประชุม

คุณร่วมงานกับผู้เชี่ยวชาญด้าน FastTrack จากระยะไกลผ่านพูดคุยด้านเทคนิค เวิร์คช้อป และการประชุมจุดตรวจสอบ รวมถึง:

  - **การพูดคุยและเวิร์คช้อปทางเทคนิค:** สิ่งเหล่านี้จะให้หัวข้อที่เจาะลึกและหลักปฏิบัติที่ดีที่สุดเพื่อเร่งการปรับใช้บริการ Microsoft 365 ของคุณ การประชุมเหล่านี้มักจะให้ภาพรวมของเอกสารที่มีให้ใช้งาน เครื่องมือซอฟต์แวร์ และคอนโซลผู้ดูแลระบบ ระยะเวลามาตรฐาน: 60 นาที
  - **การประชุม Checkpoint:** แผงสัมผัสรายสัปดาห์หรือแบบสองสัปดาห์เพื่อให้แน่ใจว่าการปรับใช้ของคุณคืบหน้า ระยะเวลามาตรฐาน: 30 นาที

### <a name="expected-outcomes-and-key-milestones"></a>ผลลัพธ์ที่คาดหมายและหลักหลักๆ ของหลักๆ 

FastTrack มอบหลักปฏิบัติที่ดีที่สุดและขั้นตอนในการแก้ไขปัญหาเพื่อติดตามเพื่อเปิดใช้งาน Microsoft 365 ขององค์กรของคุณ

 **สิ่งที่คุณคาดหวังได้จาก FastTrack:**

  - เราร่วมกับคุณเพื่อสร้างแผนในการแก้ไขที่จะมอบสภาพแวดล้อมต้นทางของคุณไปยังความต้องการขั้นต่สุดเพื่อให้การออนบอร์ดและการโยกย้ายประสบความสเร็จ
  - เรามีแนวทางในการสร้างพื้นฐานที่แข็งแกร่งสําหรับการออนบอร์ดหลัก การรวมข้อมูลเฉพาะตัว และกิจกรรมการเปิดใช้งานบริการ

**ข้อมูลล่าสุดของการมีส่วนร่วม ได้แก่:**

  - เริ่มการประชุม – การเข้าใจกระบวนการ จัดเวลา เครื่องมือ และกิจกรรมของทีม
  - การประเมิน – พัฒนารายการตรวจสอบการแก้ไข
  - การอภิปรายเกี่ยวกับความปลอดภัย - วางแผนกลยุทธ์ด้านความปลอดภัยตามหลักปฏิบัติที่ดีที่สุดด้านความปลอดภัยของ Microsoft ที่แนะนา
  - รีวิวการแก้ไข – การตรวจทานรายการในการแก้ไข
  - เวิร์คช้อปความสเร็จ – เครื่องมือ เทมเพลต และคู่มือการสร้างแผนปฏิบัติการ
  - การประชุม Checkpoint – รายงานการประชุมและตารางการประชุม
  - เปิดใช้งานกิจกรรม – เปิดใช้งานบริการให้เสร็จสมบูรณ์
  - เวิร์คช้อปความเร็วของการโยกย้าย (ไม่บังคับ Microsoft 365 เท่านั้น) - ความพร้อมในการโยกย้าย

ดูข้อมูลเพิ่มเติม ดูเรื่องราวความสเร็[จของลูกค้า Microsoft 365](https://www.microsoft.com/microsoft-365/customer-stories?rtc=1)

### <a name="customer-success-tips"></a>เคล็ดลับความสเร็จของลูกค้า

การเตรียมพร้อมของคุณเพื่อรับความช่วยเหลือของ FastTrack เป็นเรื่องที่ทําให้กระบวนการราบรื่นและประสบความสาเร็จ ก่อนที่จะมีส่วนร่วมกับ FastTrack โปรดตรวจสอบให้แน่ใจว่าได้สร้าง:

  - วันที่ที่ต้องการ (เมื่อคุณวางแผนที่จะปรับใช้บริการ)
  - จุดประสงค์ (บริการที่คุณวางแผนที่จะเปิดใช้)
  - เหตุผลทางธุรกิจ (เหตุผลที่คุณวางแผนที่จะเปิดใช้บริการให้กับผู้ใช้)
  - เจ้าของความสเร็จ (ผู้ที่รับผิดชอบความสเร็จของโครงการ)

เราขอแนะนนะให้คุณพิจารณาและระบุสิ่งต่อไปนี้:

  - ผู้เกี่ยวข้อง – การจัดหาพนักงานและการเพิ่มศักยภาพของผู้สนับสนุนระดับผู้บริหารและผู้ชนะ
  - สถานการณ์สมมติ - การจัดลาดับความจัดลาดับความความสเร็จและข้อนิยามของเกณฑ์ความสเร็จ
  - การรับรู้ - การปรับใช้แคมเปญการสื่อสารและการวางแผนเหตุการณ์การเริ่มใช้งาน
  - การฝึกอบรม – การให้ความรู้แก่ผู้ใช้และแหล่งข้อมูลบริการช่วยเหลือที่พร้อมใช้

## <a name="fasttrack-and-customer-responsibilities"></a>FastTrack และความรับผิดชอบของลูกค้า 

ความรับผิดชอบที่ชัดเจนของทุกฝ่ายมีความจําเป็นในการออนบอร์ดไปยัง Microsoft 365 ได้อย่างราบรื่น ส่วนนี้จะสรุปความรับผิดชอบเหล่านั้นโดยยึดตามขั้นตอนต่างๆ ของการมีส่วนร่วมใน FastTrack FastTrack มีหน้าที่ให้การเข้าถึงผู้เชี่ยวชาญด้าน FastTrack อย่างต่อเนื่องตลอดระยะเวลาการสมัครใช้งานของคุณเพื่อเข้าใช้บริการ Microsoft 365,Enterprise Mobility & Security (EMS) และบริการ Windows 10 ผู้เชี่ยวชาญเหล่านี้ให้การสนับสนุนระยะไกลระหว่างเวลาธุรกิจปกติที่ FastTrack Center อยู่และโยกย้ายอีเมลและไฟล์ที่ได้รับการสนับสนุน (500 ที่นั่งหรือมากกว่า) การสนับสนุนที่มีแนวทางนี้รวมถึงหลักปฏิบัติที่ดีที่สุด การเข้าถึงทรัพยากรออนไลน์ และเครื่องมือแบบบริการตนเอง Customers, or Partners working on their behalf, are responsible for project management, performing remediation and hands-on configuration, implementing change management and process optimization, on-site readiness, and training. ตารางต่อไปนี้มีรายการที่ครอบคลุมของความรับผิดชอบเหล่านี้
