# <a name="onboarding-phases"></a>ระยะการปฐมนิเทศ

เมื่อคุณใช้[บริการที่เหมาะสมและแผน](M365-eligible-services-and-plans.md)เพื่อรับ พิเศษไดเรกทอรีที่ใช้งานอยู่ของ Azure Microsoft และ Microsoft Intune ที่พร้อมสำหรับใช้ มีระยะต่าง ๆ ในกระบวนการ ส่วนต่อไปนี้อธิบายแต่ละขั้นตอนของกระบวนการปฐมนิเทศ

ทะเบียนมีหลักสี่ระยะ:

![ระยะสี่ของกระบวนการปฐมนิเทศ FastTrack](./media/O365-Onboarding-Phases.png)


## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

หลังจากที่คุณซื้อสิทธิ์การใช้งานจำนวนที่เหมาะสม ทำตามคำแนะนำจากอีเมลยืนซื้อเพื่อเชื่อมโยงใบอนุญาตให้ผู้เช่าที่มีอยู่แล้วหรือผู้เช่าใหม่ Microsoft ตรวจสอบสิทธิ์ของคุณสำหรับสวัสดิการศูนย์ FastTrack และพยายามติดต่อคุณเพื่อเสนอความช่วยเหลือในการปฐมนิเทศแล้ว

> [!NOTE]
> คุณสามารถขอความช่วยเหลือจาก[ศูนย์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ถ้าคุณพร้อมที่จะปรับใช้บริการเหล่านี้สำหรับองค์กรของคุณ

### <a name="to-request-assistance"></a>เพื่อขอความช่วยเหลือ

1. เข้าสู่ระบบไปยัง[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**บริการ**
4. เสร็จสมบูรณ์**ร้องขอความช่วยเหลือเกี่ยวกับฟอร์ม Microsoft 365**

หนึ่งครั้งเริ่มสนับสนุนปฐมนิเทศ เราจะตั้งค่าตารางเวลาของการประชุมแบบออนไลน์

> [!NOTE]
> ถ้าคุณมีคู่ค้าของ Microsoft ที่แสดงรายการอยู่ในผู้เช่า Office 365 ของคุณ คุณจะไม่เห็นตัวเลือกนี้ โปรดปรึกษาคู่ค้า Microsoft ของคุณสำหรับความช่วยเหลือ

คู่ค้าของ Microsoft สามารถจะขอความช่วยเหลือผ่าน[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ในนามของลูกค้า เมื่อต้องการทำเช่นนี้:

1. เข้าสู่ระบบไปยัง[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**ลูกค้าของฉัน**
4. ค้นหาลูกค้าของคุณ หรือเลือกจากรายการของลูกค้าของคุณ
5. เลือก**บริการ**
6. เสร็จสมบูรณ์**ร้องขอความช่วยเหลือเกี่ยวกับฟอร์ม Microsoft 365**

เมื่อมีสนับสนุนการปฐมนิเทศ เริ่มต้น FastTrack เซ็ตอัพตัวจัดกำหนดการของการประชุมแบบออนไลน์คุณจะอภิปรายกระบวนการปฐมนิเทศ การตรวจสอบข้อมูล และตั้งค่าการประชุมเตะ

![ระยะ initiate ปฐมนิเทศ](./media/ft-initiate-phase.png)

## <a name="assess-phase"></a>ขั้นตอนประเมิน

เมื่อเริ่มต้นกระบวนการปฐมนิเทศ ศูนย์ FastTrack ทำงานกับคุณสามารถประเมินของสภาพแวดล้อมของแหล่งที่มาและความต้องการ มีการเรียกใช้เครื่องมือเพื่อประเมินสภาพแวดล้อมของคุณ และผู้เชี่ยวชาญ FastTrack แนะนำคุณเกี่ยวกับการประเมินไดเรก ทอรีที่ใช้งานอยู่ของคุณในสถาน อินเทอร์เน็ตเบราว์เซอร์ ระบบปฏิบัติการของไคลเอนต์อุปกรณ์ Domain Name System (DNS), เครือข่าย โครงสร้างพื้นฐาน และระบบรหัสประจำตัว ตรวจสอบว่า การเปลี่ยนแปลงใด ๆ ที่จำเป็นสำหรับการปฐมนิเทศ

ศูนย์กลาง FastTrack ยังเชื่อมต่อคุณกับคำแนะนำเกี่ยวกับวิธีการไดรฟ์เริ่มประสบความสำเร็จของบริการเหมาะสม

ขึ้นอยู่กับการตั้งค่าปัจจุบันของคุณ เราให้แผนด้านที่เปิดแหล่งที่มาของสภาพแวดล้อมจนถึงความต้องการขั้นต่ำสำหรับการปฐมนิเทศเสร็จเรียบร้อยแล้วเพื่อ EMS หรือบริการของแต่ละรูปเมฆ เรายังติดตั้งเรียกจุดตรวจสอบที่เหมาะสมสำหรับเฟสด้าน

![ขั้นตอนประเมินปฐมนิเทศ](./media/ft-assess-phase.png)

## <a name="remediate-phase"></a>Remediate ขั้นตอน
คุณดำเนินการงานในแผนด้านบนระบบต้นทางของคุณเพื่อให้คุณตอบสนองความต้องการสำหรับการปฐมนิเทศและการใช้แต่ละบริการ (จำเป็น)

![ทะเบียน remediate ขั้นตอน](./media/ft-remediate-phase.png)

ก่อนที่คุณเริ่มต้นขั้นตอนการเปิดใช้งาน เรานิติบุคคลตรวจสอบผลลัพธ์ของกิจกรรมด้านเพื่อให้แน่ใจว่า คุณพร้อมที่จะดำเนินต่อหรือไม่

## <a name="enable-phase"></a>ขั้นตอนการเปิดใช้งาน
เมื่อกิจกรรมด้านทั้งหมดเสร็จสมบูรณ์ โครงการเลื่อน เพื่อตั้งค่าคอนฟิกโครงสร้างพื้นฐานหลักสำหรับปริมาณการใช้บริการ และแต่ละบริการ cloud EMS มีสิทธิ์ในการเตรียมใช้งาน

**เปิดใช้งานขั้นตอน - ความสามารถหลัก**

การปฐมนิเทศหลักเกี่ยวข้องกับการรวมการเตรียมใช้งาน และผู้เช่า และข้อมูลเฉพาะตัวของบริการ นอกจากนี้ยังรวมถึงขั้นตอนที่ให้พื้นฐานสำหรับบริการออนไลน์ปฐมนิเทศเช่นพิเศษโฆษณา Azure และ Intune

![เปิดใช้งานการปฐมนิเทศระยะ - ความสามารถหลัก](./media/ft-enable-phase-core-01.png)

![เปิดใช้งานการปฐมนิเทศระยะ - ความสามารถหลัก](./media/ft-enable-phase-core-02.png)
> [!NOTE]
> WAP หมายถึงสำหรับพร็อกซีโปรแกรมประยุกต์บนเว็บ SSL หมายถึง การรักษาความปลอดภัย Secure Sockets Layer SDS หมายถึงสำหรับซิงค์ข้อมูลโรงเรียน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับ SDS ดู[ยินดีต้อนรับสู่ Microsoft ซิงค์ข้อมูลของโรงเรียน](https://go.microsoft.com/fwlink/?linkid=871480)

> [!NOTE]
> วิธีการรับรองความถูกต้องที่มีการจัดการรวมถึง แต่ไม่จำกัดเฉพาะการซิงโครไนส์แฮชของรหัสผ่าน รวมข้อมูลเฉพาะตัวจะเป็นเวลากิจกรรม และไม่รวมโยกย้าย หรือ decommissioning ของวิธีการรับรองความถูกต้องที่มีอยู่ เช่นที่มีจัดการ หรือการติดต่อกับภายนอก

### <a name="enable-phase---azure-ad-premium"></a>เปิดใช้งานขั้นตอน - Azure โฆษณาพิเศษ

สามารถตั้งค่าสภาพแวดล้อมพิเศษโฆษณา Azure โดยใช้ Azure Active Directory เชื่อมต่อเครื่องมือไดเรกทอรีการซิงโครไนส์และบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) (จำเป็น)

สำหรับสถานการณ์พิเศษโฆษณา Azure ที่มีการซิงโครไนส์ข้อมูลเฉพาะตัวในสถานที่ไปยัง cloud เราช่วยคุณ ด้วยการเพิ่มผู้ดูแลระบบไอทีและผู้ใช้ให้การสมัครใช้งาน การตั้งค่าคอนฟิกการจัดการข้อกำหนดเบื้องต้น การตั้งค่าพิเศษโฆษณา Azure การตั้งค่าไดเรกทอรี ซิงโครไนส์กับการรับรองความถูกต้องที่มีการจัดการ และ FS โฆษณาโดยใช้เครื่องมือการเชื่อมต่อ AD Azure การตั้งค่าคอนฟิกผู้ใช้ทดสอบ และตรวจสอบหลักของคุณใช้กรณีและปัญหาสำหรับการบริการ

ตั้งค่าค่าจ้างพิเศษโฆษณา azure มีการเปิดใช้งานลักษณะการทำงานต่อไปนี้:

-   รีเซ็ตรหัสผ่านด้วยตนเองในไดเรกทอรีที่ใช้งานอยู่ที่ azure (SSPR)

-   ปัจจัยหลาย azure การรับรองความถูกต้อง (Azure MFA)

-   ได้ถึงสาม (3) หรือเพิ่มเติมซอฟต์แวร์เป็นการรวมแอพลิเคชันเป็นบริการ (SaaS) กับหนึ่งเข้าสู่ระบบ (SSO) จาก[Azure Marketplace ของไดเรกทอรีที่ใช้งานอยู่](https://azure.microsoft.com/marketplace/active-directory/)

-   กำลังเตรียมใช้งานผู้ใช้โดยอัตโนมัติสำหรับล่วงหน้ารวมแอพลิเคชัน SaaS ตามที่แสดงรายการอยู่ใน[รายการของบทสอนการรวมแอพลิเคชัน](https://docs.microsoft.com/en-us/azure/active-directory/saas-apps/tutorial-list)จำกัดจัดสรรขาออกเท่านั้น

-   เข้าสู่ระบบที่กำหนดเองหน้าจอแสดงผล รวมถึงโลโก้ ข้อความ และภาพพื้นหลัง

-   บริการตนเอง และแบบไดนามิกกลุ่ม (กลุ่ม)

-   พร็อกซีโปรแกรมประยุกต์ azure ไดเรกทอรีที่ใช้งานอยู่

-   ไดเรกทอรีที่ใช้งานอยู่ที่ azure เชื่อมต่อสุขภาพ

-   เข้าถึงไดเรกทอรีที่ใช้งานอยู่ที่ azure ตามเงื่อนไข

-   ไดเรกทอรีที่ใช้งานอยู่ที่ azure ข้อตกลงการใช้งาน

-   การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ที่ azure

-   ไดเรกทอรีที่ใช้งานอยู่ที่ azure มีสิทธิ์จัดการข้อมูลประจำตัว

-   บทวิจารณ์เข้าถึงไดเรกทอรีที่ใช้งานอยู่ที่ azure

![เปิดใช้งานการปฐมนิเทศระยะ - Azure โฆษณาพิเศษ](./media/ft-enable-phase_aad-premium_adconnect_adfed.png)

### <a name="enable-phase---intune"></a>เปิดใช้งานขั้นตอน - Intune

สำหรับ Intune เราแนะนำคุณเกี่ยวกับการรับพร้อมที่จะใช้ Microsoft Intune เพื่อจัดการอุปกรณ์ ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณ และขึ้นอยู่กับอุปกรณ์เคลื่อนที่ของคุณ และจำเป็นต้องจัดการแอพลิเคชันโมบาย ขั้นตอนอาจรวมถึง:

-   อนุญาตให้ใช้สิทธิ์ของผู้ใช้ นอกจากนี้เรายังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสำหรับของคุณ Microsoft cloud บริการผู้เช่า (จำเป็น) ด้วย

-   การกำหนดค่าข้อมูลเฉพาะตัวที่จะใช้ โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่ใช้งานอยู่ในสถานของคุณหรือข้อมูลเฉพาะตัวของ cloud

-   เพิ่มผู้ใช้ให้การสมัครใช้งาน Intune การกำหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และอุปกรณ์

-   ตั้งค่าคอนฟิกของผู้จัดการอุปกรณ์เคลื่อนที่ (MDM) ขึ้นอยู่กับความต้องการจัดการ รวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM เมื่อ Intune เป็นโซลูชัน MDM ของคุณเท่านั้น หรือไม่สอดคล้องกับการจัดการอุปกรณ์มือถือสำหรับ Office 365

-   ให้คำแนะนำ MDM สำหรับ:

    -   การกำหนดค่ากลุ่มทดสอบที่จะใช้ในการตรวจสอบ MDM นโยบายการจัดการ

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   ปรับใช้แอพลิเคชันสำหรับแต่ละได้รับการสนับสนุนแพลตฟอร์มผ่านทางการเชื่อมโยงเว็บหรือการเชื่อมโยงที่ซับซ้อน

        -   นโยบายการเข้าถึงแบบมีเงื่อนไข

        -   การปรับใช้ของอีเมล เครือข่ายไร้สาย และโพรไฟล์เครือข่ายส่วนตัวเสมือน (VPN) ถ้าคุณมีใบรับรองที่มีอยู่ Wi-fi หรือ VPN โครงสร้างพื้นฐานในองค์กรของคุณ

        -   การตั้งค่าที่ Microsoft Intune แลกเปลี่ยนตัวเชื่อมต่อ (เมื่อใช้ได้)

        -   เชื่อมต่อไปยังคลังสินค้าข้อมูล Intune

        -   Intune กับการรวม:
            -   ทีม Viewer สำหรับความช่วยเหลือระยะไกล (การสมัครใช้งานตัวแสดงทีมจำเป็น)

            -   ป้องกันการคุกคามโมบาย (MTD) หุ้นส่วนการแก้ไขปัญหา (การบอกรับเป็นสมาชิก Mobile ป้องกันการคุกคามจำเป็น)

            -   โซลูชันการจัดการค่าใช้จ่ายโทรคมนาคม (สมัครใช้งานโซลูชันการจัดการค่าใช้จ่ายโทรคมนาคมจำเป็น)

            -   การป้องกันภัยคุกคามขั้นสูงของ Windows Defender (ใบอนุญาต Windows E5 หรือ Microsoft 365 E5 จำเป็นต้องมี)

    -   ลงทะเบียนอุปกรณ์ของแต่ละ[แพลตฟอร์มที่ได้รับการสนับสนุน](https://technet.microsoft.com/library/dn600287.aspx)ในการ Intune

-   ให้คำแนะนำในการป้องกันโปรแกรมประยุกต์บน:

    -   การกำหนดค่านโยบายการป้องกันโปรแกรมประยุกต์สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   การกำหนดค่านโยบายการเข้าถึงแบบมีเงื่อนไขสำหรับโปรแกรมประยุกต์ที่มีการจัดการ

    -   ระบุเป้าหมายที่เป็นกลุ่มผู้ใช้ที่เหมาะสมกับนโยบาย MAM ข้างต้น

    -   ใช้รายงานการใช้งานโปรแกรมประยุกต์ที่มีจัดการ

-   ให้คำแนะนำในการจัดการพีซีบน:

    -   การติดตั้งซอฟต์แวร์ไคลเอ็นต์ Intune (เมื่อจำเป็น)

    -   ใช้ซอฟต์แวร์และฮาร์ดแวร์รายงานพร้อมใช้งานใน Intune

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนการจัดการพีซีคลาสสิค Windows 10 ด้วย Intune FastTrack สนับสนุนเฉพาะการจัดการอุปกรณ์ Windows 10 ผ่านทางการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM) เท่านั้น

#### <a name="windows-autopilot"></a>Autopilot ของ Windows

FastTrack สามารถช่วยให้คุณผ่านความซับซ้อนน้อยลงของอุปกรณ์การเตรียมใช้งานกับ Windows Autopilot และ Intune โดยนำอุปกรณ์ใหม่ของคุณแก่ผู้ใช้โดยไม่ต้องสร้าง รักษา และใช้อิมเมจระบบปฏิบัติการที่กำหนดเองไปยังอุปกรณ์ของคุณ

FastTrack สนับสนุน Autopilot สถานการณ์ต่อไปนี้:

- **ระบบบริการตนเองโฆษณา azure:** อุปกรณ์เข้าร่วมโฆษณา Azure และลงทะเบียนไว้ใน Intune สถานการณ์สมมตินี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1703 และรุ่นล่าสุด

- **ระบบบริการตนเองของไฮบริดสลี AAD:** อุปกรณ์รวมทั้งในสถานโฆษณาและโฆษณา Azure และลงทะเบียนไว้ใน Intune สถานการณ์สมมตินี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1809 และรุ่นล่าสุด

- **การเตรียมใช้งานด้วยตนเอง:** อุปกรณ์ร่วมโฆษณา Azure โดยอัตโนมัติ สถานการณ์สมมตินี้ได้รับการสนับสนุนเมื่อใช้ Windows 1809 และรุ่นล่าสุด

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนสถานการณ์สมมติ Autopilot ที่เริ่มจากตัวจัดการการตั้งค่าคอนฟิก

ขั้นตอนการติดตั้ง Windows Autopilot ขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มา และสามารถรวมถึง:

- กำหนดค่า และการตั้งค่า Microsoft Intune สำหรับ Windows Autopilot

- การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกโฆษณา Azure

- เพิ่มบริษัทของคุณกำหนดตราสินค้าไว้ในโฆษณา Azure

- สร้าง และกำหนดอุปกรณ์กับส่วนกำหนดค่า Windows Autopilot (เช่นโพรไฟล์ Windows Autopilot ที่จำกัดการสร้างบัญชีผู้ใช้ผู้ดูแลระบบท้องถิ่น)

- การกำหนดเองออกของ--ประสบการณ์ใช้งานแบบทันที (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

- การตั้งค่าคอนฟิกการลงทะเบียนอัตโนมัติ MDM ในโฆษณา Azure และ Intune

#### <a name="deploy-outlook-for-ios-and-android-securely"></a>ปรับใช้ Outlook สำหรับ iOS และ Android ได้อย่างปลอดภัย

FastTrack สามารถช่วยให้คุณ โดยใช้ Outlook สำหรับ iOS และ Android ได้อย่างปลอดภัยในองค์กรของคุณเพื่อให้แน่ใจว่า ผู้ใช้มีโปรแกรมประยุกต์ที่จำเป็นทั้งหมดที่ติดตั้ง

ขั้นตอนในการปรับใช้ Outlook Mobile อย่างปลอดภัยสำหรับ iOS และ Android กับ Intune ขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มา และสามารถรวมถึง:

- ดาวน์โหลด Outlook สำหรับ iOS และ Android การรับรองความถูก ต้องของ Microsoft และโปรแกรมประยุกต์พอร์ทัลของ บริษัท Intune ผ่าน Apple App Store หรือ Google Play Store
- นอกจากนี้ยัง ให้คำแนะนำเกี่ยวกับการตั้งค่า:
    - Outlook สำหรับ iOS และ Android การรับรองความถูก ต้องของ Microsoft และการจัดวางแอพลิเคชันพอร์ทัล บริษัท Intune กับ Intune
    - นโยบายการป้องกันโปรแกรมประยุกต์
    - นโยบายการเข้าถึงแบบมีเงื่อนไข
    - นโยบายการตั้งค่าคอนฟิกแอพลิเคชัน

    > [!IMPORTANT]
    > FastTrack ทีมสนับสนุนการรักษาความปลอดภัยของ Outlook สำหรับ iOS และ Android ด้วยอัตราแลกเปลี่ยนนโยบายกล่องจดหมายของอุปกรณ์เคลื่อนที่

#### <a name="co-management"></a>จัดการร่วม

FastTrack แนะนำคุณผ่านการรับพร้อมที่จะจัดการอุปกรณ์ Windows 10 ทั้งตัวจัดการการตั้งค่าคอนฟิกและ Intune งานพร้อม ๆ กัน ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มา และสามารถรวมถึง:

- อธิบายข้อดีของการจัดการร่วมกัน

- สิทธิ์การใช้งานแก่ผู้ใช้ของคุณ FastTrack ยังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสำหรับของคุณ Microsoft cloud บริการผู้เช่า (จำเป็น)

- กำหนดค่าข้อมูลเฉพาะตัวที่จะใช้ โดย Intune โดยใช้ประโยชน์จากอย่างใดอย่างหนึ่งของคุณในสถานไดเรกทอรีที่ใช้งานอยู่และ/หรือ cloud ผู้ใช้อีเมล

- เพิ่มผู้ใช้ให้การสมัครใช้งาน Intune การกำหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และอุปกรณ์

- ให้คำแนะนำเกี่ยวกับวิธีการย้ายจาก Intune ที่รวมเข้ากับระบบศูนย์จัดการการตั้งค่า (ไฮบริดสลี) เป็นแบบสแตนด์อโลน Intune

- ให้คำแนะนำเกี่ยวกับการตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับการลงทะเบียนอัตโนมัติ MDM

- ให้คำแนะนำการตั้งค่าค่าไฮบริดสลี Azure Active Directory เข้าร่วม

- ให้คำแนะนำเกี่ยวกับวิธีการตั้งค่าเกตเวย์จัดการ Cloud

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่า

- กำหนดค่าปริมาณที่ได้รับการสนับสนุนที่คุณต้องการสลับไปยัง Intune

- ติดตั้งไคลเอ็นต์การจัดการการตั้งค่าในอุปกรณ์ Intune ที่ลงทะเบียนไว้

- ให้คำแนะนำเกี่ยวกับวิธีการตรวจสอบกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

FastTrack ยังให้คุณคำแนะนำเกี่ยวกับวิธีการไดรฟ์เริ่มประสบความสำเร็จของบริการเหมาะสม
