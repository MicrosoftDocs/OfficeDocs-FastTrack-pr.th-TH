# <a name="onboarding-phases"></a>ระยะการปฐมนิเทศ

เมื่อคุณใช้บริการที่มี[สิทธิ์และแผน](M365-eligible-services-and-plans.md)การที่จะได้รับ Microsoft Azure ใช้งานอยู่ของไดเรกทอรีพรีเมี่ยม microsoft Intune และการป้องกันข้อมูล Azure พร้อมสำหรับการใช้งานมีหลายขั้นตอนที่เกี่ยวข้องในกระบวนการ ส่วนต่อไปนี้อธิบายถึงแต่ละระยะของกระบวนการปฐมนิเทศ

การเตรียมการมีสี่ขั้นตอนหลัก:

![ขั้นตอนที่สี่ของกระบวนการปฐมนิเทศของ FastTrack](./media/O365-Onboarding-Phases.png)


## <a name="initiate-phase"></a>เริ่มต้นเฟส

หลังจากที่คุณซื้อจำนวนใบอนุญาตที่เหมาะสมแล้วให้ทำตามคำแนะนำจากเมลยืนยันการซื้อเพื่อเชื่อมโยงใบอนุญาตกับผู้เช่าที่มีอยู่หรือผู้เช่าใหม่ จากนั้น Microsoft จะตรวจสอบสิทธิ์ของคุณสำหรับสิทธิประโยชน์ของศูนย์ FastTrack และพยายามติดต่อคุณเพื่อเสนอความช่วยเหลือในการเตรียมพร้อม

> [!NOTE]
> นอกจากนี้คุณยังสามารถร้องขอความช่วยเหลือจาก[ศูนย์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ถ้าคุณพร้อมที่จะปรับใช้บริการเหล่านี้สำหรับองค์กรของคุณ

### <a name="to-request-assistance"></a>หากต้องการขอความช่วยเหลือ

1. เข้าสู่ระบบ[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**บริการ**
4. ทำ**คำขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วยแบบฟอร์ม Microsoft ๓๖๕**

เมื่อเริ่มการสนับสนุนการปฐมนิเทศเราจะตั้งค่าตารางเวลาของการประชุมออนไลน์

คู่ค้าของ Microsoft ยังสามารถรับความช่วยเหลือผ่าน[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ในนามของลูกค้าได้ด้วย การทำเช่นนั้น:

1. เข้าสู่ระบบ[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**ลูกค้าของฉัน**
4. ค้นหาลูกค้าของคุณหรือเลือกจากรายชื่อลูกค้าของคุณ
5. เลือก**บริการ**
6. ทำ**คำขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วยแบบฟอร์ม Microsoft ๓๖๕**

เมื่อการสนับสนุนการปฐมนิเทศเริ่มต้น FastTrack จะตั้งค่าตารางการประชุมแบบออนไลน์กับคุณเพื่อหารือเกี่ยวกับกระบวนการปฐมนิเทศตรวจสอบข้อมูลและตั้งค่าการประชุมแบบเตะออก

![การเตรียมการเริ่มต้นขั้นตอน](./media/ft-initiate-phase.png)

## <a name="assess-phase"></a>ประเมินระยะ

เมื่อเริ่มกระบวนการปฐมนิเทศศูนย์ FastTrack จะทำงานร่วมกับคุณเพื่อประเมินสภาพแวดล้อมต้นทางและความต้องการของคุณ เครื่องมือจะถูกเรียกใช้เพื่อประเมินสภาพแวดล้อมของคุณและผู้เชี่ยวชาญ FastTrack แนะนำให้คุณทราบถึงการประเมินไดเรกทอรีที่ใช้งานอยู่ในสถานที่, เบราว์เซอร์อินเทอร์เน็ต, ระบบปฏิบัติการของอุปกรณ์ไคลเอ็นต์, ระบบชื่อโดเมน (DNS), เครือข่าย, โครงสร้างพื้นฐานและระบบข้อมูลประจำตัวเพื่อ ตรวจสอบว่ามีการเปลี่ยนแปลงใดๆที่จำเป็นสำหรับการเตรียมการ

ศูนย์ FastTrack ยังเชื่อมต่อคุณด้วยคำแนะนำเกี่ยวกับวิธีการกระตุ้นการใช้บริการที่มีสิทธิ์ที่ประสบความสำเร็จ

ขึ้นอยู่กับการตั้งค่าปัจจุบันของคุณเรามีแผนการแก้ไขที่นำสภาพแวดล้อมแหล่งที่มาของคุณถึงความต้องการขั้นต่ำสำหรับการเตรียมพร้อมสำหรับการทำงานของ EMS หรือบริการคลาวด์ของแต่ละบุคคล นอกจากนี้เรายังตั้งค่าการเรียกจุดตรวจสอบที่เหมาะสมสำหรับขั้นตอนการแก้ไข

![ขั้นตอนการประเมินการเตรียมการ](./media/ft-assess-phase.png)

## <a name="remediate-phase"></a>ระยะแก้ไข
คุณทำงานในแผนการแก้ไขในสภาพแวดล้อมของแหล่งที่มาของคุณเพื่อให้คุณเป็นไปตามข้อกำหนดสำหรับการปฐมนิเทศและการใช้บริการแต่ละรายการ (ตามความจำเป็น)

![ระยะแก้ไขการปฐมนิเทศ](./media/ft-remediate-phase.png)

ก่อนที่คุณจะเริ่มขั้นตอนการเปิดใช้งานเราร่วมกันตรวจสอบผลลัพธ์ของกิจกรรมการแก้ไขเพื่อให้แน่ใจว่าคุณพร้อมที่จะดำเนินการต่อ

## <a name="enable-phase"></a>เปิดใช้งานเฟส
เมื่อกิจกรรมการแก้ไขทั้งหมดเสร็จสมบูรณ์โครงการจะเลื่อนการกำหนดค่าโครงสร้างพื้นฐานหลักสำหรับการใช้บริการและการจัดสรรบริการคลาวด์ EMS ที่มีสิทธิ์แต่ละรายการ

**เปิดใช้งานความสามารถหลักของเฟส**

การเตรียมการหลักเกี่ยวข้องกับการจัดเตรียมการบริการและผู้เช่าและการรวมตัวตน นอกจากนี้ยังมีขั้นตอนสำหรับการให้รากฐานสำหรับบริการออนไลน์การเตรียมพร้อมเช่น Azure โฆษณาพรีเมี่ยม, Intune และการป้องกันข้อมูล Azure

![การเปิดใช้งานความสามารถขั้นตอนหลัก](./media/ft-enable-phase-core-01.png)

![การเปิดใช้งานความสามารถขั้นตอนหลัก](./media/ft-enable-phase-core-02.png)
> [!NOTE]
> หมายถึงพร็อกซีโปรแกรมประยุกต์บนเว็บ SSL สำหรับการรักษาความปลอดภัยเลเยอร์ซ็อกเก็ต มี SDS ยืนสำหรับการซิงค์ข้อมูลโรงเรียน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับ SDS โปรดดูที่[ยินดีต้อนรับสู่การซิงค์ข้อมูลของโรงเรียน Microsoft](https://go.microsoft.com/fwlink/?linkid=871480)

> [!NOTE]
> วิธีการรับรองความถูกต้องที่มีการจัดการรวมถึงแต่ไม่จำกัดเฉพาะการซิงโครไนส์แฮชของรหัสผ่าน การรวมข้อมูลประจำตัวเป็นกิจกรรมครั้งเดียวและไม่รวมการย้ายหรือการรื้อถอนวิธีการรับรองความถูกต้องที่มีอยู่เช่นการจัดการหรือการติดต่อกับภายนอก

### <a name="enable-phase---azure-ad-premium"></a>เปิดใช้งานระยะ-Azure โฆษณาระดับพรีเมี่ยม

สภาพแวดล้อมแบบพรีเมียม AD Azure สามารถตั้งค่าโดยใช้ Azure ที่ใช้งานอยู่ไดเรกทอรีเครื่องมือเชื่อมต่อการซิงโครไนส์และบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) (ตามความจำเป็น)

สำหรับสถานการณ์สมมติ Azure AD Premium ที่มีการซิงโครไนส์ข้อมูลเฉพาะตัวในสถานที่ไปยัง cloud เราจะช่วยคุณโดยการเพิ่มผู้ดูแลระบบไอทีและผู้ใช้ในการสมัครสมาชิกของคุณการกำหนดค่าข้อกำหนดเบื้องต้นการจัดการการตั้งค่า Azure AD Premium การตั้งค่าไดเรกทอรี การทำข้อมูลให้ตรงกันกับการรับรองความถูกต้องที่มีการจัดการและ FS โฆษณาโดยใช้เครื่องมือการเชื่อมต่อ AD Azure การกำหนดค่าผู้ใช้ทดสอบและการตรวจสอบกรณีการใช้งานหลักของคุณสำหรับการบริการ

การตั้งค่า Azure AD Premium รวมถึงการเปิดใช้งานคุณลักษณะต่อไปนี้:

-   รีเซ็ตรหัสผ่านบริการตนเองของ Azure Active Directory (SSPR)

-   การรับรองความถูกต้องด้วยหลายปัจจัย azure (เอฟเอ MFA)

-   ถึงสาม (3) หรือมากกว่าซอฟต์แวร์เป็นการรวมแอพลิเคชันบริการ (SaaS) กับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ของ Azure](https://azure.microsoft.com/marketplace/active-directory/)

-   การจัดสรรผู้ใช้โดยอัตโนมัติสำหรับแอปพลิเคชัน SaaS แบบบูรณาการล่วงหน้าตามที่ระบุไว้ใน[รายการการสอนการรวมแอป](https://docs.microsoft.com/en-us/azure/active-directory/saas-apps/tutorial-list)จำกัดการจัดสรรขาออกเท่านั้น

-   หน้าจอเข้าสู่ระบบที่กำหนดเองรวมถึงโลโก้ข้อความและรูปภาพ

-   การบริการตนเองและกลุ่มแบบไดนามิก (กลุ่ม)

-   พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ Azure

-   Azure ที่ใช้งานอยู่ไดเรกทอรีเชื่อมต่อสุขภาพ

-   การเข้าถึงแบบมีเงื่อนไข Azure ไดเรกทอรีที่ใช้งานอยู่

-   เงื่อนไขการใช้ไดเรกทอรีที่ใช้งานอยู่ของ Azure

-   การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่

-   การจัดการข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ของ Azure

-   รีวิวการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ Azure

### <a name="enable-phase---intune"></a>เปิดใช้เฟส-Intune

สำหรับ Intune เราแนะนำคุณผ่านการเตรียมพร้อมที่จะใช้ Microsoft Intune ในการจัดการอุปกรณ์ ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและจะขึ้นอยู่กับอุปกรณ์เคลื่อนที่และความต้องการในการจัดการแอพมือถือของคุณ ขั้นตอนอาจรวมถึง:

-   ให้สิทธิ์ผู้ใช้ปลายทางของคุณ นอกจากนี้เรายังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งาน volume license สำหรับผู้เช่าบริการ cloud ของ Microsoft (ตามความจำเป็น)

-   การกำหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intune โดยใช้ประโยชน์ทั้งในสถานที่ Active Directory หรือข้อมูลเฉพาะตัวของ cloud

-   การเพิ่มผู้ใช้ไปยังการบอกรับเป็นสมาชิก Intune ของคุณกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

-   การกำหนดค่าสิทธิในการจัดการอุปกรณ์เคลื่อนที่ (MDM) ตามความต้องการในการจัดการของคุณซึ่งรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงาน MDM ของคุณเมื่อ Intune เป็นโซลูชัน MDM ของคุณเท่านั้น

-   การให้คำแนะนำ MDM สำหรับ:

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   การปรับใช้โปรแกรมประยุกต์สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านการเชื่อมโยงเว็บหรือการเชื่อมโยงอย่างลึก

        -   นโยบายการเข้าถึงแบบมีเงื่อนไข

        -   การใช้งานของ e-mail เครือข่ายไร้สายและโปรไฟล์เครือข่ายส่วนตัวเสมือน (VPN) ถ้าคุณมีหน่วยงานออกใบรับรองที่มีอยู่, Wi-fi หรือ VPN โครงสร้างพื้นฐานในองค์กรของคุณ

        -   การตั้งค่าตัวเชื่อมต่อ Exchange Intune Microsoft (เมื่อมี)

        -   การเชื่อมต่อกับคลังข้อมูล Intune

        -   การรวม Intune ด้วย:
            -   ตัวแสดงทีมสำหรับความช่วยเหลือระยะไกล (ต้องมีการสมัครสมาชิกตัวแสดงทีม)

            -   โซลูชั่นคู่ค้าป้องกันภัยคุกคามมือถือ (MTD) (จำเป็นต้องมีการป้องกันภัยคุกคามมือถือ).

            -   โซลูชั่นการจัดการค่าใช้จ่ายโทรคมนาคม (จำเป็นต้องสมัครสมาชิกโซลูชันการจัดการค่าใช้จ่ายโทรคมนาคม)

            -   จำเป็นต้องมีการป้องกันภัยคุกคามขั้นสูงของ windows Defender (Windows E5 หรือ Microsoft ๓๖๕ E5)

    -   การลงทะเบียนอุปกรณ์ของแต่ละ[แพลตฟอร์มที่สนับสนุน](https://technet.microsoft.com/library/dn600287.aspx)ไปยัง Intune

-   ให้คำแนะนำในการป้องกันแอปใน:

    -   การกำหนดค่านโยบายการป้องกันแอปสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   การกำหนดค่านโยบายการเข้าถึงแบบมีเงื่อนไขสำหรับแอปที่มีการจัดการ

    -   กำหนดเป้าหมายกลุ่มผู้ใช้ที่เหมาะสมด้วยนโยบาย MAM ด้านบน

    -   การใช้รายงานการใช้โปรแกรมประยุกต์ที่มีการจัดการ

-   ให้คำแนะนำในการจัดการเครื่องคอมพิวเตอร์ใน:

    -   การติดตั้งซอฟต์แวร์ไคลเอ็นต์ Intune (เมื่อจำเป็น)

    -   การใช้รายงานซอฟต์แวร์และฮาร์ดแวร์ที่มีอยู่ใน Intune

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 กับ Intune FastTrack สนับสนุนการจัดการอุปกรณ์ Windows 10 ผ่านการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM) เท่านั้น

#### <a name="windows-autopilot"></a>วินโดวส์ออโตไพลอต

FastTrack สามารถช่วยให้คุณทำให้อุปกรณ์ของคุณใช้งานได้ง่ายขึ้นด้วย Windows ออโตไพลอตและ Intune โดยให้อุปกรณ์ใหม่แก่ผู้ใช้ของคุณโดยไม่จำเป็นต้องสร้างรักษาและใช้รูปภาพระบบปฏิบัติการที่กำหนดเองไปยังอุปกรณ์ของคุณ

FastTrack สนับสนุนสถานการณ์ต่อไปนี้ของออโตไพลอต:

- **บริการตนเองของ AZURE AD:** อุปกรณ์เข้าร่วมโฆษณา Azure และลงทะเบียนใน Intune สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๐๑๗๐๓และรุ่นล่าสุด

- **ไฮบริดสลีบริการตนเอง AAD:** อุปกรณ์เข้าร่วมทั้งโฆษณาในสถานที่และโฆษณา Azure และลงทะเบียนใน Intune สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๐๑๘๐๙และรุ่นล่าสุด

- การจัด**สรรตนเอง:** อุปกรณ์จะเข้าร่วมโฆษณา Azure โดยอัตโนมัติ สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๘๐๙และรุ่นล่าสุด

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนสถานการณ์ออโตไพลอตเริ่มต้นจากตัวจัดการการตั้งค่าคอนฟิก

ขั้นตอนการติดตั้ง Windows ออโตไพลอตขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณและอาจรวมถึง:

- กำหนดค่าและตั้งค่า Microsoft Intune สำหรับระบบออโตไพลอต Windows

- กำหนดค่ากลุ่มแบบไดนามิกของโฆษณา Azure

- เพิ่มตราสินค้าของบริษัทของคุณลงในโฆษณา Azure

- สร้างและกำหนดอุปกรณ์ให้กับโพรไฟล์ของ Windows ออโตไพลอต (เช่นโปรไฟล์ของ Windows ออโตไพลอตที่จำกัดการสร้างบัญชีผู้ดูแลภายใน)

- ปรับแต่งประสบการณ์ใช้งานแบบออกจากกล่อง (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

- การกำหนดค่า MDM อัตโนมัติลงทะเบียนในโฆษณา Azure และ Intune

#### <a name="deploy-outlook-for-ios-and-android-securely"></a>ปรับใช้ Outlook สำหรับ iOS และ Android อย่างปลอดภัย

FastTrack สามารถช่วยคุณได้โดยการปรับใช้ Outlook สำหรับ iOS และ Android อย่างปลอดภัยในองค์กรของคุณเพื่อให้แน่ใจว่าผู้ใช้ของคุณมีแอปที่จำเป็นทั้งหมดที่ติดตั้งอยู่

ขั้นตอนในการปรับใช้ Outlook Mobile สำหรับ iOS และ Android กับ Intune อย่างปลอดภัยขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและอาจรวมถึง:

- ดาวน์โหลด Outlook สำหรับ iOS และ Android, Microsoft Authenticator และแอพลิเคชันพอร์ทัลของบริษัท Intune ผ่าน Apple App Store หรือ Google Play สโตร์
- นอกจากนี้ยังให้คำแนะนำเกี่ยวกับการตั้งค่า:
    - Outlook สำหรับ iOS และ Android, Microsoft Authenticator และการปรับใช้แอพลิเคชันเว็บไซต์ของบริษัท Intune กับ Intune
    - นโยบายการป้องกันแอป
    - นโยบายการเข้าถึงแบบมีเงื่อนไข
    - นโยบายการกำหนดค่าแอป

    > [!IMPORTANT]
    > ทีมงาน FastTrack ไม่สนับสนุนการรักษาความปลอดภัย Outlook สำหรับ iOS และ Android ที่มีนโยบายกล่องจดหมายของอุปกรณ์เคลื่อนที่ Exchange

#### <a name="co-management"></a>การจัดการร่วม

FastTrack แนะนำคุณผ่านการเตรียมพร้อมในการจัดการอุปกรณ์ Windows 10 ที่มีทั้งตัวจัดการการตั้งค่าคอนฟิกและ Intune ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณและอาจรวมถึง:

- อธิบายประโยชน์ของการบริหารจัดการร่วมกัน

- อนุญาตให้ผู้ใช้ปลายทางของคุณ FastTrack ยังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งาน volume license สำหรับผู้เช่าบริการ cloud ของ Microsoft (ตามความจำเป็น)

- กำหนดค่ารหัสประจำตัวที่จะใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่มีอยู่ในสถานที่ของคุณและ/หรือข้อมูลเฉพาะตัวของ cloud

- การเพิ่มผู้ใช้ไปยังการบอกรับเป็นสมาชิก Intune ของคุณกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

- ให้คำแนะนำเกี่ยวกับวิธีการย้ายจาก Intune รวมกับตัวจัดการการตั้งค่าคอนฟิกศูนย์ระบบ (ไฮบริดสลี) ไปยัง Intune แบบสแตนด์อโลน

- ให้คำแนะนำเกี่ยวกับการตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับการลงทะเบียนอัตโนมัติ MDM

- ให้คำแนะนำการตั้งค่าการเข้าร่วมไดเรกทอรีที่ใช้งานอยู่ของไฮบริดสลี Azure

- ให้คำแนะนำเกี่ยวกับวิธีการตั้งค่าเกตเวย์จัดการคลาวด์

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่า

- กำหนดค่าปริมาณงานที่สนับสนุนที่คุณต้องการสลับไปยัง Intune

- ติดตั้งไคลเอนต์ตัวจัดการการตั้งค่าคอนฟิกในอุปกรณ์ที่ลงทะเบียน Intune

- ให้คำแนะนำเกี่ยวกับวิธีการตรวจสอบกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

นอกจากนี้ FastTrack ยังให้คำแนะนำเกี่ยวกับวิธีกระตุ้นการใช้บริการที่มีสิทธิ์ที่ประสบความสำเร็จ

#### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานระยะ–การป้องกันข้อมูล Azure

ลูกค้าจะได้รับคำแนะนำเกี่ยวกับวิธีการ: 

- เปิดใช้งานและตั้งค่าคอนฟิกผู้เช่าของพวกเขา
- สร้างและตั้งค่าป้ายชื่อและนโยบาย
- ใช้การป้องกันข้อมูลกับเอกสาร 
- จัดประเภทและข้อมูลป้ายชื่อโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทำงานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure
- ใช้แฟ้มที่เหลือโดยใช้สแกนเนอร์การป้องกันข้อมูล Azure
- ตรวจสอบจดหมายในการส่งต่อโดยใช้กฎการไหลของจดหมายของ Exchange แบบออนไลน์

คำแนะนำยังให้แก่ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ Azure ของ Microsoft (Azure RMS), Office ๓๖๕การเข้ารหัสลับข้อความ (OME), และการป้องกันการสูญหายของข้อมูล (DLP)

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมหรือไม่** ดูการ[เคลื่อนย้ายขององค์กร + ความปลอดภัย](https://www.microsoft.com/en-us/cloud-platform/enterprise-mobility)
