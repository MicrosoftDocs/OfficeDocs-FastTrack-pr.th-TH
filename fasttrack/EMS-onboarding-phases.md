# <a name="onboarding-phases"></a>ขั้นตอนการออนบอร์ด

เมื่อคุณใช้[บริการที่มีสิทธิ์และแผน](M365-eligible-services-and-plans.md)เพื่อรับ Microsoft Azure ที่ใช้งานอยู่ไดเรกทอรีค่าจ้างพิเศษ Microsoft Intune และการป้องกันข้อมูล Azure พร้อมใช้งาน มีหลายขั้นตอนที่เกี่ยวข้องในกระบวนการ ส่วนต่อไปนี้อธิบายแต่ละขั้นตอนของกระบวนการเตรียมพร้อม

การปฐมภูมิมีสี่ขั้นตอนหลัก:

![ขั้นตอน 4 ขั้นตอนของกระบวนการออนบอร์ด FastTrack](./media/O365-Onboarding-Phases.png)


## <a name="initiate-phase"></a>เริ่มต้นขั้นตอน

หลังจากที่คุณซื้อจํานวนที่เหมาะสมของใบอนุญาตให้ปฏิบัติตามคําแนะนําจากอีเมลการยืนยันการซื้อเพื่อเชื่อมโยงสิทธิ์การใช้งานกับผู้เช่าที่มีอยู่หรือผู้เช่าใหม่ของคุณ จากนั้น Microsoft จะตรวจสอบสิทธิ์ของคุณสําหรับสวัสดิการของศูนย์ FastTrack และพยายามติดต่อคุณเพื่อให้ความช่วยเหลือในการปฐมนารถ

> [!NOTE]
> คุณยังสามารถขอความช่วยเหลือจาก[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)หากคุณพร้อมที่จะปรับใช้บริการเหล่านี้สําหรับองค์กรของคุณ

### <a name="to-request-assistance"></a>เมื่อต้องการขอความช่วยเหลือ

1. ลงชื่อเข้าใช้เว็บไซต์[FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**ขอรับความช่วยเหลือสําหรับ Microsoft 365**จาก**การดําเนินการด่วน**ที่ด้านบนของหน้า Landing Page ของคุณ หรือโดยการเลือก**ขอรับความช่วยเหลือสําหรับ Microsoft 365**บนการ์ดปรับใช้
3. ดําเนินการ**ร้องขอความช่วยเหลือสําหรับฟอร์ม Microsoft 365**

เมื่อเริ่มสนับสนุนการเริ่มทํางาน เราจะตั้งค่าตารางเวลาของการประชุมแบบออนไลน์

คู่ค้าของ Microsoft ยังสามารถรับความช่วยเหลือผ่าน[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ในนามของลูกค้า เมื่อต้องการทําเช่นนั้น:

1. ลงชื่อเข้าใช้เว็บไซต์[FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**ลูกค้าของฉัน**
3. ค้นหาลูกค้าของคุณ หรือเลือกลูกค้าจากรายการลูกค้าของคุณ
4. เลือก**บริการ**
5. ดําเนินการ**ร้องขอความช่วยเหลือสําหรับฟอร์ม Microsoft 365**

เมื่อการสนับสนุนการเริ่มทํางาน FastTrack จะกําหนดตารางเวลาของการประชุมออนไลน์กับคุณเพื่อหารือเกี่ยวกับกระบวนการการออนบอร์ด

![ระยะเริ่มต้นการเริ่มทํางาน](./media/ft-initiate-phase.png)

## <a name="assess-phase"></a>ประเมินระยะ

เมื่อกระบวนการเริ่มทํางานแล้ว FastTrack Center จะทํางานร่วมกับคุณเพื่อประเมินสภาพแวดล้อมของแหล่งที่มาและข้อกําหนด เครื่องมือจะทํางานเพื่อประเมินสภาพแวดล้อมของคุณและผู้เชี่ยวชาญ FastTrack แนะนําคุณผ่านการประเมินไดเรกทอรีที่ใช้งานอยู่ในสถานที่เบราว์เซอร์อินเทอร์เน็ตระบบปฏิบัติการอุปกรณ์ของลูกค้าระบบชื่อโดเมน (DNS), เครือข่าย, โครงสร้างพื้นฐานและระบบเอกลักษณ์เพื่อกําหนดว่าการเปลี่ยนแปลงใด ๆ ที่จําเป็นสําหรับการปฐมนิเทศ

นอกจากนี้ FastTrack Center ยังเชื่อมต่อคุณเข้ากับคําแนะนําเกี่ยวกับวิธีผลักดันการนําบริการที่มีสิทธิ์มาใช้สําเร็จ

เราจัดเตรียมแผนการแก้ไขที่นําสภาพแวดล้อมต้นทางของคุณมาสู่ความต้องการขั้นต่ําสําหรับการเริ่มต้นใช้งาน EMS หรือบริการคลาวด์แต่ละแห่ง เรายังตั้งค่าจุดตรวจที่เหมาะสมสําหรับระยะการแก้ไข

![ระยะการประเมินการปฐม](./media/ft-assess-phase.png)

## <a name="remediate-phase"></a>ขั้นตอนการแก้ไข
คุณดําเนินงานในแผนการแก้ไขในสภาพแวดล้อมต้นทางของคุณเพื่อให้คุณมีคุณสมบัติตรงตามข้อกําหนดสําหรับการเริ่มต้นใช้งานและการยอมรับแต่ละบริการ (ตามความจําเป็น)

![ระยะการแก้ไขการออนบอร์ด](./media/ft-remediate-phase.png)

ก่อนที่คุณจะเริ่มขั้นตอนเปิดใช้งานเราร่วมกันตรวจสอบผลของกิจกรรมการแก้ไขเพื่อให้แน่ใจว่าคุณพร้อมที่จะดําเนินการต่อ

## <a name="enable-phase"></a>เปิดใช้งานเฟส
เมื่อกิจกรรมการแก้ไขทั้งหมดเสร็จสมบูรณ์

**เปิดใช้งานเฟส - ความสามารถหลัก**

การเตรียมพร้อมหลักเกี่ยวข้องกับการจัดเตรียมบริการและการรวมผู้เช่าและเอกลักษณ์ นอกจากนี้ยังรวมถึงขั้นตอนในการจัดเตรียมพื้นฐานสําหรับบริการออนไลน์ที่พร้อมสําหรับออนบอร์ด เช่น Azure AD Premium, Intune และการปกป้องข้อมูล Azure

![การเตรียมพร้อมสําหรับระยะ - ความสามารถหลัก](./media/ft-enable-phase-core-01.png)

![การเตรียมพร้อมสําหรับระยะ - ความสามารถหลัก](./media/ft-enable-phase-core-02.png)
> [!NOTE]
> WAP ย่อมาจากพร็อกซีโปรแกรมประยุกต์บนเว็บ SSL ย่อมาจากชั้นซ็อกเก็ตการรักษาความปลอดภัย SDS ย่อมาจากการซิงค์ข้อมูลของโรงเรียน สําหรับข้อมูลเพิ่มเติมเกี่ยวกับ SDS ให้ดูที่[ยินดีต้อนรับสู่การซิงค์ข้อมูล Microsoft School](https://go.microsoft.com/fwlink/?linkid=871480)

> [!NOTE]
> วิธีการรับรองความถูกต้องที่มีการจัดการรวมถึง แต่ไม่ จํากัด เฉพาะการซิงโครไนส์แฮรหัสผ่าน การรวมเอกลักษณ์เป็นกิจกรรมครั้งเดียว และไม่รวมการโอนย้ายหรือยกเลิกการใช้งานของวิธีการตรวจสอบสิทธิ์ที่มีอยู่ เช่น มีการจัดการหรือติดต่อกับภายนอก

### <a name="enable-phase---azure-ad-premium"></a>เปิดใช้งานเฟส - Azure โฆษณาพิเศษ

สามารถตั้งค่าสภาพแวดล้อม Azure AD Premium โดยใช้การซิงโครไนส์ Azure ไดเรกทอรีที่ใช้งานอยู่เชื่อมต่อไดเรกทอรีไดเรกทอรีไดเรกทอรีไดเรกทอรีไดเรกทอรีไดเรกทอรีไดเรกทอรีและไดเรกทอรีที่ใช้งานอยู่ (FS โฆษณา) (ตามความจําเป็น)

สําหรับสถานการณ์สมมติ Azure AD Premium ที่มีข้อมูลเฉพาะตัวในสถานที่ตรงกันกับข้อมูลเฉพาะตัวในองค์กรกับระบบคลาวด์ เราจะช่วยคุณโดยการเพิ่มผู้ดูแลระบบ IT และผู้ใช้ลงในการสมัครใช้งานของคุณ

การตั้งค่า Azure AD Premium รวมถึงการเปิดใช้งานคุณลักษณะต่อไปนี้:

-   Azure Active Directory รหัสผ่านบริการตนเอง (SSPR)

-   การรับรองความถูกต้องด้วยหลายปัจจัย azure (Azure MFA)

-   ขึ้นไปสาม (3) ซอฟต์แวร์เป็นการรวมแอพลิเคชันบริการ (SaaS) กับ Single Sign-On (SSO) จาก[Azure ที่ใช้งานอยู่ไดเรกทอรีตลาด](https://azure.microsoft.com/marketplace/active-directory/)

-   การจัดสรรผู้ใช้อัตโนมัติสําหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามที่ระบุไว้ใน[รายการบทแนะนําการรวมแอพ](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จํากัด การจัดสรรขาออกเท่านั้น

-   หน้าจอการเข้าสู่ระบบแบบกําหนดเอง รวมถึงโลโก้ ข้อความ และรูปภาพ

-   บริการตนเองและกลุ่มแบบไดนามิก (กลุ่ม)

-   พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ azure

-   azure. azure

-   การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ azure

-   ข้อตกลงการใช้งานไดเรกทอรีที่ใช้งานอยู่ของ Azure

-   การป้องกันข้อมูลประจําตัวของไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การจัดการข้อมูลประจําตัวของ Azure ที่มีสิทธิ์ของไดเรกทอรีที่ใช้งานอยู่

-   การตรวจทานการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การป้องกันรหัสผ่านของไดเรกทอรีที่ใช้งานอยู่ของ Azure

-   Azure B2B ไดเรกทอรีที่ใช้งานอยู่

### <a name="enable-phase---intune"></a>เปิดใช้งานเฟส - Intune

สําหรับ Intune เราแนะนําคุณในการเตรียมพร้อมในการใช้ Microsoft Intune เพื่อจัดการอุปกรณ์ ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาและขึ้นอยู่กับอุปกรณ์เคลื่อนที่และความต้องการการจัดการแอปบนอุปกรณ์เคลื่อนที่ของคุณ ขั้นตอนสามารถรวมถึง:

-   การให้สิทธิ์การใช้งานผู้ใช้ของคุณ เรายังให้ความช่วยเหลือเกี่ยวกับวิธีเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสําหรับผู้เช่าบริการระบบคลาวด์ของ Microsoft (ตามความจําเป็น)

-   การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่ใช้งานอยู่ในสถานที่ของคุณหรือข้อมูลเฉพาะตัวของ cloud

-   การเพิ่มผู้ใช้ในการสมัครใช้งาน Intune ของคุณ การกําหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และกลุ่มอุปกรณ์

-   การกําหนดค่าหน่วยงานการจัดการอุปกรณ์เคลื่อนที่ (MDM) ของคุณ โดยขึ้นอยู่กับความต้องการด้านการจัดการของคุณ รวมถึง:

    -   การตั้งค่า Intune เป็นอํานาจ MDM ของคุณเมื่อ Intune เป็นโซลูชัน MDM ของคุณเท่านั้น

-   ให้คําแนะนํา MDM สําหรับ:

    -   การกําหนดคอนฟิกกลุ่มการทดสอบที่จะใช้เพื่อตรวจสอบนโยบายการจัดการ MDM

    -   การกําหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   การปรับใช้แอปพลิเคชันสําหรับแต่ละแพลตฟอร์มที่สนับสนุนผ่านลิงก์เว็บหรือลิงก์ในเนื้อหา

        -   นโยบายการเข้าถึงแบบมีเงื่อนไข

        -   การปรับใช้อีเมล เครือข่ายไร้สาย และโปรไฟล์เครือข่ายส่วนตัวเสมือน (VPN) ถ้าคุณมีผู้ให้บริการออกใบรับรอง โครงสร้างพื้นฐาน Wi-Fi หรือ VPN ที่มีอยู่ในองค์กรของคุณ

        -   การตั้งค่าการเชื่อมต่อการแลกเปลี่ยน Intune Microsoft (เมื่อเกี่ยวข้อง)

        -   การเชื่อมต่อกับคลังข้อมูล Intune

        -   การรวม Intune กับ:
            -   ตัวแสดงทีมสําหรับความช่วยเหลือระยะไกล (ต้องการการสมัครสมาชิกตัวแสดงทีม)

            -   โซลูชั่นคู่ค้า (MTD) ภัยคุกคามมือถือ (ต้องสมัครรับข้อมูล)

            -   โซลูชั่นการจัดการค่าใช้จ่ายโทรคมนาคม (ค่าใช้จ่ายในการจัดการค่าใช้จ่ายโทรคมนาคมที่สมัครใช้งานเป็นสิ่งจําเป็น)

            -   การป้องกันภัยคุกคามขั้นสูงของ Microsoft Defender (จําเป็นต้องมีสิทธิ์การใช้งาน Windows E5 หรือ Microsoft 365 E5)

    -   อุปกรณ์ลงทะเบียนของแต่ละ[แพลตฟอร์มที่สนับสนุน](https://technet.microsoft.com/library/dn600287.aspx)เพื่อ Intune

-   ให้คําแนะนําในการป้องกันแอปบน:

    -   การกําหนดค่านโยบายการป้องกันแอปสําหรับแต่ละแพลตฟอร์มที่รองรับ

    -   การกําหนดค่านโยบายการเข้าถึงแบบมีเงื่อนไขสําหรับแอปที่มีการจัดการ

    -   กําหนดเป้าหมายกลุ่มผู้ใช้ที่เหมาะสมด้วยนโยบาย MAM ด้านบน

    -   การใช้รายงานการใช้งานแอปพลิเคชันที่มีการจัดการ

-   ให้คําแนะนําการจัดการคอมพิวเตอร์เกี่ยวกับ:

    -   การติดตั้งซอฟต์แวร์ไคลเอ็นต์ Intune (เมื่อจําเป็น)

    -   การใช้ซอฟต์แวร์และรายงานฮาร์ดแวร์ที่มีอยู่ใน Intune

    > [!IMPORTANT]
    > ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 ด้วย Intune FastTrack สนับสนุนการจัดการอุปกรณ์ Windows 10 ผ่านการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM) เท่านั้น

#### <a name="windows-autopilot"></a>หน้าต่างออโตไพลอต

FastTrack สามารถช่วยคุณในการจัดเตรียมอุปกรณ์ให้ง่ายขึ้นด้วย Windows Autopilot และ Intune โดยให้อุปกรณ์ใหม่ๆ แก่ผู้ใช้ปลายทางโดยไม่ต้องสร้าง ดูแลและใช้อิมเมจระบบปฏิบัติการที่กําหนดเองกับอุปกรณ์ของคุณ

FastTrack สนับสนุนสถานการณ์จําลองออโตไพลอตต่อไปนี้:

- **บริการตนเองของโฆษณา azure:** อุปกรณ์เข้าร่วม Azure AD และลงทะเบียนเข้า Intune สถานการณ์สมมตินี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1703 และรุ่นล่าสุด

- **ไฮบริด AAD บริการตนเอง:** อุปกรณ์รวมโฆษณาในสถานที่และโฆษณา Azure และลงทะเบียนเป็น Intune สถานการณ์สมมตินี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1809 และรุ่นล่าสุด

- **การเตรียมใช้งานด้วยตนเอง:** อุปกรณ์โดยอัตโนมัติเข้าร่วม Azure AD สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows 1809 และรุ่นล่าสุด

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนสถานการณ์จําลองออโตไพลอตที่เริ่มต้นจากตัวจัดการการตั้งค่าคอนฟิก

ขั้นตอนการติดตั้ง Windows Autopilot ขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณ และสามารถรวม:

- กําหนดค่าและตั้งค่า Intune ของ Microsoft สําหรับ Windows ออโตไพลอต

- การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกของโฆษณา Azure

- เพิ่มตราสินค้าของบริษัทของคุณลงใน Azure AD

- สร้าง และกําหนดอุปกรณ์ไปยังส่วนกําหนดค่าของ Windows Autopilot (เช่นส่วนกําหนดค่าออโตไพลอตของ Windows ที่จํากัดการสร้างบัญชีผู้ดูแลระบบภายในเครื่อง)

- กําหนดออกกล่องประสบการณ์ (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

- การกําหนดค่าการลงทะเบียนอัตโนมัติ MDM ในโฆษณา Azure และ Intune

#### <a name="deploy-outlook-for-ios-and-android-securely"></a>ปรับใช้ Outlook สําหรับ iOS และ Android อย่างปลอดภัย

FastTrack สามารถช่วยคุณได้โดยการปรับใช้ Outlook สําหรับ iOS และ Android อย่างปลอดภัยในองค์กรของคุณเพื่อให้แน่ใจว่าผู้ใช้ของคุณมีแอปที่จําเป็นทั้งหมดติดตั้งอยู่

ขั้นตอนในการปรับใช้ Outlook Mobile สําหรับ iOS และ Android อย่างปลอดภัยขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณ และสามารถรวม:

- ดาวน์โหลด Outlook สําหรับ iOS และ Android, Microsoft Authenticator และแอปพอร์ทัลบริษัท Intune ผ่านทาง Apple App Store หรือ Google Play สโตร์
- นอกจากนี้ยังให้คําแนะนําเกี่ยวกับการตั้งค่า:
    - Outlook สําหรับ iOS และ Android, Microsoft Authenticator และการปรับใช้แอปพอร์ทัล Intune กับ Intune
    - นโยบายการป้องกันแอป
    - นโยบายการเข้าถึงแบบมีเงื่อนไข
    - นโยบายการกําหนดค่าแอป

    > [!IMPORTANT]
    > ทีม FastTrack ไม่สนับสนุนการรักษาความปลอดภัย Outlook สําหรับ iOS และ Android กับนโยบายกล่องจดหมายอุปกรณ์เคลื่อนที่ของ Exchange

#### <a name="cloud-attach"></a>การแนบบนคลาวด์

FastTrack จะแนะนําคุณในการเตรียมพร้อมที่จะแนบสภาพแวดล้อมตัวจัดการการกําหนดค่าที่มีอยู่กับ Intune ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณ ขั้นตอนเหล่านี้อาจรวมถึง:

- อธิบายประโยชน์ของตัวจัดการการกําหนดค่าที่แนบบนระบบคลาวด์กับ Intune

- การให้สิทธิ์การใช้งานผู้ใช้ของคุณ นอกจากนี้ FastTrack ยังให้ความช่วยเหลือเกี่ยวกับวิธีเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสําหรับผู้เช่าบริการระบบคลาวด์ของ Microsoft (ตามความจําเป็น)

- การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่ใช้งานอยู่ในสถานที่ของคุณและข้อมูลเฉพาะตัวของ cloud

- การเพิ่มผู้ใช้ในการสมัครใช้งาน Intune ของคุณ การกําหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และกลุ่มอุปกรณ์

- การเปิดใช้งานการแนบบน cloud ในคอนโซลการจัดการการตั้งค่า

- ให้คําแนะนําการตั้งค่าไฮบริดสลีรวมไดเรกทอรีที่ใช้งานอยู่

- ให้คําแนะนําเกี่ยวกับการตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับการลงทะเบียนอัตโนมัติ MDM

- ให้คําแนะนําเกี่ยวกับวิธีตั้งค่าเกตเวย์การจัดการระบบคลาวด์

- การกําหนดค่าเวิร์กโหลดที่สนับสนุนที่คุณต้องการสลับไปยัง Intune

- การติดตั้งไคลเอนต์ตัวจัดการการตั้งค่าคอนฟิกบนอุปกรณ์ที่ลงทะเบียน Intune

นอกจากนี้ FastTrack ยังให้คําแนะนําเกี่ยวกับวิธีผลักดันการนําบริการที่มีสิทธิ์มาใช้อย่างประสบความสําเร็จ

#### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน – การป้องกันข้อมูล Azure

ลูกค้าจะได้รับคําแนะนําเกี่ยวกับวิธีการ: 

- เปิดใช้งานและตั้งค่าคอนฟิกผู้เช่า
- สร้างและตั้งค่าป้ายชื่อและนโยบาย
- ใช้การป้องกันข้อมูลกับเอกสาร 
- จัดประเภทและป้ายชื่อข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทํางานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure
- ใช้แฟ้มที่เหลือโดยใช้สแกนเนอร์การป้องกันข้อมูล Azure
- ตรวจสอบอีเมลในการส่งโดยใช้กฎขั้นตอนจดหมาย Exchange Online

คําแนะนํายังถูกจัดเตรียมให้แก่ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ Azure ของ Microsoft (Azure RMS), การเข้ารหัสลับข้อความ Office 365 (OME) และการป้องกันข้อมูลสูญหาย (DLP)
