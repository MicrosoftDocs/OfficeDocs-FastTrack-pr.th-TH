# <a name="onboarding-phases"></a>ขั้นตอนการออนบอร์ด

เมื่อคุณใช้บริการที่มี[สิทธิ์และวางแผน](M365-eligible-services-and-plans.md)จะได้รับ Microsoft Azure ไดเรกทอรีที่ใช้งานอยู่ที่ระดับพรีเมี่ยม microsoft Intune และการป้องกันข้อมูล Azure พร้อมสำหรับการใช้งานมีหลายขั้นตอนที่เกี่ยวข้องในกระบวนการ ส่วนต่อไปนี้อธิบายถึงแต่ละขั้นตอนของกระบวนการปฐมนิเทศ

ปฐมนิเทศมีสี่ขั้นตอนหลักคือ

![ขั้นตอนที่สี่ของกระบวนการ FastTrack ปฐมนิเทศ](./media/O365-Onboarding-Phases.png)


## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

หลังจากที่คุณซื้อจำนวนสิทธิ์การใช้งานที่เหมาะสมแล้วให้ทำตามคำแนะนำจากอีเมลยืนยันการซื้อเพื่อเชื่อมโยงใบอนุญาตกับผู้เช่ารายใหม่ของคุณ จากนั้น Microsoft จะตรวจสอบสิทธิ์ของคุณสำหรับสวัสดิการศูนย์ FastTrack และพยายามที่จะติดต่อคุณเพื่อให้ความช่วยเหลือปฐมนิเทศ

> [!NOTE]
> นอกจากนี้คุณยังสามารถขอความช่วยเหลือจาก[ศูนย์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ถ้าคุณพร้อมที่จะปรับใช้บริการเหล่านี้สำหรับองค์กรของคุณ

### <a name="to-request-assistance"></a>เพื่อขอความช่วยเหลือ

1. ลงชื่อเข้าใช้ใน[เว็บไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**บริการ**
4. ทำการ**ร้องขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วย Microsoft ๓๖๕ฟอร์ม**

เมื่อการสนับสนุนปฐมนิเทศเริ่มต้นเราจะตั้งค่าตารางการประชุมแบบออนไลน์

นอกจากนี้คู่ค้าของ Microsoft ยังสามารถรับความช่วยเหลือผ่าน[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ในนามของลูกค้าได้ด้วย เมื่อต้องการทำเช่นนั้น:

1. ลงชื่อเข้าใช้ใน[เว็บไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**FastTrack**
3. เลือก**ลูกค้าของฉัน**
4. ค้นหาลูกค้าของคุณหรือเลือกจากรายชื่อลูกค้าของคุณ
5. เลือก**บริการ**
6. ทำการ**ร้องขอความช่วยเหลือให้เสร็จสมบูรณ์ด้วย Microsoft ๓๖๕ฟอร์ม**

เมื่อการสนับสนุนปฐมนิเทศเริ่มต้น FastTrack ตั้งตารางเวลาของการประชุมแบบออนไลน์กับคุณเพื่อหารือเกี่ยวกับกระบวนการปฐมนิเทศตรวจสอบข้อมูลและตั้งค่าการประชุมเตะออก

![ปฐมนิเทศขั้นตอนการเริ่มต้น](./media/ft-initiate-phase.png)

## <a name="assess-phase"></a>ประเมินระยะ

เมื่อกระบวนการปฐมนิเทศเริ่มต้น, ศูนย์ FastTrack ทำงานร่วมกับคุณในการประเมินสภาพแวดล้อมต้นทางของคุณและความต้องการ. เครื่องมือจะทำงานเพื่อประเมินสภาพแวดล้อมของคุณและผู้เชี่ยวชาญ FastTrack แนะนำคุณผ่านการประเมินไดเรกทอรีที่ใช้งานอยู่ในสถานที่อินเทอร์เน็ตเบราว์เซอร์, ระบบปฏิบัติการของไคลเอนต์, ระบบชื่อโดเมน (DNS), เครือข่ายโครงสร้างพื้นฐานและระบบข้อมูลประจำตัวเพื่อ ตรวจสอบว่ามีการเปลี่ยนแปลงใดๆที่จำเป็นสำหรับปฐมนิเทศ

นอกจากนี้ FastTrack Center ยังเชื่อมโยงคุณกับคำแนะนำเกี่ยวกับวิธีการขับรถที่ประสบความสำเร็จในการให้บริการที่มีสิทธิ์

เรามีแผนการที่จะนำสภาพแวดล้อมต้นฉบับของคุณขึ้นอยู่กับความต้องการขั้นต่ำสำหรับการปฐมนิเทศที่สำเร็จไปยัง EMS หรือบริการคลาวด์ของแต่ละบุคคล นอกจากนี้เรายังตั้งค่าจุดตรวจสอบที่เหมาะสมสำหรับระยะของด้าน

![ขั้นตอนการประเมินปฐมนิเทศ](./media/ft-assess-phase.png)

## <a name="remediate-phase"></a>Remediate เฟส
คุณทำงานในการวางแผนในสภาพแวดล้อมต้นทางของคุณเพื่อให้คุณตอบสนองความต้องการสำหรับปฐมนิเทศและนำไปใช้แต่ละบริการ (ตามความจำเป็น)

![ปฐมนิเทศ remediate](./media/ft-remediate-phase.png)

ก่อนที่คุณจะเริ่มขั้นตอนการเปิดใช้งานเราร่วมกันตรวจสอบผลลัพธ์ของกิจกรรมด้านเพื่อให้แน่ใจว่าคุณพร้อมที่จะดำเนินการต่อไป

## <a name="enable-phase"></a>เปิดใช้งานเฟส
เมื่อกิจกรรมทุกอย่างเสร็จสมบูรณ์จะมีการเปลี่ยนแปลงการตั้งค่าโครงสร้างพื้นฐานหลักสำหรับการใช้บริการและการจัดเตรียมบริการระบบคลาวด์แบบ EMS ที่มีสิทธิ์

**เปิดใช้งานความสามารถหลักของเฟส**

หลักปฐมนิเทศเกี่ยวข้องกับการจัดเตรียมบริการและผู้เช่าและการรวมข้อมูลเฉพาะตัว นอกจากนี้ยังรวมถึงขั้นตอนสำหรับการให้รากฐานสำหรับบริการออนไลน์ปฐมนิเทศเช่น Azure โฆษณาพรีเมี่ยม, Intune และการป้องกันข้อมูล Azure

![ปฐมนิเทศเปิดใช้งานความสามารถหลักของเฟส](./media/ft-enable-phase-core-01.png)

![ปฐมนิเทศเปิดใช้งานความสามารถหลักของเฟส](./media/ft-enable-phase-core-02.png)
> [!NOTE]
> WAP ที่ยืนสำหรับพร็อกซีโปรแกรมประยุกต์บนเว็บ SSL ยืนสำหรับชั้น secure Sockets SDS สำหรับการซิงค์ข้อมูลโรงเรียน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับ SDS โปรดดู[ยินดีต้อนรับสู่การซิงค์ข้อมูลโรงเรียนของ Microsoft](https://go.microsoft.com/fwlink/?linkid=871480)

> [!NOTE]
> วิธีการรับรองความถูกต้องที่มีการจัดการประกอบด้วยแต่ไม่จำกัดการซิงโครไนส์แฮรหัสผ่าน การรวมข้อมูลเฉพาะตัวเป็นกิจกรรมเวลาหนึ่งและไม่รวมถึงการโยกย้ายหรือรื้อถอนวิธีการรับรองความถูกต้องที่มีอยู่เช่นการจัดการหรือติดต่อกับภายนอก

### <a name="enable-phase---azure-ad-premium"></a>เปิดใช้งานแบบเฟส-Azure AD พรีเมี่ยม

สภาพแวดล้อมแบบพรีเมี่ยม Azure AD สามารถตั้งค่าโดยใช้ Azure ที่ใช้งานอยู่ไดเรกทอรีเชื่อมต่อเครื่องมือการซิงโครไนส์และบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) (ตามความจำเป็น)

สำหรับสถานการณ์พรีเมี่ยม AD Azure ที่มีการซิงโครไนส์ข้อมูลเฉพาะตัวในสถานที่ไปยัง cloud เราช่วยให้คุณโดยการเพิ่มผู้ดูแลระบบและผู้ใช้ในการสมัครใช้งานของคุณการกำหนดค่าข้อกำหนดเบื้องต้นการจัดการการตั้งค่าพรีเมี่ยม Azure โฆษณาการตั้งค่าไดเรกทอรี ซิงโครไนส์กับการรับรองความถูกต้องที่มีการจัดการและ FS โฆษณาที่ใช้เครื่องมือการเชื่อมต่อ AD Azure การกำหนดค่าผู้ใช้ทดสอบและการตรวจสอบกรณีการใช้งานหลักของคุณสำหรับการบริการ

การตั้งค่าพรีเมียมโฆษณา Azure รวมถึงการเปิดใช้งานลักษณะการทำงานต่อไปนี้:

-   การตั้งค่ารหัสผ่านบริการตนเอง Azure ไดเรกทอรีที่ใช้งานอยู่ (SSPR)

-   การรับรองความถูกต้องหลายปัจจัย azure (Azure MFA)

-   สูงสุดสาม (3) โปรแกรมประยุกต์บริการ (SaaS) การรวมกับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ Azure](https://azure.microsoft.com/marketplace/active-directory/)

-   การจัดสรรผู้ใช้โดยอัตโนมัติสำหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามที่แสดงไว้ในรายการการแนะนำการ[รวมแอป](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จำกัดเฉพาะการจัดเตรียมขาออกเท่านั้น

-   หน้าจอการเข้าสู่ระบบแบบกำหนดเองรวมทั้งโลโก้ข้อความและรูปภาพ

-   กลุ่มการบริการตนเองและแบบไดนามิก (กลุ่ม)

-   พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ Azure

-   Azure ที่ใช้งานอยู่ไดเรกทอรีเชื่อมต่อสุขภาพ

-   การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ Azure

-   ข้อตกลงในการใช้งานไดเรกทอรี Active Azure

-   การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ Azure

-   การจัดการข้อมูลประจำตัวของไดเรกทอรีที่ใช้งานอยู่ Azure

-   ความคิดเห็นการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ Azure

### <a name="enable-phase---intune"></a>เปิดใช้งานขั้นตอน-Intune

สำหรับ Intune เราแนะนำคุณผ่านการรับพร้อมที่จะใช้ Microsoft Intune ในการจัดการอุปกรณ์ ขั้นตอนที่แน่นอนจะขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและจะขึ้นอยู่กับอุปกรณ์เคลื่อนที่และการจัดการแอพบนมือถือของคุณ ขั้นตอนต่างๆอาจประกอบด้วย:

-   อนุญาตให้ผู้ใช้ปลายของคุณ นอกจากนี้เรายังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งานใบอนุญาตระดับเสียงสำหรับผู้เช่าระบบ cloud บริการของ Microsoft (ตามความจำเป็น)

-   การกำหนดค่าข้อมูลประจำตัวที่จะใช้โดย Intune ของคุณด้วยใช้ประโยชน์จากทั้งไดเรกทอรีที่มีอยู่ในสถานที่หรือข้อมูลเฉพาะตัวของ cloud

-   การเพิ่มผู้ใช้ลงในการสมัครสมาชิก Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างผู้ใช้และกลุ่มอุปกรณ์

-   การกำหนดค่าหน่วยงานจัดการอุปกรณ์เคลื่อนที่ของคุณ (MDM) ตามความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM เมื่อ Intune เป็นโซลูชัน MDM ของคุณเท่านั้น

-   การให้คำแนะนำ MDM สำหรับ:

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบนโยบายการจัดการ MDM

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการต่างๆเช่น:

        -   การปรับใช้โปรแกรมประยุกต์สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านการเชื่อมโยงเว็บหรือการเชื่อมโยงลึก

        -   นโยบายการเข้าถึงแบบมีเงื่อนไข

        -   การใช้งานของอีเมลเครือข่ายไร้สายและส่วนกำหนดค่าเครือข่ายส่วนตัวเสมือน (VPN) ถ้าคุณมีผู้มีสิทธิ์ออกใบรับรองที่มีอยู่, Wi-fi หรือโครงสร้างพื้นฐาน VPN ในองค์กรของคุณ

        -   การตั้งค่าการเชื่อมต่อ Microsoft Intune แลกเปลี่ยน (เมื่อเกี่ยวข้อง)

        -   การเชื่อมต่อไปยังคลังสินค้าของ Intune ข้อมูล

        -   รวม Intune ด้วย:
            -   ตัวแสดงทีมสำหรับความช่วยเหลือระยะไกล (ต้องใช้การสมัครสมาชิกของทีมงาน)

            -   โซลูชั่นการป้องกันการคุกคามแบบเคลื่อนที่ (MTD) (การป้องกันภัยคุกคามบนมือถือ subcription จำเป็นต้องใช้)

            -   ระบบการจัดการค่าใช้จ่ายเทเลคอม (โทรคมนาคมต้องสมัครเป็นสมาชิกโซลูชันการจัดการค่าใช้จ่าย)

            -   การป้องกันการคุกคามขั้นสูงของ windows Defender (Windows E5 หรือ Microsoft ๓๖๕ E5 สิทธิ์การใช้งานจำเป็น)

    -   การลงทะเบียนอุปกรณ์ของแต่ละ[แพลตฟอร์มที่ได้รับการสนับสนุน](https://technet.microsoft.com/library/dn600287.aspx)เพื่อ Intune

-   ให้คำแนะนำในการปกป้องแอพพลิเคชั่น:

    -   การกำหนดค่านโยบายการป้องกันโปรแกรมประยุกต์สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   การกำหนดค่านโยบายการเข้าถึงแบบมีเงื่อนไขสำหรับแอปที่มีการจัดการ

    -   การกำหนดเป้าหมายกลุ่มผู้ใช้ที่เหมาะสมกับนโยบายข้างต้นแหม่ม

    -   โดยใช้รายงานการใช้งานที่มีการจัดการ

-   ให้คำแนะนำในการจัดการเครื่องคอมพิวเตอร์:

    -   การติดตั้งซอฟต์แวร์ของไคลเอ็นต์ Intune (เมื่อจำเป็น)

    -   การใช้รายงานซอฟต์แวร์และฮาร์ดแวร์ที่มีอยู่ใน Intune

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุน Windows 10 การจัดการพีซีคลาสสิกด้วย Intune FastTrack สนับสนุน Windows 10 การจัดการอุปกรณ์ผ่าน Intune การจัดการอุปกรณ์เคลื่อนที่ (MDM)

#### <a name="windows-autopilot"></a>Windows ขี่

FastTrack สามารถช่วยให้คุณผ่านการจัดสรรอุปกรณ์ของคุณกับ Windows ขี่และ Intune โดยการให้อุปกรณ์ใหม่แก่ผู้ใช้ของคุณโดยไม่จำเป็นต้องสร้างบำรุงรักษาและใช้ภาพระบบปฏิบัติการที่กำหนดเองกับอุปกรณ์ของคุณ

FastTrack สนับสนุนสถานการณ์ขี่ต่อไปนี้:

- ระบบ**โฆษณา Azure บริการตนเอง:** อุปกรณ์เข้าร่วมโฆษณา Azure และลงทะเบียนใน Intune สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๐๑๗๐๓และรุ่นล่าสุด

- **บริการตนเองของไฮบริด AAD:** อุปกรณ์จะเข้าร่วมทั้งโฆษณาในสถานที่และโฆษณา Azure และลงทะเบียนใน Intune สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๐๑๘๐๙และรุ่นล่าสุด

- การเตรียมใช้งาน**ด้วยตนเอง:** อุปกรณ์จะเข้าร่วมโฆษณา Azure โดยอัตโนมัติ สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows ๑๘๐๙และรุ่นล่าสุด

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนสถานการณ์ขี่ที่เริ่มต้นจากตัวจัดการการตั้งค่าคอนฟิก

ขั้นตอนในการติดตั้ง Windows ขี่ขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและอาจรวมถึง:

- ตั้งค่าคอนฟิกและติดตั้ง Microsoft Intune สำหรับ Windows ขี่

- การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกโฆษณา Azure

- เพิ่มตราสินค้าของบริษัทของคุณลงในโฆษณา Azure

- สร้างและกำหนดอุปกรณ์ให้กับ Windows ขี่ส่วนกำหนดค่า (เช่นส่วนกำหนดค่า Windows ขี่ที่จำกัดการสร้างบัญชีผู้ดูแลท้องถิ่น)

- กำหนดเองออกกล่อง (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

- การกำหนดค่าการลงทะเบียนอัตโนมัติของ MDM ในโฆษณา Azure และ Intune

#### <a name="deploy-outlook-for-ios-and-android-securely"></a>ปรับใช้ Outlook สำหรับ iOS และ Android อย่างปลอดภัย

FastTrack สามารถช่วยให้คุณโดยการปรับใช้ Outlook สำหรับ iOS และ Android อย่างปลอดภัยในองค์กรของคุณเพื่อให้แน่ใจว่าผู้ใช้ของคุณมีปพลิเคชันที่จำเป็นทั้งหมดที่ติดตั้ง

ขั้นตอนในการปรับใช้ Outlook Mobile สำหรับ iOS และ Android ด้วย Intune จะขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและอาจรวมถึง:

- ดาวน์โหลด Outlook สำหรับ iOS และ Android, Microsoft Authenticator และแอพพลิเคชั่นของเว็บไซต์ของ Intune ผ่านทาง Apple App Store หรือ Google Play สโตร์
- นอกจากนี้ยังให้คำแนะนำเกี่ยวกับการตั้งค่า:
    - Outlook สำหรับ iOS และ Android, Microsoft Authenticator และการปรับใช้แอพลิเคชันของเว็บไซต์บริษัท Intune กับ Intune
    - นโยบายการป้องกันแอป
    - นโยบายการเข้าถึงแบบมีเงื่อนไข
    - นโยบายการกำหนดค่าโปรแกรมประยุกต์

    > [!IMPORTANT]
    > ทีม FastTrack ไม่สนับสนุนการรักษาความปลอดภัย Outlook สำหรับ iOS และ Android ด้วยการแลกเปลี่ยนอุปกรณ์เคลื่อนที่นโยบายกล่องจดหมาย

#### <a name="co-management"></a>การจัดการร่วม

FastTrack แนะนำคุณผ่านการเตรียมพร้อมที่จะพร้อมสำหรับการจัดการ Windows 10 อุปกรณ์ที่มีทั้งตัวจัดการการตั้งค่าคอนฟิกและ Intune ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณและอาจรวมถึง:

- อธิบายประโยชน์ของการจัดการร่วม

- ใบอนุญาตผู้ใช้ของคุณ FastTrack ยังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งานใบอนุญาตระดับเสียงสำหรับ Microsoft cloud บริการผู้เช่าของคุณ (ตามความจำเป็น)

- กำหนดค่าข้อมูลเฉพาะตัวที่ใช้โดย Intune ของคุณด้วยใช้งานไดเรกทอรีและ/

- การเพิ่มผู้ใช้ลงในการสมัครสมาชิก Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างผู้ใช้และกลุ่มอุปกรณ์

- ให้คำแนะนำเกี่ยวกับวิธีการย้ายจาก Intune รวมกับระบบศูนย์ตัวจัดการการตั้งค่าคอนฟิก (ไฮบริดสลี) เพื่อ Intune แบบสแตนด์อโลน

- ให้คำแนะนำเกี่ยวกับการตั้งค่าไดเรกทอรีที่ใช้งานอยู่ Azure สำหรับการลงทะเบียน MDM อัตโนมัติ

- ให้คำแนะนำการตั้งค่าการเข้าร่วมไดเรกทอรีที่ใช้งานอยู่ Azure ไฮบริดสลี

- ให้คำแนะนำเกี่ยวกับวิธีการตั้งค่าเกตเวย์การจัดการระบบคลาวด์

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่าคอนฟิก

- กำหนดค่าปริมาณงานที่ได้รับการสนับสนุนที่คุณต้องการสลับไปยัง Intune

- ติดตั้งไคลเอ็นต์ตัวจัดการการตั้งค่าคอนฟิกใน Intune การลงทะเบียนอุปกรณ์

- ให้คำแนะนำเกี่ยวกับวิธีการตรวจสอบกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

FastTrack ยังให้คำแนะนำเกี่ยวกับวิธีการขับรถที่ประสบความสำเร็จในการรับบริการที่มีสิทธิ์

#### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานเฟส–การป้องกันข้อมูล Azure

ลูกค้าให้คำแนะนำเกี่ยวกับวิธีการ: 

- เปิดใช้งานและกำหนดค่าผู้เช่า
- สร้างและตั้งค่าป้ายชื่อและนโยบาย
- ใช้การป้องกันข้อมูลกับเอกสาร 
- โดยอัตโนมัติจัดประเภทและป้ายชื่อข้อมูลในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทำงานอยู่บน Windows และการใช้ไคลเอ็นต์การป้องกันข้อมูล Azure
- ใช้แฟ้มในส่วนที่เหลือโดยใช้สแกนเนอร์การป้องกันข้อมูล Azure
- ตรวจสอบอีเมลในการส่งต่อโดยใช้กฎการไหลของจดหมายแลกเปลี่ยนแบบออนไลน์

นอกจากนี้คำแนะนำยังให้กับลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการการจัดการสิทธิ์ Azure Microsoft (Azure RMS), Office ๓๖๕การเข้ารหัสลับข้อความ (OME), และป้องกันข้อมูลสูญหาย (DLP)
