# <a name="onboarding-phases"></a>ขั้นตอนการออนบอร์ด

เมื่อคุณใช้[บริการที่มีสิทธิ์และแผน](M365-eligible-services-and-plans.md)เพื่อรับ Microsoft Azure Active Directory Premium, Microsoft Intunนและการป้องกันข้อมูล Azure พร้อมสําหรับการใช้งาน มีหลายขั้นตอนที่เกี่ยวข้องในกระบวนการ ส่วนต่อไปนี้อธิบายแต่ละขั้นตอนของกระบวนการออนบอร์ด

การขึ้นเครื่องบินมีสี่ขั้นตอนหลัก:

![สี่ขั้นตอนของกระบวนการเริ่มต้น FastTrack](./media/O365-Onboarding-Phases.png)


## <a name="initiate-phase"></a>เริ่มต้นเฟส

หลังจากที่คุณซื้อจํานวนที่เหมาะสมของสิทธิ์การใช้งาน ให้ทําตามคําแนะนําจากอีเมลยืนยันการซื้อเพื่อเชื่อมโยงสิทธิ์การใช้งานกับผู้เช่าที่มีอยู่หรือผู้เช่ารายใหม่ของคุณ จากนั้น Microsoft จะตรวจสอบสิทธิ์ของคุณสําหรับสิทธิประโยชน์ของศูนย์ FastTrack และพยายามติดต่อคุณเพื่อเสนอความช่วยเหลือบนระบบ

> [!NOTE]
> คุณยังสามารถขอความช่วยเหลือจาก[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)หากคุณพร้อมที่จะปรับใช้บริการเหล่านี้สําหรับองค์กรของคุณ

### <a name="to-request-assistance"></a>เมื่อต้องการขอความช่วยเหลือ

1. ลงชื่อเข้าใช้[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**ขอความช่วยเหลือสําหรับ Microsoft 365**จาก**การดําเนินการด่วน**ที่ด้านบนของหน้า Landing Page ของคุณ หรือโดยการเลือก**ขอความช่วยเหลือสําหรับ Microsoft 365**บนการ์ดการปรับใช้
3. กรอกข้อมูล**ในคําร้องขอความช่วยเหลือสําหรับฟอร์ม Microsoft 365**

เมื่อเริ่มการสนับสนุนบนระบบเราจะตั้งกําหนดการการประชุมแบบออนไลน์

คู่ค้าของ Microsoft ยังสามารถขอความช่วยเหลือผ่านทาง[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)ในนามของลูกค้า เมื่อต้องการทําเช่นนั้น:

1. ลงชื่อเข้าใช้[ไซต์ FastTrack](https://go.microsoft.com/fwlink/?linkid=780698)
2. เลือก**ลูกค้าของฉัน**
3. ค้นหาลูกค้าของคุณหรือเลือกจากรายชื่อลูกค้าของคุณ
4. เลือก**บริการ**
5. กรอกข้อมูล**ในคําร้องขอความช่วยเหลือสําหรับฟอร์ม Microsoft 365**

เมื่อการสนับสนุนการเตรียมพร้อมเริ่มต้นขึ้น FastTrack จะตั้งค่ากําหนดการการประชุมแบบออนไลน์กับคุณเพื่อหารือเกี่ยวกับกระบวนการเริ่มต้นดําเนินการ

![ขั้นตอนการเริ่มต้นการเริ่มการเริ่มทํางาน](./media/ft-initiate-phase.png)

## <a name="assess-phase"></a>ประเมินระยะ

เมื่อเริ่มต้นกระบวนการเริ่มต้นขึ้น FastTrack Center จะทํางานร่วมกับคุณเพื่อประเมินสภาพแวดล้อมต้นทางและข้อกําหนดของคุณ เครื่องมือจะเรียกใช้เพื่อประเมินสภาพแวดล้อมของคุณ และ FastTrack ผู้เชี่ยวชาญแนะนําคุณผ่านการประเมินของคุณในสถานที่ Active Directory, เบราว์เซอร์อินเทอร์เน็ต, ระบบปฏิบัติการของอุปกรณ์ไคลเอ็นต์, ระบบชื่อโดเมน (DNS), เครือข่ายโครงสร้างพื้นฐานและระบบ identity เพื่อตรวจสอบว่าการเปลี่ยนแปลงใด ๆ ที่จําเป็นสําหรับการ onboarding

นอกจากนี้ FastTrack Center ยังเชื่อมโยงคุณด้วยคําแนะนําเกี่ยวกับวิธีการผลักดันการนําบริการที่มีสิทธิ์มาใช้ให้สําเร็จ

เราจัดเตรียมแผนแก้ไขที่นําสภาพแวดล้อมต้นทางของคุณขึ้นตามความต้องการขั้นต่ําสําหรับการเริ่มต้นที่ประสบความสําเร็จใน EMS หรือบริการคลาวด์แต่ละแห่ง นอกจากนี้เรายังตั้งค่าสายจุดตรวจที่เหมาะสมสําหรับระยะการแก้ไข

![ขั้นตอนการเริ่มใช้ประเมิน](./media/ft-assess-phase.png)

## <a name="remediate-phase"></a>เฟสของการแก้ไข
คุณทํางานในแผนการแก้ไขในสภาพแวดล้อมของแหล่งที่มาของคุณเพื่อให้คุณตอบสนองความต้องการสําหรับ onboarding และการนําแต่ละบริการ (ตามความจําเป็น)

![ขั้นตอนการแก้ไขการเตรียมพร้อม](./media/ft-remediate-phase.png)

ก่อนที่คุณจะเริ่มขั้นตอนการเปิดใช้งานเราร่วมกันตรวจสอบผลของกิจกรรมการแก้ไขเพื่อให้แน่ใจว่าคุณพร้อมที่จะดําเนินการต่อ

## <a name="enable-phase"></a>เปิดใช้งานเฟส
เมื่อกิจกรรมการแก้ไขทั้งหมดเสร็จสมบูรณ์โครงการจะเลื่อนไปกําหนดค่าโครงสร้างพื้นฐานหลักสําหรับการใช้บริการและการจัดสรรบริการเมฆ EMS ที่มีสิทธิ์แต่ละ

**เปิดใช้งานขั้นตอน - ความสามารถหลัก**

การเตรียมพร้อมหลักเกี่ยวข้องกับการจัดเตรียมบริการและการรวมผู้เช่าและเอกลักษณ์ นอกจากนี้ยังมีขั้นตอนสําหรับการให้บริการออนไลน์บนพื้นฐานเช่น Azure AD Premium, Intun, และการป้องกันข้อมูล Azure

![การเตรียมพร้อมสําหรับขั้นตอน - ความสามารถหลัก](./media/ft-enable-phase-core-01.png)

![การเตรียมพร้อมสําหรับขั้นตอน - ความสามารถหลัก](./media/ft-enable-phase-core-02.png)
> [!NOTE]
> WAP ย่อมาจากพร็อกซีโปรแกรมประยุกต์บนเว็บ SSL ย่อมาจากเลเยอร์ซ็อกเก็ตที่ปลอดภัย SDS ย่อมาจากการซิงค์ข้อมูลของโรงเรียน สําหรับข้อมูลเพิ่มเติมเกี่ยวกับ SDS ให้ดูที่[ยินดีต้อนรับสู่ Microsoft School Data Sync](https://go.microsoft.com/fwlink/?linkid=871480)

> [!NOTE]
> วิธีการรับรองความถูกต้องที่มีการจัดการรวมถึง แต่ไม่จํากัดเพียงการซิงโครไนส์แฮชรหัสผ่าน การรวมข้อมูลประจําตัวเป็นกิจกรรมครั้งเดียว และไม่รวมการย้ายหรือปลดระวางของวิธีการตรวจสอบความถูกต้องที่มีอยู่ เช่น การจัดการ หรือ fedderated

### <a name="enable-phase---azure-ad-premium"></a>เปิดใช้งานเฟส - พรีเมี่ยมโฆษณา Azure

สภาพแวดล้อมแบบพรีเมี่ยมโฆษณา Azure สามารถตั้งค่า โดยใช้ Azure ไดเรกทอรีที่ใช้งานอยู่เชื่อมต่อไดเรกทอรีไดเรกทอรีไดเรกทอรีไดเรกทอรีซิงโครไนส์ไดเรกทอรีและบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (FS โฆษณา) (ตามความจําเป็น)

สําหรับสถานการณ์พิเศษโฆษณา Azure ซึ่งรวมถึงการซิงโครไนส์ข้อมูลเฉพาะตัวในสถานที่ไปยัง Cloud เราช่วยคุณโดยการเพิ่มผู้ดูแลระบบ IT และผู้ใช้ในการสมัครสมาชิกของคุณ

การตั้งค่าพรีเมี่ยมโฆษณา azure รวมถึงการเปิดใช้งานลักษณะการทํางานต่อไปนี้:

-   การตั้งค่ารหัสผ่านบริการตนเองของ azure ไดเรกทอรีที่ใช้งานอยู่ (SSPR)

-   การรับรองความถูกต้องด้วยหลายปัจจัยของ azure (MFA Azure)

-   ซอฟต์แวร์ที่รวมเป็นบริการ (SaaS) กับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ของ Azure](https://azure.microsoft.com/marketplace/active-directory/)

-   การจัดสรรผู้ใช้โดยอัตโนมัติสําหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามรายการ[บทแนะนําการรวมแอป](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จํากัด เฉพาะการจัดสรรขาออกเท่านั้น

-   หน้าจอการเข้าสู่ระบบที่กําหนดเองรวมถึงโลโก้ข้อความและภาพ

-   กลุ่มบริการตนเองและแบบไดนามิก (กลุ่ม)

-   พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ azure

-   ความสมบูรณ์ของการเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ของ azure

-   ข้อกําหนดการใช้งานไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การป้องกันข้อมูลประจําตัวของไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การจัดการข้อมูลประจําตัวที่ได้รับสิทธิพิเศษของ azure ไดเรกทอรีที่ใช้งานอยู่

-   ความคิดเห็นการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ azure

-   การป้องกันรหัสผ่านของไดเรกทอรีที่ใช้งานอยู่ของ azure

-   ไดเรกทอรีที่ใช้งานอยู่ของ azure B2B.

### <a name="enable-phase---intune"></a>เปิดใช้งานเฟส - Intun

สําหรับ Intun เน เราแนะนําคุณตลอดการเตรียมพร้อมใช้ Microsoft Intun เนเพื่อจัดการอุปกรณ์ ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมต้นทางของคุณ และขึ้นอยู่กับอุปกรณ์เคลื่อนที่และความต้องการในการจัดการแอปบนอุปกรณ์เคลื่อนที่ของคุณ ขั้นตอนต่างๆ มีดังนี้

-   การให้สิทธิ์ผู้ใช้ปลายทางของคุณ เรายังให้ความช่วยเหลือเกี่ยวกับวิธีเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสําหรับผู้เช่าบริการ Cloud Microsoft ของคุณ (ตามความจําเป็น)

-   การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intun

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intuner ของคุณ

-   การกําหนดค่าสิทธิ์การจัดการอุปกรณ์เคลื่อนที่ (MDM) ของคุณ ตามความต้องการการจัดการของคุณ รวมถึง:

    -   การตั้งค่า Intun เป็นอํานาจ MDM ของคุณเมื่อ Intun เป็นโซลูชัน MDM ของคุณเท่านั้น

-   ให้คําแนะนํา MDM สําหรับ:

    -   การกําหนดค่ากลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

    -   การกําหนดค่านโยบายการจัดการ MDM และบริการต่างๆ เช่น

        -   การปรับใช้โปรแกรมประยุกต์สําหรับแต่ละแพลตฟอร์มที่สนับสนุนผ่านการเชื่อมโยงเว็บหรือการเชื่อมโยงลึก

        -   นโยบายการเข้าถึงตามเงื่อนไข

        -   การปรับใช้โปรไฟล์อีเมล เครือข่ายไร้สาย และโปรไฟล์เครือข่ายส่วนตัวเสมือน (VPN) ถ้าคุณมีผู้ให้บริการออกใบรับรอง Wi-Fi หรือโครงสร้างพื้นฐาน VPN ที่มีอยู่ในองค์กรของคุณ

        -   การตั้งค่าตัวเชื่อมต่อการแลกเปลี่ยน Intun เน Microsoft (เมื่อเกี่ยวข้อง)

        -   การเชื่อมต่อกับคลังข้อมูล Intun

        -   การผสานรวม Intun กับ:
            -   ตัวแสดงทีมสําหรับความช่วยเหลือระยะไกล (จําเป็นต้องมีการสมัครสมาชิกของตัวแสดงทีม)

            -   มือถือภัยคุกคามป้องกัน (MTD) โซลูชั่นพันธมิตร (มือถือภัยคุกคามป้องกันการสมัครเป็นสิ่งจําเป็น)

            -   โซลูชันการจัดการค่าใช้จ่ายโทรคมนาคม (ต้องสมัครรับโซลูชันการจัดการค่าใช้จ่ายโทรคมนาคม)

            -   การป้องกันภัยคุกคามขั้นสูงของ Microsoft Defender (ต้องมีสิทธิ์การใช้งาน Windows E5 หรือ Microsoft 365 E5)

    -   การลงทะเบียนอุปกรณ์ของแต่ละ[แพลตฟอร์มที่รองรับ](https://technet.microsoft.com/library/dn600287.aspx)กับ Intun

-   ให้คําแนะนําในการป้องกันแอป:

    -   การกําหนดค่านโยบายการป้องกันแอปสําหรับแต่ละแพลตฟอร์มที่รองรับ

    -   การกําหนดค่านโยบายการเข้าถึงแบบมีเงื่อนไขสําหรับแอปที่มีการจัดการ

    -   การกําหนดเป้าหมายกลุ่มผู้ใช้ที่เหมาะสมด้วยนโยบาย MAM ข้างต้น

    -   การใช้รายงานการใช้งานแอปพลิเคชันที่มีการจัดการ

-   ให้คําแนะนําการจัดการพีซีเกี่ยวกับ:

    -   การติดตั้งซอฟต์แวร์ไคลเอ็นต์ Intun

    -   การใช้ซอฟต์แวร์และฮาร์ดแวร์รายงานที่มีอยู่ใน Intun

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 ด้วย Intun FastTrack สนับสนุนเฉพาะการจัดการอุปกรณ์ Windows 10 ผ่านการจัดการอุปกรณ์เคลื่อนที่ Intun

#### <a name="windows-autopilot"></a>หน้าต่างออโตไพลอต

FastTrack สามารถช่วยคุณในการจัดเตรียมอุปกรณ์ด้วย Windows Autopilot และ Intuni โดยให้อุปกรณ์ใหม่แก่ผู้ใช้ของคุณโดยไม่จําเป็นต้องสร้าง บํารุงรักษา และใช้อิมเมจระบบปฏิบัติการแบบกําหนดเองกับอุปกรณ์ของคุณ

FastTrack สนับสนุนสถานการณ์ออโตไพลอตต่อไปนี้:

- **บริการตนเองโฆษณา azure:** อุปกรณ์ที่เข้าร่วม Azure AD และลงทะเบียนเข้า Intun สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1703 และรุ่นล่าสุด

- **ไฮบริด AAD บริการตนเอง:** อุปกรณ์ที่เข้าร่วมทั้งโฆษณาในสถานที่และโฆษณา Azure และลงทะเบียนเข้า Intun สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows 10 1809 และรุ่นล่าสุด

- **การจัดเตรียมด้วยตนเอง:** อุปกรณ์เข้าร่วมโฆษณา Azure โดยอัตโนมัติ สถานการณ์นี้ได้รับการสนับสนุนเมื่อใช้ Windows 1809 และรุ่นล่าสุด

    > [!IMPORTANT]
    > FastTrack ไม่สนับสนุนสถานการณ์ออโตไพลอตที่เริ่มต้นจากตัวจัดการการตั้งค่าคอนฟิก

ขั้นตอนการตั้งค่า Windows Autopilot ขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณ และอาจรวมถึง:

- กําหนดค่าและตั้งค่าอินทูนไมโครซอฟท์สําหรับ Windows หม้อแปลงไฟล

- กําหนดค่ากลุ่มแบบไดนามิกของ Azure AD

- เพิ่มการสร้างแบรนด์บริษัทของคุณลงในโฆษณา Azure

- สร้างและกําหนดอุปกรณ์ให้กับส่วนกําหนดค่าระบบอัตโนมัติของ Windows (เช่น ส่วนกําหนดค่า Windows Autopilot ที่จํากัดการสร้างบัญชีผู้ดูแลระบบภายในเครื่อง)

- กําหนดประสบการณ์นอกกรอบ (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

- การกําหนดค่าการลงทะเบียนอัตโนมัติ MDM ใน Azure AD และ Intun

#### <a name="deploy-outlook-for-ios-and-android-securely"></a>ปรับใช้ Outlook สําหรับ iOS และ Android อย่างปลอดภัย

FastTrack สามารถช่วยคุณได้โดยการปรับใช้ Outlook สําหรับ iOS และ Android อย่างปลอดภัยในองค์กรของคุณเพื่อให้แน่ใจว่าผู้ใช้ของคุณมีแอปที่จําเป็นทั้งหมดติดตั้งไว้

ขั้นตอนในการปรับใช้ Outlook Mobile สําหรับ iOS และ Android อย่างปลอดภัยด้วย Intun

- ดาวน์โหลด Outlook สําหรับ iOS และ Android, ไมโครซอฟท์ตัวรับรองความถูกต้องและแอปพอร์ทัลบริษัท Intuni ผ่านแอปเปิ้ล App Store หรือ Google Play สโตร์
- นอกจากนี้ยังให้คําแนะนําเกี่ยวกับการตั้งค่า:
    - Outlook สําหรับ iOS และ Android, ตัวรับรองความถูกต้องของ Microsoft และการปรับใช้แอพลิเคชันพอร์ทัลบริษัท Intun
    - นโยบายการป้องกันแอป
    - นโยบายการเข้าถึงตามเงื่อนไข
    - นโยบายการกําหนดค่าแอป

    > [!IMPORTANT]
    > ทีม FastTrack ไม่สนับสนุนการรักษาความปลอดภัย Outlook สําหรับ iOS และ Android ด้วยนโยบายกล่องจดหมายอุปกรณ์เคลื่อนที่ของ Exchange

#### <a name="cloud-attach"></a>ติดคลาวด์

FastTrack แนะนําคุณตลอดการเตรียมพร้อมสําหรับสภาพแวดล้อมตัวจัดการการกําหนดค่าที่มีอยู่กับ Intun ขั้นตอนที่แน่นอนขึ้นอยู่กับสภาพแวดล้อมของแหล่งที่มาของคุณ ขั้นตอนเหล่านี้อาจรวมถึง:

- อธิบายประโยชน์ของตัวจัดการการกําหนดค่าที่ติดกับระบบคลาวด์ด้วย Intun

- การให้สิทธิ์ผู้ใช้ปลายทางของคุณ นอกจากนี้ FastTrack ยังให้ความช่วยเหลือเกี่ยวกับวิธีการเปิดใช้งานสิทธิ์การใช้งานไดรฟ์ข้อมูลสําหรับผู้เช่าบริการ Cloud Microsoft ของคุณ (ตามความจําเป็น)

- การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intun

- การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intuner ของคุณ

- การเปิดใช้งานการแนบกับระบบคลาวด์ในคอนโซลการจัดการการตั้งค่า

- ให้คําแนะนําในการติดตั้งการรวมไดเรกทอรีที่ใช้งานอยู่ของ Azure ไฮบริดสลี

- ให้คําแนะนําเกี่ยวกับการตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับการลงทะเบียน MDM อัตโนมัติ

- ให้คําแนะนําเกี่ยวกับวิธีการตั้งค่าเกตเวย์การจัดการระบบคลาวด์

- การกําหนดค่าปริมาณงานที่สนับสนุนที่คุณต้องการสลับไปยัง Intun

- การติดตั้งไคลเอ็นต์ตัวจัดการการตั้งค่าคอนฟิกบนอุปกรณ์ที่ลงทะเบียน Intun

นอกจากนี้ FastTrack ยังให้คําแนะนําเกี่ยวกับวิธีการผลักดันการนําบริการที่มีสิทธิ์มาใช้ให้ประสบความสําเร็จ

#### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน – การป้องกันข้อมูล Azure

ลูกค้าได้รับคําแนะนําเกี่ยวกับวิธีการ: 

- เปิดใช้งานและกําหนดค่าผู้เช่า
- สร้างและตั้งค่าป้ายชื่อและนโยบาย
- นําการป้องกันข้อมูลไปใช้กับเอกสาร 
- จัดประเภทและป้ายชื่อข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทํางานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure
- ใช้แฟ้มที่เหลือโดยใช้สแกนเนอร์การป้องกันข้อมูล Azure
- ตรวจสอบอีเมลในระหว่างการขนส่งโดยใช้กฎการรับส่งจดหมายแบบออนไลน์ของ Exchange

คําแนะนํายังให้ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ของ Microsoft Azure (Azure RMS), การเข้ารหัสลับข้อความ Office 365 (OME) และข้อมูลการป้องกันการสูญหาย (DLP)
