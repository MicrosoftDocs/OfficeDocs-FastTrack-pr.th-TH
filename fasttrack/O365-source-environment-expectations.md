# <a name="source-environment-expectations"></a>ข้อกำหนดของสภาพแวดล้อมต้นทาง

ประโยชน์ของ FastTrack Center ให้คําแนะนําสําหรับการตั้งค่าระดับการผสานรวมกับสภาพแวดล้อมของแหล่งที่มาของคุณ (ตัวอย่างเช่น ถ้าคุณมีบริการในสภาพแวดล้อมแหล่งข้อมูลที่คุณต้องการย้ายไปยัง Office 365)
  
> [!NOTE]
> ถ้าจําเป็นต้องรวม 
  
ตารางต่อไปนี้แสดงความคาดหวังสําหรับสภาพแวดล้อมแหล่งข้อมูลที่มีอยู่ของคุณสําหรับ onboarding\*

|**หลักการเตรียมพร้อม** | ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่กับฟอเรสต์ระดับการทํางานที่ตั้งค่าเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:  <br/>      ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เดียว  <br/>    ฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่เดียวและฟอเรสต์ทรัพยากร (อัตราแลกเปลี่ยนและ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  ฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่หลายและฟอเรสต์ทรัพยากร (อัตราแลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์หนึ่งเป็นฟอเรสต์บัญชี Active Directory ส่วนกลางที่มีการแลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจอย่างใดอย่างหนึ่ง  <br/>  ฟอเรสต์บัญชี Active Directory หลาย แต่ละ องค์กรแลกเปลี่ยน ของตนเอง <br/> <br/> **หมายเหตุ**  *สําหรับสถานการณ์ไดเรกทอรีที่ใช้งานอยู่ฟอเรสต์หลายถ้า Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจถูกปรับใช้ นั้นต้องถูกปรับใช้ในฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เหมือนกับอัตราแลกเปลี่ยน*   <br/>  <br/>      **หมายเหตุ***เมื่อใช้งานฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่หลายองค์กร Exchange ในการกําหนดค่าแบบหลายไฮบริดสลีอัตราแลกเปลี่ยน namespaces ชื่อหลัก (UPN) ที่ใช้ร่วมกันระหว่างฟอเรสต์ต้นทางไม่ได้รับการสนับสนุน ควรแยกพื้นที่ของ SMTP หลักระหว่างองค์กร Exchange สําหรับข้อมูลเพิ่มเติม ให้ดูที่[การปรับใช้ไฮบริดสลีกับฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่หลาย](https://go.microsoft.com/fwlink/?linkid=845444)*       <br/>   <br/>   **หมายเหตุ**  *สําหรับการกําหนดค่าฟอเรสต์หลายทั้งหมด FS โฆษณาจะอยู่นอกขอบเขตสําหรับประโยชน์ศูนย์ FastTrack*           |
