# <a name="source-environment-expectations"></a>ข้อกำหนดของสภาพแวดล้อมต้นทาง

ประโยชน์จากศูนย์ FastTrack จะให้คําแนะนําแก่คุณในการตั้งค่าระดับการรวมเข้ากับสภาพแวดล้อมต้นทางของคุณ (ตัวอย่างเช่น ถ้าคุณมีบริการในสภาพแวดล้อมต้นทางที่คุณต้องการย้ายไปยัง Office 365 อยู่แล้ว)
  
> [!NOTE]
> ถ้าจําเป็นต้องมีการรวม 
  
ตารางต่อไปนี้แสดงความคาดหวังสําหรับสภาพแวดล้อมต้นทางที่มีอยู่สําหรับการเตรียมพร้อม\*

|**การเตรียมพร้อมหลัก** | ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่กับระดับฟอเรสต์การทํางานที่ตั้งค่าเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:  <br/>      ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เดียว  <br/>    ฟอเรสต์และทรัพยากรฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่เดียว (Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  หลายฟอเรสต์บัญชี Active Directory และฟอเรสต์ทรัพยากร (แลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  หลายฟอเรสต์บัญชี Active Directory กับฟอเรสต์เป็นฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่แบบรวมศูนย์ที่มีการแลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ  <br/>  หลายฟอเรสต์บัญชี Active Directory แต่ละองค์กรแลกเปลี่ยนของตนเอง <br/> <br/> **หมายเหตุ**  *สําหรับหลายฟอเรสต์ที่ใช้งานอยู่ไดเรกทอรีสถานการณ์ ถ้า Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจจะปรับใช้ นั้นต้องถูกปรับใช้ในฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เหมือนกับการแลกเปลี่ยน*   <br/>  <br/>      **หมายเหตุ***เมื่อนํามาใช้หลายฟอเรสต์ Active Directory กับองค์กรแลกเปลี่ยนหลายในการกําหนดค่าแบบหลายไฮบริดอัตราแลกเปลี่ยน namespaces ชื่อผู้ใช้หลักที่ใช้ร่วมกัน (UPN) ระหว่างฟอเรสต์ต้นทางไม่ได้รับการสนับสนุน ช่องว่างของ SMTP หลักระหว่างองค์กร Exchange ควรถูกแยกออกด้วย สําหรับข้อมูลเพิ่มเติม โปรดดูที่[การปรับใช้ไฮบริดสลีกับฟอเรสต์ Active Directory หลาย](https://go.microsoft.com/fwlink/?linkid=845444)*       <br/>   <br/>   **หมายเหตุ**  *สําหรับการตั้งค่าคอนฟิกฟอเรสต์หลายหลาย*           |
