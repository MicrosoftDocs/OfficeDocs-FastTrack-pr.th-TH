# <a name="source-environment-expectations-for-office-365-us-government"></a>ความคาดหวังต่อสภาพแวดล้อมต้นทางของ Office 365 ของรัฐบาลสหรัฐอเมริกา

สิทธิประโยชน์ของศูนย์ FastTrack จะให้แนวทางสําหรับคุณในการตั้งค่าระดับการรวมกับสภาพแวดล้อมต้นทางของคุณ (ตัวอย่างเช่น ถ้าคุณมีบริการในสภาพแวดล้อมต้นทางที่คุณต้องการย้ายไปยัง Office 365 อยู่แล้ว)
  
> [!NOTE]
> ถ้าต้องรวม สภาพแวดล้อมต้นทางของคุณต้องอยู่ที่ระดับต่สุดของแอปพลิเคชันนั้น 
  
ตารางต่อไปนี้แสดงความคาดหวังของสภาพแวดล้อมต้นทางที่มีอยู่ของคุณในการออนบอร์ด

|**กิจกรรม**|**ความคาดหวังของสภาพแวดล้อมต้นทาง**|
|:-----|:-----|
|การออนบอร์ดหลัก | Active Directory ฟอเรสต์ที่ตั้งค่าระดับฟอเรสต์ฟังก์ชันเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:  <br/>      ฟอเรสต์ Active Directory เดี่ยว  <br/>    บัญชีผู้ใช้ Active Directory ฟอเรสต์เดียวและฟอเรสต์ทรัพยากร (การpologies ของบัญชีผู้ใช้ Active Directory และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business)  <br/>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์และฟอเรสต์ทรัพยากร (การปลอมแปลง Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business)  <br/>  หลายฟอเรสต์บัญชีผู้ใช้ Active Directory กับหนึ่งในฟอเรสต์เป็นฟอเรสต์บัญชีผู้ใช้ Active Directory ส่วนกลางที่มี Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business  <br/>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์ แต่ละฟอเรสต์มี Exchange องค์กรของตนเอง <br/> <br/> **หมายเหตุ** ถ้ามีการปรับใช้สถานการณ์สมมติของ Active Directory แบบหลายฟอเรสต์ ถ้า *Lync 2010, Lync 2013* หรือ Skype for Business ถูกปรับใช้ในฟอเรสต์ Active Directory เดียวกันกับ Exchange     <br/>  <br/>      **หมายเหตุ** เมื่อปรับใช้ฟอเรสต์ Active Directory หลายฟอเรสต์กับองค์กร Exchange หลายองค์กรในการกําหนดค่าไฮบริดหลายแบบ Exchange Name หลักของผู้ใช้ที่แชร์ *(UPN) ระหว่างฟอเรสต์ต้นฉบับไม่ได้รับการสนับสนุน Namespace SMTP หลักระหว่างองค์กร Exchange ควรแยกออกจากกันด้วย For more information, see [Hybrid deployments with multiple Active Directory forests](https://go.microsoft.com/fwlink/?linkid=845444).*       <br/>   <br/>   **หมายเหตุ**  *For all multiple forests configurations, AD FS deployment is out of scope for the FastTrack Center Benefit.*           |
|การออนบอร์ด Exchange Online | สภาพแวดล้อมของคุณต้องมีอย่างน้อยหนึ่งระดับขั้นต่อไปนี้:  <br/>  องค์กร Exchange เดียวหรือหลายองค์กรที่มี Exchange Server 2003 เป็นต้นไป  <br/>  สภาพแวดล้อมอีเมลที่สามารถใช้งาน Internet Message Access Protocol (IMAP) แบบเดี่ยวได้  <br/>  สภาพแวดล้อม G Suite เดียว (Gmail, ที่ติดต่อ และปฏิทินเท่านั้น)             |
|การออนบอร์ด SharePoint Online และ OneDrive for Business  | **SharePoint แบบไฮบริด**  <br/>  การกําหนดค่า SharePoint แบบไฮบริดรวมถึงการกําหนดค่าการค้นหาแบบไฮบริด ไซต์ การจัดโครงสร้าง ชนิดเนื้อหา OneDrive for Business ตัวเปิดใช้แอปเพิ่มเติม ไซต์เอกซ์ทราเน็ต และการสร้างไซต์แบบบริการตนเองที่เชื่อมต่อจากภายในองค์กรไปยังสภาพแวดล้อม SharePoint Online เป้าหมายเดียว เมื่อต้องการเปิดใช้งาน SharePoint แบบไฮบริด คุณต้องมีสภาพแวดล้อม SharePoint Server ภายในองค์กรอย่างใดอย่างหนึ่งต่อไปนี้:  <br/> <br/> **_SharePoint Server 2013_*_ <br/> การอัปเดตสาธารณะเดือนมิถุนายน 2017 (PU) ของชนิดเนื้อหาแบบไฮบริด For more information, see [Configure hybrid SharePoint taxonomy and hybrid content types](https://go.microsoft.com/fwlink/?linkid=853547).  PU ของเดือนมีนาคม 2017 for การกําหนดค่าการสร้างไซต์ด้วยตนเอง การสร้างไซต์ด้วยตนเองแบบไฮบริดได้รับ <br/> การสนับสนุนกับ SharePoint Server 2013 เท่านั้น For more information, see [Hybrid self-service site creation](https://go.microsoft.com/fwlink/?linkid=846015).  <br/>PU ในเดือนพฤศจิกายน 2016 เป็นต้นไปพร้อมกับ Taxonomy แบบไฮบริด For more information, see [Plan hybrid SharePoint taxonomy](https://go.microsoft.com/fwlink/?linkid=844867).  <br/>PU ของเดือนกรกฎาคม 2016 เป็นต้นไปในสถานการณ์สมมติการกําหนดค่าแบบไฮบริดอื่นๆ ทั้งหมด  <br/> <br/> _* หมายเหตุ***สถานการณ์สมมติ SharePoint Server 2013 แบบไฮบริดได้รับการสนับสนุนกับ SharePoint Server 2013 เท่านั้น สถานการณ์สมมติเหล่านี้ไม่ได้รับการสนับสนุนใน SharePoint Foundation 2013*    <br/>   <br/>    ***SharePoint Server 2016** _  <br/>  PU ในเดือนมิถุนายน 2017 for hybrid content types For more information, see [Configure hybrid SharePoint taxonomy and hybrid content types](https://go.microsoft.com/fwlink/?linkid=853547).  <br/>  PU 2016 พฤศจิกายน (Feature Pack 1) ที่มี Taxonomy แบบไฮบริด For more information, see [Plan hybrid SharePoint taxonomy](https://go.microsoft.com/fwlink/?linkid=844867).  <br/>  PU ของเดือนกรกฎาคม 2016 เป็นต้นไปในสถานการณ์สมมติการกําหนดค่าแบบไฮบริดอื่นๆ ทั้งหมด  <br/><br/> _ *การ* อัปเกรดสภาพแวดล้อม SharePoint ภายในองค์กรเป็น *   *SharePoint Server 2013 หรือ SharePoint Server 2016 ไม่มีให้ในสิทธิประโยชน์ของศูนย์ FastTrack For more information, see [Minimum public update levels for SharePoint hybrid features](https://go.microsoft.com/fwlink/?linkid=853548).*             |
|การออนบอร์ด Skype for Business Online  | **การประเมินเครือข่าย**  <br/>  การตรวจสอบพอร์ตและจุดสิ้นสุด  <br/>  ตรวจสอบคุณภาพการเชื่อมต่อ  <br/>  การประเมินแบนด์วิดท์  <br/><br/>  **Lync แบบไฮบริด**  <br/>  ฟอเรสต์ Active Directory ภายในองค์กรแบบเดียว  <br/>  สภาพแวดล้อม Lync 2010 Server ที่มีเครื่องมือผู้ดูแลระบบ Lync 2013 หรือเครื่องมือผู้ดูแลระบบ Skype for Business 2015 และบทบาทเซิร์ฟเวอร์ขอบของ Lync 2010  <br/>  สภาพแวดล้อม Lync 2013 Server และบทบาทเซิร์ฟเวอร์ขอบของ Lync 2013  <br/><br/>  **Skype for Business Online แบบไฮบริด**  <br/>  ฟอเรสต์ Active Directory ภายในองค์กรแบบเดียว  <br/>  ฟอเรสต์บัญชี Active Directory เดียวเป็นต้นไป และฟอเรสต์ทรัพยากร (การpologies Exchange และ/หรือ Skype for Business)  <br/>  หลายฟอเรสต์บัญชี Active Directory ที่มีฟอเรสต์เดียวเป็นฟอเรสต์บัญชี Active Directory แบบรวมศูนย์ที่มี Exchange และ/หรือ Skype for Business ในฟอเรสต์นั้น  <br/>  สภาพแวดล้อม Skype for Business Server 2015 รวมถึงบทบาทเซิร์ฟเวอร์ขอบของ Skype for Business  <br/><br/> หมายเหตุ บริการเพิ่มเติมนี้เป็นบริการเพิ่มเติมที่ใช้กําหนดค่าและการตรวจสอบความถูกต้องของงานโดเมนแยก (ไฮบริด) และไม่รวมการแนะนําคอมโพเนนต์ภายในองค์กร (ตัวอย่างเช่น เครื่องมือผู้ดูแลระบบ *Lync 2013 หรือเซิร์ฟเวอร์ขอบของ Skype for Business Online หรือ Lync 2010, Lync 2013* หรือเซิร์ฟเวอร์ขอบของ Skype for Business)      <br/><br/>        **อุปกรณ์ห้องประชุม**  <br/>  การสร้างบัญชีออนไลน์ที่ต้องใช้ในอุปกรณ์ห้องประชุมที่ได้รับการสนับสนุนจะแสดงอยู่บนแท็บ ระบบห้องประชุม ในแค็ตตาล็อก[โซลูชันของ Skype for Business](https://go.microsoft.com/fwlink/?LinkId=615775)  <br/><br/>  **การเปิดใช้งานการประชุมทางเสียง**  <br/>  การตั้งค่าองค์กรของการตั้งค่าบริดจ์การประชุมเริ่มต้น  <br/>  การมอบหมายบริดจ์การประชุมให้กับผู้ใช้ที่มีสิทธิ์การใช้งาน  <br/><br/>  **การเปิดใช้งานแนวทางระบบโทรศัพท์และแผนการโทร (สหรัฐอเมริกาเท่านั้น – ไม่พร้อมใช้งานในแผน GCC High หรือ DoD)**  <br/>  การตั้งค่าองค์กรในการตั้งค่าเริ่มต้นของ Cloud Voice  <br/>  การมอบหมายหมายเลขให้กับผู้ใช้ที่มีสิทธิ์การใช้งาน  <br/>  Local number porting guidance through user interface (UI) up to 999.  <br/>  การสนับสนุนการร้องขอบริการพอร์ตหมายเลขในพื้นที่ (SR) มากกว่า 999  <br/><br/>  **การเปิดใช้งานการแนะนําการออกอากาศการประชุม Skype for Business (ไม่พร้อมใช้งานในแผน GCC High หรือ DoD)**  <br/>  การตั้งค่าองค์กรเพื่อติดต่อกับภายนอกด้วยบริการการออกอากาศการประชุม   |
|การออนบอร์ดของ Microsoft Teams (ไม่มีให้บริการในแผน GCC High หรือ DoD) | ข้อมูลเฉพาะตัวที่เปิดใช้งานใน Azure Active Directory for Office 365  <br/>  ผู้ใช้เปิดใช้งาน SharePoint Online  <br/>  กล่องจดหมาย Exchange จะแสดง (ออนไลน์และ/หรือภายในองค์กรในการกําหนดค่าไฮบริดของ Exchange)  <br/>  เปิดใช้งานแล้วใน กลุ่ม Office 365  <br/><br/> **หมายเหตุ**  *ถ้าไม่มีการมอบหมายและเปิดใช้งานผู้ใช้ด้วยสิทธิ์การใช้งาน SharePoint Online พวกเขาจะไม่มีที่เก็บข้อมูล OneDrive for Business ใน Office 365 การแชร์ไฟล์จะยังคงสามารถใช้งานในแชนเนลได้ แต่ผู้ใช้ไม่สามารถแชร์ไฟล์ในการแชทได้ถ้าไม่มีที่เก็บข้อมูล OneDrive for Business ใน Office 365 Microsoft Teams ไม่สนับสนุน SharePoint ภายในองค์กร*   <br/> <br/>       **หมายเหตุ**  *สถานะเหมาะสมคือให้ผู้ใช้ทุกคนมีกล่องจดหมายของพวกเขาอยู่ใน Exchange Online ผู้ใช้ที่มีกล่องจดหมายอยู่ในองค์กรต้องซิงโครไนซ์ข้อมูลเฉพาะตัวของพวกเขากับไดเรกทอรี Office 365 ผ่าน Azure Active Directory Connect For these Exchange hybrid customers, if the user's mailbox is on-premises, the user can't add or configure Connectors.*          |
