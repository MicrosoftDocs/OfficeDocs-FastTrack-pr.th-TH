# <a name="source-environment-expectations-for-office-365-us-government"></a>ความคาดหวังต่อสภาพแวดล้อมต้นทางของ Office 365 ของสหรัฐอเมริกา

สิทธิประโยชน์ของ FastTrack Center จะให้แนวทางสําหรับคุณในการตั้งค่าระดับการรวมกับสภาพแวดล้อมต้นทางของคุณ (ตัวอย่างเช่น ถ้าคุณมีบริการในสภาพแวดล้อมต้นทางที่คุณต้องการย้ายไปยัง Office 365 อยู่แล้ว)
  
> [!NOTE]
> ถ้าต้องใช้การรวม สภาพแวดล้อมต้นทางของคุณต้องอยู่ที่ระดับต่่สุดของแอปพลิเคชันนั้น 
  
ตารางต่อไปนี้แสดงความคาดหวังของสภาพแวดล้อมต้นทางที่มีอยู่ของคุณในการออนบอร์ด

|**กิจกรรม**|**ความคาดหวังของสภาพแวดล้อมต้นทาง**|
|:-----|:-----|
|การออนบอร์ดหลัก | Active Directory ฟอเรสต์ที่ระดับฟอเรสต์ฟังก์ชันตั้งค่าเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:  <br/>      ฟอเรสต์ Active Directory เดียว  <br/>    ฟอเรสต์บัญชีผู้ใช้ Active Directory เดี่ยวและฟอเรสต์ทรัพยากร (การบล็อกของบัญชีผู้ใช้ Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business)  <br/>  ฟอเรสต์บัญชีผู้ใช้ Active Directory หลายฟอเรสต์และฟอเรสต์ทรัพยากร (Exchange และ/หรือ Topologies ของ Lync 2010, Lync 2013 หรือ Skype for Business)  <br/>  ฟอเรสต์บัญชีผู้ใช้ Active Directory หลายฟอเรสต์ที่เป็นหนึ่งในฟอเรสต์เป็นฟอเรสต์บัญชีผู้ใช้ Active Directory แบบรวมศูนย์ที่มี Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype for Business  <br/>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์ แต่ละฟอเรสต์มี Exchange ขององค์กรของตัวเอง <br/> <br/> **หมายเหตุ** ถ้าสถานการณ์สมมติของ Active Directory แบบหลายฟอเรสต์ ถ้าปรับใช้ *Lync 2010, Lync 2013* หรือ Skype for Business จะต้องถูกปรับใช้ในฟอเรสต์ Active Directory เดียวกันกับ Exchange     <br/>  <br/>      **หมายเหตุ** เมื่อปรับใช้ฟอเรสต์ Active Directory หลายฟอเรสต์กับองค์กร Exchange หลายองค์กรในการกําหนดค่าแบบหลายไฮบริดของ Exchange Namespace ชื่อหลักของผู้ใช้ (UPN) ที่แชร์ระหว่างฟอเรสต์ต้นฉบับ *ไม่ได้รับการสนับสนุน Namespace SMTP หลักระหว่างองค์กร Exchange ควรแยกออกจากกันด้วย For more information, see [Hybrid deployments with multiple Active Directory forests](https://go.microsoft.com/fwlink/?linkid=845444).*       <br/>   <br/>   **หมายเหตุ**  *For all multiple forests configurations, AD FS deployment is out of scope for the FastTrack Center Benefit.*           |
|การออนบอร์ด Exchange Online | สภาพแวดล้อมของคุณต้องมีระดับขั้นต่>อย่างใดอย่างหนึ่งต่อไปนี้:  <br/>  องค์กร Exchange เดียวหรือหลายองค์กรที่มี Exchange Server 2003 เป็นต้นไป  <br/>  สภาพแวดล้อมอีเมลที่สามารถใช้งาน Internet Message Access Protocol (IMAP) ได้รายการเดียว  <br/>  สภาพแวดล้อม G Suite เดียว (Gmail, ที่ติดต่อ และปฏิทินเท่านั้น)             |

|การออนบอร์ด Skype for Business Online  | **แบบประเมินเครือข่าย**  <br/>  การตรวจสอบพอร์ตและจุดสิ้นสุด  <br/>  ตรวจสอบคุณภาพการเชื่อมต่อ  <br/>  แบนด์วิดท์โดยประมาณ  <br/><br/>  **Lync แบบไฮบริด**  <br/>  ฟอเรสต์ Active Directory ภายในองค์กรแบบไซต์เดียว  <br/>  สภาพแวดล้อม Lync 2010 Server ที่มีเครื่องมือผู้ดูแลระบบ Lync 2013 หรือเครื่องมือผู้ดูแลระบบ Skype for Business 2015 ภายในองค์กรแบบไซต์เดียว  <br/>  บัญชี Active Directory หนึ่งฟอเรสต์เป็นต้นไป และฟอเรสต์ทรัพยากร (Exchange และ/หรือ Skype for Business) การออนไลน์  <br/>  ฟอเรสต์บัญชี Active Directory หลายฟอเรสต์ ที่มีฟอเรสต์หนึ่งเป็นฟอเรสต์บัญชีผู้ใช้ Active Directory แบบรวมศูนย์ที่มี Exchange และ/หรือ Skype for Business ในฟอเรสต์นั้น  <br/>  สภาพแวดล้อม Skype for Business Server 2015 เครื่องมือผู้ดูแลระบบ *Lync 2013 หรือเซิร์ฟเวอร์ขอบของ Skype for Business Online หรือ Lync 2010, Lync 2013 หรือเซิร์ฟเวอร์ขอบของ Skype for Business)*      <br/><br/>        **อุปกรณ์ห้องประชุม** <br/><br/>  **การเปิดใช้งานการประชุมทางเสียง**  <br/>  การตั้งค่าองค์กรเพื่อการตั้งค่าบริดจ์การประชุมเริ่มต้น  <br/>  การมอบหมายบริดจ์การประชุมให้กับผู้ใช้ที่มีสิทธิ์การใช้งาน  <br/><br/> การมอบหมายหมายเลขให้กับผู้ใช้ที่มีสิทธิ์การใช้งาน  <br/>  แนวทางพอร์ตหมายเลขภายในผ่านส่วนติดต่อผู้ใช้ (UI) สูงสุดถึง 999  <br/>  การสนับสนุนการร้องขอบริการพอร์ตหมายเลขในพื้นที่ (SR) มากกว่า 999
