# <a name="source-environment-expectations-for-office-365-us-government"></a>แหล่งความคาดหวังของสภาพแวดล้อมสำหรับ Office 365 รัฐบาลของสหรัฐอเมริกา

สวัสดิการศูนย์ FastTrack ให้คำแนะนำให้คุณตั้งค่าระดับของรวมกับสภาพแวดล้อมต้นทาง (ตัวอย่างเช่น ถ้าคุณมีบริการอยู่ในสภาพแวดล้อมของแหล่งที่มาที่คุณต้องการย้ายไปยัง Office 365)
  
> [!NOTE]
> ถ้าจำเป็นต้องรวม สภาพแวดล้อมต้นทางของคุณต้องอยู่ในระดับต่ำสุดสำหรับโปรแกรมประยุกต์นั้น 
  
ตารางต่อไปนี้แสดงความคาดหวังสำหรับสภาพแวดล้อมต้นฉบับของคุณที่มีอยู่สำหรับการปฐมนิเทศ

|**กิจกรรม**|**ความคาดหวังของสภาพแวดล้อมต้นทาง**|
|:-----|:-----|
|การปฐมนิเทศหลัก | ฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่กับระดับการทำงานของฟอเรสต์ตั้งกับ Windows Server 2003 เป็นต้นไป ด้วยการตั้งค่าคอนฟิกฟอเรสต์ดังต่อไปนี้:  <br/>      ฟอเรสต์ Active Directory เดียวกัน  <br/>    ไดเรกทอรีที่ใช้งานอยู่หนึ่งบัญชีฟอเรสต์และทรัพยากรโทฟอเรสต์ (อัตราแลกเปลี่ยนและ/หรือ Lync 2010 Lync 2013 หรือ Skype สำหรับธุรกิจ) ให้  <br/>  Active Directory บัญชีป่าไม้และทรัพยากรฟอเรสต์ (อัตราแลกเปลี่ยนและ/หรือ Lync 2010 Lync 2013 หรือ Skype สำหรับธุรกิจ) โทหลายครั้ง  <br/>  หลายป่าไม้บัญชี Active Directory ของป่าไม้กำลังฟอเรสต์บัญชี Active Directory จากส่วนกลางที่มีอัตราแลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สำหรับธุรกิจ  <br/>  หลาย Active Directory บัญชีฟอเรสต์ แต่ละชนิด มีการแลกเปลี่ยนองค์กรของตนเอง <br/> <br/> **หมายเหตุ** *สำหรับสถานการณ์จำลอง Active Directory ฟอเรสต์หลาย ถ้า Lync 2010, Lync 2013 หรือ Skype สำหรับธุรกิจการจัดวาง จะต้องถูกปรับใช้ในฟอเรสต์เดียวกันกับไดเรกทอรีที่ใช้งานอยู่เป็นอัตราแลกเปลี่ยน*     <br/>  <br/>      **หมายเหตุ** Namespaces ชื่อหลัก (UPN) ของผู้ใช้*เมื่อใช้ฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่หลาย ด้วยอัตราแลกเปลี่ยนหลายองค์กรในโครงแบบของ Exchange ไฮบริดสลีหลาย ที่ใช้ร่วมกันระหว่างฟอเรสต์ต้นทางไม่ได้รับการสนับสนุน ควรถูกแยก namespaces SMTP หลักระหว่างองค์กรแลกเปลี่ยน สำหรับข้อมูลเพิ่มเติม ให้ดูที่[การปรับใช้ไฮบริดสลีกับฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่หลาย](https://go.microsoft.com/fwlink/?linkid=845444)ครั้ง*       <br/>   <br/>   **หมายเหตุ** *สำหรับหลายป่าไม้โครงแบบ ปรับใช้ AD FS อยู่นอกขอบเขตสำหรับสวัสดิการศูนย์ FastTrack*             |
|ทะเบียนการแลกเปลี่ยนแบบออนไลน์ | สภาพแวดล้อมของคุณต้องมีวิธีการอย่างใดอย่างหนึ่งในระดับต่ำสุดต่อไปนี้:  <br/>  ชุดเดียวหรือหลายอัตราแลกเปลี่ยนองค์กรที่ มี Exchange Server 2003 เป็นต้นไป  <br/>  แบบเดี่ยว IBM Domino 7.0.3 onward สภาพแวดล้อม ([ภาคผนวก A - โยกย้ายจาก IBM Domino การแลกเปลี่ยนแบบออนไลน์](O365-from-ibm-domino-to-exchange-online.md))  <br/>  แบบเดี่ยวข้อความเข้าถึงอินเทอร์เน็ตโพรโทคอล (IMAP) -สภาพแวดล้อมทางอีเมลสามารถ  <br/>  แบบเดี่ยวชุด G สภาพแวดล้อม (Gmail ผู้ติดต่อ และปฏิทินเท่านั้น)  <br/>  แบบเดี่ยว Novell GroupWise 7.0.4 onward สภาพแวดล้อม            |
|SharePoint แบบออนไลน์และ OneDrive สำหรับธุรกิจปฐมนิเทศ  | **ไฮบริดสลี SharePoint**  <br/>  การตั้งค่าคอนฟิกไฮบริดสลี SharePoint รวมถึงการตั้งค่าคอนฟิกการค้นหาไฮบริดสลี ไซต์ ระบบภาษี ชนิด OneDrive สำหรับธุรกิจ มีตัวเปิดใช้โปรแกรมประยุกต์แบบขยาย ไซต์เอ็กซ์ทราเน็ต และการสร้างไซต์ด้วยตนเองที่เชื่อมโยงจากเนื้อหาในสถานที่จะกำหนดเป้าหมายเดียวกับ SharePoint สภาพแวดล้อมแบบออนไลน์ เมื่อต้องการเปิดใช้งานไฮบริดสลี SharePoint คุณต้องมีหนึ่งในสภาพแวดล้อมเซิร์ฟเวอร์ SharePoint ในสถานที่ต่อไปนี้:  <br/> <br/> ***2013 เซิร์ฟเวอร์ SharePoint***  <br/>  เดือน 2017 มิถุนายนสาธารณะ Update (PU) สำหรับไฮบริดสลีเนื้อหาชนิด สำหรับข้อมูลเพิ่มเติม ดู[ไฮบริการตั้งค่าคอนฟิก SharePoint การจำแนกประเภทและชนิดเนื้อหาของไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=853547)  <br/>  2017 มีนาคม PU สำหรับการกำหนดค่าการสร้างไซต์ด้วยตนเอง การสร้างไซต์ด้วยตนเองไฮบริดสลีเท่านั้นได้รับการสนับสนุนกับ 2013 เซิร์ฟเวอร์ SharePoint สำหรับข้อมูลเพิ่มเติม ดู[การสร้างไซต์ด้วยตนเองไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=846015)  <br/>  2016 พฤศจิกายน PU เป็นต้นไปกับระบบภาษีไฮบริดสลี สำหรับข้อมูลเพิ่มเติม ดู[แผนไฮบริดสลี SharePoint taxonomy](https://go.microsoft.com/fwlink/?linkid=844867)  <br/>  2016 กรกฎาคม PU เป็นต้นไปสำหรับการตั้งค่าคอนฟิกการสถานการณ์ไฮบริดสลีอื่น ๆ ทั้งหมด  <br/><br/> **หมายเหตุ** สถานการณ์สมมติไฮบริดสลี*2013 เซิร์ฟเวอร์ SharePoint เท่านั้นได้รับการสนับสนุนกับ 2013 เซิร์ฟเวอร์ SharePoint สถานการณ์เหล่านี้จะไม่ได้รับการสนับสนุนใน SharePoint Foundation 2013*    <br/>   <br/>    ***2016 เซิร์ฟเวอร์ SharePoint***  <br/>  2017 มิถุนายน PU สำหรับชนิดเนื้อหาของไฮบริดสลี สำหรับข้อมูลเพิ่มเติม ดู[ไฮบริการตั้งค่าคอนฟิก SharePoint การจำแนกประเภทและชนิดเนื้อหาของไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=853547)  <br/>  2016 พฤศจิกายน PU (คุณลักษณะ Pack 1) กับระบบภาษีไฮบริดสลี สำหรับข้อมูลเพิ่มเติม ดู[แผนไฮบริดสลี SharePoint taxonomy](https://go.microsoft.com/fwlink/?linkid=844867)  <br/>  2016 กรกฎาคม PU เป็นต้นไปสำหรับการตั้งค่าคอนฟิกการสถานการณ์ไฮบริดสลีอื่น ๆ ทั้งหมด  <br/><br/> **หมายเหตุ** *รุ่นในสถานที่สภาพแวดล้อมของ SharePoint 2013 เซิร์ฟเวอร์ SharePoint หรือ 2016 เซิร์ฟเวอร์ SharePoint ไม่ได้มีให้ โดย FastTrack ศูนย์สวัสดิการ สำหรับข้อมูลเพิ่มเติม ให้ดู[ระดับการปรับปรุงสาธารณะต่ำสุดสำหรับคุณลักษณะไฮบริดสลี SharePoint](https://go.microsoft.com/fwlink/?linkid=853548)*               |
|Skype สำหรับธุรกิจแบบออนไลน์ปฐมนิเทศ  | **ประเมินเครือข่าย**  <br/>  การตรวจสอบพอร์ตและปลายทาง  <br/>  ตรวจสอบคุณภาพเชื่อมต่อหรือไม่  <br/>  แบนด์วิดท์ที่ประเมิน  <br/><br/>  **ไฮบริดสลี Lync**  <br/>  เดียวในสถานฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่  <br/>  Lync 2010 Server ล้อม ด้วยเครื่องมือการดูแล Lync 2013 หรือ Skype สำหรับเครื่องมือ admin 2015 ธุรกิจและบทบาท Lync 2010 ขอบเซิร์ฟเวอร์  <br/>  สภาพแวดล้อมแบบเซิร์ฟเวอร์ 2013 Lync และบทบาท Lync 2013 ขอบเซิร์ฟเวอร์  <br/><br/>  **Skype สำหรับไฮบริดสลีธุรกิจออนไลน์**  <br/>  เดียวในสถานฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่  <br/>  เดี่ยว Active Directory บัญชีฟอเรสต์เป็นต้นไปและโทฟอเรสต์ (อัตราแลกเปลี่ยนและ/หรือ Skype สำหรับธุรกิจ) ของทรัพยากร  <br/>  หลาย Active Directory บัญชีป่าไม้ กับฟอเรสต์หนึ่งกำลังส่วนกลาง Active Directory บัญชีฟอเรสต์ มีอัตราแลกเปลี่ยนและ/หรือ Skype สำหรับธุรกิจในนั้น  <br/>  การ Skype สำหรับสภาพแวดล้อมทางธุรกิจเซิร์ฟเวอร์ 2015 รวมกับ Skype สำหรับธุรกิจบทบาทเซิร์ฟเวอร์ของขอบ  <br/><br/> **หมายเหตุ** *บริการนี้เพิ่มเติมสำหรับการกำหนดค่า และตรวจสอบโดเมนแยก (ไฮบริดสลี) งาน และไม่มีการแนะนำส่วนประกอบในสถาน (ตัวอย่าง เครื่องมือ admin Lync 2013 หรือ Lync 2013/Skype สำหรับเซิร์ฟเวอร์ธุรกิจออนไลน์ หรือ Lync 2010, Lync 2013 หรือ Skype สำหรับเซิร์ฟเวอร์ขอบธุรกิจ)*      <br/><br/>        **อุปกรณ์ห้องประชุม**  <br/>  การสร้างบัญชีออนไลน์ที่จำเป็นสำหรับอุปกรณ์ห้องประชุมที่ได้รับการสนับสนุนที่แสดงรายการอยู่บนแท็บระบบห้องประชุมใน[Skype สำหรับแค็ตตาล็อกโซลูชันทางธุรกิจ](https://go.microsoft.com/fwlink/?LinkId=615775)  <br/><br/>  **การเปิดใช้งานการประชุมเสียง**  <br/>  การตั้งค่าองค์กรสำหรับการตั้งค่าเริ่มต้นของการประชุมบริดจ์  <br/>  การมอบหมายของบริดจ์การประชุมไปยังผู้ใช้ที่มีลิขสิทธิ์  <br/><br/>  **การเปิดใช้งานระบบโทรศัพท์และแผนการเรียกคำแนะนำ (พวกเราเท่านั้น – ไม่พร้อมใช้งานในแผนสูง GCC หรือ DoD)**  <br/>  การตั้งค่าองค์กรสำหรับการตั้งค่าเริ่มต้นของเสียงเมฆ  <br/>  การกำหนดหมายเลขให้แก่ผู้ใช้ได้รับอนุญาต  <br/>  หมายเลขท้องถิ่นที่ปรับเปลี่ยนคำแนะนำผ่านทางส่วนติดต่อผู้ใช้ (UI) จนถึง 999  <br/>  ภายในหมายเลข++:การสร้างพอร์ตร้องขอบริการ (SR) สนับสนุนมากกว่า 999 |ทีมงาน Microsoft ปฐมนิเทศ (ไม่พร้อมใช้งานได้ในแผนสูง GCC หรือ DoD) | รหัสประจำตัวในการเปิดใช้งานในไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับ Office 365  <br/>  ผู้ใช้ที่เปิดใช้งานสำหรับ SharePoint แบบออนไลน์  <br/>  กล่องจดหมาย Exchange อยู่ (ออนไลน์ และ/หรือในสถานที่ในการตั้งค่าคอนฟิกไฮบริดสลีอัตราแลกเปลี่ยน)  <br/>  เปิดใช้งานสำหรับกลุ่ม Office 365  <br/><br/> **หมายเหตุ** *ถ้าผู้ใช้ไม่ได้กำหนด และเปิดใช้งาน ด้วยสิทธิ์การใช้งาน SharePoint แบบออนไลน์ พวกเขาจะไม่มี OneDrive สำหรับเก็บข้อมูลธุรกิจใน Office 365 แฟ้มที่ใช้ร่วมกันจะทำงานต่อในสถานี แต่ผู้ใช้ไม่สามารถใช้ร่วมกันแฟ้มในแผนภูมิโดยไม่มี OneDrive สำหรับเก็บข้อมูลธุรกิจใน Office 365 ทีมงาน Microsoft ไม่สนับสนุน SharePoint ในสถาน*     <br/> <br/>       **หมายเหตุ** สถานะ*ที่ดีที่สุดคือผู้ใช้ทุกคนต้องมี homed กล่องจดหมายของพวกเขาในการแลกเปลี่ยนแบบออนไลน์ ผู้ใช้ที่ มีกล่องจดหมายที่ homed ในสถานต้องมีการซิงโครไนส์กับไดเรกทอรี 365 Office ผ่าน Azure Active Directory เชื่อมต่อข้อมูลเฉพาะตัว สำหรับลูกค้าไฮบริดสลีแลกเปลี่ยนเหล่านี้ ถ้ากล่องจดหมายของผู้ใช้ที่อยู่ในสถาน ผู้ใช้ไม่สามารถเพิ่ม หรือกำหนดค่าตัวเชื่อมต่อได้*            |
