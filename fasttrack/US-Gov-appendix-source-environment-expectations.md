# <a name="source-environment-expectations-for-office-365-us-government"></a>ความคาดหวังด้านสิ่งแวดล้อมที่มาสําหรับรัฐบาลสหรัฐ Office 365

ประโยชน์จากศูนย์ FastTrack จะให้คําแนะนําแก่คุณในการตั้งค่าระดับการรวมเข้ากับสภาพแวดล้อมต้นทางของคุณ (ตัวอย่างเช่น ถ้าคุณมีบริการในสภาพแวดล้อมต้นทางที่คุณต้องการย้ายไปยัง Office 365 อยู่แล้ว)
  
> [!NOTE]
> ถ้าจําเป็นต้องมีการรวม 
  
ตารางต่อไปนี้แสดงความคาดหวังสําหรับสภาพแวดล้อมต้นทางที่มีอยู่สําหรับการเตรียมพร้อม

|**กิจกรรม**|**ความคาดหวังของสภาพแวดล้อมต้นทาง**|
|:-----|:-----|
|การเตรียมพร้อมหลัก | ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่กับระดับฟอเรสต์การทํางานที่ตั้งค่าเป็น Windows Server 2003 เป็นต้นไป ด้วยการกําหนดค่าฟอเรสต์ต่อไปนี้:  <br/>      ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เดียว  <br/>    ฟอเรสต์และทรัพยากรฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่เดียว (Exchange และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  หลายฟอเรสต์บัญชี Active Directory และฟอเรสต์ทรัพยากร (แลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ)  <br/>  หลายฟอเรสต์บัญชี Active Directory กับฟอเรสต์เป็นฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่แบบรวมศูนย์ที่มีการแลกเปลี่ยน และ/หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจ  <br/>  หลายฟอเรสต์บัญชี Active Directory แต่ละองค์กรแลกเปลี่ยนของตนเอง <br/> <br/> **หมายเหตุ**  *สําหรับหลายฟอเรสต์ที่ใช้งานอยู่ไดเรกทอรีสถานการณ์ ถ้า Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจจะปรับใช้ นั้นต้องถูกปรับใช้ในฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่เหมือนกับการแลกเปลี่ยน*   <br/>  <br/>      **หมายเหตุ***เมื่อนํามาใช้หลายฟอเรสต์ Active Directory กับองค์กรแลกเปลี่ยนหลายในการกําหนดค่าแบบหลายไฮบริดอัตราแลกเปลี่ยน namespaces ชื่อผู้ใช้หลักที่ใช้ร่วมกัน (UPN) ระหว่างฟอเรสต์ต้นทางไม่ได้รับการสนับสนุน ช่องว่างของ SMTP หลักระหว่างองค์กร Exchange ควรถูกแยกออกด้วย สําหรับข้อมูลเพิ่มเติม โปรดดูที่[การปรับใช้ไฮบริดสลีกับฟอเรสต์ Active Directory หลาย](https://go.microsoft.com/fwlink/?linkid=845444)*       <br/>   <br/>   **หมายเหตุ**  *สําหรับการตั้งค่าคอนฟิกฟอเรสต์หลายหลาย*           |
|การเริ่มการออนไลน์ของอัตราแลกเปลี่ยน | สภาพแวดล้อมของคุณต้องมีหนึ่งในระดับต่ําสุดต่อไปนี้:  <br/>  องค์กร Exchange เดียว หรือหลาย Exchange กับ Exchange Server 2003 เป็นต้นไป  <br/>  IBM Domino 7.0.3 สภาวะแวดล้อมเป็นต้นไป ([ภาคผนวก A - การโอนย้ายจาก IBM Domino เพื่อแลกเปลี่ยนแบบออนไลน์](O365-from-ibm-domino-to-exchange-online.md))  <br/>  สภาพแวดล้อมอีเมลที่สามารถใช้งานโพรโทคอลการเข้าถึงข้อความอินเทอร์เน็ตโพรโทคอล (IMAP) เดียว  <br/>  สภาพแวดล้อม G Suite เดียว (เฉพาะ Gmail รายชื่อ และปฏิทินเท่านั้น)  <br/>  สภาพแวดล้อมใหม่ๆ ที่โนเวล            |
|SharePoint แบบออนไลน์และ OneDrive สําหรับออนบอร์ดธุรกิจ  | **ไฮบริดสลีของ SharePoint**  <br/>  การตั้งค่าคอนฟิกไฮบริด SharePoint รวมถึงการกําหนดค่าการค้นหาไฮบริดสลี, ไซต์, ระบบภาษี, ชนิดเนื้อหา, OneDrive for Business, เมื่อต้องการเปิดใช้งานไฮบริดสลี SharePoint คุณต้องมีสภาพแวดล้อม SharePoint ในสถานต่อไปนี้อย่างใดอย่างหนึ่ง:  <br/> <br/> ***เซิร์ฟเวอร์ SharePoint 2013***  <br/>  มิถุนายน 2017 การปรับปรุงสาธารณะ (PU) สําหรับชนิดเนื้อหาไฮบริด สําหรับข้อมูลเพิ่มเติม โปรดดูที่[การตั้งค่าคอนฟิกชนิดเนื้อหาของ SharePoint แบบไฮบริดและไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=853547)  <br/>  มีนาคม 2017 PU สําหรับการกําหนดค่าการสร้างไซต์ด้วยตนเอง การสร้างไซต์ด้วยตนเองแบบไฮบริดได้รับการสนับสนุนเฉพาะกับ SharePoint Server 2013 สําหรับข้อมูลเพิ่มเติม ให้ดูที่[การสร้างไซต์ระบบตนเองแบบไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=846015)  <br/>  พฤศจิกายน 2016 PU เป็นต้นไปด้วยระบบภาษีไฮบริด สําหรับข้อมูลเพิ่มเติม ให้ดูที่[วางแผนการจําแนกประเภท SharePoint แบบไฮบริด](https://go.microsoft.com/fwlink/?linkid=844867)  <br/>  2016 กรกฎาคม PU เป็นต้นไปสําหรับสถานการณ์จําลองการตั้งค่าคอนฟิกไฮบริดสลีอื่น ๆ ทั้งหมด  <br/><br/> **หมายเหตุ**  *สถานการณ์สมมติไฮบริดสลี 2013 เซิร์ฟเวอร์ SharePoint ได้รับการสนับสนุนกับ 2013 เซิร์ฟเวอร์ SharePoint เท่านั้น สถานการณ์สมมติเหล่านี้ไม่ได้รับการสนับสนุนใน 2013 พื้นฐานของ SharePoint*  <br/>   <br/>    ***เซิร์ฟเวอร์ SharePoint 2016***  <br/>  มิถุนายน 2017 PU สําหรับชนิดเนื้อหาไฮบริด สําหรับข้อมูลเพิ่มเติม โปรดดูที่[การตั้งค่าคอนฟิกชนิดเนื้อหาของ SharePoint แบบไฮบริดและไฮบริดสลี](https://go.microsoft.com/fwlink/?linkid=853547)  <br/>  พฤศจิกายน 2016 PU (คุณลักษณะ Pack 1) กับระบบภาษีไฮบริด สําหรับข้อมูลเพิ่มเติม ให้ดูที่[วางแผนการจําแนกประเภท SharePoint แบบไฮบริด](https://go.microsoft.com/fwlink/?linkid=844867)  <br/>  2016 กรกฎาคม PU เป็นต้นไปสําหรับสถานการณ์จําลองการตั้งค่าคอนฟิกไฮบริดสลีอื่น ๆ ทั้งหมด  <br/><br/> **หมายเหตุ***:การปรับรุ่นของสภาพแวดล้อมของ SharePoint ในสถานที่ 2013 เซิร์ฟเวอร์ SharePoint หรือ SharePoint Server 2016 ไม่ได้ให้ประโยชน์จากศูนย์ FastTrack สําหรับข้อมูลเพิ่มเติม ให้ดูที่[ระดับการปรับปรุงสาธารณะต่ําสุดสําหรับคุณลักษณะไฮบริดสลี SharePoint](https://go.microsoft.com/fwlink/?linkid=853548)*               |
|การเริ่มต้นใช้งานแบบออนไลน์ของ Skype สําหรับธุรกิจ  | **การประเมินเครือข่าย**  <br/>  การตรวจสอบพอร์ตและปลายทาง  <br/>  การตรวจสอบคุณภาพการเชื่อมต่อ  <br/>  ประมาณการแบนด์วิดธ์  <br/><br/>  **ไฮบริดสลี Lync**  <br/>  ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ในสถานเดียว  <br/>  สภาพแวดล้อม Lync 2010 Server ด้วยเครื่องมือผู้ดูแลระบบ Lync 2013 หรือ Skype สําหรับเครื่องมือการจัดการ 2015 ธุรกิจและบทบาทเซิร์ฟเวอร์ Lync 2010 ขอบ  <br/>  สภาพแวดล้อม Lync 2013 Server และบทบาทเซิร์ฟเวอร์ Lync 2013 ขอบ  <br/><br/>  **Skype สําหรับธุรกิจออนไลน์ไฮบริด**  <br/>  ฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ในสถานเดียว  <br/>  ฟอเรสต์บัญชีไดเรกทอรีที่ใช้งานอยู่เดียวเป็นต้นไป และทรัพยากรฟอเรสต์ (แลกเปลี่ยน และ/หรือ Skype สําหรับธุรกิจ)  <br/>  หลายฟอเรสต์บัญชี Active Directory กับฟอเรสต์หนึ่งถูกฟอเรสต์บัญชี Active Directory ส่วนกลางกับ Exchange และ/หรือ Skype สําหรับธุรกิจในนั้น  <br/>  Skype สําหรับสภาพแวดล้อมทางธุรกิจเซิร์ฟเวอร์ 2015 รวมทั้งบทบาทเซิร์ฟเวอร์ขอบ Lync  <br/><br/> **หมายเหตุ**  *บริการเพิ่มเติมนี้สําหรับการกําหนดค่า และตรวจสอบงานแยกโดเมน (ไฮบริด) และไม่รวมคอมโพเนนต์ในสถานที่แนะนํา (ตัวอย่างเช่น เครื่องมือการดูแล Lync 2013 หรือ Lync 2013/Skype สําหรับเซิร์ฟเวอร์ออนไลน์ของธุรกิจ หรือ Lync 2010, Lync 2013 หรือ Skype สําหรับธุรกิจเซิร์ฟเวอร์ขอบ)*    <br/><br/>        **อุปกรณ์ห้องประชุม**  <br/>  การสร้างบัญชีออนไลน์ที่จําเป็นสําหรับอุปกรณ์ห้องประชุมที่ได้รับการสนับสนุนที่แสดงอยู่บนแท็บ ระบบห้องประชุม ใน[Skype สําหรับธุรกิจโซลูชันแค็ตตาล็อก](https://go.microsoft.com/fwlink/?LinkId=615775)  <br/><br/>  **การเปิดใช้งานการประชุมทางเสียง**  <br/>  การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นของบริดจ์การประชุม  <br/>  การกําหนดของสะพานประชุมกับผู้ใช้ที่ได้รับอนุญาต  <br/><br/>  **การเปิดใช้งานระบบโทรศัพท์และคําแนะนําแผนการโทร (สหรัฐอเมริกาเท่านั้น - ไม่สามารถใช้ได้ในแผน GCC สูงหรือ DoD)**  <br/>  การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นของ Cloud Voice  <br/>  การกําหนดหมายเลขให้กับผู้ใช้ที่ได้รับอนุญาต  <br/>  คําแนะนําเกี่ยวกับพอร์ตหมายเลขท้องถิ่นผ่านส่วนติดต่อผู้ใช้ (UI) ถึง 999  <br/>  บริการ porting หมายเลขท้องถิ่น (SR) สนับสนุนผ่าน 999  <br/><br/>  **การเปิดใช้งาน Skype สําหรับคําแนะนําในการออกอากาศการประชุมทางธุรกิจ (ไม่พร้อมใช้งานในแผนสูงหรือ DoD GCC)**  <br/>  การตั้งค่าองค์กรสําหรับสหพันธรัฐกับบริการการออกอากาศการประชุม   |
|การเริ่มต้นใช้งาน Microsoft Teams (ไม่มีในแผน GCC สูงหรือ DoD) | ผู้ใช้อีเมลที่เปิดใช้งานในไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับ Office 365  <br/>  ผู้ใช้ที่เปิดใช้งานสําหรับ SharePoint แบบออนไลน์  <br/>  กล่องจดหมาย Exchange มีอยู่ (ออนไลน์ และ/หรือในสถานที่ในการกําหนดค่าไฮบริดสลีอัตราแลกเปลี่ยน)  <br/>  เปิดใช้งานสําหรับกลุ่ม Office 365  <br/><br/> **หมายเหตุ**  *ถ้าผู้ใช้ไม่ได้กําหนด และเปิดใช้งานด้วยสิทธิ์การใช้งาน SharePoint Online ผู้ใช้เหล่านั้นจะไม่มี OneDrive สําหรับการจัดเก็บธุรกิจใน Office 365 การแชร์ไฟล์จะยังคงทํางานต่อไปในแชนเนล แต่ผู้ใช้ไม่สามารถแชร์ไฟล์ในแชทได้โดยไม่ต้อง OneDrive สําหรับที่เก็บข้อมูลธุรกิจใน Office 365 ทีม Microsoft ไม่สนับสนุน SharePoint ในองค์กร*   <br/> <br/>       **หมายเหตุ**  *:สถานะเหมาะสําหรับผู้ใช้ทั้งหมดมีกล่องจดหมาย homed ในการแลกเปลี่ยนแบบออนไลน์ ผู้ใช้ที่มีกล่องจดหมาย homed ในสถานต้องมีข้อมูลประจําตัวของพวกเขาซิงโครไนส์ไปยังไดเรกทอรี Office 365 ผ่านการเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับลูกค้าเหล่านี้ไฮบริดสลีแลกเปลี่ยนถ้ากล่องจดหมายของผู้ใช้อยู่ในสถาน*          |
| การเตรียมพร้อมในการให้บริการ รวมถึง:  <br/>  *แลกเปลี่ยน<br/>แบบออนไลน์<br/>ของ SharePoint OneDrive<br/>สําหรับธุรกิจ Skype<br/>สําหรับธุรกิจ<br/>แบบออนไลน์<br/>Microsoft<br/>ทีม<br/>BI ออนไลน์โครงการ Yammer Office 365 ProPlus<br/>*   |ซอฟต์แวร์ไคลเอนต์ออนไลน์เช่นโครงการสําหรับ Office 365, ไคลเอนต์ Outlook, OneDrive สําหรับไคลเอ็นต์ซิงค์ธุรกิจ เดสก์ท็อป Bi และ Skype สําหรับธุรกิจต้องอยู่ในระดับต่ําสุดตามที่กําหนดไว้[ในความต้องการของระบบสําหรับ Office](https://go.microsoft.com/fwlink/?LinkID=723597)  <br/> โปรแกรมติดตั้งสําหรับ Microsoft ทีม Windows และเดสก์ท็อป Mac ไคลเอนต์สามารถ[https://go.microsoft.com/fwlink/?linkid=839411](https://go.microsoft.com/fwlink/?linkid=839411)ดาวน์โหลดได้จาก   |
