# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

FastTrack มีความรับผิดชอบต่อไปนี้ระหว่างการปฐมนิเทศ

## <a name="general"></a>ทั่ว ไป

-   ให้ความช่วยเหลือด้านการสนับสนุนระยะไกลแก่คุณสำหรับกิจกรรมการกำหนดค่าที่จำเป็นตามที่ระบุไว้ในคำอธิบายระยะโดยละเอียด

-   ให้เอกสารประกอบเครื่องมือซอฟต์แวร์และคอนโซลผู้ดูแลระบบที่พร้อมใช้งานเพื่อช่วยคุณในการลดหรือกำจัดงานการกำหนดค่า

## <a name="initiate-phase"></a>เริ่มต้นเฟส

-   ทำงานร่วมกับคุณเพื่อเริ่มการเตรียมพร้อม

-   กำหนดบริการที่มีสิทธิ์ที่คุณต้องการบนเครื่องบิน

## <a name="assess-phase"></a>ประเมินระยะ

-   แสดงภาพรวมของการดูแลระบบ

-   ให้คำแนะนำเกี่ยวกับ:

    -   ความต้องการของ DNS เครือข่ายและโครงสร้างพื้นฐาน

    -   ความต้องการของลูกค้า (อินเทอร์เน็ตเบราว์เซอร์ระบบปฏิบัติการไคลเอ็นต์และความต้องการของบริการ)

    -   ข้อมูลประจำตัวของผู้ใช้และการจัดสรร

    -   การเปิดใช้งานบริการที่มีสิทธิ์ซึ่งได้ซื้อและกำหนดให้เป็นส่วนหนึ่งของการปฐมนิเทศ

-   สร้างเส้นเวลาสำหรับกิจกรรมด้านการแก้ไข

-   ระบุรายการตรวจสอบสำหรับทั้ง Intune และ Azure โฆษณาพรีเมี่ยม

## <a name="remediate-phase"></a>ระยะแก้ไข

-   ถือการประชุมทางโทรศัพท์กับคุณตามตารางเวลาที่ตกลงกันเพื่อตรวจสอบความคืบหน้าของกิจกรรมการแก้ไขตัวอย่างเช่นแนะนำให้คุณผ่านการติดตั้งก่อนการเตรียมการบริการระบบคลาวด์ของ Microsoft

## <a name="enable-phase"></a>เปิดใช้งานเฟส
ให้คำแนะนำเกี่ยวกับ:

-   การเปิดใช้งานผู้เช่าบริการออนไลน์ของ Microsoft หรือการสมัครสมาชิกของคุณ

-   การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตของไฟร์วอลล์

-   การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

-   ตรวจสอบการเชื่อมต่อกับบริการออนไลน์ของ Microsoft

-   สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

    -   การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และบริการออนไลน์ของ Microsoft ที่มีสิทธิ์ (เฉพาะคำแนะนำถ้าจำเป็น)

    -   การกำหนดค่าการรับรองความถูกต้องที่มีการจัดการ (การซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องแบบพาส-ทรู) ด้วยเครื่องมือ Azure เชื่อมต่อไดเรกทอรี (เฉพาะการแนะนำถ้าจำเป็น)

        > [!NOTE]
        > การพัฒนาและการดำเนินการสำหรับส่วนขยายกฎที่กำหนดเองอยู่นอกขอบเขต

-   สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลประจำตัวที่ติดต่อกับภายนอก: การติดตั้งและการตั้งค่าคอนฟิกบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) สำหรับการรับรองความถูกต้องของโดเมนภายในเครื่องกับ Intune ในไซต์เดียวการกำหนดค่าแบบทนต่อความผิดพลาดถ้าจำเป็น

    > [!NOTE]
    > สำหรับการตั้งค่าคอนฟิกฟอเรสต์หลายการปรับใช้ FS โฆษณาอยู่นอกขอบเขต

-   การทดสอบฟังก์ชันการเข้าสู่ระบบแบบครั้งเดียว (SSO) ถ้ามีการปรับใช้

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งานเฟส-Microsoft Azure ที่ใช้งานอยู่ไดเรกทอรีพรีเมี่ยม

ให้คำแนะนำเกี่ยวกับ:

- การเปิดใช้งานผู้เช่าพรีเมี่ยมโฆษณา Azure ของคุณ

- การกำหนดค่าพอร์ตไฟร์วอลล์

- การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

- ตรวจสอบการเชื่อมต่อกับบริการ Azure AD Premium

- สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

  -   การติดตั้งไดเรกทอรีให้ตรงกันระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และการเชื่อมต่อ AD Azure ถ้าจำเป็น

  -   การกำหนดค่าวิธีการรับรองความถูกต้อง (การซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องแบบพาส-ทรู) ด้วยเครื่องมือการเชื่อมต่อ AD Azure

- สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการซิงโครไนส์การเชื่อมต่อ AD Azure ตั้งค่าสำหรับสถานการณ์ป่าหลาย
- สำหรับสภาพแวดล้อมแบบเดี่ยวและหลายฟอเรสต์:
  - การกำหนดค่าการรับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ของ Azure ถ้าจำเป็น
  - การกำหนดค่า Azure ที่ใช้งานไดเรกทอรีการเข้าสู่ระบบแบบครั้งเดียว (SSO) ถ้าจำเป็น
    > [!NOTE]
    > การรับรองความถูกต้องแบบพาส-ทรูของไดเรกทอรี Azure สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์ได้รับการสนับสนุนถ้ามีความเชื่อถือฟอเรสต์ระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ของคุณและถ้ามีการกำหนดค่าการกำหนดเส้นทางต่อท้ายชื่อ ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์หลายในสถานที่เพื่อให้มีความพร้อมใช้งานสูงสำหรับการร้องขอการเข้าสู่ระบบ

  - สำหรับข้อมูลเพิ่มเติมให้ดูการ[รับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)และ[Azure ที่ใช้งานอยู่การเข้าสู่ระบบแบบครั้งเดียว: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับขีดจำกัดการรับรองความถูกต้องแบบพาส-ทรูโปรดดูที่การ[รับรองความถูกต้องแบบพาสไดเรกทอรีที่ใช้งานอยู่ของ Azure: ข้อจำกัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO แบบไร้รอยต่อโปรดดูที่[การแก้ไขปัญหา Azure ที่ใช้งานไดเรกทอรีการเข้าสู่ระบบแบบครั้งเดียว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > การซิงค์แฮชของรหัสผ่านและรหัสผ่านกลับสนับสนุนป่าไม้หลาย อย่างไรก็ตามสถานการณ์เขียนกลับอื่นๆไม่ได้รับการสนับสนุน

  - การตั้งค่าคอนฟิกการซิงโครไนส์ระหว่างในสถานป่าไม้ไดเรกทอรีที่ใช้งานอยู่และไดเรกทอรีของ Azure ไดเรกทอรีที่ใช้งานอยู่ของ Microsoft

    > [!NOTE]
    > การพัฒนาและการดำเนินการสำหรับส่วนขยายกฎที่กำหนดเองอยู่นอกขอบเขต

- สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก:

  -   การติดตั้งและตั้งค่า AD FS สำหรับการรับรองความถูกต้องของโดเมนภายในเครื่องด้วย Azure AD Premium ในไซต์เดียวการกำหนดค่าแบบทนต่อความผิดพลาด (ถ้าจำเป็น)

  > [!NOTE]
  > สำหรับการตั้งค่าคอนฟิกฟอเรสต์หลายการปรับใช้ FS โฆษณาอยู่นอกขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้ามีการปรับใช้)

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งานระยะ-Azure โฆษณาพรีเมี่ยม-ด้วยการเชื่อมต่อ AD Azure และ FS โฆษณา

ให้คำแนะนำเกี่ยวกับการตั้งค่า:

- การจัดสรรผู้ใช้รวมถึงการให้ใบอนุญาต

- การซิงโครไนส์ไดเรกทอรีการเชื่อมต่อ AD Azure (ด้วยการเขียนกลับรหัสผ่านและซิงค์แฮชของรหัสผ่าน)

  - รีเซ็ตรหัสผ่านบริการตนเองของไดเรกทอรีที่ใช้งานอยู่ของ Azure (SSPR)

  - การรับรองความถูกต้องด้วยหลายปัจจัย Azure

  - ถึงสาม (3) หรือมากกว่าซอฟต์แวร์เป็นการรวมแอพลิเคชันบริการ (SaaS) กับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ของ Azure](https://azure.microsoft.com/marketplace/active-directory/)

  - การจัดสรรผู้ใช้โดยอัตโนมัติสำหรับแอปพลิเคชัน SaaS แบบบูรณาการล่วงหน้าตามที่ระบุไว้ใน[รายการการสอนการรวมแอป](https://docs.microsoft.com/en-us/azure/active-directory/saas-apps/tutorial-list)จำกัดการจัดสรรขาออกเท่านั้น

  - หน้าจอเข้าสู่ระบบที่กำหนดเองรวมถึงโลโก้ข้อความและรูปภาพ

  - การบริการตนเองและกลุ่มแบบไดนามิก (กลุ่ม)

  - พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - Azure ที่ใช้งานอยู่ไดเรกทอรีเชื่อมต่อสุขภาพ

  - การเข้าถึงแบบมีเงื่อนไข Azure ไดเรกทอรีที่ใช้งานอยู่

  - เงื่อนไขการใช้ไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่

  - การจัดการข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - รีวิวการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ Azure

### <a name="enable-phase---intune"></a>เปิดใช้เฟส-Intune

> [!IMPORTANT]
> FastTrack ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 กับ Intune FastTrack สนับสนุนเฉพาะการจัดการ Windows 10 ผ่านการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM) เท่านั้น

ให้คำแนะนำเกี่ยวกับ:

-   การตั้งค่าคอนฟิกข้อมูลเฉพาะตัวที่จะใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรี Active Directory ของคุณในสถานที่หรือข้อมูลเฉพาะตัวของระบบคลาวด์ (Azure Active Directory)

-   ให้สิทธิ์ผู้ใช้ปลายทางของคุณ

-   การเพิ่มผู้ใช้ไปยังการบอกรับเป็นสมาชิก Intune ของคุณกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

-   การตั้งค่าคอนฟิกสิทธิ์การจัดการอุปกรณ์เคลื่อนที่ของคุณ) ขึ้นอยู่กับความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงาน MDM ของคุณ

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

    -   การนำทางพอร์ทัล admin Intune เพื่อค้นหาข้อมูลบนผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ตัวดำเนินการโต๊ะช่วยเหลือผู้ดูแลระบบฯลฯ)

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   การปรับใช้แอปสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านการเชื่อมโยงเว็บ, MSI และ/หรือลิงก์ลึก

        -   การปรับใช้ Office ProPlus บนอุปกรณ์ Windows 10

        -   โปรแกรมการซื้อระดับเสียงสำหรับการปรับใช้แอปรวมถึง VPP ของ Apple, Windows Store สำหรับธุรกิจและการเล่นของ Google สำหรับร้านทำงาน

        -   การใช้งานของ e-mail เครือข่ายไร้สายและโปรไฟล์ VPN ถ้าคุณมีหน่วยงานออกใบรับรองที่มีอยู่, Wi-fi หรือ VPN โครงสร้างพื้นฐานในองค์กรของคุณ

        -   การตั้งค่าตัวเชื่อมต่อ Exchange Intune Microsoft (เมื่อมี)

        -   โปรไฟล์การกำหนดค่าอุปกรณ์สำหรับแพลตฟอร์มอุปกรณ์ที่รองรับ

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   การกำหนดค่าและการปรับใช้นโยบายการป้องกันแอป Intune สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   กำลังจัดเตรียมแอปบรรทัดของธุรกิจ (LOB) สำหรับนโยบายการป้องกันแอป Intune พร้อมด้วยคำแนะนำเกี่ยวกับตัวเลือกที่มีอยู่

    -   ลงทะเบียนอุปกรณ์ของแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนไปยัง Intune หรือตัวจัดการการตั้งค่าคอนฟิกของคุณกับบริการ Microsoft Intune

    -   การเชื่อมต่อกับคลังข้อมูล Intune

    -   การรวม Intune ด้วย:
        -   ตัวแสดงทีมสำหรับความช่วยเหลือระยะไกล (จำเป็นต้องสมัครสมาชิกตัวแสดงทีม)

        -   โซลูชั่นคู่ค้าป้องกันภัยคุกคามมือถือ (การสมัครสมาชิกโซลูชันป้องกันภัยคุกคามมือถือเป็นสิ่งจำเป็น)

        -   โซลูชั่นการจัดการค่าใช้จ่ายโทรคมนาคม (จำเป็นต้องสมัครสมาชิกโซลูชันการจัดการค่าใช้จ่ายโทรคมนาคม)

        -   จำเป็นต้องมีการป้องกันภัยคุกคามขั้นสูงของ windows Defender (Windows E5 หรือ Microsoft ๓๖๕ E5)

    -   การกำหนดค่าการปรับปรุงซอฟต์แวร์สำหรับแพลตฟอร์มที่สนับสนุนที่เกี่ยวข้อง

    -   ทรัพยากรสำหรับการวางแผนการนำผู้ใช้

- การตั้งค่า Windows ออโตไพลอต:

    - กำหนดค่าและตั้งค่า Microsoft Intune สำหรับระบบออโตไพลอต Windows

    - กำหนดค่ากลุ่มแบบไดนามิกของโฆษณา Azure

    - เพิ่มตราสินค้าของบริษัทของคุณลงในโฆษณา Azure

    - สร้างและกำหนดอุปกรณ์ให้กับโพรไฟล์ของ Windows ออโตไพลอต (เช่นโปรไฟล์ของ Windows ออโตไพลอตที่จำกัดการสร้างบัญชีผู้ดูแลภายใน)

    - ปรับแต่งประสบการณ์ใช้งานแบบออกจากกล่อง (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การกำหนดค่า MDM อัตโนมัติลงทะเบียนในโฆษณา Azure และ Intune

    > [!NOTE]
    > การตั้งค่า Windows ออโตไพลอตภายนอก Intune อยู่นอกขอบเขตสำหรับสวัสดิการของ FastTrack

### <a name="enable-phase---co-management"></a>เปิดใช้งานระยะ-การจัดการร่วม

ให้คำแนะนำเกี่ยวกับ:

-   ให้สิทธิ์ผู้ใช้ปลายทางของคุณ

-   การเพิ่มผู้ใช้ไปยังการบอกรับเป็นสมาชิก Intune ของคุณกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์ (ถ้า Intune ไม่ได้ติดตั้ง)

-   การตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับการลงทะเบียนอัตโนมัติ MDM

-   ตั้งค่าการเข้าร่วมไดเรกทอรีที่ใช้งานอยู่ของไฮบริดสลี Azure

-   ตั้งค่าเกตเวย์จัดการคลาวด์

-   การเพิ่มผู้ใช้ไปยังการบอกรับเป็นสมาชิก Intune ของคุณกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

-   เตรียม Intune (ถ้าไม่ได้ติดตั้ง Intune):

    -   การตั้งค่าคอนฟิกสิทธิ์การจัดการอุปกรณ์เคลื่อนที่ของคุณ) ขึ้นอยู่กับความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงาน MDM ของคุณ

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

    -   การนำทางพอร์ทัล admin Intune เพื่อค้นหาข้อมูลบนผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ตัวดำเนินการโต๊ะช่วยเหลือผู้ดูแลระบบฯลฯ)

    -   การกำหนดค่าและการปรับใช้นโยบายการป้องกันแอป Intune สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   การลงทะเบียนอุปกรณ์ Windows 10 ไปยัง Intune ของคุณ

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่า

- เปลี่ยนเวิร์กโหลดเป็น Intune

- ติดตามกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานระยะ–การป้องกันข้อมูล Azure

ให้คำแนะนำเกี่ยวกับ: 

- การเปิดใช้งานและการตั้งค่าคอนฟิกผู้เช่าลูกค้า

- การสร้างและการตั้งค่าป้ายชื่อและนโยบาย

- การใช้การป้องกันข้อมูลกับเอกสาร 

- การจัดประเภทและการติดฉลากข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทำงานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure

- การใช้แฟ้มที่เหลือด้วยเครื่องสแกนเนอร์การป้องกันข้อมูล Azure

- การตรวจสอบจดหมายในการส่งต่อโดยใช้กฎของโฟลว์จดหมาย Exchange แบบออนไลน์

คำแนะนำยังให้แก่ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ Azure ของ Microsoft (Azure RMS), Office ๓๖๕การเข้ารหัสลับข้อความ (OME), และการป้องกันการสูญหายของข้อมูล (DLP)

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมหรือไม่** ดูการ[เคลื่อนย้ายขององค์กร + ความปลอดภัย](https://www.microsoft.com/cloud-platform/enterprise-mobility)
