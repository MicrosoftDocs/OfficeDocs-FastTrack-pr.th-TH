# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบ FastTrack

FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างการปฐมนิเทศ

## <a name="general"></a>ทั่วไป

-   ให้ความช่วยเหลือระยะไกลสนับสนุนแก่คุณสำหรับกิจกรรมที่ตั้งค่าคอนฟิกที่จำเป็นตามที่แสดงรายการในคำอธิบายขั้นตอนโดยละเอียด

-   แสดงเอกสารที่พร้อมใช้งาน เครื่องมือซอฟต์แวร์ และคอนโซล admin เพื่อช่วยให้คุณสามารถลด หรือกำจัดการตั้งค่าคอนฟิกภารกิจ

## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

-   ทำงานกับคุณเพื่อเริ่มการปฐมนิเทศ

-   กำหนดว่าบริการใดมีสิทธิ์คุณต้องเป็น onboard

## <a name="assess-phase"></a>ขั้นตอนประเมิน

-   แสดงภาพรวมสำหรับผู้ดูแล

-   ให้คำแนะนำเกี่ยวกับ:

    -   DNS เครือข่าย และโครงสร้างพื้นฐานของความต้องการ

    -   ไคลเอนต์ความต้องการ (อินเทอร์เน็ตเบราว์เซอร์ ระบบปฏิบัติการของไคลเอนต์ และความต้องการของบริการ)

    -   ข้อมูลเฉพาะตัวของผู้ใช้และการเตรียมใช้งาน

    -   การเปิดใช้งานสิทธิ์ได้รับบริการที่มีการสั่งซื้อ และกำหนดไว้เป็น ส่วนหนึ่งของการปฐมนิเทศ

-   สร้างเส้นเวลาสำหรับกิจกรรมด้าน

-   แสดงรายการตรวจสอบด้านทั้ง Intune และ Azure โฆษณาพิเศษ

## <a name="remediate-phase"></a>Remediate ขั้นตอน

-   หยุดการเรียกประชุมกับคุณตามกำหนดเวลาที่ตกลงไว้เพื่อตรวจสอบความคืบหน้าของกิจกรรมด้าน เช่น แนะนำคุณเกี่ยวกับการติดตั้งเบื้องต้นก่อนหน้าปฐมนิเทศบริการ cloud Microsoft

## <a name="enable-phase"></a>ขั้นตอนการเปิดใช้งาน
ให้คำแนะนำเกี่ยวกับ:

-   เปิดใช้งานผู้เช่าบริการแบบออนไลน์ของ Microsoft หรือสมัครใช้งานของคุณ

-   การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์

-   การกำหนดค่า DNS สำหรับบริการที่เหมาะสม

-   ตรวจสอบการเชื่อมต่อกับบริการแบบออนไลน์ของ Microsoft

-   สำหรับสภาพแวดล้อมของฟอเรสต์เดียว:

    -   ติดตั้งเซิร์ฟเวอร์ไดเรกตอรีซิงโครไนส์ระหว่างของบริการไดเรกทอรีโดเมน (AD DS) ที่ใช้งานอยู่และเหมาะสม Microsoft บริการแบบออนไลน์ (เฉพาะคำแนะนำถ้าจำเป็น)

    -   กำหนดค่าการพิสูจน์ตัวจริงที่มีการจัดการ (ซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องของพาส-ทรู) โดยใช้เครื่องมือ Azure Active Directory เชื่อมต่อ (เฉพาะคำแนะนำถ้าจำเป็น)

        > [!NOTE]
        > การพัฒนาและการใช้งานสำหรับส่วนขยายของกฎที่กำหนดเองอยู่นอกขอบเขต

-   สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นผู้ใช้อีเมลที่ติดต่อกับภายนอก: การติดตั้ง และตั้งค่าคอนฟิกบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) สำหรับการตรวจสอบโดเมนเฉพาะกับ Intune ในการกำหนดค่า ไซต์เดียว แบบทนต่อ ถ้าจำเป็น

    > [!NOTE]
    > สำหรับหลายฟอเรสต์โครงแบบ จัดวางโฆษณา FS อยู่นอกขอบเขต

-   การทดสอบเดียวเข้าสู่ระบบ (SSO) ฟังก์ชัน ถ้ามีการปรับใช้

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งานขั้นตอน - Microsoft Azure พิเศษไดเรกทอรีที่ใช้งานอยู่

ให้คำแนะนำเกี่ยวกับ:

- เปิดใช้งานผู้เช่าพิเศษโฆษณา Azure ของคุณ

- ตั้งค่าคอนฟิกพอร์ตไฟร์วอลล์

- การกำหนดค่า DNS สำหรับบริการที่เหมาะสม

- ตรวจสอบการเชื่อมต่อกับบริการพิเศษโฆษณา Azure

- สำหรับสภาพแวดล้อมของฟอเรสต์เดียว:

  -   ติดตั้งไดเรกทอรีที่ปรับให้ตรงกันระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) ของคุณและเชื่อมต่อ AD Azure ถ้าจำเป็น

  -   ตั้งค่าคอนฟิกวิธีการรับรองความถูกต้อง (ซิงค์แฮชของรหัสผ่านหรือการรับรองความถูกต้องของพาส-ทรู) ด้วยเครื่องมือการเชื่อมต่อ AD Azure

- สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการซิงโครไนส์เชื่อมต่อ AD Azure ตั้งค่าฟอเรสต์หลายสถานการณ์
- สำหรับสภาพแวดล้อมของฟอเรสต์เดียว และหลาย:
  - การกำหนดค่า Azure Active Directory พาส-ทรูรับรองความถูกต้อง ถ้าจำเป็น
  - การกำหนดค่า Azure Active Directory อย่างราบเดียวเข้าสู่ระบบ (SSO), ถ้าจำเป็น
    > [!NOTE]
    > Azure Active Directory พาส-ทรูรับรองความถูกต้องสำหรับสภาพแวดล้อมหลายฟอเรสต์ได้รับการสนับสนุนถ้ามีความเชื่อถือฟอเรสต์ระหว่างฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่ของคุณและ ถ้ามีการกำหนดเส้นทางส่วนต่อท้ายชื่อถูกกำหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายตัวเพื่อให้พร้อมใช้งานสูงสำหรับการร้องขอการเข้าสู่ระบบ

  - สำหรับข้อมูลเพิ่มเติม ดู[Azure Active Directory พาส-ทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)และ[Azure Active Directory อย่างราบเดียวเข้าสู่ระบบ: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจำกัดของการรับรองความถูกต้องแบบพาส-ทรู ดู[Azure Active Directory พาส-ทรูรับรองความถูกต้อง: ข้อจำกัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหาอย่างราบ SSO ดู[การแก้ปัญหา Azure ใช้งานไดเรกทอรีอย่างราบเดียวเข้าสู่ระบบ](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > ซิงค์แฮชของรหัสผ่านและรหัสผ่านการเขียนกลับสนับสนุนหลายป่าไม้ อย่างไรก็ตาม สถานการณ์อื่น ๆ เขียนกลับไม่ได้รับการสนับสนุน

  - การกำหนดค่าให้ตรงกันระหว่างฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่ในสถานและไดเรกทอรีของ Microsoft Azure พิเศษไดเรกทอรีที่ใช้งานอยู่ (Azure ไดเรกทอรีที่ใช้งานอยู่)

    > [!NOTE]
    > การพัฒนาและการใช้งานสำหรับส่วนขยายของกฎที่กำหนดเองอยู่นอกขอบเขต

- สำหรับข้อมูลเฉพาะตัวของฟอเรสต์เดียวเมื่อมีเป้าหมายติดต่อกับภายนอก:

  -   การติดตั้ง และตั้งค่าคอนฟิก FS โฆษณาสำหรับการรับรองความถูกต้องของโดเมนเฉพาะ ด้วยค่าจ้างพิเศษโฆษณา Azure ในการกำหนดค่า ไซต์เดียว แบบทนต่อ (ถ้าจำเป็น)

  > [!NOTE]
  > สำหรับหลายฟอเรสต์โครงแบบ จัดวางโฆษณา FS อยู่นอกขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้าใช้)

### <a name="enable-phase---azure-ad-premium--with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งานขั้นตอน - พิเศษโฆษณา Azure - มีการเชื่อมต่อ AD Azure และ FS โฆษณา

ให้คำแนะนำเกี่ยวกับการตั้งค่า:

- ผู้ใช้การเตรียมใช้งาน รวมถึงการอนุญาตให้ใช้สิทธิ์

- ตรงไดเรกทอรีการเชื่อมต่อ AD azure (มีการเขียนกลับรหัสผ่านและซิงค์แฮชของรหัสผ่าน)

  - ไดเรกทอรีที่ใช้งานอยู่ที่ azure Self บริการรีเซ็ตรหัสผ่าน (SSPR)

  - พิสูจน์ตัวจริงด้วยหลายปัจจัย azure

  - ได้ถึงสาม (3) หรือเพิ่มเติมซอฟต์แวร์เป็นการรวมแอพลิเคชันเป็นบริการ (SaaS) กับหนึ่งเข้าสู่ระบบ (SSO) จาก[Azure Marketplace ของไดเรกทอรีที่ใช้งานอยู่](https://azure.microsoft.com/marketplace/active-directory/)

  - กำลังเตรียมใช้งานผู้ใช้โดยอัตโนมัติสำหรับล่วงหน้ารวมแอพลิเคชัน SaaS ตามที่แสดงรายการอยู่ใน[รายการของบทสอนการรวมแอพลิเคชัน](https://docs.microsoft.com/en-us/azure/active-directory/saas-apps/tutorial-list)จำกัดจัดสรรขาออกเท่านั้น

  - เข้าสู่ระบบที่กำหนดเองหน้าจอแสดงผล รวมถึงโลโก้ ข้อความ และภาพพื้นหลัง

  - บริการตนเอง และแบบไดนามิกกลุ่ม (กลุ่ม)

  - พร็อกซีโปรแกรมประยุกต์ azure ไดเรกทอรีที่ใช้งานอยู่

  - ไดเรกทอรีที่ใช้งานอยู่ที่ azure เชื่อมต่อสุขภาพ

  - เข้าถึงไดเรกทอรีที่ใช้งานอยู่ที่ azure ตามเงื่อนไข

  - ไดเรกทอรีที่ใช้งานอยู่ที่ azure ข้อตกลงการใช้งาน

  - การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ที่ azure

  - ไดเรกทอรีที่ใช้งานอยู่ที่ azure มีสิทธิ์จัดการข้อมูลประจำตัว

  - บทวิจารณ์เข้าถึงไดเรกทอรีที่ใช้งานอยู่ที่ azure

### <a name="enable-phase---intune"></a>เปิดใช้งานขั้นตอน - Intune

> [!IMPORTANT]
> FastTrack ไม่สนับสนุนการจัดการพีซีคลาสสิค Windows 10 ด้วย Intune FastTrack สนับสนุนเฉพาะ Windows 10 จัดการผ่านทางการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM) เท่านั้น

ให้**คำแนะนำ**บน:

-   การกำหนดค่าข้อมูลเฉพาะตัวที่จะใช้ โดยการใช้ประโยชน์ของคุณในสถานไดเรกทอรีที่ใช้งานอยู่หรือก้อนเมฆรหัสประจำตัว (Azure Active Directory) จาก Intune

-   อนุญาตให้ใช้สิทธิ์ของผู้ใช้

-   เพิ่มผู้ใช้ให้การสมัครใช้งาน Intune การกำหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และอุปกรณ์

-   ตั้งค่าคอนฟิก MDM การจัดการอุปกรณ์มือถือของคุณ) หน่วยงาน ตามความต้องการจัดการ รวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM

    -   การกำหนดค่ากลุ่มทดสอบที่จะใช้ในการตรวจสอบ MDM นโยบายการจัดการ

    -   การนำทางในพอร์ทัล admin Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ผู้ให้บริการความช่วยเหลือโต๊ะทำงาน admins เป็นต้น)

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   ปรับใช้แอพลิเคชันสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านการเชื่อมโยงเว็บ MSI และ/หรือความลึกของการเชื่อมโยง

        -   ปรับใช้ Office ProPlus บนอุปกรณ์ Windows 10

        -   โปรแกรมการซื้อไดรฟ์ข้อมูลสำหรับการปรับใช้โปรแกรมประยุกต์ รวมถึงของ Apple VPP ร้านค้า Windows สำหรับธุรกิจ และเล่นของ Google ของร้านทำงาน

        -   การปรับใช้ของอีเมล เครือข่ายไร้สาย และโปรไฟล์ VPN ถ้าคุณมีใบรับรองที่มีอยู่ Wi-fi หรือ VPN โครงสร้างพื้นฐานในองค์กรของคุณ

        -   การตั้งค่าที่ Microsoft Intune แลกเปลี่ยนตัวเชื่อมต่อ (เมื่อใช้ได้)

        -   โพรไฟล์การตั้งค่าคอนฟิกอุปกรณ์สำหรับแพลตฟอร์มของอุปกรณ์ที่ได้รับการสนับสนุน

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   ตั้งค่าคอนฟิก และปรับใช้นโยบายการป้องกันโปรแกรมประยุกต์ Intune สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   กำลังเตรียมบรรทัดของธุรกิจ (LOB) apps สำหรับ Intune app ป้องกันนโยบาย ด้วยคำแนะนำในตัวเลือกที่พร้อมใช้งาน

    -   ลงทะเบียนอุปกรณ์ของแต่ละรายการได้รับการสนับสนุนแพลตฟอร์ม Intune หรือตัวจัดการการตั้งค่าคอนฟิกบริการ Microsoft Intune ของคุณ

    -   การเชื่อมต่อกับคลังสินค้า Intune ข้อมูล

    -   Intune กับการรวม:
        -   ทีม Viewer สำหรับความช่วยเหลือระยะไกล (การบอกรับเป็นสมาชิกทีม Viewer จำเป็น)

        -   ป้องกันการคุกคามมือถือหุ้นส่วนการแก้ไขปัญหา (Mobile ป้องกันการคุกคามคู่ค้าโซลูชันบอกรับเป็นสมาชิกที่จำเป็น)

        -   โทรคมนาคมค่าใช้จ่ายบริหารการแก้ไขปัญหา (สมัครใช้งานโซลูชันการจัดการค่าใช้จ่ายโทรคมนาคมจำเป็น)

        -   การป้องกันภัยคุกคามขั้นสูงของ Windows Defender (ใบอนุญาต Windows E5 หรือ Microsoft 365 E5 จำเป็นต้องมี)

    -   การกำหนดค่าการปรับปรุงซอฟต์แวร์สำหรับแพลตฟอร์มที่ได้รับการสนับสนุนที่เกี่ยวข้อง

    -   ทรัพยากรสำหรับผู้ใช้เริ่มการวางแผน

- การตั้งค่า Windows Autopilot:

    - กำหนดค่า และการตั้งค่า Microsoft Intune สำหรับ Windows Autopilot

    - การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกโฆษณา Azure

    - เพิ่มบริษัทของคุณกำหนดตราสินค้าไว้ในโฆษณา Azure

    - สร้าง และกำหนดอุปกรณ์กับส่วนกำหนดค่า Windows Autopilot (เช่นโพรไฟล์ Windows Autopilot ที่จำกัดการสร้างบัญชีผู้ใช้ผู้ดูแลระบบท้องถิ่น)

    - การกำหนดเองออกของ--ประสบการณ์ใช้งานแบบทันที (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การตั้งค่าคอนฟิกการลงทะเบียนอัตโนมัติ MDM ในโฆษณา Azure และ Intune

    > [!NOTE]
    > การตั้งค่า Windows Autopilot นอก Intune อยู่นอกขอบเขตสำหรับสวัสดิการ FastTrack

### <a name="enable-phase---co-management"></a>เปิดใช้งานขั้นตอน - การจัดการร่วม

ให้คำแนะนำเกี่ยวกับ:

-   อนุญาตให้ใช้สิทธิ์ของผู้ใช้

-   เพิ่มผู้ใช้ให้การสมัครใช้งาน Intune การกำหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และอุปกรณ์ (ถ้าไม่ได้ติดตั้ง Intune)

-   การตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับการลงทะเบียนอัตโนมัติ MDM

-   ตั้งค่าไฮบริดสลี Azure Active Directory เข้าร่วม

-   ตั้งค่าเกตเวย์จัดการ Cloud

-   เพิ่มผู้ใช้ให้การสมัครใช้งาน Intune การกำหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และอุปกรณ์

-   เตรียม Intune (ถ้าไม่ได้ติดตั้ง Intune):

    -   ตั้งค่าคอนฟิก MDM การจัดการอุปกรณ์มือถือของคุณ) หน่วยงาน ตามความต้องการจัดการ รวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM

    -   การกำหนดค่ากลุ่มทดสอบที่จะใช้ในการตรวจสอบ MDM นโยบายการจัดการ

    -   การนำทางในพอร์ทัล admin Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ผู้ให้บริการความช่วยเหลือโต๊ะทำงาน admins เป็นต้น)

    -   ตั้งค่าคอนฟิก และปรับใช้นโยบายการป้องกันโปรแกรมประยุกต์ Intune สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   ลงทะเบียนอุปกรณ์ Windows 10 Intune ของคุณ

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่า

- สลับเวิร์กโหลดไปที่ Intune

- ตรวจสอบกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน – ป้องกันข้อมูล Azure

ให้การสนับสนุนสำหรับ: 

- ลูกค้าเพื่อจัดประเภท และป้ายชื่อข้อมูลในโปรแกรมประยุกต์ Office (เช่น Word, PowerPoint, Excel และ Outlook) โดยอัตโนมัติทำงานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure 
- แฟ้มที่เหลือที่ใช้สแกนเนอร์ป้องกันข้อมูล Azure
- อีเมล์ในการส่งต่อโดยใช้กฎกระแสจดหมาย Exchange แบบออนไลน์ 

นอกจากนี้ยังสนับสนุนที่ให้แก่ลูกค้าที่ต้องการใช้การป้องกันโดยใช้ บริการจัดการสิทธิ์ Azure ของ Microsoft (Azure RMS), การเข้ารหัสลับข้อความใน 365 Office (าน), และป้องกันการสูญหายของข้อมูล (DLP) 

ลูกค้าจะมีคำแนะนำเกี่ยวกับวิธีการ: 

- เปิดใช้งาน และตั้งค่าคอนฟิกของผู้เช่า
- สร้าง และตั้งค่าป้ายชื่อและนโยบาย
- ป้องกันข้อมูลให้กับเอกสาร 

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมได้อย่างไร** ดู[องค์กร Mobility + ความปลอดภัย](https://www.microsoft.com/cloud-platform/enterprise-mobility)
