# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างการเตรียมพร้อม

## <a name="general"></a>ทั่วไป

-   ให้ความช่วยเหลือสนับสนุนระยะไกลแก่คุณสําหรับกิจกรรมการกําหนดค่าที่จําเป็นตามที่ระบุไว้ในคําอธิบายเฟสโดยละเอียด

-   จัดเตรียมเอกสาร เครื่องมือซอฟต์แวร์ และคอนโซลผู้ดูแลระบบที่พร้อมให้ใช้งาน เพื่อช่วยคุณลดหรือกําจัดงานการกําหนดค่า

## <a name="initiate-phase"></a>เริ่มต้นเฟส

-   ทํางานร่วมกับคุณเพื่อเริ่มใช้งาน

-   กําหนดบริการที่มีสิทธิ์ที่คุณต้องการขึ้นเครื่อง

## <a name="assess-phase"></a>ประเมินระยะ

-   แสดงภาพรวมการบริหาร

-   ให้คําแนะนําเกี่ยวกับ:

    -   DNS เครือข่าย และโครงสร้างพื้นฐานที่ต้องการ

    -   ความต้องการของลูกค้า (อินเทอร์เน็ตเบราว์เซอร์ระบบปฏิบัติการของลูกค้าและความต้องการของบริการ')

    -   ข้อมูลเฉพาะตัวของผู้ใช้และการจัดเตรียม

    -   การเปิดใช้งานบริการที่มีสิทธิ์ซึ่งซื้อและกําหนดให้เป็นส่วนหนึ่งของการเตรียมพร้อม

-   สร้างไทม์ไลน์สําหรับกิจกรรมการแก้ไข

-   จัดเตรียมรายการตรวจสอบการแก้ไขสําหรับทั้ง Intun หรือ Azure AD Premium

## <a name="remediate-phase"></a>เฟสของการแก้ไข

-   จัดการประชุมทางโทรศัพท์กับคุณตามกําหนดการที่ตกลงกันไว้เพื่อทบทวนความคืบหน้าของกิจกรรมการแก้ไขเช่นแนะนําคุณผ่านการติดตั้ง pre -requisites ก่อน onboarding บริการเมฆ Microsoft

## <a name="enable-phase"></a>เปิดใช้งานเฟส
ให้คําแนะนําเกี่ยวกับ:

-   การเปิดใช้งานผู้เช่าบริการออนไลน์ของ Microsoft หรือการสมัครสมาชิก

-   การกําหนดค่าโปรโตคอล TCP/IP และพอร์ตไฟร์วอลล์

-   การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์

-   กําลังตรวจสอบความถูกต้องของการเชื่อมต่อกับบริการออนไลน์ของ Microsoft

-   สําหรับสภาพแวดล้อมของฟอเรสต์เดียว:

    -   การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และมีสิทธิ์ Microsoft online Services (คําแนะนําเฉพาะถ้าจําเป็น)

    -   การกําหนดค่าการรับรองความถูกต้องที่มีการจัดการ (รหัสผ่านแฮซิงค์หรือการรับรองความถูกต้องผ่าน) ด้วยเครื่องมือการเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ของ Azure (เฉพาะคําแนะนําถ้าจําเป็น)

        > [!NOTE]
        > การพัฒนาและการใช้งานสําหรับส่วนขยายกฎที่กําหนดเองอยู่นอกขอบเขต

-   สําหรับฟอเรสต์เดียวเมื่อปลายทางเป็น identities fededrated: การติดตั้ง และการตั้งค่าคอนฟิกบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) สําหรับการรับรองความถูกต้องโดเมนท้องถิ่นกับ Intun เย้ในการกําหนดค่าแบบเดียวไซต์

    > [!NOTE]
    > สําหรับการกําหนดค่าฟอเรสต์หลายทั้งหมด FS โฆษณาจะอยู่นอกขอบเขต

-   ทดสอบฟังก์ชันการลงชื่อเพียงครั้งเดียว (SSO) ถ้าปรับใช้

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งานเฟส - พรีเมี่ยมของไดเรกทอรีที่ใช้งานอยู่ของ Microsoft Azure

ให้คําแนะนําเกี่ยวกับ:

- การเปิดใช้งานผู้เช่าพรีเมี่ยมโฆษณา Azure ของคุณ

- การกําหนดค่าพอร์ตไฟร์วอลล์

- การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์

- กําลังตรวจสอบความถูกต้องของการเชื่อมต่อกับบริการ Azure AD Premium

- สําหรับสภาพแวดล้อมของฟอเรสต์เดียว:

  -   การติดตั้งการซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และการเชื่อมต่อโฆษณา Azure ถ้าจําเป็น

  -   การกําหนดค่าวิธีการรับรองความถูกต้อง (รหัสผ่านแฮชซิงค์หรือการรับรองความถูกต้องผ่าน) ด้วยเครื่องมือการเชื่อมต่อโฆษณา Azure

- สําหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการซิงโครไนส์การเชื่อมต่อ AD Azure เซ็ตอัพสําหรับสถานการณ์ฟอเรสต์หลาย
- สําหรับสภาพแวดล้อมแบบเดี่ยวและฟอเรสต์หลาย:
  - การกําหนดค่า Azure ไดเรกทอรีที่ใช้งานอยู่แบบพาส-ทรูรับรองความถูกต้อง ถ้าจําเป็น
  - การกําหนดค่า Azure ไดเรกทอรีที่ใช้งานอยู่แบบจําแนกแบบครั้งเดียวเข้าสู่ระบบ (SSO), ถ้าจําเป็น
    > [!NOTE]
    > รับรองความถูกต้องของ azure ไดเรกทอรีที่ใช้งานอยู่แบบพาส-ทรูสําหรับสภาพแวดล้อมแบบหลายฟอเรสต์ได้รับการสนับสนุนถ้ามีฟอเรสต์เชื่อถือระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ของคุณ และถ้าเส้นทางคําต่อท้ายชื่อถูกกําหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายเพื่อให้พร้อมใช้งานสูงสําหรับการร้องขอการลงชื่อเข้าใช้

  - สําหรับข้อมูลเพิ่มเติม ให้ดูที่[Azure ที่ใช้งานอยู่ไดเรกทอรีแบบพาส-ทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)และ[Azure ที่ใช้งานอยู่ไดเรกทอรีแบบราบรื่นแบบครั้งเดียวเข้าสู่ระบบ: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สําหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจํากัดการรับรองความถูกต้องแบบพาส-ทรู โปรดดูที่[การรับรองความถูกต้องผ่านไดเรกทอรีที่ใช้งานอยู่ของ Azure: ข้อจํากัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สําหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO แบบไม่มีรอยต่อ ให้ดูที่[การแก้ไขปัญหาการลงชื่อเพียงครั้งเดียวแบบไม่มีรอยต่อของ Azure ไดเรกทอรีที่ใช้งานอยู่](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > รหัสผ่านแฮชซิงค์และรหัสผ่านเขียนกลับสนับสนุนป่าหลาย อย่างไรก็ตาม สถานการณ์การเขียนกลับอื่น ๆ ไม่ได้รับการสนับสนุน

  - การกําหนดค่าการทําข้อมูลให้ตรงกันระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ในสถานที่และไดเรกทอรีพรีเมี่ยมไดเรกทอรีที่ใช้งานอยู่ของ Microsoft Azure (Azure Active Directory)

    > [!NOTE]
    > การพัฒนาและการใช้งานสําหรับส่วนขยายกฎที่กําหนดเองอยู่นอกขอบเขต

- สําหรับฟอเรสต์เดียวเมื่อเป้าหมายมีบุคลิกติดต่อกับภายนอก:

  -   การติดตั้งและการกําหนดค่า FS โฆษณาสําหรับการรับรองความถูกต้องโดเมนท้องถิ่นด้วย Azure AD Premium ในการกําหนดค่าแบบเดียวไซต์

  > [!NOTE]
  > สําหรับการกําหนดค่าฟอเรสต์หลายทั้งหมด FS โฆษณาจะอยู่นอกขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้าปรับใช้)

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งานเฟส - พรีเมี่ยมโฆษณา Azure - ด้วยการเชื่อมต่อโฆษณา Azure และ FS โฆษณา

ให้คําแนะนําเกี่ยวกับการตั้งค่า:

- การจัดสรรผู้ใช้ รวมถึงการให้สิทธิ์ใช้งาน

- ซิงโครไนส์ไดเรกทอรี AD เชื่อมต่อ azure (ด้วยรหัสผ่านเขียนกลับและรหัสผ่านแฮซิงค์)

  - การตั้งค่ารหัสผ่านบริการตนเองของ Azure ไดเรกทอรีที่ใช้งานอยู่ (SSPR)

  - การรับรองความถูกต้องด้วยหลายปัจจัยของ azure

  - ซอฟต์แวร์ที่รวมเป็นบริการ (SaaS) กับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ของ Azure](https://azure.microsoft.com/marketplace/active-directory/)

  - การจัดสรรผู้ใช้โดยอัตโนมัติสําหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามรายการ[บทแนะนําการรวมแอป](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จํากัด เฉพาะการจัดสรรขาออกเท่านั้น

  - หน้าจอการเข้าสู่ระบบที่กําหนดเองรวมถึงโลโก้ข้อความและภาพ

  - กลุ่มบริการตนเองและแบบไดนามิก (กลุ่ม)

  - พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ azure

  - ความสมบูรณ์ของการเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ของ azure

  - การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ของ azure

  - ข้อกําหนดการใช้งานไดเรกทอรีที่ใช้งานอยู่ของ azure

  - การป้องกันข้อมูลประจําตัวของไดเรกทอรีที่ใช้งานอยู่ของ azure

  - การจัดการข้อมูลประจําตัวที่ได้รับสิทธิพิเศษของ azure ไดเรกทอรีที่ใช้งานอยู่

  - ความคิดเห็นการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ azure

  -   การป้องกันรหัสผ่านของไดเรกทอรีที่ใช้งานอยู่ของ azure

  -   ไดเรกทอรีที่ใช้งานอยู่ของ azure B2B.

### <a name="enable-phase---intune"></a>เปิดใช้งานเฟส - Intun

> [!IMPORTANT]
> FastTrack ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 ด้วย Intun FastTrack สนับสนุนเฉพาะการจัดการ Windows 10 ผ่านการจัดการอุปกรณ์เคลื่อนที่ Intun

ให้คําแนะนําเกี่ยวกับ:

-   การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้ โดย Intun

-   การให้สิทธิ์ผู้ใช้ปลายทางของคุณ

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intuner ของคุณ

-   การกําหนดค่าหน่วยงานจัดเก็บอุปกรณ์เคลื่อนที่ MDM) ของคุณ ตามความต้องการการจัดการของคุณ รวมถึง

    -   การตั้งค่า Intun เป็นหน่วยงาน MDM ของคุณ

    -   การกําหนดค่ากลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

    -   การนําทางพอร์ทัลผู้ดูแลระบบ Intun เนเพื่อค้นหาข้อมูลบนผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intunet (ผู้ให้บริการช่วยเหลือ ผู้ดูแลระบบ ฯลฯ)

    -   การกําหนดค่านโยบายการจัดการ MDM และบริการต่างๆ เช่น

        -   การปรับใช้โปรแกรมประยุกต์สําหรับแต่ละแพลตฟอร์มที่สนับสนุนผ่านเว็บลิงค์, MSI และ / หรือการเชื่อมโยงลึก

        -   การปรับใช้ Office ProPlus ไปยังอุปกรณ์ Windows 10

        -   โปรแกรมการซื้อแบบปริมาณสําหรับการปรับใช้แอป รวมถึง VPP ของ Apple, Windows Store สําหรับธุรกิจ และ Play for Work Store ของ Google

        -   การปรับใช้อีเมล เครือข่ายไร้สาย และโปรไฟล์ VPN ถ้าคุณมีผู้ให้บริการออกใบรับรอง Wi-Fi หรือ VPN ที่มีอยู่ในองค์กรของคุณ

        -   การตั้งค่าตัวเชื่อมต่อการแลกเปลี่ยน Intun เน Microsoft (เมื่อเกี่ยวข้อง)

        -   โปรไฟล์การกําหนดค่าอุปกรณ์สําหรับแพลตฟอร์มอุปกรณ์ที่รองรับ

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   การกําหนดค่าและปรับใช้นโยบายการป้องกันแอป Intun

    -   การเตรียมแอป line-of-business (LOB) สําหรับนโยบายการป้องกันแอป Intun

    -   การลงทะเบียนอุปกรณ์ของแต่ละแพลตฟอร์มที่สนับสนุนกับ Intun หรือตัวจัดการการตั้งค่าคอนฟิกของคุณกับบริการ Microsoft Intun

    -   การเชื่อมต่อกับคลังข้อมูล Intun

    -   การผสานรวม Intun กับ:
        -   ตัวแสดงทีมสําหรับความช่วยเหลือระยะไกล (จําเป็นต้องมีการสมัครสมาชิก Team Viewer)

        -   โซลูชั่นพันธมิตรการป้องกันภัยคุกคามมือถือ (การสมัครสมาชิกโซลูชั่นพันธมิตร Mobile Defense เป็นสิ่งจําเป็น)

        -   โซลูชั่นการจัดการค่าใช้จ่ายโทรคมนาคม (ค่าใช้จ่ายในการสมัครเทเลคอมโซลูชั่นการจัดการเป็นสิ่งจําเป็น)

        -   การป้องกันภัยคุกคามขั้นสูงของ Microsoft Defender (ต้องมีสิทธิ์การใช้งาน Windows E5 หรือ Microsoft 365 E5)

    -   การกําหนดค่าการอัพเดตซอฟต์แวร์สําหรับแพลตฟอร์มที่สนับสนุนที่เกี่ยวข้อง

    -   ทรัพยากรสําหรับการวางแผนการนําผู้ใช้มาใช้

- การตั้งค่า Windows อัตโนมัติ:

    - กําหนดค่าและตั้งค่าอินทูนไมโครซอฟท์สําหรับ Windows หม้อแปลงไฟล

    - กําหนดค่ากลุ่มแบบไดนามิกของ Azure AD

    - เพิ่มการสร้างแบรนด์บริษัทของคุณลงในโฆษณา Azure

    - สร้างและกําหนดอุปกรณ์ให้กับส่วนกําหนดค่าระบบอัตโนมัติของ Windows (เช่น ส่วนกําหนดค่า Windows Autopilot ที่จํากัดการสร้างบัญชีผู้ดูแลระบบภายในเครื่อง)

    - กําหนดประสบการณ์นอกกรอบ (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การกําหนดค่าการลงทะเบียนอัตโนมัติ MDM ใน Azure AD และ Intun

    > [!NOTE]
    > การตั้งค่า Windows Autopilot นอก Intuni อยู่นอกขอบเขตสําหรับประโยชน์ FastTrack

### <a name="enable-phase---cloud-attach"></a>เปิดใช้งานเฟส - ติดคลาวด์

ให้คําแนะนําเกี่ยวกับ:

-   การให้สิทธิ์ผู้ใช้ปลายทางของคุณ

-   การใช้การแนบบนคลาวด์ในคอนโซลการจัดการการตั้งค่าคอนฟิก

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intunn ของคุณ

-   การตั้งค่าการรวมไดเรกทอรีที่ใช้งานอยู่ของ Azure ไฮบริดสลี

-   การตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับการลงทะเบียน MDM อัตโนมัติ

-   การตั้งค่าเกตเวย์การจัดการระบบคลาวด์

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intuner ของคุณ

-   กําลังเตรียมบริการ Intun

-   การตั้งค่าสิทธิการจัดการอุปกรณ์เคลื่อนที่ (MDM) เป็น Intun

-   การกําหนดค่ากลุ่มการทดสอบที่จะใช้ในการตรวจสอบความถูกต้องของนโยบายการจัดการ MDM

-   การนําทางพอร์ทัลผู้ดูแลระบบ Intun เนเพื่อค้นหาข้อมูลบนผู้ใช้และอุปกรณ์

-   การตั้งค่าบทบาท Intun หรือ Intun (ผู้ดําเนินการช่วยเหลือ ผู้ดูแลระบบ และอื่นๆ)

-   การลงทะเบียนอุปกรณ์ Windows 10 ไปยัง Intun

-   การสลับปริมาณงานสําหรับการจัดการโดย Intun

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน – การป้องกันข้อมูล Azure

คําแนะนําที่จัดเตรียมไว้: 

- การเปิดใช้งานและการตั้งค่าคอนฟิกผู้เช่าของลูกค้า

- การสร้างและการตั้งค่าป้ายชื่อและนโยบาย

- การใช้การป้องกันข้อมูลกับเอกสาร 

- การจัดประเภทและป้ายชื่อข้อมูลในแอป Office โดยอัตโนมัติ (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทํางานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure

- การใช้แฟ้มที่เหลือกับสแกนเนอร์การป้องกันข้อมูล Azure

- การตรวจสอบอีเมลในระหว่างการขนส่งโดยใช้กฎการรับส่งจดหมายแบบออนไลน์ของ Exchange

คําแนะนํายังให้ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ของ Microsoft Azure (Azure RMS), การเข้ารหัสลับข้อความ Office 365 (OME) และข้อมูลการป้องกันการสูญหาย (DLP)

> **ต้องการเรียนรู้เพิ่มเติมหรือไม่** ดู[ความคล่องตัวขององค์กร + การรักษาความปลอดภัย](https://www.microsoft.com/cloud-platform/enterprise-mobility)
