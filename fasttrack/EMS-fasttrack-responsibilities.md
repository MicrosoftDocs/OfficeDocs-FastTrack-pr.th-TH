# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

มีความรับผิดชอบต่อไปนี้ในระหว่างการปฐมนารถ

## <a name="general"></a>ทั่ว ไป

-   ให้ความช่วยเหลือสนับสนุนระยะไกลสําหรับกิจกรรมการกําหนดค่าที่จําเป็นตามที่แสดงในคําอธิบายระยะโดยละเอียด

-   จัดเตรียมเอกสาร เครื่องมือซอฟต์แวร์ และคอนโซลผู้ดูแลระบบเพื่อช่วยคุณลดหรือกําจัดงานการกําหนดค่า

## <a name="initiate-phase"></a>เริ่มต้นขั้นตอน

-   ทํางานร่วมกับคุณเพื่อเริ่มการปฐม

-   กําหนดบริการที่มีสิทธิ์ที่คุณต้องการออนบอร์ด

## <a name="assess-phase"></a>ประเมินระยะ

-   ให้ภาพรวมด้านการดูแลระบบ

-   ให้คําแนะนําเกี่ยวกับ:

    -   ความต้องการ DNS เครือข่าย และโครงสร้างพื้นฐาน

    -   ความต้องการของลูกค้า (อินเทอร์เน็ตเบราเซอร์, ระบบปฏิบัติการของลูกค้า, และความต้องการบริการ).

    -   ข้อมูลประจําตัวของผู้ใช้และการเตรียมใช้งาน

    -   การเปิดใช้งานบริการที่มีสิทธิ์ซึ่งได้สั่งซื้อและกําหนดให้เป็นส่วนหนึ่งของการเริ่มต้นใช้งาน

-   กําหนดเส้นเวลาสําหรับกิจกรรมการแก้ไข

-   แสดงรายการตรวจสอบการแก้ไขสําหรับทั้ง Intune และ Azure AD Premium

## <a name="remediate-phase"></a>ขั้นตอนการแก้ไข

-   จัดการประชุมทางโทรศัพท์กับคุณตามกําหนดเวลาที่ตกลงกันเพื่อทบทวนความคืบหน้าของกิจกรรมการแก้ไข เช่น แนะนําคุณเกี่ยวกับข้อกําหนดก่อนการติดตั้งก่อนการเริ่มใช้บริการระบบคลาวด์ของ Microsoft

## <a name="enable-phase"></a>เปิดใช้งานเฟส
ให้คําแนะนําเกี่ยวกับ:

-   การเปิดใช้งานผู้เช่าบริการออนไลน์ของ Microsoft หรือการสมัครใช้งาน

-   การกําหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์

-   การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์

-   การตรวจสอบความถูกต้องของการเชื่อมต่อกับบริการออนไลน์ของ Microsoft

-   สําหรับสภาพแวดล้อมของฟอเรสต์เดียว:

    -   การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และบริการแบบออนไลน์ของ Microsoft ที่มีสิทธิ์ (เฉพาะคําแนะนําถ้าจําเป็น)

    -   การตั้งค่าคอนฟิกการรับรองความถูกต้องที่มีการจัดการ (รหัสผ่านแฮซิงค์หรือการรับรองความถูกต้องแบบพาส-ทรู) ด้วยเครื่องมือ Azure Active Directory เชื่อมต่อ (เฉพาะคําแนะนําถ้าจําเป็น)

        > [!NOTE]
        > การพัฒนาและการดําเนินการสําหรับส่วนขยายกฎที่กําหนดเองจะออกจากขอบเขต

-   สําหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก: การติดตั้ง และการตั้งค่าคอนฟิก Active Directory สหพันธรัฐบริการ (AD FS) สําหรับการรับรองความถูกต้องของโดเมนเฉพาะที่ Intune ในไซต์เดียว, การกําหนดค่าทนต่อข้อบกพร่อง ถ้าจําเป็น

    > [!NOTE]
    > สําหรับการตั้งค่าคอนฟิกฟอเรสต์หลายการตั้งค่าคอนฟิก AD FS จะออกจากขอบเขต

-   ทดสอบฟังก์ชันการเข้าสู่ระบบแบบครั้งเดียว (SSO) ถ้าปรับใช้

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งานระยะ - Microsoft Azure แบบพรีเมียมไดเรกทอรีที่ใช้งานอยู่

ให้คําแนะนําเกี่ยวกับ:

- การเปิดใช้งานผู้เช่า Azure AD Premium ของคุณ

- การกําหนดค่าพอร์ตไฟร์วอลล์

- การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์

- การตรวจสอบความถูกต้องของการเชื่อมต่อกับบริการ Azure Premium

- สําหรับสภาพแวดล้อมของฟอเรสต์เดียว:

  -   การติดตั้งการซิงโครไนส์ของไดเรกทอรีระหว่างบริการการโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และการเชื่อมต่อ AD Azure ถ้าจําเป็น

  -   การกําหนดค่าวิธีการการรับรองความถูกต้อง (รหัสผ่านแฮซิงค์หรือรหัสผ่านผ่านการรับรองความถูกต้อง) ด้วยเครื่องมือ Azure AD Connect

- สําหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการทําข้อมูลการเชื่อมต่อโฆษณา Azure ทําข้อมูลให้ตรงกัน
- สําหรับสภาพแวดล้อมแบบฟอเรสต์เดี่ยวและหลายฟอเรสต์:
  - การตั้งค่าคอนฟิก Azure Active Directory ทรูพาส-ทรูรับรองความถูกต้อง ถ้าจําเป็น
  - การตั้งค่าคอนฟิก Azure ไดเรกทอรีที่ใช้งานอยู่แบบ Seamless เดียวเข้าสู่ระบบ (SSO), ถ้าจําเป็น
    > [!NOTE]
    > Azure Active Directory พาส-ทรูรับรองความถูกต้องสําหรับสภาพแวดล้อมหลายฟอเรสต์ได้รับการสนับสนุนถ้าไม่มีความเชื่อถือฟอเรสต์ระหว่างฟอเรสต์ Active Directory ของคุณ และถ้าการกําหนดเส้นทางส่วนต่อท้ายชื่อถูกกําหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายเพื่อให้มีพร้อมใช้งานสูงสําหรับคําขอการเข้าสู่ระบบ

  - สําหรับข้อมูลเพิ่มเติม ให้ดู[Azure Active Directory ทรูทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)และ[Azure Active Directory สู่ระบบเดียว: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สําหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจํากัดการรับรองความถูกต้องแบบพาส-ทรู ให้ดู[Azure Active Directory ทรูพาส-ทรูรับรองความถูกต้อง: ข้อจํากัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สําหรับข้อมูลเพิ่มเติมเกี่ยวกับการตัดสินค้าจากคลัง SSO แบบราบรื่น ให้ดูที่ การแก้ไขปัญหา Azure Active Directory แบบ[ไม่มีรอยต่อครั้งเดียว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > การซิงค์แฮรหัสผ่านและการเขียนรหัสผ่านสนับสนุนหลายฟอเรสต์ อย่างไรก็ตาม สถานการณ์ writeback อื่น ๆ ไม่ได้รับการสนับสนุน

  - การตั้งค่าคอนฟิกการซิงโครไนส์ระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ในสถานที่และไดเรกทอรีของ Microsoft Azure ที่ใช้งานอยู่ไดเรกทอรีค่าจ้างไดเรกทอรี (Azure ไดเรกทอรีที่ใช้งานอยู่)

    > [!NOTE]
    > การพัฒนาและการดําเนินการสําหรับส่วนขยายกฎที่กําหนดเองจะออกจากขอบเขต

- สําหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก:

  -   การติดตั้ง และการตั้งค่าคอนฟิก FS โฆษณาสําหรับการรับรองความถูกต้องของโดเมนเฉพาะที่ ด้วย Azure โฆษณาพิเศษในการกําหนดค่าแบบทนต่อความผิดพลาดไซต์เดียว (ถ้าจําเป็น)

  > [!NOTE]
  > สําหรับการตั้งค่าคอนฟิกฟอเรสต์หลายการตั้งค่าคอนฟิก AD FS จะออกจากขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้าปรับใช้)

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งานเฟส - Azure โฆษณาพิเศษ - กับ Azure AD เชื่อมต่อและ FS โฆษณา

ให้คําแนะนําเกี่ยวกับการตั้งค่า:

- การจัดสรรผู้ใช้ รวมถึงการให้สิทธิ์การใช้งาน

- ซิงโครไนส์ไดเรกทอรีของ Azure AD เชื่อมต่อ (มีรหัสผ่านและรหัสผ่านที่ซิงค์แฮ)

  - Azure ใช้งานไดเรกทอรีบริการตนเองรหัสผ่านใหม่ (SSPR)

  - การรับรองความถูกต้องด้วยหลายปัจจัย azure

  - ขึ้นไปสาม (3) ซอฟต์แวร์เป็นการรวมแอพลิเคชันบริการ (SaaS) กับ Single Sign-On (SSO) จาก[Azure ที่ใช้งานอยู่ไดเรกทอรีตลาด](https://azure.microsoft.com/marketplace/active-directory/)

  - การจัดสรรผู้ใช้อัตโนมัติสําหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามที่ระบุไว้ใน[รายการบทแนะนําการรวมแอพ](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จํากัด การจัดสรรขาออกเท่านั้น

  - หน้าจอการเข้าสู่ระบบแบบกําหนดเอง รวมถึงโลโก้ ข้อความ และรูปภาพ

  - บริการตนเองและกลุ่มแบบไดนามิก (กลุ่ม)

  - พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ของ azure

  - azure. azure

  - การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ azure

  - ข้อตกลงการใช้งานไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - การป้องกันข้อมูลประจําตัวของไดเรกทอรีที่ใช้งานอยู่ของ azure

  - การจัดการข้อมูลประจําตัวของ Azure ที่มีสิทธิ์ของไดเรกทอรีที่ใช้งานอยู่

  - การตรวจทานการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ azure

  -   การป้องกันรหัสผ่านของไดเรกทอรีที่ใช้งานอยู่ของ Azure

  -   Azure B2B ไดเรกทอรีที่ใช้งานอยู่

### <a name="enable-phase---intune"></a>เปิดใช้งานเฟส - Intune

> [!IMPORTANT]
> ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 ด้วย Intune FastTrack สนับสนุนการจัดการ Windows 10 เท่านั้นผ่านการจัดการอุปกรณ์เคลื่อนที่ Intune (MDM)

ให้คําแนะนําเกี่ยวกับ:

-   การกําหนดค่าข้อมูลเฉพาะตัวที่จะใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่ใช้งานอยู่ในสถานที่ของคุณหรือข้อมูลเฉพาะตัวของ cloud (Azure Active Directory)

-   การให้สิทธิ์การใช้งานผู้ใช้ของคุณ

-   การเพิ่มผู้ใช้ในการสมัครใช้งาน Intune ของคุณ การกําหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และกลุ่มอุปกรณ์

-   การกําหนดค่าอํานาจการจัดการอุปกรณ์เคลื่อนที่ MDM) ของคุณขึ้นอยู่กับความต้องการการจัดการของคุณ รวมถึง:

    -   การตั้งค่า Intune เป็นอํานาจ MDM ของคุณ

    -   การกําหนดคอนฟิกกลุ่มการทดสอบที่จะใช้เพื่อตรวจสอบนโยบายการจัดการ MDM

    -   การนําทางพอร์ทัลผู้ดูแลระบบ Intune เพื่อค้นหาข้อมูลผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ตัวดําเนินการของฝ่ายให้ความช่วยเหลือ ผู้ดูแลระบบ ฯลฯ)

    -   การกําหนดค่านโยบายการจัดการ MDM และบริการเช่น:

        -   การปรับใช้แอปสําหรับแต่ละแพลตฟอร์มที่สนับสนุนผ่านลิงก์เว็บ MSI และ/หรือลิงก์ในลึก

        -   การปรับใช้ Office ProPlus ลงในอุปกรณ์ Windows 10

        -   โปรแกรมการซื้อจํานวนมากสําหรับการปรับใช้แอป รวมถึง VPP ของ Apple, Windows Store สําหรับธุรกิจ และ Play for Work Store ของ Google

        -   การปรับใช้อีเมล เครือข่ายไร้สาย และโปรไฟล์ VPN ถ้าคุณมีผู้ให้บริการออกใบรับรอง Wi-Fi หรือ VPN ที่มีอยู่ในองค์กรของคุณ

        -   การตั้งค่าการเชื่อมต่อการแลกเปลี่ยน Intune Microsoft (เมื่อเกี่ยวข้อง)

        -   โปรไฟล์การกําหนดค่าอุปกรณ์สําหรับแพลตฟอร์มอุปกรณ์ที่สนับสนุน

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   การกําหนดค่าและการปรับใช้นโยบายการป้องกันแอป Intune สําหรับแต่ละแพลตฟอร์มที่สนับสนุน

    -   การเตรียมแอปสายธุรกิจ (LOB) สําหรับนโยบายการป้องกันแอป Intune พร้อมคําแนะนําเกี่ยวกับตัวเลือกที่พร้อมใช้งาน

    -   ลงทะเบียนอุปกรณ์ของแต่ละแพลตฟอร์มได้รับการสนับสนุนไปยัง Intune หรือตัวจัดการการตั้งค่าคอนฟิกของคุณด้วยบริการ Microsoft Intune

    -   การเชื่อมต่อกับคลังข้อมูล Intune

    -   การรวม Intune กับ:
        -   ตัวแสดงทีมสําหรับความช่วยเหลือระยะไกล (ต้องการการสมัครสมาชิก Team Viewer)

        -   โซลูชั่นพันธมิตรการป้องกันภัยคุกคามมือถือ (Mobile Threat กลาโหมคู่ค้าสมัครใช้งานต้อง)

        -   โซลูชั่นการจัดการค่าใช้จ่ายโทรคมนาคม (ค่าใช้จ่ายในการจัดการค่าใช้จ่ายโทรคมนาคมที่สมัครใช้งานเป็นสิ่งจําเป็น)

        -   การป้องกันภัยคุกคามขั้นสูงของ Microsoft Defender (จําเป็นต้องมีสิทธิ์การใช้งาน Windows E5 หรือ Microsoft 365 E5)

    -   การกําหนดค่าการอัปเดตซอฟต์แวร์สําหรับแพลตฟอร์มที่สนับสนุนที่เกี่ยวข้อง

    -   ทรัพยากรสําหรับการวางแผนการปรับใช้สําหรับผู้ใช้

- การตั้งค่าออโตไพลอตของ Windows:

    - กําหนดค่าและตั้งค่า Intune ของ Microsoft สําหรับ Windows ออโตไพลอต

    - การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกของโฆษณา Azure

    - เพิ่มตราสินค้าของบริษัทของคุณลงใน Azure AD

    - สร้าง และกําหนดอุปกรณ์ไปยังส่วนกําหนดค่าของ Windows Autopilot (เช่นส่วนกําหนดค่าออโตไพลอตของ Windows ที่จํากัดการสร้างบัญชีผู้ดูแลระบบภายในเครื่อง)

    - กําหนดออกกล่องประสบการณ์ (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การกําหนดค่าการลงทะเบียนอัตโนมัติ MDM ในโฆษณา Azure และ Intune

    > [!NOTE]
    > การตั้งค่าออโตไพลอตของ Windows นอก Intune อยู่นอกขอบเขตสําหรับประโยชน์ FastTrack

### <a name="enable-phase---cloud-attach"></a>เปิดใช้งานเฟส - การแนบบนระบบคลาวด์

ให้คําแนะนําเกี่ยวกับ:

-   การให้สิทธิ์การใช้งานผู้ใช้ของคุณ

-   การใช้การแนบบนคลาวด์ในคอนโซลการจัดการการตั้งค่า

-   การเพิ่มผู้ใช้ในการสมัครใช้งาน Intune ของคุณ การกําหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และกลุ่มอุปกรณ์ (หาก Intune ไม่ได้ติดตั้งอยู่)

-   การตั้งค่าไฮบริดสลีรวมไดเรกทอรีที่ใช้งานอยู่

-   การตั้งค่าไดเรกทอรีที่ใช้งานอยู่ของ Azure สําหรับการลงทะเบียนอัตโนมัติ MDM

-   การตั้งค่าเกตเวย์การจัดการระบบคลาวด์

-   การเพิ่มผู้ใช้ในการสมัครใช้งาน Intune ของคุณ การกําหนดบทบาทผู้ดูแลระบบ IT และสร้างกลุ่มผู้ใช้และกลุ่มอุปกรณ์

-   กําลังเตรียมบริการ Intune สําหรับการจัดการอุปกรณ์:

    -   การตั้งค่าสิทธิ์การจัดการอุปกรณ์เคลื่อนที่ (MDM) เป็น Intune

    -   การกําหนดคอนฟิกกลุ่มการทดสอบที่จะใช้เพื่อตรวจสอบนโยบายการจัดการ MDM

    -   การนําทางพอร์ทัลผู้ดูแลระบบ Intune เพื่อค้นหาข้อมูลผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาท Intune (ผู้ดําเนินการช่วยเหลือ ผู้ดูแลระบบ และอื่นๆ)

    -   ลงทะเบียนอุปกรณ์ Windows 10 เพื่อ Intune

    -   การสลับปริมาณงานสําหรับการจัดการโดย Intune ตามต้องการ

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน – การป้องกันข้อมูล Azure

ให้คําแนะนําเกี่ยวกับ: 

- การเปิดใช้งานและการตั้งค่าคอนฟิกผู้เช่าของลูกค้า

- การสร้างและการตั้งค่าป้ายชื่อและนโยบาย

- การใช้การป้องกันข้อมูลกับเอกสาร 

- จัดประเภทและติดป้ายข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทํางานบน Windows และใช้ไคลเอ็นต์การป้องกันข้อมูล Azure

- การใช้แฟ้มต่างๆ กับสแกนเนอร์การป้องกันข้อมูล Azure

- ตรวจสอบอีเมลในระหว่างขนส่งโดยใช้กฎขั้นตอนจดหมาย Exchange Online

คําแนะนํายังถูกจัดเตรียมให้แก่ลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการจัดการสิทธิ์ Azure ของ Microsoft (Azure RMS), การเข้ารหัสลับข้อความ Office 365 (OME) และการป้องกันข้อมูลสูญหาย (DLP)

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมหรือไม่** ดู[องค์กร Mobility + การรักษาความปลอดภัย](https://www.microsoft.com/cloud-platform/enterprise-mobility)
