# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

> [!CAUTION]
> เนื้อหานี้ไม่ได้อยู่ในปัจจุบันอีกต่อไปและมีการจัดกำหนดการสำหรับการเอาออก ใช้สารบัญในการนำทางด้านซ้ายสำหรับเนื้อหาปัจจุบัน

FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างการปฐมนิเทศ

## <a name="general"></a>ทั่ว

-   ให้ความช่วยเหลือด้านการสนับสนุนระยะไกลแก่คุณสำหรับกิจกรรมการกำหนดค่าที่จำเป็นตามที่แสดงอยู่ในคำอธิบายขั้นตอนโดยละเอียด

-   มีเอกสารประกอบที่พร้อมใช้งานเครื่องมือซอฟต์แวร์และคอนโซลผู้ดูแลระบบเพื่อช่วยให้คุณลดหรือกำจัดงานการกำหนดค่า

## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

-   ทำงานร่วมกับคุณเพื่อเริ่มต้นปฐมนิเทศ

-   กำหนดบริการที่มีสิทธิ์ที่คุณต้องการให้กับออนบอร์ด

## <a name="assess-phase"></a>การประเมินขั้นตอน

-   แสดงภาพรวมของการดูแลระบบ

-   ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้

    -   ความต้องการของ DNS, เครือข่ายและโครงสร้างพื้นฐาน

    -   ความต้องการของลูกค้า (อินเทอร์เน็ตเบราว์เซอร์ระบบปฏิบัติการไคลเอ็นต์และความต้องการของบริการ)

    -   ข้อมูลประจำตัวของผู้ใช้และการเตรียมใช้งาน

    -   การเปิดใช้งานบริการที่มีสิทธิ์ที่ได้รับการซื้อและกำหนดให้เป็นส่วนหนึ่งของปฐมนิเทศ

-   สร้างไทม์ไลน์สำหรับกิจกรรมที่มีส่วนร่วม

-   ให้รายการตรวจสอบด้านการทำงานสำหรับทั้ง Intune และ Azure AD Premium

## <a name="remediate-phase"></a>เฟส Remediate

-   ถือการประชุมทางโทรศัพท์กับคุณตามกำหนดการที่ตกลงกันเพื่อตรวจสอบความคืบหน้าของกิจกรรมทางด้านตัวอย่างเช่นให้คำแนะนำคุณเกี่ยวกับการติดตั้งก่อน requisites ปฐมนิเทศบริการ cloud ของ Microsoft

## <a name="enable-phase"></a>เปิดใช้งานขั้นตอน
ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้

-   การเปิดใช้งานผู้เช่า Microsoft online service หรือการสมัครใช้งานของคุณ

-   การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์

-   การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

-   การตรวจสอบความถูกต้องของการเชื่อมต่อกับบริการ Microsoft online

-   สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

    -   การติดตั้งเซิร์ฟเวอร์การซิงโครไนซ์ไดเรกทอรีระหว่างบริการโดเมน active Directory ของคุณ (AD DS) และบริการ Microsoft online ที่มีสิทธิ์ (เฉพาะคำแนะนำถ้าจำเป็น)

    -   การกำหนดค่าการรับรองความถูกต้องที่มีการจัดการ (รหัสผ่านแฮชหรือการรับรองความถูกต้องแบบพาส-ทรู) ที่มีเครื่องมือการเชื่อมต่อ Azure Active directory (เฉพาะคำแนะนำถ้าจำเป็น)

        > [!NOTE]
        > การพัฒนาและการดำเนินการสำหรับส่วนขยายกฎแบบกำหนดเองอยู่นอกขอบเขต

-   สำหรับฟอเรสต์เดียวเมื่อเป้าหมายคือข้อมูลประจำตัวที่ติดต่อกับภายนอก: การติดตั้งและการกำหนดค่าบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) สำหรับการรับรองความถูกต้องของโดเมนภายในเครื่องที่มี Intune ในไซต์เดียว, การกำหนดค่าการทนต่อข้อบกพร่องถ้าจำเป็น

    > [!NOTE]
    > สำหรับการกำหนดค่าทั้งหมดของฟอเรสต์การปรับใช้ FS โฆษณาอยู่นอกขอบเขต

-   การทดสอบฟังก์ชันการลงชื่อเข้าระบบครั้งเดียว (SSO) ถ้าปรับใช้

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งาน Microsoft Azure Active Directory Premium

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้

- การเปิดใช้งานผู้เช่า Azure AD Premium ของคุณ

- การกำหนดค่าพอร์ตไฟร์วอลล์

- การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

- การตรวจสอบความถูกต้องของการเชื่อมต่อกับ Azure AD Premium service

- สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

  -   การติดตั้งการซิงโครไนซ์ไดเรกทอรีระหว่างบริการโดเมนของ Active Directory ของคุณ (AD DS) และ Azure AD Connect ถ้าจำเป็น

  -   การกำหนดค่าวิธีการรับรองความถูกต้อง (การซิงค์แฮหัสผ่านหรือการรับรองความถูกต้องแบบพาส-ทรู) ที่มีเครื่องมือการเชื่อมต่อ AD Azure

- สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการซิงโครไนซ์การเชื่อมต่อ AD Azure ตั้งค่าสำหรับสถานการณ์ต่างๆของฟอเรสต์
- สำหรับสภาพแวดล้อมแบบเดี่ยวและแบบหลายฟอเรสต์:
  - การกำหนดค่าการรับรองความถูกต้องแบบพาส-ทรูของ Azure Active Directory ถ้าจำเป็น
  - การกำหนดค่า Azure Active Directory อย่างราบรื่นการลงชื่อเข้าใช้ครั้งเดียว (SSO) ถ้าจำเป็น
    > [!NOTE]
    > การรับรองความถูกต้องแบบพาส-ทรูของไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์จะได้รับการสนับสนุนถ้ามีความเชื่อถือของฟอเรสต์ระหว่างฟอเรสต์ Active Directory ของคุณและถ้ามีการกำหนดเส้นทางคำต่อท้ายชื่อ คุณสามารถติดตั้งตัวแทนเพิ่มเติมบนเซิร์ฟเวอร์ภายในองค์กรหลายรายการเพื่อให้มีความพร้อมใช้งานสูงสำหรับการร้องขอการลงชื่อเข้าใช้

  - สำหรับข้อมูลเพิ่มเติมให้ดูที่การ[รับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ของ azure: เริ่มต้น](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)ใช้งานด่วนและ[azure Active Directory ได้อย่างราบรื่นการลงชื่อเข้าใช้แบบครั้งเดียว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับขีดจำกัดการรับรองความถูกต้องแบบพาส-ทรูให้ดูที่การ[รับรองความถูกต้องแบบพาส-ทรูของ Azure Active directory: ข้อจำกัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO ที่ไม่ราบรื่นให้ดูที่การ[แก้ไขปัญหาการลงชื่อเข้าใช้ Azure active directory อย่างราบรื่น](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > การซิงค์แฮชของรหัสผ่านและการสนับสนุน writeback ของรหัสผ่านที่สนับสนุนหลาย forests อย่างไรก็ตามสถานการณ์สมมติ writeback อื่นไม่ได้รับการสนับสนุน

  - การกำหนดค่าการซิงโครไนซ์ระหว่างฟอเรสต์ Active Directory ภายในองค์กรและไดเรกทอรี Active directory ของ Microsoft Azure (Azure Active directory)

    > [!NOTE]
    > การพัฒนาและการดำเนินการสำหรับส่วนขยายกฎแบบกำหนดเองอยู่นอกขอบเขต

- สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก:

  -   การติดตั้งและการกำหนดค่า AD FS สำหรับการรับรองความถูกต้องของโดเมนภายในเครื่องที่มี Azure AD Premium ในไซต์เดียวการกำหนดค่าแบบทนต่อข้อบกพร่อง (ถ้าจำเป็น)

  > [!NOTE]
  > สำหรับการกำหนดค่าทั้งหมดของฟอเรสต์การปรับใช้ FS โฆษณาอยู่นอกขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้ามีการจัดวาง)

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งาน Azure AD Premium-ด้วย Azure AD Connect และ FS โฆษณา

ให้คำแนะนำเกี่ยวกับการตั้งค่า:

- การเตรียมใช้งานของผู้ใช้รวมถึงสิทธิ์การใช้งาน

- การซิงโครไนซ์ไดเรกทอรีของ Azure AD Connect (ด้วยรหัสผ่าน writeback และการซิงค์แฮชของรหัสผ่าน)

  - Azure Active Directory ตั้งค่ารหัสผ่านบริการตนเอง (SSPR)

  - การรับรองความถูกต้องแบบหลายปัจจัยของ Azure

  - สูงสุดสาม (3) หรือซอฟต์แวร์เพิ่มเติมเป็นแอปพลิเคชันบริการ (SaaS) ที่รวมกับการลงชื่อเข้าระบบครั้งเดียว (SSO) จากการใช้[งานไดเรกทอรีที่ใช้งานอยู่ของ Azure Marketplace](https://azure.microsoft.com/marketplace/active-directory/)

  - การเตรียมการของผู้ใช้โดยอัตโนมัติสำหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามที่แสดงอยู่ใน [รายการบทช่วยสอนการรวมแอป](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จำกัดเฉพาะการจัดเตรียมขาออกเท่านั้น

  - หน้าจอการเข้าสู่ระบบแบบกำหนดเองรวมถึงโลโก้ข้อความและรูปภาพ

  - กลุ่มแบบบริการตนเองและแบบไดนามิก (กลุ่ม)

  - พร็อกซีแอปพลิเคชันไดเรกทอรีของ Azure ที่ใช้งานอยู่

  - Azure Active Directory เชื่อมต่อสถานภาพ

  - การเข้าถึงตามเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - ข้อกำหนดการใช้ของไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - การป้องกันข้อมูลประจำตัวของไดเรกทอรีที่ใช้งานอยู่ของ Azure

  - การจัดการข้อมูลประจำตัวของ Azure Active Directory ที่มีสิทธิ์การใช้งาน

  - การเข้าถึงไดเรกทอรีที่ใช้งานอยู่ของ Azure

  -   การป้องกันด้วยรหัสผ่านของ Azure Active Directory

  -   ไดเรกทอรีที่ใช้งานอยู่ของ Azure B2B

### <a name="enable-phase---intune"></a>เปิดใช้งานขั้นตอน-Intune

> [!IMPORTANT]
> FastTrack ไม่สนับสนุนการจัดการพีซีแบบคลาสสิกของ Windows 10 ด้วย Intune FastTrack สนับสนุนการจัดการ Windows 10 ผ่าน Intune การจัดการอุปกรณ์เคลื่อนที่ (MDM) เท่านั้น

ให้คำแนะนำเกี่ยวกับ:

-   การกำหนดค่าผู้ใช้อีเมลของคุณจะถูกใช้โดย Intune โดยใช้ประโยชน์จากไดเรกทอรีที่ใช้งานอยู่ภายในองค์กรของคุณหรือผู้ใช้อีเมล cloud (Azure Active directory)

-   สิทธิ์การใช้งานผู้ใช้ของคุณ

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

-   การกำหนดค่าหน่วยงานการจัดการอุปกรณ์เคลื่อนที่ของคุณโดยยึดตามความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นผู้ให้บริการ MDM ของคุณ

    -   การกำหนดค่ากลุ่มการทดสอบที่จะใช้ในการตรวจสอบนโยบายการจัดการ MDM

    -   การนำทางในพอร์ทัลผู้ดูแลระบบ Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาทของ Intune (ตัวดำเนินการเจ้าหน้าที่ให้ความช่วยเหลือผู้ดูแลระบบฯลฯ)

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการต่างๆเช่น:

        -   การปรับใช้แอปสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านลิงก์เว็บ, MSI และ/หรือลิงก์ลึก

        -   การปรับใช้ Office ProPlus บนอุปกรณ์ Windows 10

        -   โปรแกรมการซื้อระดับเสียงสำหรับการปรับใช้แอปรวมถึง Apple ของ Apple Store for Business และการเล่นของ Google สำหรับที่เก็บงานของ Google

        -   การปรับใช้อีเมลเครือข่ายไร้สายและโปรไฟล์ VPN ถ้าคุณมีผู้มีสิทธิ์ออกใบรับรองที่มีอยู่, Wi-fi หรือโครงสร้างพื้นฐาน VPN ในองค์กรของคุณ

        -   การตั้งค่าตัวเชื่อมต่อ Exchange Intune ของ Microsoft (เมื่อสามารถใช้งานได้)

        -   โปรไฟล์การกำหนดค่าอุปกรณ์สำหรับแพลตฟอร์มสำหรับอุปกรณ์ที่ได้รับการสนับสนุน

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   การกำหนดค่าและการปรับใช้โปรแกรมประยุกต์ Intune การป้องกันสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   การจัดเตรียมแอป line-of-business (LOB) สำหรับนโยบายการป้องกันแอป Intune ที่มีคำแนะนำเกี่ยวกับตัวเลือกที่พร้อมใช้งาน

    -   การลงทะเบียนอุปกรณ์ของแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนไปยัง Intune ของคุณหรือตัวจัดการการกำหนดค่าด้วย Microsoft Intune service

    -   การเชื่อมต่อกับคลังข้อมูล Intune

    -   รวม Intune ด้วย:
        -   ตัวแสดงทีมสำหรับความช่วยเหลือระยะไกล (จำเป็นต้องใช้การสมัครใช้งานของทีม)

        -   โซลูชั่นการป้องกันภัยคุกคามของอุปกรณ์เคลื่อนที่ (จำเป็นต้องมีการสมัครใช้งานโซลูชันคู่ค้าที่มีการป้องกัน)

        -   โซลูชันการจัดการค่าใช้จ่ายของเทเลคอม (จำเป็นต้องมีการสมัครใช้งานโซลูชันการจัดการค่าใช้จ่ายในการโทรคมนาคม)

        -   การป้องกันการคุกคามขั้นสูงของ microsoft Defender (Windows E5 หรือ Microsoft ๓๖๕ E5 สิทธิ์การใช้งานจำเป็น)

    -   การกำหนดค่าการอัปเดตซอฟต์แวร์สำหรับแพลตฟอร์มที่ได้รับการสนับสนุน

    -   แหล่งข้อมูลสำหรับการวางแผนการใช้งานของผู้ใช้

- การตั้งค่า Windows Autopilot ให้ทำดังนี้

    - กำหนดค่าและตั้งค่า Microsoft Intune สำหรับ Windows Autopilot

    - กำหนดค่ากลุ่ม Azure AD dynamic

    - เพิ่มตราสินค้าของบริษัทของคุณลงใน Azure AD

    - สร้างและกำหนดอุปกรณ์ให้กับ Windows Autopilot โปรไฟล์ (เช่นโปรไฟล์ Windows Autopilot ที่จำกัดการสร้างบัญชีผู้ดูแลภายในเครื่อง)

    - กำหนดค่าประสบการณ์การใช้งาน (OOBE) ที่กำหนดเองเพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การกำหนดค่าการลงทะเบียนโดยอัตโนมัติของ MDM ใน Azure AD และ Intune

    > [!NOTE]
    > การตั้งค่า Windows Autopilot ภายนอก Intune อยู่นอกขอบเขตสำหรับสิทธิประโยชน์ FastTrack

### <a name="enable-phase---cloud-attach"></a>เปิดใช้งานเฟส-ระบบคลาวด์ที่แนบมา

ให้คำแนะนำเกี่ยวกับ:

-   สิทธิ์การใช้งานผู้ใช้ของคุณ

-   การใช้งาน cloud แนบในคอนโซลตัวจัดการการกำหนดค่า

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างกลุ่มผู้ใช้และอุปกรณ์ (ถ้าไม่ได้ติดตั้ง Intune)

-   การตั้งค่าการรวมไดเรกทอรีที่ใช้งานอยู่ของ Azure แบบไฮบริด

-   การตั้งค่า Azure Active Directory สำหรับการลงทะเบียนโดยอัตโนมัติ MDM

-   การตั้งค่าเกตเวย์การจัดการระบบคลาวด์

-   การเพิ่มผู้ใช้ลงในการสมัครใช้งาน Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบไอทีและการสร้างกลุ่มผู้ใช้และอุปกรณ์

-   การเตรียมบริการ Intune สำหรับการจัดการอุปกรณ์

-   การตั้งค่าสิทธิ์การจัดการอุปกรณ์เคลื่อนที่ (MDM) เป็น Intune

-   การกำหนดค่ากลุ่มการทดสอบที่จะใช้ในการตรวจสอบนโยบายการจัดการ MDM

-   การนำทางในพอร์ทัลผู้ดูแลระบบ Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

-   การตั้งค่าบทบาทของ Intune (ตัวดำเนินการ helpdesk ผู้ดูแลระบบและอื่นๆ)

-   การลงทะเบียนอุปกรณ์ Windows 10 เพื่อ Intune

-   การสลับปริมาณสำหรับการจัดการโดย Intune ตามต้องการ

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานขั้นตอน–การป้องกันข้อมูล Azure

คำแนะนำที่มีให้ใช้งาน: 

- การเปิดใช้งานและการกำหนดค่าผู้เช่าลูกค้า

- การสร้างและการตั้งค่าป้ายชื่อและนโยบาย

- การนำการป้องกันข้อมูลไปใช้กับเอกสาร 

- จัดประเภทและการติดป้ายข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทำงานบน Windows และการใช้ไคลเอ็นต์การป้องกันข้อมูลของ Azure

- การใช้ไฟล์ที่เหลือด้วยสแกนเนอร์การป้องกันข้อมูลของ Azure

- การตรวจสอบอีเมลในการส่งต่อโดยใช้กฎการรับส่งจดหมายของ Exchange Online

คำแนะนำนี้ยังมีให้กับลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการการจัดการสิทธิ์ของ Microsoft Azure (Azure RMS), การเข้ารหัสลับข้อความของ Office ๓๖๕ (OME) และการป้องกันข้อมูลสูญหาย (DLP)

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมใช่หรือไม่** ดูการ[เคลื่อนไหวขององค์กร + ความปลอดภัย](https://www.microsoft.com/cloud-platform/enterprise-mobility)
