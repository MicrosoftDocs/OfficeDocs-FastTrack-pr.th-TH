# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างปฐมนิเทศ

## <a name="general"></a>ทั่ว ไป

-   ให้ความช่วยเหลือในการสนับสนุนระยะไกลแก่คุณสำหรับกิจกรรมการกำหนดค่าที่จำเป็นตามที่แสดงในคำอธิบายขั้นตอนโดยละเอียด

-   มีเอกสารประกอบเครื่องมือซอฟต์แวร์และคอนโซลผู้ดูแลระบบเพื่อช่วยให้คุณลดหรือกำจัดงานการตั้งค่าคอนฟิก

## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

-   ทำงานร่วมกับคุณเพื่อเริ่มต้นปฐมนิเทศ

-   กำหนดบริการที่มีสิทธิ์ที่คุณต้องการไปยังบนเครื่อง

## <a name="assess-phase"></a>ประเมินระยะ

-   แสดงภาพรวมของผู้ดูแล

-   ให้คำแนะนำเกี่ยวกับ:

    -   DNS, เครือข่ายและความต้องการโครงสร้างพื้นฐาน

    -   ความต้องการของลูกค้า (อินเทอร์เน็ตเบราว์เซอร์ระบบปฏิบัติการไคลเอนต์และความต้องการของบริการ)

    -   ข้อมูลเฉพาะตัวของผู้ใช้และการจัดเตรียม

    -   เปิดใช้งานบริการที่มีสิทธิ์ที่ได้รับการซื้อและกำหนดให้เป็นส่วนหนึ่งของปฐมนิเทศ

-   สร้างเส้นเวลาสำหรับกิจกรรมด้าน

-   ให้รายการตรวจสอบด้านสำหรับทั้ง Intune และ Azure โฆษณาพรีเมี่ยม

## <a name="remediate-phase"></a>Remediate เฟส

-   ถือการประชุมสายกับคุณตามตารางเวลาที่ตกลงกันเพื่อตรวจสอบความคืบหน้าของกิจกรรมด้านตัวอย่างเช่นแนะนำคุณในการติดตั้งก่อน requisites ปฐมนิเทศบริการระบบคลาวด์ของ Microsoft

## <a name="enable-phase"></a>เปิดใช้งานเฟส
ให้คำแนะนำเกี่ยวกับ:

-   การเปิดใช้งานผู้เช่าบริการออนไลน์ของ Microsoft หรือการสมัครใช้งาน

-   การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์

-   การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

-   การตรวจสอบการเชื่อมต่อกับบริการออนไลน์ของ Microsoft

-   สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

    -   การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการการโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และบริการออนไลน์ของ Microsoft ที่มีสิทธิ์ (เฉพาะคำแนะนำถ้าจำเป็น)

    -   การตั้งค่าคอนฟิกจัดการการรับรองความถูกต้อง (รหัสผ่านแฮซิงค์หรือการรับรองความถูกต้องทรู) ด้วยเครื่องมือ Azure เชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ (เฉพาะคำแนะนำถ้าจำเป็น)

        > [!NOTE]
        > การพัฒนาและการดำเนินการสำหรับส่วนขยายของกฎที่กำหนดเองอยู่นอกขอบเขต

-   สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก: การติดตั้งและการตั้งค่าคอนฟิกบริการสหพันธรัฐไดเรกทอรีที่ใช้งานอยู่ (AD FS) สำหรับการรับรองความถูกต้องโดเมนเฉพาะที่มี Intune ในไซต์เดียวการกำหนดค่าการทนต่อความผิดพลาดถ้าจำเป็น

    > [!NOTE]
    > สำหรับการตั้งค่าคอนฟิกฟอเรสต์หลายทั้งหมดใช้ FS โฆษณาอยู่นอกขอบเขต

-   การทดสอบฟังก์ชันการเข้าสู่ระบบแบบครั้งเดียว (SSO) ถ้ามีการจัดวาง

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>เปิดใช้งานระดับพรีเมี่ยม Azure ไดเรกทอรีที่ใช้งานอยู่เฟส

ให้คำแนะนำเกี่ยวกับ:

- การเปิดใช้งานผู้เช่าพรีเมี่ยมโฆษณา Azure ของคุณ

- การกำหนดค่าพอร์ตไฟร์วอลล์

- การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์

- การตรวจสอบการเชื่อมต่อกับบริการพรีเมี่ยม AD Azure

- สำหรับสภาพแวดล้อมแบบฟอเรสต์เดียว:

  -   การติดตั้งไดเรกทอรีการซิงโครไนส์ระหว่างบริการการโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และเชื่อมต่อ Azure โฆษณาถ้าจำเป็น

  -   การตั้งค่าคอนฟิกวิธีการรับรองความถูกต้อง (รหัสผ่านแฮซิงค์หรือการรับรองความถูกต้องทรู) กับเครื่องมือการเชื่อมต่อ AD Azure

- สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์:

  -   การติดตั้งการซิงโครไนส์การเชื่อมต่อ AD Azure ตั้งค่าสำหรับสถานการณ์ต่างๆของฟอเรสต์
- สำหรับสภาพแวดล้อมแบบเดี่ยวและแบบหลายป่า:
  - การกำหนดค่าการรับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ Azure
  - การกำหนดค่า Azure ไดเรกทอรีที่ใช้งานอยู่อย่างราบรื่นเข้าสู่ระบบแบบครั้งเดียว (SSO), ถ้าจำเป็น
    > [!NOTE]
    > การรับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ Azure สำหรับสภาพแวดล้อมของฟอเรสต์หลายได้รับการสนับสนุนถ้ามี trusts ฟอเรสต์ระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ของคุณและถ้ามีการกำหนดเส้นทางส่วนต่อท้ายของชื่อ ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายเพื่อให้พร้อมใช้งานสูงสำหรับการร้องขอการเข้าสู่ระบบ

  - สำหรับข้อมูลเพิ่มเติมให้ดูการ[รับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ azure: เริ่มต้นอย่างรวดเร็ว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites)และ[Azure ไดเรกทอรีที่ใช้งานอยู่โดยไม่ต้องเข้าสู่ระบบแบบครั้งเดียว:](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับขีดจำกัดการรับรองความถูกต้องทรูให้ดูที่การ[รับรองความถูกต้องแบบพาส-ทรู Azure ไดเรกทอรีที่ใช้งานอยู่: ข้อจำกัดปัจจุบัน](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations)
  - สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO ไม่ราบรื่นให้ดู[ที่การแก้ไขปัญหา Azure ไดเรกทอรีที่ใช้งานอยู่โดยไม่ต้องเข้าสู่ระบบแบบครั้งเดียว](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso)

      > [!NOTE]
      > รหัสผ่านแฮชของการซิงค์และรหัสผ่าน writeback สนับสนุนหลาย forests อย่างไรก็ตามสถานการณ์ writeback อื่นๆไม่ได้รับการสนับสนุน

  - การกำหนดค่าการซิงโครไนส์ระหว่างฟอเรสต์ที่ใช้งานอยู่ในสถานที่และไดเรกทอรีของ Microsoft Azure ไดเรกทอรีที่ใช้งานอยู่ (Azure Active Directory)

    > [!NOTE]
    > การพัฒนาและการดำเนินการสำหรับส่วนขยายของกฎที่กำหนดเองอยู่นอกขอบเขต

- สำหรับฟอเรสต์เดียวเมื่อเป้าหมายเป็นข้อมูลเฉพาะตัวที่ติดต่อกับภายนอก:

  -   การติดตั้งและการตั้งค่าคอนฟิก FS โฆษณาสำหรับการรับรองความถูกต้องของโดเมนท้องถิ่นด้วย Azure AD Premium ในไซต์เดียวการกำหนดค่าการทนต่อความผิดพลาด (ถ้าจำเป็น)

  > [!NOTE]
  > สำหรับการตั้งค่าคอนฟิกฟอเรสต์หลายทั้งหมดใช้ FS โฆษณาอยู่นอกขอบเขต

- การทดสอบฟังก์ชัน SSO (ถ้ามีการจัดวาง)

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>เปิดใช้งานเฟส-Azure AD พรีเมียม-ด้วยการเชื่อมต่อโฆษณา Azure และ FS โฆษณา

ให้คำแนะนำเกี่ยวกับการตั้งค่า:

- การจัดเตรียมผู้ใช้รวมถึงการให้สิทธิ์

- การซิงโครไนส์ของไดเรกทอรีการเชื่อมต่อ Azure AD (ด้วยการเขียนกลับรหัสผ่านและแฮชของการซิงค์รหัสผ่าน)

  - ไดเรกทอรีที่ใช้งานอยู่ Azure รีเซ็ตรหัสผ่านบริการตนเอง (SSPR)

  - การรับรองความถูกต้องหลายปัจจัย Azure

  - สูงสุดสาม (3) โปรแกรมประยุกต์บริการ (SaaS) การรวมกับการเข้าสู่ระบบแบบครั้งเดียว (SSO) จาก[ตลาดไดเรกทอรีที่ใช้งานอยู่ Azure](https://azure.microsoft.com/marketplace/active-directory/)

  - การจัดสรรผู้ใช้โดยอัตโนมัติสำหรับแอปพลิเคชัน SaaS ที่รวมไว้ล่วงหน้าตามที่แสดงไว้ในรายการการแนะนำการ[รวมแอป](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list)จำกัดเฉพาะการจัดเตรียมขาออกเท่านั้น

  - หน้าจอการเข้าสู่ระบบแบบกำหนดเองรวมทั้งโลโก้ข้อความและรูปภาพ

  - กลุ่มการบริการตนเองและแบบไดนามิก (กลุ่ม)

  - พร็อกซีโปรแกรมประยุกต์ไดเรกทอรีที่ใช้งานอยู่ Azure

  - Azure ที่ใช้งานอยู่ไดเรกทอรีเชื่อมต่อสุขภาพ

  - การเข้าถึงแบบมีเงื่อนไขของไดเรกทอรีที่ใช้งานอยู่ Azure

  - ข้อตกลงในการใช้งานไดเรกทอรี Active Azure

  - การป้องกันข้อมูลเฉพาะตัวของไดเรกทอรีที่ใช้งานอยู่ Azure

  - การจัดการข้อมูลประจำตัวของไดเรกทอรีที่ใช้งานอยู่ Azure

  - ความคิดเห็นการเข้าถึงไดเรกทอรีที่ใช้งานอยู่ Azure

### <a name="enable-phase---intune"></a>เปิดใช้งานขั้นตอน-Intune

> [!IMPORTANT]
> FastTrack ไม่สนับสนุน Windows 10 การจัดการพีซีคลาสสิกด้วย Intune FastTrack สนับสนุน Windows 10 การจัดการผ่าน Intune การจัดการอุปกรณ์เคลื่อนที่ (MDM)

ให้คำแนะนำเกี่ยวกับ:

-   การกำหนดค่าข้อมูลประจำตัวที่จะใช้โดย Intune โดยวิธีการที่ใช้งานอยู่ในสถานที่ไดเรกทอรีของคุณเองหรือข้อมูลเฉพาะตัวของ cloud (Azure ไดเรกทอรีที่ใช้งานอยู่)

-   อนุญาตให้ผู้ใช้ปลายของคุณ

-   การเพิ่มผู้ใช้ลงในการสมัครสมาชิก Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างผู้ใช้และกลุ่มอุปกรณ์

-   การกำหนดค่าหน่วยงานจัดการอุปกรณ์เคลื่อนที่ของคุณ MDM) ตามความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบนโยบายการจัดการ MDM

    -   การนำทางไปยังเว็บไซต์ผู้ดูแลระบบของ Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาทของ Intune (ตัวดำเนินการเคาน์เตอร์ช่วยเหลือผู้ดูแลฯลฯ)

    -   การกำหนดค่านโยบายการจัดการ MDM และบริการต่างๆเช่น:

        -   การปรับใช้โปรแกรมประยุกต์สำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนผ่านการเชื่อมโยงเว็บ, MSI และ/หรือการเชื่อมโยงที่ลึก

        -   การใช้งาน Office ProPlus บนอุปกรณ์ Windows 10.

        -   โปรแกรมการซื้อจำนวนมากสำหรับการปรับใช้แอปรวมถึงของ Apple และ Windows Store สำหรับธุรกิจและการเล่นของ Google สำหรับร้านค้าที่ทำงาน

        -   การใช้งานอีเมลเครือข่ายไร้สายและโปรไฟล์ VPN ถ้าคุณมีผู้มีสิทธิ์ออกใบรับรองที่มีอยู่, Wi-fi หรือโครงสร้างพื้นฐาน VPN ในองค์กรของคุณ

        -   การตั้งค่าการเชื่อมต่อ Microsoft Intune แลกเปลี่ยน (เมื่อเกี่ยวข้อง)

        -   โปรไฟล์การกำหนดค่าอุปกรณ์สำหรับแพลตฟอร์มของอุปกรณ์ที่สนับสนุน

    -   การตั้งค่านโยบายการเข้าถึงแบบมีเงื่อนไข

    -   การตั้งค่าคอนฟิกและปรับใช้โปรแกรมประยุกต์ Intune นโยบายการป้องกันสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   กำลังเตรียมโปรแกรมประยุกต์ที่มีบรรทัดของธุรกิจ (LOB) สำหรับการป้องกันโปรแกรมประยุกต์ Intune ที่มีคำแนะนำเกี่ยวกับตัวเลือกที่พร้อมใช้งาน

    -   การลงทะเบียนอุปกรณ์ของแต่ละแพลตฟอร์มที่ได้รับการสนับสนุนให้กับ Intune ของคุณหรือตัวจัดการการตั้งค่าคอนฟิกกับบริการ Microsoft Intune

    -   การเชื่อมต่อไปยังคลังสินค้าของ Intune ข้อมูล

    -   รวม Intune ด้วย:
        -   ตัวแสดงทีมสำหรับความช่วยเหลือระยะไกล (ต้องใช้การสมัครสมาชิกของทีมงาน)

        -   การป้องกันการคุกคามทางโทรศัพท์มือถือโซลูชั่น (การป้องกันภัยคุกคามมือถือการสมัครสมาชิกโซลูชั่นที่จำเป็น)

        -   ระบบการจัดการค่าใช้จ่ายเทเลคอม (โทรคมนาคมจำเป็นต้องมีการสมัครใช้งาน)

        -   การป้องกันการคุกคามขั้นสูงของ windows Defender (Windows E5 หรือ Microsoft ๓๖๕ E5 สิทธิ์การใช้งานจำเป็น)

    -   การกำหนดค่าการปรับปรุงซอฟต์แวร์สำหรับแพลตฟอร์มที่สนับสนุนที่เกี่ยวข้อง

    -   ทรัพยากรสำหรับการวางแผนการยอมรับของผู้ใช้

- การตั้งค่า Windows ขี่:

    - ตั้งค่าคอนฟิกและติดตั้ง Microsoft Intune สำหรับ Windows ขี่

    - การตั้งค่าคอนฟิกกลุ่มแบบไดนามิกโฆษณา Azure

    - เพิ่มตราสินค้าของบริษัทของคุณลงในโฆษณา Azure

    - สร้างและกำหนดอุปกรณ์ให้กับ Windows ขี่ส่วนกำหนดค่า (เช่นส่วนกำหนดค่า Windows ขี่ที่จำกัดการสร้างบัญชีผู้ดูแลท้องถิ่น)

    - กำหนดเองออกกล่อง (OOBE) เพื่อให้สอดคล้องกับความต้องการขององค์กร

    - การกำหนดค่าการลงทะเบียนอัตโนมัติของ MDM ในโฆษณา Azure และ Intune

    > [!NOTE]
    > การตั้งค่า Windows ขี่ภายนอก Intune จะอยู่นอกขอบเขตสำหรับประโยชน์ FastTrack

### <a name="enable-phase---co-management"></a>เปิดใช้งานการจัดการขั้นตอน

ให้คำแนะนำเกี่ยวกับ:

-   อนุญาตให้ผู้ใช้ปลายของคุณ

-   การเพิ่มผู้ใช้ไปยังการสมัครใช้งาน Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างผู้ใช้และกลุ่มอุปกรณ์ (ถ้าไม่มีการติดตั้ง Intune)

-   การตั้งค่าไดเรกทอรีที่ใช้งานอยู่ Azure สำหรับการลงทะเบียนอัตโนมัติของ MDM

-   ตั้งค่าการเข้าร่วมไดเรกทอรีที่ใช้งานอยู่ Azure ไฮบริดสลี

-   ตั้งค่าเกตเวย์การจัดการเมฆ

-   การเพิ่มผู้ใช้ลงในการสมัครสมาชิก Intune ของคุณการกำหนดบทบาทผู้ดูแลระบบและการสร้างผู้ใช้และกลุ่มอุปกรณ์

-   เตรียม Intune (ถ้าไม่มีการติดตั้ง Intune):

    -   การกำหนดค่าหน่วยงานจัดการอุปกรณ์เคลื่อนที่ของคุณ MDM) ตามความต้องการในการจัดการของคุณรวมถึง:

    -   การตั้งค่า Intune เป็นหน่วยงานของคุณ MDM

    -   การตั้งค่าคอนฟิกกลุ่มการทดสอบที่จะใช้ในการตรวจสอบนโยบายการจัดการ MDM

    -   การนำทางไปยังเว็บไซต์ผู้ดูแลระบบของ Intune เพื่อค้นหาข้อมูลเกี่ยวกับผู้ใช้และอุปกรณ์

    -   การตั้งค่าบทบาทของ Intune (ตัวดำเนินการเคาน์เตอร์ช่วยเหลือผู้ดูแลฯลฯ)

    -   การตั้งค่าคอนฟิกและปรับใช้โปรแกรมประยุกต์ Intune นโยบายการป้องกันสำหรับแต่ละแพลตฟอร์มที่ได้รับการสนับสนุน

    -   ลงทะเบียนอุปกรณ์ Windows 10 ไปยัง Intune ของคุณ

- เปิดใช้งานการจัดการร่วมในคอนโซลการจัดการการตั้งค่าคอนฟิก

- สลับปริมาณงานไปยัง Intune

- ติดตามกิจกรรมการจัดการร่วมในสภาพแวดล้อมของคุณ

### <a name="enable-phase--azure-information-protection"></a>เปิดใช้งานเฟส–การป้องกันข้อมูล Azure

คำแนะนำที่ให้ไว้บน: 

- การเปิดใช้งานและการกำหนดค่าผู้เช่าลูกค้า

- การสร้างและการตั้งค่าป้ายชื่อและนโยบาย

- การประยุกต์ใช้การป้องกันข้อมูลไปยังเอกสาร 

- การจัดประเภทและการติดป้ายข้อมูลโดยอัตโนมัติในแอป Office (เช่น Word, PowerPoint, Excel และ Outlook) ที่ทำงานอยู่บน Windows และการใช้ไคลเอ็นต์การป้องกันข้อมูล Azure

- การใช้แฟ้มในส่วนที่เหลือด้วยสแกนเนอร์การป้องกันข้อมูล Azure

- การตรวจสอบอีเมลในการส่งต่อโดยใช้กฎการไหลของจดหมายแลกเปลี่ยนแบบออนไลน์

นอกจากนี้คำแนะนำยังให้กับลูกค้าที่ต้องการใช้การป้องกันโดยใช้บริการการจัดการสิทธิ์ Azure Microsoft (Azure RMS), Office ๓๖๕การเข้ารหัสลับข้อความ (OME), และป้องกันข้อมูลสูญหาย (DLP)

> [!NOTE]
> **ต้องการเรียนรู้เพิ่มเติมหรือไม่** ดู[ความปลอดภัยขององค์กร +](https://www.microsoft.com/cloud-platform/enterprise-mobility)
