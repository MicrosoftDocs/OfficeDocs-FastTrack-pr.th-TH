# <a name="fasttrack-responsibilities-for-office-365-us-government"></a>ความรับผิดชอบของ FastTrack สำหรับ Office ๓๖๕รัฐบาลสหรัฐอเมริกา

ผู้เชี่ยวชาญ FastTrack มีหน้าที่รับผิดชอบต่อไปนี้ในระหว่างการปฐมนิเทศ  
  
## <a name="general"></a>ทั่ว

- ให้ความช่วยเหลือด้านการสนับสนุนระยะไกลแก่คุณในการพัฒนาการวางแผนและการดำเนินการที่ประสบความสำเร็จและสำหรับกิจกรรมการกำหนดค่าที่จำเป็นตามรายละเอียดในคำอธิบายขั้นตอน
- มีเอกสารประกอบและเครื่องมือซอฟต์แวร์ที่พร้อมใช้งาน, คอนโซลผู้ดูแลระบบและสคริปต์เพื่อให้คำแนะนำสำหรับคุณในการลดหรือกำจัดงานการกำหนดค่าและแหล่งข้อมูลการวางแผนสำเร็จ   
    
## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

- ทำงานร่วมกับคุณเพื่อทำความเข้าใจเกี่ยวกับความตั้งใจเป้าหมายขององค์กรและแผนการใช้งานของคุณสำหรับบริการ 
- ทำงานกับคุณโดยใช้บริการการทำงานร่วมกันของ Office ๓๖๕ (เช่นทีม Microsoft) เพื่อเริ่มต้นปฐมนิเทศ 
- กำหนดบริการที่มีสิทธิ์ที่คุณต้องการให้กับออนบอร์ด 
    
## <a name="assess-phase"></a>การประเมินขั้นตอน

- ถือการวางแผนความสำเร็จเพื่อให้คำแนะนำแก่คุณในการนำผู้ใช้ที่ประสบความสำเร็จมาใช้  
- แสดงภาพรวมของการดูแลระบบ  
- ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้ 
  - ระบบชื่อโดเมน (DNS), เครือข่ายและความต้องการของโครงสร้างพื้นฐาน  
  - ความต้องการของลูกค้า (อินเทอร์เน็ตเบราว์เซอร์ระบบปฏิบัติการไคลเอ็นต์อุปกรณ์เคลื่อนที่และความต้องการของบริการ)
  - ข้อมูลประจำตัวของผู้ใช้และการเตรียมใช้งาน 
  - การเปิดใช้งานบริการที่มีสิทธิ์ที่ซื้อและกำหนดให้เป็นส่วนหนึ่งของปฐมนิเทศ
  - การขับขี่ที่ประสบความสำเร็จในการใช้บริการและค่า   
- สร้างไทม์ไลน์สำหรับกิจกรรมที่มีส่วนร่วม
- แสดงรายการตรวจสอบด้าน   
- ประเมินโครงสร้างพื้นฐานของ SharePoint Server ๒๐๑๓หรือ SharePoint Server ๒๐๑๖ที่มีอยู่รวมถึง:  
  - ข้อกำหนดเบื้องต้นสำหรับ SharePoint Online ไฮบริด  
  - ความพร้อมใช้งานของโครงสร้างพื้นฐานภายในองค์กรสำหรับฟีเจอร์ไฮบริดของ SharePoint Online  
  - เข้าถึงจุดสิ้นสุดของ SharePoint Online ที่จำเป็น 
  - ผู้ชมสำหรับ OneDrive for Business แบบไฮบริด    
- ประเมินโครงสร้างพื้นฐานของ Lync หรือ Skype for Business Online ที่มีอยู่รวมถึง:  
  - กลยุทธ์การปรับใช้ไคลเอ็นต์ Skype for Business ที่ได้รับการสนับสนุน  
  - เข้าถึงจุดสิ้นสุด  
  - คุณภาพการเชื่อมต่อ  
  - การประเมินแบนด์วิดธ์  
  - ข้อกำหนดเบื้องต้นเพื่อสนับสนุนการกำหนดค่าเซิร์ฟเวอร์โดเมนแยก  
  - ความพร้อมของผู้ใช้ที่ระบุที่จะย้ายไปยัง Skype for Business Online  
- ประเมินโครงสร้างพื้นฐานของการส่งข้อความรวมถึง:   
  - การไหลของจดหมายโดยรวมและหลักการกำหนดเส้นทาง  
  - การเข้าถึงไคลเอ็นต์ (รวมถึงจุดสิ้นสุดของการเข้าถึงไคลเอ็นต์ที่เผยแพร่แล้ว)  
  - สภาพแวดล้อมการส่งข้อความต้นฉบับสำหรับความต้องการของการรวม 
- ให้การโยกย้ายข้อมูลถ้ามีการใช้บริการการโยกย้ายข้อมูล FastTrack Center และถ้าคุณพบสิทธิ์
    
## <a name="remediate-phase"></a>เฟส Remediate

- ถือการประชุมทางโทรศัพท์กับคุณตามกำหนดการที่ตกลงไว้เพื่อตรวจสอบความคืบหน้าของกิจกรรมและการวางแผนความสำเร็จ   
- แนะนำคุณเกี่ยวกับการเรียกใช้เครื่องมือการประเมินเพื่อระบุและ remediate ปัญหาและแปลผลลัพธ์
    
## <a name="enable-phase"></a>เปิดใช้งานขั้นตอน

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้ 
- ประเมินความคืบหน้าในการวางแผนความสำเร็จและการกำหนดสิ่งที่คุณต้องการความช่วยเหลือเพิ่มเติม    
- การเปิดใช้งานผู้เช่า Office ๓๖๕ของคุณ
- การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์   
- การกำหนดค่า DNS สำหรับบริการที่มีสิทธิ์   
- การตรวจสอบความถูกต้องของการเชื่อมต่อไปยัง Office ๓๖๕   
- การเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ภายในองค์กรของคุณกับ Azure Active Directory:   
  - การติดตั้งเซิร์ฟเวอร์การซิงโครไนซ์ไดเรกทอรีระหว่างบริการโดเมน active Directory ของคุณ (AD DS) และ Office ๓๖๕ถ้าจำเป็น   
  - การกำหนดค่าการซิงโครไนซ์รหัสผ่าน (แฮชของรหัสผ่าน) ไปยัง Office ๓๖๕ (Azure Active directory) ที่มีเครื่องมือเชื่อมต่อไดเรกทอรีของ Azure ที่ใช้งานอยู่ถ้าจำเป็น  
  - สำหรับสภาพแวดล้อมแบบเดี่ยวและแบบหลายฟอเรสต์:
      - การกำหนดค่าการรับรองความถูกต้องแบบพาส-ทรูของ Azure Active Directory ถ้าจำเป็น (ไม่มีให้ใช้งานในแผนสูงหรือ DoD)
      - การกำหนดค่า Azure Active Directory ที่ไม่ราบรื่น Sign-On (SSO) ถ้าจำเป็น (ไม่มีให้ใช้งานในแผนสูงหรือ DoD ใน GCC)
    > [!NOTE]
    > การรับรองความถูกต้องแบบพาส-ทรูของไดเรกทอรีที่ใช้งานอยู่ของ Azure สำหรับสภาพแวดล้อมแบบหลายฟอเรสต์จะได้รับการสนับสนุนถ้ามีความเชื่อถือของฟอเรสต์ระหว่างฟอเรสต์ Active Directory ของคุณและถ้ามีการกำหนดเส้นทางคำต่อท้ายชื่อ คุณสามารถติดตั้งตัวแทนเพิ่มเติมบนเซิร์ฟเวอร์ภายในองค์กรหลายรายการเพื่อให้มีความพร้อมใช้งานสูงสำหรับการร้องขอการลงชื่อเข้าใช้ สำหรับข้อมูลเพิ่มเติมให้ดูที่การ[รับรองความถูกต้องแบบพาส-ทรูไดเรกทอรีที่ใช้งานอยู่ของ azure: เริ่มต้น](https://go.microsoft.com/fwlink/?linkid=860094)ใช้งานด่วนและ[azure Active Directory ได้อย่างราบรื่นการลงชื่อเข้าใช้แบบครั้งเดียว](https://go.microsoft.com/fwlink/?linkid=860095) 
    > [!NOTE]
    > สำหรับข้อมูลเพิ่มเติมเกี่ยวกับขีดจำกัดการรับรองความถูกต้องแบบพาส-ทรูให้ดูที่การ[รับรองความถูกต้องแบบพาส-ทรูของ Azure Active directory: ข้อจำกัดปัจจุบัน](https://go.microsoft.com/fwlink/?linkid=860356) 
    > [!NOTE]
    > สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO ที่ไม่ราบรื่นให้ดูที่การ[แก้ไขปัญหาการลงชื่อเข้าใช้ Azure active directory อย่างราบรื่น](https://go.microsoft.com/fwlink/?linkid=841926) 
- สำหรับฟอเรสต์เดียวเมื่อข้อมูลเฉพาะตัวที่ติดต่อกับภายนอกคือเป้าหมาย: 
  - การติดตั้งและการกำหนดค่า AD FS สำหรับการรับรองความถูกต้องของโดเมนภายในเครื่องที่มี Office ๓๖๕ในไซต์เดียวการกำหนดค่าแบบทนต่อข้อบกพร่องถ้าจำเป็น  
  - การติดตั้งและการกำหนดค่า WAP สำหรับการประกาศโครงสร้างพื้นฐาน FS โฆษณาของคุณไปยังอินเทอร์เน็ตเมื่อจำเป็น 
    > [!NOTE]
    > สำหรับการกำหนดค่าทั้งหมดของฟอเรสต์การปรับใช้ FS โฆษณาจะอยู่นอกขอบเขต 
- การทดสอบฟังก์ชัน SSO ถ้าปรับใช้   
- การขับขี่ที่ประสบความสำเร็จในการใช้บริการและค่า
    
## <a name="exchange-online"></a>Exchange Online

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้ 
- การสร้างหรือการอัปเดตระเบียน DNS    
- การเปิดใช้งานการกำหนดเส้นทางอีเมลระหว่างระบบการส่งข้อความต้นฉบับและสภาพแวดล้อมของ Office ๓๖๕    
- การกำหนดค่าฟีเจอร์การป้องกัน Exchange Online (รวมถึงฟีเจอร์การป้องกันภัยคุกคามขั้นสูงของ Exchange Online ถ้าพร้อมใช้งานในการสมัครใช้งานของคุณ) และการตรวจสอบระเบียน MX ของคุณไปยัง Office ๓๖๕สำหรับโดเมนที่เปิดใช้งานจดหมายที่ผ่านการตรวจสอบ   
- การกำหนดค่าการตั้งค่าแบบไฮบริดในระหว่างองค์กร Exchange ภายในองค์กรเดียวและ Office ๓๖๕  *หรือ*  ระหว่างองค์กร exchange ภายในองค์กรหลายองค์กรและ office ๓๖๕ 
- การตั้งค่าการส่งข้อความแบบรวม (UM) กับ Exchange Online (UM ไม่พร้อมใช้งานในแผนการ DoD ของ GCC) 
    
สำหรับข้อมูลเพิ่มเติมเกี่ยวกับความรับผิดชอบในการโยกย้ายข้อมูลให้ดูที่การ[โยกย้ายข้อมูล](data-migration.md)
  
## <a name="sharepoint-online"></a>SharePoint Online

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้
- การตั้งค่าการเตรียมใช้งานของผู้ใช้รวมถึงสิทธิ์การใช้งาน   
- การเปิดใช้งานการสร้างไซต์สำหรับผู้ดูแลระบบ SharePoint Online ของคุณ   
- การวางแผนไซต์คอลเลกชัน   
- การรักษาความปลอดภัยเนื้อหาและการจัดการสิทธิ์   
- การเปิดใช้งานไซต์ส่วนบุคคลและฟีเจอร์ทางสังคม   
- การกำหนดค่าฟีเจอร์ SharePoint Online    
- การจัดเตรียมการโยกย้ายข้อมูลถ้ามีการใช้บริการการโยกย้ายข้อมูล FastTrack Center และถ้าคุณพบสิทธิ์  
- การประเมินการกำหนดค่าโครงสร้างพื้นฐานของฟาร์ม SharePoint ภายในองค์กรที่จำเป็นสำหรับ SharePoint Online ไฮบริด    
- การใช้เครื่องมือและการทำงานอัตโนมัติเพื่อ: 
  - กำหนดค่าแอปพลิเคชันบริการการค้นหาระบบคลาวด์ภายในองค์กร    
  - กำหนดค่าความน่าเชื่อถือระหว่างสภาพแวดล้อมของ SharePoint ภายในองค์กรและระบบคลาวด์   
- การกำหนดค่าไซต์ SharePoint ภายในองค์กรให้ใช้ฟีเจอร์ไฮบริดของ SharePoint Online
    
## <a name="onedrive-for-business"></a>OneDrive for Business

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้ 
- การระบุตัวเลือกเวอร์ชันและการรวมของ SharePoint ภายในองค์กร    
- การระบุตัวเลือกการซิงโครไนซ์และข้อมูลประจำตัว   
- การเลือกตัวเลือกไวร์:   
  - ไวร์เพียงครั้งเดียว  
  - ไวร์แบบกำหนดลำดับขั้น (เรียงลำดับและเลิก)   
- การจัดเตรียมสภาพแวดล้อมภายในองค์กรสำหรับการปรับใช้ OneDrive for Business ให้ทำดังนี้  
  - การระบุไคลเอ็นต์การซิงค์ OneDrive for Business ที่ถูกต้อง 
  - การกำหนดค่า DNS, พอร์ตเครือข่ายและไฟร์วอลล์   
- การกำหนดสิทธิ์การใช้งานของผู้ใช้ปลายทาง   
- การตั้งค่าผู้ชม SharePoint Online เพื่อควบคุมและควบคุมผู้ที่จะได้รับ OneDrive for Business    
- การปรับใช้ไคลเอ็นต์การซิงค์ OneDrive for Business กับเดสก์ท็อป   
- วิธีกำหนดค่าการเปลี่ยนเส้นทาง SharePoint Online ไฮบริดของ OneDrive for Business (SharePoint ๒๐๑๓และ SharePoint ๒๐๑๖เท่านั้น)  
- การโยกย้ายข้อมูลถ้ามีการใช้บริการการโยกย้ายข้อมูลของศูนย์ FastTrack และถ้าคุณพบสิทธิ์
    
## <a name="skype-for-business-online"></a>Skype for Business Online

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้
- การเตรียมใช้งานข้อมูลประจำตัวของ Skype for Business สำหรับ Office ๓๖๕   
- การเปิดใช้งานการประชุมแบบออนไลน์การส่งข้อความโต้ตอบแบบทันที (IM) และฟีเจอร์การแสดงตนสำหรับ Office ๓๖๕  
- การสร้างบัญชีผู้ใช้เพื่อเชื่อมโยงกับอุปกรณ์ระบบห้องที่ได้รับการสนับสนุน (สูงสุด10บัญชี)    
- การกำหนดค่าสภาพแวดล้อมเซิร์ฟเวอร์โดเมนแยกเพื่อสนับสนุนสถานการณ์สมมติไฮบริดของ Lync หรือ Skype for Business Online (ถ้ามี)   
- การเปิดใช้งานการประชุมทางเสียง:   
  - การตั้งค่าองค์กรสำหรับการตั้งค่าเริ่มต้นของบริดจ์การประชุม   
  - งานที่มอบหมายของบริดจ์การประชุมให้กับผู้ใช้ที่ได้รับสิทธิ์การใช้งาน 
- การเปิดใช้งานระบบโทรศัพท์ (ไม่มีให้ใช้งานในแผนของ GCC สูงหรือ DoD):  
  - การเปิดใช้งานระบบโทรศัพท์และแผนการโทรปฐมนิเทศ (ในตลาดที่พร้อมใช้งาน) 
  - การมอบหมายตัวเลขให้กับผู้ใช้ที่ได้รับสิทธิ์การใช้งาน  
  - คำแนะนำโอนย้ายหมายเลขภายในเครื่องผ่าน UI สูงสุดถึง๙๙๙  
  - การสนับสนุนหมายเลขภายในโอนย้าย SR มากกว่า๙๙๙  
- การเปิดใช้งานการออกอากาศการประชุม Skype for Business (ไม่พร้อมใช้งานในแผนการใช้งานที่สูงหรือ DoD ของ GCC):  
  - การเปิดใช้งานการออกอากาศการประชุม Skype for Business ปฐมนิเทศคำแนะนำ  
  - การตั้งค่าองค์กรสำหรับสหพันธรัฐที่มีบริการออกอากาศการประชุม
    
## <a name="microsoft-teams"></a>Microsoft Teams

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้
- การยืนยันความต้องการขั้นต่ำ   
- การกำหนดค่าพอร์ตไฟร์วอลล์  
- การตั้งค่า DNS  
- การยืนยันว่าทีม Microsoft เปิดใช้งานแล้วในผู้เช่า Office ๓๖๕ของคุณ   
- การเปิดใช้งานหรือปิดใช้งานสิทธิ์การใช้งานของผู้ใช้  
- การแจกจ่ายไคลเอ็นต์ของทีม Microsoft    
- ฟีเจอร์ผู้เชี่ยวชาญด้าน IT และผู้ดูแลระบบ 
- ฟีเจอร์ผลิตภัณฑ์หลัก  
- เทมเพลตที่สำเร็จของลูกค้า
    
## <a name="power-bi"></a>Power BI

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้
- ตรวจทานแผนการสมัครใช้งาน Power BI    
- การเพิ่มบริการ Power BI    
- การดาวน์โหลดแอป Power BI บนเดสก์ท็อป
    
## <a name="project-online"></a>Project Online

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้  
- ตรวจทานแผนการสมัครใช้งาน  
- การตรวจสอบฟังก์ชันการทำงานพื้นฐานของ SharePoint    
- การเพิ่มบริการ Project Online  
- การเพิ่มผู้ใช้ลงใน Project Online รวมถึงการซิงค์ ERP  
- การตรวจสอบฟังก์ชันการทำงานพื้นฐานของ Project Online โดยการสร้างโครงการ
    
## <a name="yammer-enterprise"></a>Yammer Enterprise

ให้คำแนะนำเกี่ยวกับการแปลงเครือข่าย Yammer พื้นฐานเดียวของคุณให้เป็นเครือข่ายขององค์กร Yammer เดียว

> [!NOTE]
> Yammer Enterprise ไม่ใช่องค์ประกอบของ Office ๓๖๕รัฐบาลสหรัฐอเมริกาแต่สามารถซื้อได้โดยไม่มีค่าใช้จ่ายเป็นข้อเสนอแบบสแตนด์อโลนสำหรับผู้ใช้แต่ละคนที่ได้รับอนุญาตสำหรับ Office ๓๖๕ใน GCC ข้อเสนอนี้จะถูกจำกัดไว้ที่ลูกค้าที่ซื้อ Office ๓๖๕ GCC ภายใต้ข้อตกลงขององค์กรและข้อตกลงการสมัครใช้งานขององค์กร Yammer ไม่พร้อมใช้งานในแผนสูงหรือ DoD ของ GCC
  
## <a name="microsoft-365-apps"></a>แอป Microsoft ๓๖๕

ให้คำแนะนำเกี่ยวกับสิ่งต่อไปนี้
- การจัดการปัญหาการปรับใช้   
- การกำหนดสิทธิ์การใช้งานของผู้ใช้ปลายทางโดยใช้ [ศูนย์การจัดการ Microsoft ๓๖๕](https://go.microsoft.com/fwlink/?linkid=2032704) และ Windows PowerShell  
- การติดตั้งแอป Microsoft ๓๖๕จากพอร์ทัล Office ๓๖๕โดยใช้คลิก-ทู-รัน   
- การติดตั้งแอป Office สำหรับอุปกรณ์เคลื่อนที่ (เช่น Outlook Mobile, Word mobile, Excel Mobile และ PowerPoint Mobile) บนอุปกรณ์เคลื่อนที่ iOS, Android หรือ Windows Mobile ของคุณ   
- การกำหนดค่าการตั้งค่าการอัปเดตโดยใช้เครื่องมือการปรับใช้ Office ๒๐๑๖หรือแม่แบบของนโยบายกลุ่ม   
- การตั้งค่าเซิร์ฟเวอร์การแจกจ่ายในไซต์เดียวสำหรับแอป Microsoft ๓๖๕รวมถึงความช่วยเหลือในการสร้างไฟล์ที่ configuration.xml สำหรับใช้กับเครื่องมือการปรับใช้ Office ๓๖๕   
- การปรับใช้โดยใช้ตัวจัดการการกำหนดค่า Microsoft จุดสิ้นสุดรวมถึงความช่วยเหลือในการสร้างแพคเกจตัวจัดการการกำหนดค่าของ Microsoft จุดสิ้นสุด
