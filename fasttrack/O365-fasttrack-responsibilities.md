# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

ผู้เชี่ยวชาญ FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างการขึ้นเรือ\*
  
## <a name="general"></a>ทั่วไป

- ให้ความช่วยเหลือสนับสนุนระยะไกลเพื่อคุณในการพัฒนาและดําเนินการวางแผนความสําเร็จและสําหรับกิจกรรมการกําหนดค่าที่จําเป็นตามรายละเอียดในคําอธิบายขั้นตอน
- จัดเตรียมเครื่องมือสําหรับจัดทําเอกสารและซอฟต์แวร์ คอนโซลผู้ดูแลระบบ และสคริปต์ที่พร้อมจะให้คําแนะนําแก่คุณในการลดหรือกําจัดงานการกําหนดค่า และทรัพยากรการวางแผนความสําเร็จ 
    
## <a name="initiate-phase"></a>เริ่มต้นเฟส

- ทํางานร่วมกับคุณเพื่อให้เข้าใจถึงความตั้งใจเป้าหมายขององค์กรและแผนการใช้งานสําหรับบริการ
- ทํางานร่วมกับคุณโดยใช้บริการการทํางานร่วมกันของ Office 365 (เช่น Microsoft Teams) เพื่อเริ่มใช้งาน   
- กําหนดบริการที่มีสิทธิ์ที่คุณต้องการขึ้นเครื่อง 
    
## <a name="assess-phase"></a>ประเมินระยะ

- ถือสายการวางแผนความสําเร็จเพื่อให้คําแนะนํากับคุณสําหรับการยอมรับผู้ใช้ที่ประสบความสําเร็จ  
- แสดงภาพรวมการบริหาร
- ให้คําแนะนําเกี่ยวกับ: 
  - โดเมนเนมระบบ (DNS), เครือข่ายและโครงสร้างพื้นฐานความต้องการ
  - ความต้องการของลูกค้า (อินเทอร์เน็ตเบราเซอร์ระบบปฏิบัติการของลูกค้าโทรศัพท์มือถือและความต้องการของบริการ')
  - ข้อมูลเฉพาะตัวของผู้ใช้และการจัดเตรียม
  - การเปิดใช้งานบริการที่มีสิทธิ์ซึ่งซื้อและกําหนดเป็นส่วนหนึ่งของการเตรียมพร้อม 
  - ผลักดันการยอมรับบริการที่ประสบความสําเร็จและคุณค่า  
- สร้างไทม์ไลน์สําหรับกิจกรรมการแก้ไข
- จัดเตรียมรายการตรวจสอบการแก้ไข 
- ประเมินโครงสร้างพื้นฐาน SharePoint Server 2013 หรือ SharePoint Server 2016 ที่มีอยู่รวมถึง:
  - ข้อกําหนดเบื้องต้นสําหรับไฮบริดสลี SharePoint แบบออนไลน์ 
  - ความพร้อมโครงสร้างพื้นฐานในสถานที่สําหรับคุณลักษณะไฮบริดสลี SharePoint แบบออนไลน์
  - การเข้าถึงปลายทางของ SharePoint แบบออนไลน์ที่จําเป็น
  - ผู้ชมสําหรับ OneDrive สําหรับธุรกิจไฮบริดสลี 
- ประเมินโครงสร้างพื้นฐาน Lync, Skype สําหรับธุรกิจออนไลน์ หรือ Microsoft Teams ที่มีอยู่ รวมถึง:
  - Skype สนับสนุนสําหรับไคลเอ็นต์ธุรกิจหรือทีมกลยุทธ์การปรับใช้ไคลเอ็นต์
  - การเข้าถึงปลายทาง
  - คุณภาพการเชื่อมต่อ
  - แบนด์วิดธ์ประมาณการ
  - ข้อกําหนดเบื้องต้นเพื่อสนับสนุนการกําหนดค่าเซิร์ฟเวอร์โดเมนที่แยก
  - ความพร้อมของผู้ใช้ที่ระบุเพื่อย้ายไปยัง Skype สําหรับธุรกิจออนไลน์หรือทีม
- ประเมินโครงสร้างพื้นฐานของการส่งข้อความ รวมถึง: 
  - การไหลของจดหมายโดยรวมและหลักการเส้นทาง
  - การเข้าถึงไคลเอ็นต์ (รวมถึงปลายทางการเข้าถึงไคลเอ็นต์ที่มีอยู่)
  - สภาพแวดล้อมการส่งข้อความต้นทางสําหรับความต้องการการรวม
- ให้ย้ายข้อมูลถ้ามีใช้บริการย้ายข้อมูล FastTrack Center และถ้าคุณมีคุณสมบัติตรงตามคุณสมบัติ
    
## <a name="remediate-phase"></a>เฟสของการแก้ไข

- จัดการประชุมสายกับคุณตามกําหนดการตกลงกันเพื่อตรวจสอบความคืบหน้าของกิจกรรมการแก้ไขและการวางแผนความสําเร็จ 
- แนะนําคุณผ่านการเรียกใช้เครื่องมือประเมินเพื่อระบุและแก้ไขปัญหาและตีความผล
    
## <a name="enable-phase"></a>เปิดใช้งานเฟส

ให้คําแนะนําเกี่ยวกับ: 
- การประเมินความคืบหน้าในการวางแผนความสําเร็จและการกําหนดสิ่งที่ความช่วยเหลือเพิ่มเติมที่คุณต้องการ
- การเปิดใช้งานผู้เช่า Office 365 ของคุณ  
- การกําหนดค่าโปรโตคอล TCP/IP และพอร์ตไฟร์วอลล์
- การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์ 
- กําลังตรวจสอบความถูกต้องของการเชื่อมต่อกับ Office 365
- การเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ในสถานของคุณด้วยไดเรกทอรีที่ใช้งานอยู่ของ Azure:
  - การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และ Office 365 ถ้าจําเป็น 
  - การกําหนดค่าการซิงโครไนส์รหัสผ่าน (แฮรหัสผ่าน) กับ Office 365 (Azure Active Directory) ด้วยเครื่องมือการเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ของ Azure ถ้าจําเป็น
  - สําหรับสภาพแวดล้อมแบบเดี่ยวและฟอเรสต์หลาย:
      - การกําหนดค่า Azure ไดเรกทอรีที่ใช้งานอยู่แบบพาส-ทรูรับรองความถูกต้อง ถ้าจําเป็น\*\*
      - การกําหนดค่า Azure ไดเรกทอรีที่ใช้งานอยู่แบบจําแนกแบบครั้งเดียวเข้าสู่ระบบ (SSO), ถ้าจําเป็น\*\*\*
    > [!NOTE]
    > รับรองความถูกต้องของ azure ไดเรกทอรีที่ใช้งานอยู่แบบพาส-ทรูสําหรับสภาพแวดล้อมแบบหลายฟอเรสต์ได้รับการสนับสนุนถ้ามีฟอเรสต์เชื่อถือระหว่างฟอเรสต์ไดเรกทอรีที่ใช้งานอยู่ของคุณ และถ้าเส้นทางคําต่อท้ายชื่อถูกกําหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายเพื่อให้พร้อมใช้งานสูงสําหรับการร้องขอการลงชื่อเข้าใช้ สําหรับข้อมูลเพิ่มเติม ให้ดูที่[Azure ที่ใช้งานอยู่ไดเรกทอรีแบบพาส-ทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860094)และ[Azure ที่ใช้งานอยู่ไดเรกทอรีแบบราบรื่นแบบครั้งเดียวเข้าสู่ระบบ: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860095) 
- สําหรับฟอเรสต์เดียวเมื่อติดต่อกับภายนอกเป็นเป้าหมาย: 
  - การติดตั้งและการกําหนดค่า FS โฆษณาสําหรับการรับรองความถูกต้องโดเมนท้องถิ่นกับ Office 365 ในการกําหนดค่าแบบเดียวไซต์
  - การติดตั้งและกําหนดค่า WAP สําหรับการเผยแพร่โครงสร้างพื้นฐาน FS โฆษณาไปยังอินเทอร์เน็ตเมื่อจําเป็น
    > [!NOTE]
    > สําหรับการกําหนดค่าฟอเรสต์หลายทั้งหมด FS โฆษณาการจัดวางจะออกของขอบเขต 
- การทดสอบฟังก์ชัน SSO แบบไร้รอยต่อหากใช้งาน
- ผลักดันการยอมรับบริการที่ประสบความสําเร็จและคุณค่า
    
\*\*สําหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจํากัดการรับรองความถูกต้องแบบพาส-ทรู โปรดดูที่[การรับรองความถูกต้องผ่านไดเรกทอรีที่ใช้งานอยู่ของ Azure: ข้อจํากัดปัจจุบัน](https://go.microsoft.com/fwlink/?linkid=860356) 

\*\*\*สําหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหา SSO แบบไม่มีรอยต่อ ให้ดูที่[การแก้ไขปัญหาการลงชื่อเพียงครั้งเดียวแบบไม่มีรอยต่อของ Azure ไดเรกทอรีที่ใช้งานอยู่](https://go.microsoft.com/fwlink/?linkid=841926)

## <a name="exchange-online"></a>การแลกเปลี่ยนแบบออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- การสร้างหรือการปรับปรุงระเบียน DNS 
- การเปิดใช้งานการกําหนดเส้นทางอีเมลระหว่างระบบการส่งข้อความต้นทางและสภาพแวดล้อมของ Office 365 
- การกําหนดค่าการป้องกัน Exchange Online การป้องกันข้อมูลสูญหาย (DLP) และการเข้ารหัสลับข้อความ Office 365 (OME) และการป้องกันภัยคุกคามขั้นสูงของ Office 365 (ATP) (ถ้ามีอยู่ในการสมัครใช้งานของคุณ) และการตรวจสอบระเบียน MX ของคุณชี้ไปที่ Office 365 สําหรับโดเมนที่เปิดใช้งานจดหมายที่ผ่านการตรวจสอบความถูกต้องทั้งหมด
- การกําหนดค่าการตั้งค่าไฮบริดสลีระหว่างองค์กร Exchange ในสถานที่เดียวและ Office 365*หรือ*ระหว่างองค์กร Exchange ในสถานที่หลายองค์กรและ Office 365 
- การกําหนดค่าไคลเอ็นต์กล่องจดหมาย (Outlook สําหรับ Windows, Outlook บนเว็บ และ Outlook สําหรับ iOS และ Android)
- การกําหนดค่าระบบอัตโนมัติ การตรวจสอบ และการตอบสนองสําหรับ ATP ของ Office 365 (ถ้ามีอยู่ในการสมัครใช้งานของคุณ)
    
สําหรับข้อมูลเพิ่มเติมเกี่ยวกับความรับผิดชอบในการย้ายข้อมูล ให้ดูที่[การย้ายข้อมูล](O365-data-migration.md)
  
## <a name="microsoft-365-apps"></a>แอปไมโครซอฟท์ 365

ให้คําแนะนําเกี่ยวกับ:
- แก้ไขปัญหาการปรับใช้
- การกําหนดสิทธิ์การใช้งานของผู้ใช้และอุปกรณ์โดยใช้ศูนย์[การจัดการ Microsoft 365](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- การติดตั้งแอป Microsoft 365 จากพอร์ทัล Office 365 โดยใช้คลิกทูรัน
- การติดตั้งแอป Office สําหรับอุปกรณ์เคลื่อนที่ (เช่น Outlook สําหรับ iOS และ Android, Word Mobile, Excel Mobile และ PowerPoint Mobile) บน iOS หรือ Android ของคุณ 
- การกําหนดค่าการตั้งค่าการปรับปรุงโดยใช้เครื่องมือการปรับใช้ Office 365
- เลือกและตั้งค่าการติดตั้งภายในหรือบนคลาวด์
- สร้าง XML การตั้งค่าคอนฟิกเครื่องมือการปรับใช้ Office ด้วยเครื่องมือกําหนดเองของ Office หรือ XML ดั้งเดิมเพื่อกําหนดค่าแพคเกจการปรับใช้
- การปรับใช้โดยใช้ตัวจัดการการตั้งค่าคอนฟิกปลายทางของ Microsoft รวมทั้งความช่วยเหลือเกี่ยวกับการสร้างบรรจุภัณฑ์ตัวจัดการการตั้งค่าคอนฟิกปลายทางของ Microsoft

## <a name="microsoft-information-governance"></a>การกํากับดูแลข้อมูลไมโครซอฟท์ 

ให้คําแนะนําเกี่ยวกับ:
- การจัดการระเบียน
  - การนําสิทธิ์ไปใช้กับการจัดการระเบียน
  - คําแนะนําเกี่ยวกับการแปลแผนไฟล์และกําหนดการเก็บรักษาลงในป้ายกํากับและนโยบาย
  - การสร้างป้ายกํากับและนโยบายการเก็บรักษา
  - กําลังสร้างนโยบายการลบ
  - ตรวจทานรายการสําหรับการจัดการ
- การบริหารความเสี่ยงภายใน
  - การเปิดใช้งานบันทึกการตรวจสอบ Office 365
  - การกําหนดค่าการตั้งค่าภายในการจัดการความเสี่ยงภายใน
  - การสร้างนโยบายความเสี่ยงภายในโดยใช้ playbooks ในตัว
  - การกําหนดค่าสิทธิ์สําหรับการปฏิบัติตามข้อกําหนดด้านการสื่อสาร
  - การสร้างนโยบายการปฏิบัติตามข้อกําหนดด้านการสื่อสารโดยใช้เทมเพลตที่ปรับแต่งได้
  - การตรวจสอบและการตรวจสอบการแจ้งเตือน
- การกํากับดูแลข้อมูล
  - การใช้สิทธิ์สําหรับการกํากับดูแลข้อมูล
  - การสร้างป้ายกํากับการเก็บรักษา
  - การประกาศป้ายชื่อการเก็บข้อมูล (ด้วยตนเองและอัตโนมัติ)
  - กําลังสร้างงานนําเข้า

  - การสร้างกรณีและปัญหา
  - การเพิ่มผู้ดูแล

  - ค้น หา
  - ชุดตรวจทาน
  - กําลังส่งออกเนื้อหา

## <a name="microsoft-information-protection"></a>การปกป้องข้อมูลไมโครซอฟท์

## <a name="microsoft-teams"></a>Microsoft Teams

ให้คําแนะนําเกี่ยวกับ:
- ยืนยันข้อกําหนดขั้นต่ํา
- การกําหนดค่าพอร์ตไฟร์วอลล์
- การตั้งค่า DNS  
- การยืนยัน Teams ถูกเปิดใช้งานบนผู้เช่า Office 365 ของคุณ
- การเปิดใช้งานหรือปิดใช้งานสิทธิ์การใช้งานของผู้ใช้
- การกระจายไคลเอนต์ของทีม
- คุณสมบัติด้านไอทีและผู้ดูแลระบบ
- คุณสมบัติผลิตภัณฑ์หลัก
- เท็มเพลตความสําเร็จของลูกค้า
- การสร้างบัญชีเพื่อเชื่อมโยงกับอุปกรณ์ระบบห้องที่รองรับ (สูงสุด 10 บัญชี) 
- การเปิดใช้งานการกําหนดเส้นทางโดยตรง
- การเปิดใช้งานการประชุมทางเสียง
- การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นบริดจ์การประชุม
- การกําหนดบริดจ์การประชุมให้กับผู้ใช้ที่ได้รับสิทธิ์การใช้งาน
- การเปิดใช้งานระบบโทรศัพท์
- การเปิดใช้งานระบบโทรศัพท์และแผนการโทร (ในตลาดที่มีอยู่)
- การกําหนดหมายเลขให้กับผู้ใช้ที่ได้รับอนุญาตให้ใช้งาน
- หมายเลขท้องถิ่นพอร์ตคําแนะนําผ่าน UI ถึง 999
- หมายเลขท้องถิ่นสนับสนุน SR พอร์ตมากกว่า 999 
- การเปิดใช้งาน Teams Live Events 
- การตั้งค่าและการรวมองค์กรลงในกระแสข้อมูล Microsoft

## <a name="office-365-advanced-threat-protection"></a>การป้องกันภัยคุกคามขั้นสูงของ Office 365

ให้คําแนะนําเกี่ยวกับ:
- การเปิดใช้งานการเชื่อมโยงที่ปลอดภัย
- การเปิดใช้งานสิ่งที่แนบมาอย่างปลอดภัย
- การเปิดใช้งานนโยบายการป้องกันฟิชชิ่ง
- การกําหนดค่าระบบอัตโนมัติ การตรวจสอบ และการตอบสนอง
- ใช้โปรแกรมจําลองการโจมตี
- การรายงานและการวิเคราะห์ภัยคุกคาม
    
## <a name="onedrive-for-business"></a>วันไดรฟ์สําหรับธุรกิจ

ให้คําแนะนําเกี่ยวกับ:
- การระบุเวอร์ชัน SharePoint ภายในองค์กรและตัวเลือกการรวม 
- การระบุตัวเลือกการซิงโครไนซ์และข้อมูลประจําตัว
- การเลือกตัวเลือกการเริ่มใช้:   
  - เปิดตัวเพียงในเวลา
  - การเปิดตัวแบบแบ่งเป็นฉาก (ลําดับและแบ่งเป็นช่วง)
- การเตรียมสภาพแวดล้อมในสถานที่สําหรับ OneDrive สําหรับการปรับใช้ธุรกิจ:
  - การระบุ OneDrive ที่ถูกต้องสําหรับไคลเอ็นต์การซิงค์ธุรกิจ
  - การกําหนดค่า DNS พอร์ตเครือข่าย และไฟร์วอลล์ 
- การกําหนดสิทธิ์การใช้งานสําหรับผู้ใช้ 
- การตั้งค่าผู้ชม SharePoint Online เพื่อควบคุมและควบคุมผู้ที่รับ OneDrive สําหรับธุรกิจ 
- การปรับใช้ OneDrive สําหรับไคลเอ็นต์การซิงค์ธุรกิจไปยังเดสก์ท็อป   
- วิธีการกําหนดค่า SharePoint ไฮบริดสลีแบบออนไลน์ OneDrive สําหรับการเปลี่ยนเส้นทางธุรกิจ (SharePoint 2013 และ SharePoint 2016 เท่านั้น)
- การย้ายข้อมูลหากมีการใช้บริการย้ายข้อมูล FastTrack Center และหากคุณมีคุณสมบัติตรงตามคุณสมบัติ

## <a name="project-online"></a>โครงการออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- ตรวจทานแผนการสมัครใช้งาน
- การตรวจสอบฟังก์ชันพื้นฐานของ SharePoint
- เพิ่มบริการออนไลน์ของโครงการ
- การเพิ่มผู้ใช้ไปยัง Project Online รวมทั้ง ERP Sync
- การตรวจสอบฟังก์ชันพื้นฐานของโครงการออนไลน์ โดยการสร้างโครงการ
    
## <a name="project-online-professional-and-project-online-premium"></a>โครงการออนไลน์มืออาชีพและโครงการออนไลน์พรีเมี่ยม

ให้คําแนะนําเกี่ยวกับ:
- แก้ไขปัญหาการปรับใช้
- การกําหนดสิทธิ์การใช้งานของผู้ใช้โดยใช้[ศูนย์การจัดการ Microsoft 365](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- การดาวน์โหลด และการติดตั้งไคลเอนต์เดสก์ท็อปแบบออนไลน์ของโครงการจากพอร์ทัล   
- การกําหนดค่าการตั้งค่าการปรับปรุงโดยใช้เครื่องมือการปรับใช้ Office 365 หรือแม่แบบของนโยบายกลุ่ม
- การตั้งค่าเซิร์ฟเวอร์การกระจายบนเซิร์ฟเวอร์เดียวในไซต์สําหรับไคลเอนต์เดสก์ท็อปแบบออนไลน์ของโครงการ รวมทั้งคําแนะนําสําหรับการสร้างแฟ้มconfiguration.xmlสําหรับเครื่องมือการปรับใช้ Office 2016 
- การเชื่อมต่อไคลเอนต์เดสก์ท็อปโครงการแบบออนไลน์ไปยังโครงการแบบออนไลน์

## <a name="sharepoint-online"></a>SharePoint Online

ให้คําแนะนําเกี่ยวกับ:
- การตั้งค่าการจัดสรรผู้ใช้รวมทั้งการให้สิทธิ์การใช้งาน
- การเปิดใช้งานการสร้างไซต์สําหรับผู้ดูแลระบบ SharePoint Online ของคุณ    
- การวางแผนไซต์คอลเลกชัน 
- การรักษาความปลอดภัยเนื้อหาและการจัดการสิทธิ์
- การเปิดใช้งานไซต์ส่วนบุคคลและคุณลักษณะทางสังคม
- การกําหนดค่าคุณลักษณะ SharePoint แบบออนไลน์ 
- การโยกย้ายข้อมูลหากมีการใช้บริการย้ายข้อมูล FastTrack Center และหากคุณมีคุณสมบัติตรงตามคุณสมบัติ
- การประเมินการกําหนดค่าโครงสร้างพื้นฐานของ SharePoint ในสถานที่จําเป็นสําหรับไฮบริดสลี SharePoint แบบออนไลน์ 
- การใช้เครื่องมือและระบบอัตโนมัติเพื่อ:
  - กําหนดค่าแอปพลิเคชันบริการการค้นหาบนระบบคลาวด์ในองค์กร 
  - กําหนดค่าความน่าเชื่อถือระหว่างสภาพแวดล้อมในองค์กรและสภาพแวดล้อมใน SharePoint
- การกําหนดค่าไซต์ SharePoint ในสถานที่เพื่อใช้คุณลักษณะไฮบริดสลี SharePoint แบบออนไลน์
    
## <a name="skype-for-business-online"></a>Skype สําหรับธุรกิจออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- การเตรียมใช้งาน Skype สําหรับข้อมูลเฉพาะตัวของธุรกิจสําหรับ Office 365 
- การเปิดใช้งานการประชุมแบบออนไลน์ 
- การสร้างบัญชีเพื่อเชื่อมโยงกับอุปกรณ์ระบบห้องที่รองรับ (สูงสุด 10 บัญชี) 
- การกําหนดค่าสภาพแวดล้อมของเซิร์ฟเวอร์โดเมนแยกเพื่อสนับสนุนไฮบริดสลี Lync หรือ Skype สําหรับธุรกิจออนไลน์ไฮบริดสลีสถานการณ์สมมติ (ถ้ามี)
- การเปิดใช้งานการประชุมทางเสียง:
  - การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นบริดจ์การประชุม
  - การกําหนดบริดจ์การประชุมให้กับผู้ใช้ที่ได้รับสิทธิ์การใช้งาน
- การเปิดใช้งานระบบโทรศัพท์:
  - การเปิดใช้งานระบบโทรศัพท์และแผนการโทร (ในตลาดที่มีอยู่)
  - การกําหนดหมายเลขให้กับผู้ใช้ที่ได้รับอนุญาตให้ใช้งาน
  - หมายเลขท้องถิ่นพอร์ตคําแนะนําผ่าน UI ถึง 999
  - หมายเลขท้องถิ่นสนับสนุน SR พอร์ตมากกว่า 999

  - การเปิดใช้งาน Skype สําหรับคําแนะนําในการออกอากาศการประชุมทางธุรกิจ
  - การตั้งค่าองค์กรสําหรับสหพันธรัฐด้วยบริการออกอากาศการประชุม
