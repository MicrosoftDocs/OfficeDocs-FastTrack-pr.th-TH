# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

ผู้เชี่ยวชาญ FastTrack มีความรับผิดชอบต่อไปนี้ในระหว่างการปฐมนิเทศ\*
  
## <a name="general"></a>ทั่วไป

- ให้ความช่วยเหลือระยะไกลสนับสนุนกับคุณ ในความสำเร็จที่วางแผนการพัฒนาและการใช้งาน และ สำหรับกิจกรรมการตั้งค่าคอนฟิกที่จำเป็นเป็นรายละเอียดในคำอธิบายของขั้นตอน
- แสดงเอกสารที่พร้อมใช้งาน และเครื่องมือซอฟต์แวร์ คอนโซล admin และสคริปต์เพื่อให้คำแนะนำให้คุณสามารถลด หรือกำจัดการตั้งค่าคอนฟิกภารกิจ และทรัพยากรการวางแผนของความสำเร็จ 
    
## <a name="initiate-phase"></a>ขั้นตอนการเริ่มต้น

- ทำงานกับคุณเข้าใจเจตนา เป้าหมายขององค์กร และแผนการใช้สำหรับการบริการ
- ทำงานกับที่คุณใช้บริการร่วมกันใน Office 365 (เช่น Microsoft ทีม) เพื่อเริ่มต้นการปฐมนิเทศ   
- กำหนดว่าบริการใดมีสิทธิ์คุณต้องเป็น onboard 
    
## <a name="assess-phase"></a>ขั้นตอนประเมิน

- เก็บความสำเร็จโทรเพื่อให้คำแนะนำกับคุณสำหรับผู้ใช้เสร็จเรียบร้อยแล้วเริ่มการวางแผน  
- แสดงภาพรวมสำหรับผู้ดูแล
- ให้คำแนะนำเกี่ยวกับ: 
  - โดเมน Name System (DNS), เครือข่าย และโครงสร้างพื้นฐานของความต้องการ
  - ไคลเอนต์ความต้องการ (อินเทอร์เน็ตเบราว์เซอร์ ระบบปฏิบัติการของไคลเอนต์ อุปกรณ์เคลื่อนที่ และความต้องการของบริการ)
  - ข้อมูลเฉพาะตัวของผู้ใช้และการเตรียมใช้งาน
  - การเปิดใช้งานสิทธิ์ได้รับบริการที่ถูกซื้อ หรือกำหนดให้เป็นส่วนหนึ่งของการปฐมนิเทศ 
  - การขับเริ่มบริการที่ประสบความสำเร็จและมูลค่า  
- สร้างเส้นเวลาสำหรับกิจกรรมด้าน
- แสดงรายการตรวจสอบด้าน 
- ประเมินอยู่ SharePoint Server 2013 หรือ 2016 เซิร์ฟเวอร์ SharePoint โครงสร้างพื้นฐานรวมทั้ง:
  - ข้อกำหนดเบื้องต้นสำหรับไฮบริดสลีออนไลน์ของ SharePoint 
  - ความพร้อมของโครงสร้างพื้นฐานในสถานที่สำหรับคุณลักษณะของไฮบริดสลีออนไลน์ของ SharePoint
  - การเข้าถึงปลายทางออนไลน์ของ SharePoint ที่จำเป็น
  - ผู้ชมสำหรับ OneDrive สำหรับไฮบริดสลีธุรกิจ 
- ประเมินอยู่ Lync หรือ Skype สำหรับธุรกิจออนไลน์โครงสร้างพื้นฐานรวมถึง:
  - ได้รับการสนับสนุน Skype สำหรับธุรกิจกลยุทธ์การปรับใช้ไคลเอนต์
  - การเข้าถึงปลายทาง
  - คุณภาพการเชื่อมต่อ
  - แบนด์วิดท์ที่ประเมิน
  - ข้อกำหนดเบื้องต้นเพื่อสนับสนุนการตั้งค่าคอนฟิกเซิร์ฟเวอร์ของโดเมนแยก
  - ความพร้อมของผู้ใช้ที่ระบุเมื่อต้องการย้ายไปยัง Skype สำหรับธุรกิจออนไลน์
- โครงสร้างพื้นฐานการส่งข้อความ ประเมินรวมถึง: 
  - ส่งขั้นตอนและหลักการสายงานการผลิตโดยรวม
  - การเข้าถึงไคลเอ็นต์ (รวมไปถึงปลายทางการเข้าถึงไคลเอ็นต์ที่เผยแพร่แล้วที่มีอยู่)
  - สภาพแวดล้อมสำหรับความต้องการรวมข้อความต้นฉบับ
- ให้ย้ายข้อมูล ถ้ามีใช้บริการ FastTrack ศูนย์การโยกย้ายข้อมูล และถ้าคุณมีสิทธิ์
    
## <a name="remediate-phase"></a>Remediate ขั้นตอน

- หยุดการเรียกประชุมกับคุณตามกำหนดเวลาที่ตกลงกันไว้เพื่อตรวจสอบความคืบหน้าของด้านกิจกรรมและความสำเร็จในการวางแผน 
- แนะนำคุณเกี่ยวกับการเรียกใช้เครื่องมือการประเมินที่ระบุ และ remediate ปัญหา และแปลผลลัพธ์
    
## <a name="enable-phase"></a>ขั้นตอนการเปิดใช้งาน

ให้คำแนะนำเกี่ยวกับ: 
- การประเมินความคืบหน้าของความสำเร็จในการวางแผน และกำหนดว่าความช่วยเหลือเพิ่มเติมได้
- การเรียกใช้ของผู้เช่า Office 365  
- การกำหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์
- การกำหนดค่า DNS สำหรับบริการที่เหมาะสม 
- ตรวจสอบการเชื่อมต่อกับ Office 365
- การเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ในสถานของคุณ มีไดเรกทอรีที่ใช้งานอยู่ของ Azure:
  - ติดตั้งเซิร์ฟเวอร์ไดเรกทอรีให้ตรงกันระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) ของคุณและ Office 365 ถ้าจำเป็น 
  - ตั้งค่าคอนฟิกซิงโครไนส์ของรหัสผ่าน (รหัสผ่านแฮ) กับ Office 365 (Azure ไดเรกทอรีที่ใช้งานอยู่) โดยใช้เครื่องมือ Azure Active Directory เชื่อมต่อ ถ้าจำเป็น
  - สำหรับสภาพแวดล้อมของฟอเรสต์เดียว และหลาย:
      - การกำหนดค่า Azure Active Directory พาส-ทรูรับรองความถูกต้อง ถ้าจำเป็น\*\*
      - การกำหนดค่า Azure Active Directory อย่างราบเดียวเข้าสู่ระบบ (SSO), ถ้าจำเป็น\*\*\*
    > [!NOTE]
    > Azure Active Directory พาส-ทรูรับรองความถูกต้องสำหรับสภาพแวดล้อมหลายฟอเรสต์ได้รับการสนับสนุนถ้ามีความเชื่อถือฟอเรสต์ระหว่างฟอเรสต์เป็นไดเรกทอรีที่ใช้งานอยู่ของคุณและ ถ้ามีการกำหนดเส้นทางส่วนต่อท้ายชื่อถูกกำหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายตัวเพื่อให้พร้อมใช้งานสูงสำหรับการร้องขอการเข้าสู่ระบบ สำหรับข้อมูลเพิ่มเติม ดู[Azure Active Directory พาส-ทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860094)และ[Azure Active Directory อย่างราบเดียวเข้าสู่ระบบ: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860095) 
- สำหรับฟอเรสต์เดียวเมื่อผู้ใช้อีเมลที่ติดต่อกับภายนอกคือ เป้าหมาย: 
  - การติดตั้ง และตั้งค่าคอนฟิก FS โฆษณาสำหรับการรับรองความถูกต้องของโดเมนเฉพาะกับ Office 365 ในการกำหนดค่า ไซต์เดียว แบบทนต่อ ถ้าจำเป็น
  - การติดตั้ง และตั้งค่าคอนฟิก WAP สำหรับการเผยแพร่โครงสร้างพื้นฐาน FS โฆษณาของคุณกับอินเทอร์เน็ต เมื่อจำเป็น
    > [!NOTE]
    > สำหรับโครงแบบป่าไม้หลายทั้งหมด จัดวาง FS โฆษณาจะออกของขอบเขต 
- การทดสอบฟังก์ชัน SSO ถ้าปรับใช้
- การขับเริ่มบริการที่ประสบความสำเร็จและมูลค่า
    
\*\*สำหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจำกัดของการรับรองความถูกต้องแบบพาส-ทรู ดู[Azure Active Directory พาส-ทรูรับรองความถูกต้อง: ข้อจำกัดปัจจุบัน](https://go.microsoft.com/fwlink/?linkid=860356) 

\*\*\*สำหรับข้อมูลเพิ่มเติมเกี่ยวกับปัญหาอย่างราบ SSO ดู[การแก้ปัญหา Azure ใช้งานไดเรกทอรีอย่างราบเดียวเข้าสู่ระบบ](https://go.microsoft.com/fwlink/?linkid=841926)

## <a name="exchange-online"></a>การแลกเปลี่ยนแบบออนไลน์

ให้คำแนะนำเกี่ยวกับ:
- การสร้าง หรือปรับปรุงระเบียน DNS 
- เปิดใช้งานอีเมลสายงานการผลิตระหว่างระบบการส่งข้อความต้นฉบับและสภาพแวดล้อมของ Office 365 
- การกำหนดค่าคุณลักษณะการป้องกันแบบออนไลน์ของอัตราแลกเปลี่ยน (รวมทั้งแลกเปลี่ยนออนไลน์ขั้นสูงคุกคามคุณลักษณะป้องกันถ้าพร้อมใช้งานในการสมัครใช้งาน) และการตรวจสอบระเบียน MX ของคุณชี้ไปที่ Office 365 สำหรับทั้งหมดถูกตรวจสอบโดเมนที่เปิดใช้งานจดหมาย
- การกำหนดค่าการตั้งค่าไฮบริดสลี ระหว่างองค์กร Exchange เดียวในสถานที่และ Office 365*หรือ*ระหว่างองค์กร Exchange ในสถานที่และ Office 365 หลายอย่างใดอย่างหนึ่ง 
- การกำหนดค่าไคลเอ็นต์จดหมาย (Outlook สำหรับ Windows, Outlook บนเว็บ และ Outlook สำหรับ iOS และ Android)
    
สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการโยกย้ายข้อมูลที่รับผิดชอบ ดู[การโยกย้ายข้อมูล](O365-data-migration.md)
  
## <a name="sharepoint-online"></a>SharePoint แบบออนไลน์

ให้คำแนะนำเกี่ยวกับ:
- การตั้งค่าการเตรียมใช้งานรวมถึงการอนุญาตให้ใช้สิทธิ์ของผู้ใช้
- การเปิดใช้งานการสร้างไซต์สำหรับผู้ดูแลระบบ SharePoint แบบออนไลน์ของคุณ    
- ไซต์คอลเลกชันในการวางแผน 
- การรักษาความปลอดภัยเนื้อหา และการจัดการสิทธิ์
- เปิดใช้งานไซต์ส่วนบุคคลและคุณลักษณะทางสังคม
- ตั้งค่าคอนฟิกลักษณะการทำงานแบบออนไลน์ของ SharePoint 
- ให้ย้ายข้อมูล ถ้ามีใช้บริการ FastTrack ศูนย์การโยกย้ายข้อมูล และถ้าคุณมีสิทธิ์
- ประเมินในสถานที่ SharePoint โครงสร้างพื้นฐานการกำหนดค่าฟาร์มจำเป็นสำหรับไฮบริดสลีออนไลน์ของ SharePoint 
- ใช้เครื่องมือและระบบอัตโนมัติเพื่อ:
  - กำหนดค่าโปรแกรมประยุกต์บริการค้นหาของ cloud ในสถานที่ 
  - การตั้งค่าคอนฟิกความน่าเชื่อถือระหว่าง SharePoint ในสถานและเมฆสภาพแวดล้อม
- การกำหนดค่าไซต์ SharePoint ในสถานที่จะใช้ลักษณะการทำงานของไฮบริดสลีออนไลน์ของ SharePoint
    
## <a name="onedrive-for-business"></a>OneDrive สำหรับธุรกิจ

ให้คำแนะนำเกี่ยวกับ:
- ระบุในสถานที่ SharePoint รุ่นและสามารถรวมตัวเลือก 
- ระบุตัวเลือกการซิงโครไนส์และรหัสประจำตัว
- เลือกตัวเลือกไวร์:   
  - ไวร์เพิ่งเวลา
  - แบ่งระยะไวร์ (เรียงลำดับ และสามารถยุติ)
- เตรียมสภาพแวดล้อมในสถานที่สำหรับ OneDrive สำหรับธุรกิจการปรับใช้:
  - ระบุ OneDrive ถูกต้องสำหรับไคลเอ็นต์ซิงค์ธุรกิจ
  - ตั้งค่าคอนฟิก DNS พอร์ตเครือข่าย และไฟร์วอลล์ 
- กำหนดสิทธิ์การใช้งานของผู้ใช้ 
- การตั้งค่าผู้ชมแบบออนไลน์ของ SharePoint เพื่อควบคุม และควบคุมว่าผู้ใดมี OneDrive สำหรับธุรกิจ 
- ปรับใช้การ OneDrive สำหรับไคลเอ็นต์ซิงค์ธุรกิจกับคอมพิวเตอร์แบบตั้งโต๊ะ   
- วิธีการตั้งค่าคอนฟิก OneDrive ไฮบริดสลีออนไลน์ของ SharePoint สำหรับการเปลี่ยนเส้นทางธุรกิจ (SharePoint 2013 และ 2016 SharePoint เท่านั้น)
- โยกย้ายข้อมูล ถ้ามีใช้บริการ FastTrack ศูนย์การโยกย้ายข้อมูล และถ้าคุณมีสิทธิ์
    
## <a name="skype-for-business-online"></a>Skype สำหรับธุรกิจแบบออนไลน์

ให้คำแนะนำเกี่ยวกับ:
- เตรียมใช้งาน Skype สำหรับข้อมูลเฉพาะตัวของธุรกิจสำหรับ Office 365 
- เปิดใช้งานการประชุมทางออนไลน์ ส่งข้อความโต้ตอบแบบทันที (IM), และสถานะการออนไลน์คุณลักษณะสำหรับ Office 365 
- สร้างบัญชีเพื่อเชื่อมโยงกับอุปกรณ์ระบบห้อง (สูงสุด 10 บัญชี) ได้รับการสนับสนุน 
- การกำหนดค่าสภาพแวดล้อมเซิร์ฟเวอร์โดเมนแยกเพื่อสนับสนุนไฮบริดสลี Lync หรือ Skype สำหรับธุรกิจออนไลน์สถานการณ์ไฮบริดสลี (ถ้ามี)
- เปิดใช้งานการประชุมเสียง:
  - การตั้งค่าองค์กรสำหรับการตั้งค่าเริ่มต้นของการประชุมบริดจ์
  - การมอบหมายของบริดจ์เครือประชุมการอนุญาตผู้ใช้
- การเปิดใช้งานระบบโทรศัพท์:
  - การเปิดใช้งานระบบโทรศัพท์และเรียกแผนปฐมนิเทศ (ในตลาดที่พร้อมใช้งาน)
  - การกำหนดหมายเลขให้แก่ผู้ใช้ได้รับอนุญาต
  - หมายเลขท้องถิ่นที่ปรับเปลี่ยนคำแนะนำผ่าน UI จนถึง 999
  - ภายในหมายเลข++:การสร้างพอร์ต SR สนับสนุนมากกว่า 999
- เปิดใช้งาน Skype สำหรับการออกอากาศการประชุมทางธุรกิจ:
  - เปิดการใช้งานของ Skype สำหรับธุรกิจการออกอากาศการประชุมปฐมนิเทศให้คำแนะนำ
  - การตั้งค่าองค์กรกับภายนอกกับบริการการออกอากาศการประชุม
    
## <a name="microsoft-teams"></a>ทีมงานของ Microsoft

ให้คำแนะนำเกี่ยวกับ:
- ยืนยันความต้องการต่ำสุด
- ตั้งค่าคอนฟิกพอร์ตไฟร์วอลล์
- การตั้งค่า DNS  
- ทีมงาน Microsoft ยืนยันว่า มีการเปิดใช้ผู้เช่า Office 365 ของคุณ
- การเปิดใช้งาน หรือปิดใช้งานสิทธิ์การใช้งานของผู้ใช้
- การกระจายของไคลเอนต์ Microsoft ทีมงาน
- IT Pro และการดูแลลักษณะการทำงาน
- คุณลักษณะของผลิตภัณฑ์หลัก
- ต้นแบบความสำเร็จของลูกค้า
    
## <a name="power-bi"></a>พลังงาน BI

- ตรวจทานแผนการบอกรับเป็นสมาชิกใช้พลังงาน BI 
- เพิ่มบริการพลังงาน BI 
- ดาวน์โหลด app เดสก์ท็อปใช้พลังงาน BI
    
## <a name="project-online"></a>โครงการแบบออนไลน์

ให้คำแนะนำเกี่ยวกับ:
- ตรวจทานแผนการบอกรับเป็นสมาชิก
- การตรวจสอบฟังก์ชันการทำงานพื้นฐานของ SharePoint
- เพิ่มบริการออนไลน์ของโครงการ
- เพิ่มผู้ใช้ไปยังโครงการ Online รวมซิงค์ ERP
- การตรวจสอบว่าโครงการออนไลน์ฟังก์ชันพื้นฐาน โดยการสร้างโครงการ
    
## <a name="project-online-professional-and-project-online-premium"></a>โครงการ Professional ออนไลน์และค่าจ้างพิเศษออนไลน์ของโครงการ

ให้คำแนะนำเกี่ยวกับ:
- กำหนดประเด็นการปรับใช้
- กำหนดสิทธิ์การใช้งานของผู้ใช้โดยใช้[Microsoft 365 admin center](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- ดาวน์โหลด และติดตั้งไคลเอ็นต์เดสก์ท็อปออนไลน์ของโครงการจากเว็บไซต์   
- การกำหนดค่าการตั้งค่าการปรับปรุงโดยใช้ต้นแบบของเครื่องมือการปรับใช้ Office หรือ'นโยบายกลุ่ม'
- การตั้งค่าเซิร์ฟเวอร์การกระจายภายในอาคารเดียวสำหรับโครงการออนไลน์เดสก์ท็อปไคลเอนต์ รวมถึงคำแนะนำสำหรับการสร้างแฟ้ม configuration.xml สำหรับเครื่องมือการปรับใช้ Office 2016 
- การเชื่อมต่อโครงการออนไลน์เดสก์ท็อปไคลเอ็นต์ไปยังโครงการแบบออนไลน์

## <a name="office-365-proplus"></a>Office 365 ProPlus

ให้คำแนะนำเกี่ยวกับ:
- กำหนดประเด็นการปรับใช้
- กำหนดสิทธิ์การใช้งานของผู้ใช้โดยใช้[Microsoft 365 admin center](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- ติดตั้ง Office 365 ProPlus จากพอร์ทัล Office 365 โดยใช้คลิกทูรัน
- การติดตั้ง Office Mobile apps (เช่น Outlook สำหรับ iOS และ Android, Word Mobile, Excel Mobile และ PowerPoint Mobile) บน iOS, Android หรือ Windows Mobile อุปกรณ์ 
- การกำหนดค่าการตั้งค่าการปรับปรุงโดยใช้ต้นแบบของเครื่องมือการปรับใช้ของ Office 2016 หรือ'นโยบายกลุ่ม'
- การตั้งค่าเซิร์ฟเวอร์การกระจายภายในอาคารเดียวสำหรับ Office 365 ProPlus รวมทั้งความช่วยเหลือในการสร้างแฟ้ม configuration.xml เพื่อใช้กับเครื่องมือการปรับใช้ Office
- การปรับใช้ที่ใช้ระบบศูนย์การตั้งค่าคอนฟิกตัวจัดการ รวมทั้งความช่วยเหลือในการสร้างตัวจัดการการตั้งค่าคอนฟิกศูนย์ระบบบรรจุ
