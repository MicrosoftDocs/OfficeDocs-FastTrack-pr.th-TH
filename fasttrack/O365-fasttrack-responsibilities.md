# <a name="fasttrack-responsibilities"></a>ความรับผิดชอบของ FastTrack

ผู้เชี่ยวชาญ FastTrack มีหน้าที่รับผิดชอบต่อไปนี้ระหว่างการปฐมนต\*
  
## <a name="general"></a>ทั่ว ไป

- ให้ความช่วยเหลือสนับสนุนระยะไกลแก่คุณในการพัฒนาการวางแผนความสําเร็จและการดําเนินการและสําหรับกิจกรรมการกําหนดค่าที่จําเป็นตามรายละเอียดในรายละเอียดขั้นตอน
- จัดเตรียมเครื่องมือสําหรับเอกสารและเครื่องมือซอฟต์แวร์ คอนโซลผู้ดูแลระบบ และสคริปต์เพื่อให้คําแนะนําสําหรับการลดหรือกําจัดงานการกําหนดค่า และทรัพยากรการวางแผนความสําเร็จ 
    
## <a name="initiate-phase"></a>เริ่มต้นขั้นตอน

- ทํางานร่วมกับคุณเพื่อให้เข้าใจถึงเจตนา เป้าหมายขององค์กร และแผนการใช้งานสําหรับบริการ
- ทํางานร่วมกับคุณโดยใช้บริการการทํางานร่วมกันของ Office 365 (เช่น Microsoft Teams) เพื่อเริ่มต้นการเริ่มทํางาน   
- กําหนดบริการที่มีสิทธิ์ที่คุณต้องการออนบอร์ด 
    
## <a name="assess-phase"></a>ประเมินระยะ

- ถือสายการวางแผนความสําเร็จเพื่อให้คําแนะนํากับคุณสําหรับการนําผู้ใช้ที่ประสบความสําเร็จ  
- ให้ภาพรวมด้านการดูแลระบบ
- ให้คําแนะนําเกี่ยวกับ: 
  - ความต้องการของระบบชื่อโดเมน (DNS) เครือข่ายและโครงสร้างพื้นฐาน
  - ความต้องการของลูกค้า (อินเทอร์เน็ตเบราเซอร์, ระบบปฏิบัติการของลูกค้า, อุปกรณ์มือถือ, และความต้องการบริการ).
  - ข้อมูลประจําตัวของผู้ใช้และการเตรียมใช้งาน
  - การเปิดใช้งานบริการที่มีสิทธิ์ซึ่งซื้อและกําหนดเป็นส่วนหนึ่งของการเริ่มต้นใช้งาน 
  - การขับเคลื่อนการรับเอาบริการที่ประสบความสําเร็จและคุ้มค่า  
- กําหนดเส้นเวลาสําหรับกิจกรรมการแก้ไข
- แสดงรายการตรวจสอบการแก้ไข 
- ประเมินโครงสร้างพื้นฐาน SharePoint Server 2013 หรือ SharePoint Server 2016 ที่มีอยู่รวมถึง:
  - ข้อกําหนดเบื้องต้นสําหรับไฮบริดสลี SharePoint แบบออนไลน์ 
  - ความพร้อมในโครงสร้างพื้นฐานในสถานที่สําหรับคุณลักษณะไฮบริด SharePoint Online
  - การเข้าถึงปลายทาง SharePoint แบบออนไลน์ที่จําเป็น
  - ผู้ชมสําหรับ OneDrive สําหรับไฮบริดสลีธุรกิจ 
- ประเมินที่มีอยู่ Lync, Skype สําหรับธุรกิจออนไลน์ หรือ Microsoft Teams โครงสร้างพื้นฐานรวมถึง:
  - สนับสนุน Skype สําหรับไคลเอ็นต์ธุรกิจหรือกลยุทธ์การปรับใช้ไคลเอ็นต์ Teams
  - การเข้าถึงปลายทาง
  - คุณภาพการเชื่อมต่อ
  - ประมาณการแบนด์วิดธ์
  - ข้อกําหนดเบื้องต้นเพื่อสนับสนุนการตั้งค่าคอนฟิกเซิร์ฟเวอร์โดเมนแยก
  - ความพร้อมของผู้ใช้ที่ระบุเพื่อย้ายไปยัง Skype สําหรับธุรกิจออนไลน์หรือทีมงาน
- ประเมินโครงสร้างพื้นฐานของการส่งข้อความ รวมถึง: 
  - ขั้นตอนและหลักการของเส้นทางจดหมายโดยรวม
  - การเข้าถึงไคลเอ็นต์ (รวมถึงปลายทางการเข้าถึงไคลเอ็นต์ที่เผยแพร่แล้ว)
  - สภาพแวดล้อมการส่งข้อความต้นทางสําหรับความต้องการการรวม
- ให้การโยกย้ายข้อมูลหากใช้บริการย้ายข้อมูล FastTrack Center และหากคุณมีคุณสมบัติตรงตามสิทธิ์
    
## <a name="remediate-phase"></a>ขั้นตอนการแก้ไข

- จัดการประชุมสายกับคุณตามกําหนดการที่ตกลงกันเพื่อทบทวนความคืบหน้าของกิจกรรมการแก้ไขและการวางแผนความสําเร็จ 
- แนะนําคุณเกี่ยวกับการเรียกใช้เครื่องมือการประเมินเพื่อระบุและแก้ไขปัญหาและตีความผลลัพธ์
    
## <a name="enable-phase"></a>เปิดใช้งานเฟส

ให้คําแนะนําเกี่ยวกับ: 
- การประเมินความคืบหน้าในการวางแผนความสําเร็จและการกําหนดความช่วยเหลือเพิ่มเติมที่คุณต้องการ
- การเปิดใช้งานผู้เช่า Office 365 ของคุณ  
- การกําหนดค่าโพรโทคอล TCP/IP และพอร์ตไฟร์วอลล์
- การกําหนดค่า DNS สําหรับบริการที่มีสิทธิ์ 
- การตรวจสอบความถูกต้องของการเชื่อมต่อกับ Office 365
- การเชื่อมต่อไดเรกทอรีที่ใช้งานอยู่ในสถานที่ของคุณกับ Azure ไดเรกทอรีที่ใช้งานอยู่:
  - การติดตั้งเซิร์ฟเวอร์การซิงโครไนส์ไดเรกทอรีระหว่างบริการระบบโดเมนไดเรกทอรีที่ใช้งานอยู่ (AD DS) และ Office 365 ถ้าจําเป็น 
  - การตั้งค่าคอนฟิกการซิงโครไนส์รหัสผ่าน (รหัสผ่านแฮ) เพื่อ Office 365 (Azure Active Directory) ด้วยเครื่องมือ Azure Active Directory เชื่อมต่อ ถ้าจําเป็น
  - สําหรับสภาพแวดล้อมแบบฟอเรสต์เดี่ยวและหลายฟอเรสต์:
      - การตั้งค่าคอนฟิก Azure Active Directory ทรูพาส-ทรูรับรองความถูกต้อง ถ้าจําเป็น\*\*
      - การตั้งค่าคอนฟิก Azure ไดเรกทอรีที่ใช้งานอยู่แบบ Seamless เดียวเข้าสู่ระบบ (SSO), ถ้าจําเป็น\*\*\*
    > [!NOTE]
    > Azure Active Directory พาส-ทรูรับรองความถูกต้องสําหรับสภาพแวดล้อมหลายฟอเรสต์ได้รับการสนับสนุนถ้าไม่มีความเชื่อถือฟอเรสต์ระหว่างฟอเรสต์ Active Directory ของคุณ และถ้าการกําหนดเส้นทางส่วนต่อท้ายชื่อถูกกําหนดค่าอย่างถูกต้อง ตัวแทนเพิ่มเติมสามารถติดตั้งบนเซิร์ฟเวอร์ในสถานที่หลายเพื่อให้มีพร้อมใช้งานสูงสําหรับคําขอการเข้าสู่ระบบ สําหรับข้อมูลเพิ่มเติม ให้ดู[Azure Active Directory ทรูทรูรับรองความถูกต้อง: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860094)และ[Azure Active Directory สู่ระบบเดียว: เริ่มต้นอย่างรวดเร็ว](https://go.microsoft.com/fwlink/?linkid=860095) 
- สําหรับฟอเรสต์เดียวเมื่อข้อมูลเฉพาะตัวที่ติดต่อกับภายนอกคือเป้าหมาย: 
  - การติดตั้งและการตั้งค่าคอนฟิก FS โฆษณาสําหรับการรับรองความถูกต้องของโดเมนเฉพาะกับ Office 365 ในการกําหนดค่าแบบทนต่อความผิดพลาดของไซต์เดียวถ้าจําเป็น
  - การติดตั้งและกําหนดค่า WAP สําหรับการเผยแพร่โครงสร้างพื้นฐาน AD FS ของคุณไปยังอินเทอร์เน็ตเมื่อจําเป็น
    > [!NOTE]
    > สําหรับการตั้งค่าคอนฟิกฟอเรสต์หลายหลาย 
- การทดสอบฟังก์ชัน SSO ที่ไร้รอยต่อหากนําไปใช้
- การขับเคลื่อนการรับเอาบริการที่ประสบความสําเร็จและคุ้มค่า
    
\*\*สําหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อจํากัดการรับรองความถูกต้องแบบพาส-ทรู ให้ดู[Azure Active Directory ทรูพาส-ทรูรับรองความถูกต้อง: ข้อจํากัดปัจจุบัน](https://go.microsoft.com/fwlink/?linkid=860356) 

\*\*\*สําหรับข้อมูลเพิ่มเติมเกี่ยวกับการตัดสินค้าจากคลัง SSO แบบราบรื่น ให้ดูที่ การแก้ไขปัญหา Azure Active Directory แบบ[ไม่มีรอยต่อครั้งเดียว](https://go.microsoft.com/fwlink/?linkid=841926)

## <a name="exchange-online"></a>แลกเปลี่ยนแบบออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- การสร้างหรือการปรับปรุงระเบียน DNS 
- การเปิดใช้งานการกําหนดเส้นทางอีเมลระหว่างระบบการส่งข้อความต้นทางและสภาพแวดล้อม Office 365 
- การกําหนดค่าการป้องกันแบบออนไลน์ของ Exchange การป้องกันข้อมูลสูญหาย (DLP) และการเข้ารหัสลับข้อความ Office 365 (OME) และการป้องกันภัยคุกคามขั้นสูงของ Office 365 (ATP) (ถ้ามีให้ในการสมัครใช้งานของคุณ) และยืนยันระเบียน MX ของคุณชี้ไปที่ Office 365 สําหรับโดเมนที่เปิดใช้งานจดหมายทั้งหมดที่มีการตรวจสอบ
- การตั้งค่าคอนฟิกการตั้งค่าไฮบริดสลีระหว่างองค์กร Exchange ในสถานที่เดียว และ Office 365*หรือ*ระหว่างองค์กร Exchange ในสถานที่หลายแห่งและ Office 365 
- การกําหนดค่าไคลเอนต์กล่องจดหมาย (Outlook สําหรับ Windows, Outlook บนเว็บ และ Outlook สําหรับ iOS และ Android)
- การกําหนดค่าการทํางานอัตโนมัติ
    
สําหรับข้อมูลเพิ่มเติมเกี่ยวกับความรับผิดชอบในการย้ายข้อมูล ให้ดูที่[การโยกย้ายข้อมูล](O365-data-migration.md)
  
## <a name="microsoft-365-apps"></a>ไมโครซอฟท์ 365 ปพลิเคชัน

ให้คําแนะนําเกี่ยวกับ:
- การแก้ไขปัญหาการปรับใช้
- การกําหนดสิทธิ์การใช้งานตามอุปกรณ์และผู้ใช้โดยใช้[ศูนย์การจัดการ Microsoft 365](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- การติดตั้งแอป Microsoft 365 จากพอร์ทัล Office 365 โดยใช้คลิก-ทู-รัน
- การติดตั้งแอป Office Mobile (เช่น Outlook สําหรับ iOS และ Android, มือถือด้วยมือถือของ Excel, และ PowerPoint Mobile) บน iOS หรือ Android ของคุณ 
- การกําหนดค่าการตั้งค่าการปรับปรุงโดยใช้เครื่องมือการปรับใช้ Office 365
- เลือกและตั้งค่าการติดตั้งภายในเครื่องหรือบนคลาวด์
- สร้าง XML การกําหนดค่าเครื่องมือการปรับใช้ Office ด้วยเครื่องมือกําหนดเอง Office หรือ XML ดั้งเดิมเพื่อกําหนดค่าแพคเกจการปรับใช้
- การปรับใช้โดยใช้ตัวจัดการการตั้งค่าคอนฟิกปลายทางของ Microsoft รวมทั้งความช่วยเหลือกับการสร้างบรรจุภัณฑ์ของตัวจัดการการตั้งค่าคอนฟิกปลายทางของ Microsoft

## <a name="microsoft-teams"></a>Microsoft Teams

ให้คําแนะนําเกี่ยวกับ:
- ยืนยันความต้องการขั้นต่ํา
- การกําหนดค่าพอร์ตไฟร์วอลล์
- การตั้งค่า DNS  
- การยืนยัน Teams ถูกเปิดใช้งานบนผู้เช่า Office 365 ของคุณ
- การเปิดใช้งานหรือปิดใช้งานสิทธิ์การใช้งานสําหรับผู้ใช้
- การกระจายไคลเอ็นต์ของทีม
- ผู้เชี่ยวชาญด้านไอทีและคุณสมบัติผู้ดูแลระบบ
- คุณสมบัติผลิตภัณฑ์หลัก
- เทมเพลตความสําเร็จของลูกค้า
- การสร้างบัญชีเพื่อเชื่อมโยงกับอุปกรณ์ระบบห้องที่รองรับ (สูงสุด 10 บัญชี) 
- การเปิดใช้งานการกําหนดเส้นทางโดยตรง
- การเปิดใช้งานการประชุมทางเสียง
- การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นของบริดจ์การประชุม
- การกําหนดบริดจ์การประชุมให้กับผู้ใช้ที่ได้รับอนุญาต
- การเปิดใช้งานระบบโทรศัพท์
- การเปิดใช้งานระบบโทรศัพท์และแผนการโทรการเริ่มใช้งาน (ในตลาดที่มีอยู่)
- การกําหนดหมายเลขให้กับผู้ใช้ที่ได้รับอนุญาต
- คําแนะนํา porting หมายเลขท้องถิ่นผ่าน UI ถึง 999
- พอร์ตหมายเลขท้องถิ่นสนับสนุน SR กว่า 999 
- การเปิดใช้งานกิจกรรมถ่ายทอดสดของ Teams 
- การตั้งค่าองค์กรและการรวมเข้ากับกระแสข้อมูล Microsoft

## <a name="office-365-advanced-threat-protection"></a>การป้องกันภัยคุกคามขั้นสูงของ Office 365

ให้คําแนะนําเกี่ยวกับ:
- การเปิดใช้งานการเชื่อมโยงที่ปลอดภัย
- การเปิดใช้งานสิ่งที่แนบมาที่ปลอดภัย
- การเปิดใช้งานนโยบายการป้องกันฟิชชิ่ง
- การกําหนดค่าระบบอัตโนมัติ
- การใช้จําลองการโจมตี
- การรายงานและการวิเคราะห์ภัยคุกคาม
    
## <a name="onedrive-for-business"></a>วันไดรฟ์สําหรับธุรกิจ

ให้คําแนะนําเกี่ยวกับ:
- การระบุตัวเลือกเวอร์ชัน SharePoint และการรวมภายในองค์กร 
- การระบุตัวเลือกการซิงโครไนซ์และเอกลักษณ์
- การเลือกตัวเลือกเปิดตัว:   
  - เปิดตัวในเวลาเพียง
  - การเปิดตัวแบบกําหนดขั้น (ตามลําดับและแบ่ง)
- การเตรียมสภาพแวดล้อมในสถานที่สําหรับ OneDrive สําหรับการปรับใช้ธุรกิจ:
  - ระบุ OneDrive ถูกต้องสําหรับไคลเอ็นต์ซิงค์ธุรกิจ
  - การกําหนดค่า DNS พอร์ตเครือข่าย และไฟร์วอลล์ 
- การกําหนดสิทธิ์การใช้งานของผู้ใช้ 
- การตั้งค่าผู้ชม SharePoint Online เพื่อควบคุมและควบคุมผู้ที่ได้รับ OneDrive สําหรับธุรกิจ 
- การปรับใช้ OneDrive สําหรับไคลเอ็นต์ซิงค์ธุรกิจไปยังเดสก์ท็อป   
- วิธีการกําหนดค่า OneDrive ไฮบริดสลี SharePoint แบบออนไลน์สําหรับการเปลี่ยนเส้นทางธุรกิจ (SharePoint 2013 และ SharePoint 2016 เท่านั้น)
- การย้ายข้อมูลหากใช้บริการย้ายข้อมูล FastTrack Center และหากคุณมีคุณสมบัติตรงตามสิทธิ์
    
## <a name="outlook-for-ios-and-android"></a>แนวโน้มสําหรับ iOS และแอนดรอยด์

ให้คําแนะนําเกี่ยวกับ:
- ดาวน์โหลด Outlook ลงใน iOS และอุปกรณ์แอนดรอยด์
- การกําหนดค่าบัญชีอีเมลภายใน Outlook

## <a name="power-bi"></a>พลังงาน BI

ให้คําแนะนําเกี่ยวกับ:
- การทบทวนแผนการสมัครใช้งาน Power BI 
- การเพิ่มบริการ Power BI 
- กําลังดาวน์โหลดแอป Power BI บนเดสก์ท็อป
    
## <a name="project-online"></a>โครงการออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- กําลังตรวจทานแผนการสมัครใช้งาน
- การตรวจสอบฟังก์ชันการทํางานของ SharePoint พื้นฐาน
- เพิ่มบริการแบบออนไลน์ของโครงการ
- การเพิ่มผู้ใช้ไปยังโครงการออนไลน์รวมถึง ERP Sync
- การตรวจสอบฟังก์ชันพื้นฐานของโครงการออนไลน์ โดยการสร้างโครงการ
    
## <a name="project-online-professional-and-project-online-premium"></a>โครงการออนไลน์ระดับมืออาชีพและโครงการพรีเมี่ยม

ให้คําแนะนําเกี่ยวกับ:
- การแก้ไขปัญหาการปรับใช้
- การกําหนดสิทธิ์การใช้งานสําหรับผู้ใช้โดยใช้[ศูนย์การจัดการ Microsoft 365](https://go.microsoft.com/fwlink/?linkid=2032704)และ Windows PowerShell
- การดาวน์โหลด และติดตั้งไคลเอ็นต์เดสก์ท็อปออนไลน์ของโครงการจากพอร์ทัล   
- การตั้งค่าคอนฟิกการตั้งค่าการปรับปรุงโดยใช้เครื่องมือการปรับใช้ Office 365 หรือแม่แบบของนโยบายกลุ่ม
- การตั้งค่าเซิร์ฟเวอร์การกระจายไซต์เดียวสําหรับ Project Online Desktop Client รวมถึงคําแนะนําสําหรับการสร้างแฟ้ม configuration.xml สําหรับเครื่องมือการปรับใช้ Office 2016 
- การเชื่อมต่อไคลเอนต์เดสก์ท็อปออนไลน์ของโครงการกับโครงการแบบออนไลน์

## <a name="sharepoint-online"></a>SharePoint Online

ให้คําแนะนําเกี่ยวกับ:
- การตั้งค่าการจัดสรรผู้ใช้รวมทั้งการออกใบอนุญาต
- การเปิดใช้งานการสร้างไซต์สําหรับผู้ดูแลระบบ SharePoint Online ของคุณ    
- กําลังวางแผนไซต์คอลเลกชัน 
- การรักษาความปลอดภัยเนื้อหาและการจัดการสิทธิ์
- การเปิดใช้งานไซต์ส่วนบุคคลและฟีเจอร์ทางสังคม
- การกําหนดค่าคุณลักษณะ SharePoint แบบออนไลน์ 
- การจัดเตรียมการย้ายข้อมูลหากใช้บริการย้ายข้อมูล FastTrack Center และหากคุณมีคุณสมบัติตรงตามเกณฑ์
- การประเมินการกําหนดค่าโครงสร้างพื้นฐานฟาร์ม SharePoint ในสถานที่ที่จําเป็นสําหรับไฮบริดสลี SharePoint Online 
- การใช้เครื่องมือและระบบอัตโนมัติเพื่อ:
  - กําหนดค่าแอปพลิเคชันบริการการค้นหาบนระบบคลาวด์ในสถานที่ 
  - กําหนดค่าความน่าเชื่อถือระหว่าง SharePoint ในองค์กรและสภาพแวดล้อมแบบคลาวด์
- การกําหนดค่าไซต์ SharePoint ในสถานที่เพื่อใช้คุณลักษณะไฮบริดสลี SharePoint แบบออนไลน์
    
## <a name="skype-for-business-online"></a>Skype สําหรับธุรกิจออนไลน์

ให้คําแนะนําเกี่ยวกับ:
- การเตรียมใช้งาน Skype สําหรับข้อมูลเฉพาะตัวของธุรกิจสําหรับ Office 365 
- การเปิดใช้งานการประชุมแบบออนไลน์ 
- การสร้างบัญชีเพื่อเชื่อมโยงกับอุปกรณ์ระบบห้องที่รองรับ (สูงสุด 10 บัญชี) 
- การตั้งค่าคอนฟิกสภาพแวดล้อมเซิร์ฟเวอร์โดเมนแยกเพื่อสนับสนุนไฮบริดสลี Lync หรือ Skype สําหรับสถานการณ์สมมติไฮบริดสลีออนไลน์ธุรกิจ (ถ้าเกี่ยวข้อง)
- การเปิดใช้งานการประชุมทางเสียง:
  - การตั้งค่าองค์กรสําหรับการตั้งค่าเริ่มต้นของบริดจ์การประชุม
  - การกําหนดบริดจ์การประชุมให้กับผู้ใช้ที่ได้รับอนุญาต
- การเปิดใช้งานระบบโทรศัพท์:
  - การเปิดใช้งานระบบโทรศัพท์และแผนการโทรการเริ่มใช้งาน (ในตลาดที่มีอยู่)
  - การกําหนดหมายเลขให้กับผู้ใช้ที่ได้รับอนุญาต
  - คําแนะนํา porting หมายเลขท้องถิ่นผ่าน UI ถึง 999
  - พอร์ตหมายเลขท้องถิ่นสนับสนุน SR กว่า 999
- การเปิดใช้งาน Skype สําหรับการออกอากาศการประชุมทางธุรกิจ:
  - การเปิดใช้งาน Skype สําหรับคําแนะนําการออกอากาศการประชุมทางธุรกิจ
  - การตั้งค่าองค์กรสําหรับสหพันธรัฐกับบริการการออกอากาศการประชุม
